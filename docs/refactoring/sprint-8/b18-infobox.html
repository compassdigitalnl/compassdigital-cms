  • Rich Text Description: Test with bold, italic, links, inline code. Verify text color (grey-dark), line-height (1.7), code background (light variant color). Test long text (wraps correctly), short text (looks good).

-- Create infobox_blocks table
CREATE TABLE IF NOT EXISTS infobox_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  block_id UUID NOT NULL REFERENCES blocks(id) ON DELETE CASCADE,

  variant VARCHAR(20) NOT NULL DEFAULT 'info'
    CHECK (variant IN ('info', 'success', 'warning', 'error')),
  icon VARCHAR(50), -- Lucide icon name
  title VARCHAR(100) NOT NULL,
  description JSONB NOT NULL, -- Lexical rich text JSON
  dismissible BOOLEAN NOT NULL DEFAULT FALSE,
  persistent BOOLEAN NOT NULL DEFAULT FALSE,
  storage_key VARCHAR(100), -- For localStorage

  -- Design settings
  max_width VARCHAR(20) NOT NULL DEFAULT 'wide'
    CHECK (max_width IN ('narrow', 'wide', 'full')),
  margin_top VARCHAR(10) NOT NULL DEFAULT 'md'
    CHECK (margin_top IN ('none', 'sm', 'md', 'lg')),
  margin_bottom VARCHAR(10) NOT NULL DEFAULT 'md'
    CHECK (margin_bottom IN ('none', 'sm', 'md', 'lg')),

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create indexes
CREATE INDEX idx_infobox_blocks_block_id ON infobox_blocks(block_id);
CREATE INDEX idx_infobox_blocks_variant ON infobox_blocks(variant);
CREATE INDEX idx_infobox_blocks_dismissible ON infobox_blocks(dismissible);

-- Composite index for persistent dismissible notifications
CREATE INDEX idx_infobox_persistent ON infobox_blocks(dismissible, persistent, storage_key)
  WHERE dismissible = TRUE AND persistent = TRUE;

-- Full-text search on title + description (for search)
CREATE INDEX idx_infobox_title_search ON infobox_blocks
  USING gin(to_tsvector('english', title || ' ' || description::text));

-- Trigger for updated_at
CREATE OR REPLACE FUNCTION update_infobox_blocks_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_infobox_blocks_updated_at
  BEFORE UPDATE ON infobox_blocks

7 Troubleshooting Guide
Icon not rendering or showing [object Object]
Cause: Invalid Lucide icon name or incorrect import.
Fix: Verify icon name matches Lucide docs exactly (case-sensitive). Check lucide-react is installed: npm list lucide-react. Test with default icons first (leave icon field empty). Add error boundary for invalid icons.
Colors not matching design (blue/green/amber/coral)
Cause: Tailwind config missing color extensions or wrong color values.
Fix: Add blue-50/500/900, green-50/500/900, amber-50/500/900, coral-50/500/900 to tailwind.config.js. Verify hex values match design tokens (blue: #2196F3, green: #00C853, amber: #F59E0B, coral: #FF6B6B). Run npm run build to regenerate CSS.
Dismissible close button not working
Cause: Component not marked 'use client' or onClick handler missing.
Fix: Add 'use client' directive at top of Component.tsx. Verify useState and handleDismiss are implemented. Check onClick handler is attached to button. Test in browser console: verify no React errors.
Persistent dismissal not working (reappears on reload)
Cause: localStorage not set, wrong key, or browser blocking localStorage.
Fix: Verify storageKey is unique (e.g., "dismissed-db-warning"). Check localStorage in DevTools → Application → Local Storage. Test in incognito (localStorage disabled? Use cookies instead). Add try/catch for localStorage errors (Safari private mode blocks).
Multiple infoboxes dismiss together
Cause: Same storageKey used for different notifications.
Fix: Ensure each dismissible infobox has unique storageKey. Use descriptive keys: "dismissed-db-warning", "dismissed-payment-info", etc. Add validation in Payload config to prevent duplicate keys.
Rich text description not rendering or shows JSON
Cause: serializeLexical not called or description is null/undefined.
Fix: Import serializeLexical from @/utilities/serializeLexical. Call serializeLexical({ nodes: description }). Check description field is not empty in Payload. Verify Lexical JSON structure is valid.
Border-left not visible or wrong color
Cause: border-l-[3px] not applying or variant color mapping incorrect.
Fix: Verify Tailwind supports arbitrary values (border-l-[3px]). Check variantConfig borderColor values (border-blue-500, etc.). Test with fixed border-l-4 first. Inspect computed styles in DevTools.
Width variants not responsive on mobile
Cause: Missing mx-auto px-6 or max-width overriding on small screens.
Fix: Add mx-auto px-6 to wrapper div. Use responsive classes: max-w-full sm:max-w-2xl. Test at <640px breakpoint. Ensure parent container doesn't have overflow:hidden.
Inline code background color not matching variant
Cause: Custom prose styles needed for inline code in colored backgrounds.
Fix: Add variant-specific code styles: info → bg-blue-100, success → bg-green-100, etc. Use data-variant attribute on wrapper. Apply conditional classes based on variant. Test with long code snippets.
Close button overlaps long titles
Cause: Absolute positioning of close button without padding compensation.
Fix: Add pr-10 (padding-right) to content wrapper when dismissible=true. Use conditional padding: dismissible ? 'pr-10' : ''. Test with very long titles (80+ chars). Consider truncating title with text-ellipsis if too long.
Accessibility warnings (contrast ratio, focus states)
Cause: Text color not sufficient contrast on light backgrounds or missing focus ring.
Fix: Use darker text variants: blue-900, green-900, amber-900, coral-900 for title/icon. Verify contrast ratio ≥ 4.5:1 with WebAIM contrast checker. Add focus-visible:ring-2 to close button. Test with keyboard navigation.

Usage Guidelines
  • Variant Selection: Use info (blue) for general tips, notes, documentation links. Use success (green) for confirmations ("Order placed!", "Profile updated"). Use warning (amber) for cautions ("Limited stock", "Payment pending"). Use error (coral) for failures ("Payment failed", "Invalid input").
  • Icon Choice: Leave icon field empty for default icons (info → info, success → check-circle, warning → alert-triangle, error → x-circle). Use custom icons sparingly for brand consistency. Test icon visibility at 20px size.
  • Title Length: Keep titles under 50 characters for readability. Titles are bold and colored to match variant. Use sentence case (not ALL CAPS).
  • Description Content: Use rich text features (bold for emphasis, inline code for tech terms, links for external resources). Keep descriptions concise (1-3 sentences). For longer content, link to separate page.
  • Dismissible Best Practices: Enable dismissible for non-critical notifications (tips, announcements). DON'T make critical errors dismissible (user might miss important info). Use persistent for low-priority recurring notifications.
  • Persistent Storage: Use descriptive storageKeys: "dismissed-db-warning", "dismissed-promo-banner", etc. Avoid generic keys like "box1", "notification". Consider adding version to key if content changes: "dismissed-db-warning-v2".
  • Width Selection: Use narrow (640px) for focused in-content notifications. Use wide (900px, default) for page-level announcements. Use full for banner-style notifications at top of page.
  • Accessibility: Always provide meaningful title (screen readers announce this first). Use semantic HTML (button for close, not div). Ensure 4.5:1 contrast ratio. Test keyboard navigation (Tab to close button, Enter/Space to dismiss).
  • Common Use Cases: Info → Environment setup instructions, feature announcements, helpful tips. Success → Order confirmations, profile updates, successful saves. Warning → Stock alerts, payment pending, important notices. Error → Payment failures, form errors, system errors.

7 Troubleshooting Guide
Icon not rendering or showing [object Object]
Cause: Invalid Lucide icon name or incorrect import.
Fix: Verify icon name matches Lucide docs exactly (case-sensitive). Check lucide-react is installed: npm list lucide-react. Test with default icons first (leave icon field empty). Add error boundary for invalid icons.
Colors not matching design (blue/green/amber/coral)
Cause: Tailwind config missing color extensions or wrong color values.
Fix: Add blue-50/500/900, green-50/500/900, amber-50/500/900, coral-50/500/900 to tailwind.config.js. Verify hex values match design tokens (blue: #2196F3, green: #00C853, amber: #F59E0B, coral: #FF6B6B). Run npm run build to regenerate CSS.
Dismissible close button not working
Cause: Component not marked 'use client' or onClick handler missing.
Fix: Add 'use client' directive at top of Component.tsx. Verify useState and handleDismiss are implemented. Check onClick handler is attached to button. Test in browser console: verify no React errors.
Persistent dismissal not working (reappears on reload)
Cause: localStorage not set, wrong key, or browser blocking localStorage.
Fix: Verify storageKey is unique (e.g., "dismissed-db-warning"). Check localStorage in DevTools → Application → Local Storage. Test in incognito (localStorage disabled? Use cookies instead). Add try/catch for localStorage errors (Safari private mode blocks).
Multiple infoboxes dismiss together
Cause: Same storageKey used for different notifications.
Fix: Ensure each dismissible infobox has unique storageKey. Use descriptive keys: "dismissed-db-warning", "dismissed-payment-info", etc. Add validation in Payload config to prevent duplicate keys.
Rich text description not rendering or shows JSON
Cause: serializeLexical not called or description is null/undefined.
Fix: Import serializeLexical from @/utilities/serializeLexical. Call serializeLexical({ nodes: description }). Check description field is not empty in Payload. Verify Lexical JSON structure is valid.
Border-left not visible or wrong color
Cause: border-l-[3px] not applying or variant color mapping incorrect.
Fix: Verify Tailwind supports arbitrary values (border-l-[3px]). Check variantConfig borderColor values (border-blue-500, etc.). Test with fixed border-l-4 first. Inspect computed styles in DevTools.
Width variants not responsive on mobile
Cause: Missing mx-auto px-6 or max-width overriding on small screens.
Fix: Add mx-auto px-6 to wrapper div. Use responsive classes: max-w-full sm:max-w-2xl. Test at <640px breakpoint. Ensure parent container doesn't have overflow:hidden.
Inline code background color not matching variant
Cause: Custom prose styles needed for inline code in colored backgrounds.
Fix: Add variant-specific code styles: info → bg-blue-100, success → bg-green-100, etc. Use data-variant attribute on wrapper. Apply conditional classes based on variant. Test with long code snippets.
Close button overlaps long titles
Cause: Absolute positioning of close button without padding compensation.
Fix: Add pr-10 (padding-right) to content wrapper when dismissible=true. Use conditional padding: dismissible ? 'pr-10' : ''. Test with very long titles (80+ chars). Consider truncating title with text-ellipsis if too long.
Accessibility warnings (contrast ratio, focus states)
Cause: Text color not sufficient contrast on light backgrounds or missing focus ring.
Fix: Use darker text variants: blue-900, green-900, amber-900, coral-900 for title/icon. Verify contrast ratio ≥ 4.5:1 with WebAIM contrast checker. Add focus-visible:ring-2 to close button. Test with keyboard navigation.

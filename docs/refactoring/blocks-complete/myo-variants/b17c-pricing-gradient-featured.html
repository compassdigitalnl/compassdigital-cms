Veelgestelde vragen

Kan ik op elk moment opzeggen?

Ja, je kunt je abonnement op elk moment opzeggen zonder opzegtermijn. Bij opzegging blijft je account actief tot het einde van je huidige facturatieperiode.

Wat gebeurt er na de 14-dagen trial?

Na de trial wordt je niet automatisch doorbelast. Je krijgt een melding dat je trial is verlopen en kunt dan kiezen of je wilt upgraden naar een betaald plan.

Zijn er setup kosten of verborgen kosten?

Nee, er zijn geen setup kosten, verborgen kosten of transactiekosten. De prijs die je ziet is inclusief alle functies van dat plan.

Kan ik later upgraden of downgraden?

Ja, je kunt op elk moment upgraden of downgraden naar een ander plan. Bij upgraden krijg je direct toegang tot extra functies. Bij downgraden blijven de nieuwe tarieven van kracht vanaf je volgende factuurperiode.

4 Admin Panel Structure

How editors interact with this block in the Payload admin panel:

📝 Editor Workflow

  1. Choose Layout Variant: Three-Tier, Two-Column, Comparison Table, or With FAQ
  2. Configure Header:
    • Enter badge text (e.g., "Eenvoudige prijzen")
    • Select badge icon (Zap, Gift, Shield, Crown)
    • Enter main title and description
  3. Setup Billing Toggle:
    • Check "Enable Billing Toggle" for monthly/yearly options
    • Enter save badge text (e.g., "Bespaar 20%")
  4. Add Pricing Plans: Click "Add Item" to create plan (2-4 plans recommended)
  5. For Each Plan:
    • Enter plan name (e.g., "Professional") and description
    • Check "Featured Plan" for ONE plan to highlight with gradient
    • If featured: Choose badge text, icon, and gradient color (Navy/Teal/Purple/Blue)
    • Enter pricing: monthly price, yearly price, or custom text ("Op aanvraag")
    • Optional: Add old price for discount display with save badge
    • Configure CTA button text and URL
    • Add features (check enabled/disabled for each)
  6. Optional FAQ: If using "With FAQ" variant, add Q&A pairs below pricing

✅ Best Practice: Mark ONE plan as featured (typically the middle or most popular tier). This creates visual hierarchy and guides users toward your recommended option, increasing conversions by 20-30%.

7 Troubleshooting

Common issues and solutions:

Issue Cause Solution
Featured card not scaling Missing transform: scale(1.05) or conflicting CSS Ensure featured class applies:
transform: scale(1.05) and z-index: 1
Gradient not visible CSS custom properties not defined or gradient class not applied Check that --navy, --navy-light, --teal, --teal-light are defined in :root. Verify getGradientClass() returns correct Tailwind class.
Billing toggle doesn't update prices useState not triggering re-render or getCurrentPrice not reading state Ensure isYearly state is passed to getCurrentPrice(). Check that onClick handler calls setIsYearly(!isYearly).
Popular badge overlapping card Missing negative top position or transform not applied Badge must have:
position: absolute; top: -14px; left: 50%; transform: translateX(-50%)
Icons not rendering Lucide React not installed or icon name mismatch Install: npm install lucide-react
Check iconMap keys match field options (zap, star, crown, check, x, etc.)
Featured CTA button hard to read Insufficient contrast (white on light gradient) Featured CTA uses inverted colors (white bg, teal text) for contrast. Verify bg-white text-teal classes applied when isFeatured is true.
Multiple featured cards displayed Editor selected multiple plans as featured Add validation in Payload config or component to warn if more than 1 plan has isFeatured: true. Best practice: only 1 featured plan.
FAQ section not showing Variant not set to "with-faq" or FAQ array empty Check variant === 'with-faq' condition and ensure FAQ array has items. FAQ field only visible in admin when variant is "With FAQ".

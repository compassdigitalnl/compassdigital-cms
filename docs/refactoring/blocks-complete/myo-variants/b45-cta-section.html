Admin Panel Structure
How editors see and use the block
Admin Panel Layout:
📝 Overline [text input, optional]
🎯 Title [text input, required]
📄 Description [textarea, 3 rows, optional]
🎨 Background Variant [select: Navy | Teal | White]
🔘 Buttons [array, 1-2 items]
├─ Button Text [text input]
├─ Link URL [text input]
├─ Button Style [select: Primary | Secondary]
└─ Icon [select: None or 6 icon options]
💡 Editor Guidance:
  • Title: kort en krachtig — max 8-10 woorden voor impact
  • 1 primaire button is vaak genoeg — te veel keuze vermindert conversie
  • Dark variants (navy/teal) → gebruik aan einde pagina voor conversie
  • White variant → geschikt voor mid-page CTA's (newsletter, etc.)
  • Icons: gebruik met mate — alleen bij eerste button voor visuele richting

Testing Checklist
Verifieer deze punten voor deployment
✅ Functionality
  • Navy variant renders correctly (gradient + glow)
  • Teal variant renders correctly (gradient + glow)
  • White variant renders correctly (borders)
  • Buttons link to correct URLs
  • Icons render when selected (6 variants)
  • Primary button styling correct per variant
  • Secondary button styling correct per variant
📱 Responsive
  • Desktop (1024px+): buttons inline (flex-row)
  • Mobile (< 768px): buttons stacked (flex-col)
  • Title font size adapts (36px → 28px)
  • Description font size adapts (16px → 14px)
  • Glow circles repositioned on mobile
  • Padding reduced on mobile (80px → 56px)
♿ Accessibility
  • Color contrast voldoet (white text on dark bg = 21:1, dark text on white = 12:1)
  • Buttons keyboard accessible (focus states visible)
  • Link text descriptive (no "klik hier")
  • Icons decorative only (not required for understanding)
  • Heading hierarchy correct (h2 for title)

Common Issues & Solutions
Troubleshooting guide
Issue Solution
Glow circles not visible Check overflow: hidden on section.
Verify pseudo-element has pointer-events: none
Gradient not showing CSS gradients require Tailwind config or inline styles.
Use: bg-gradient-to-br from-navy to-navy-light
Or define custom gradient utilities in tailwind.config.ts
Buttons wrong color on white variant Check conditional styling: variant === 'white' ? 'bg-teal' : 'bg-white'
Primary button should be teal on white bg, white on dark bg
Icons not rendering Verify Lucide React import: npm install lucide-react
Check iconMap has all 6 icon options
Ensure icon value in database matches iconMap keys
Buttons not stacking on mobile Add responsive classes: flex-wrap on container
Or force column: md:flex-row flex-col
Text hard to read on gradient Dark variants MUST use white text: text-white
Add opacity to description: text-white/80
If still unclear, adjust gradient colors or add text-shadow

Admin Panel Structure
How editors see and use the block
Admin Panel Layout:
📝 Section Title [text input]
📄 Subtitle [textarea, 2 rows, optional]
🎨 Background Color [select: Grey (default) | White]
📋 Process Steps [array, 2-4 items]
├─ Icon [select dropdown with emoji previews]
├─ Step Title [text input]
└─ Step Description [textarea, 3 rows]
💡 Editor Guidance:
  • 3 steps werkt het beste — middle step wordt automatisch highlighted
  • Korte, actie-georiënteerde titels (3-4 woorden)
  • Beschrijvingen: 2-3 zinnen voor optimale leesbaarheid
  • Icons: kies visueel passend bij de stap (Search → Settings → Check)

Common Issues & Solutions
Troubleshooting guide
Issue Solution
Icons not rendering Check Lucide React is installed: npm install lucide-react
Verify icon import in component file
Center step not highlighted Only works with exactly 3 steps. Check: steps.length === 3 && index === 1
Verify Tailwind classes: lg:scale-105
Grid layout broken on mobile Tailwind responsive classes missing. Add: lg:grid-cols-[1fr_400px_1fr]
Check grid falls back to single column below 1024px
Step numbers wrong Using index + 1 in map. Verify array starts at 0.
Check stepNumber field is NOT being used (auto-number via index)
White text on white background Center step uses text-white. Ensure backgroundColor is NOT white when using center highlight.
Use backgroundColor='grey' for best contrast
Block doesn't appear in admin Verify block is registered in Payload config:
blocks: [Hero, Features, ProcessSteps, ...]
Restart dev server after adding new block

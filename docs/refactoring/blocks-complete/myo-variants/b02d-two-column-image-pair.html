4 Admin Panel Structure

How editors interact with this block in the Payload admin panel:

📝 Editor Workflow

  1. Choose Layout: Select from Equal Width, Left Large, Right Large, or Comparison
  2. Choose Image Type: Standard (hover caption) or With Text Overlay
  3. Set Aspect Ratio: Landscape (4:3), Portrait (3:4), or Square (1:1)
  4. Upload Image 1: Click "Choose File" and select from Media Library or upload new image
  5. Add Image 1 Content:
    • For Standard type: Enter optional hover caption
    • For Overlay type: Choose icon, enter title and description
    • For Comparison layout: Enter badge text (default: "Voor")
  6. Upload Image 2: Click "Choose File" for second image
  7. Add Image 2 Content: Same options as Image 1
  8. Adjust Spacing: Choose Compact (16px), Default (32px), or Spacious (48px) gap between images

✅ Best Practice: Use Equal Width for balanced layouts, Left/Right Large to emphasize one image, and Comparison for before/after scenarios.

  • ✅ Left large layout shows 1.5:1 ratio, right large shows 1:1.5
  • ✅ Comparison layout displays badges correctly positioned
  • ✅ Mobile breakpoint (<768px) stacks images vertically
  • ✅ Spacing variants (compact/default/spacious) apply correct gaps

  • ✅ Images load with correct aspect ratios (4:3, 3:4, 1:1)
  • ✅ object-fit: cover crops images properly without distortion
  • ✅ Image optimization (Next.js Image) loads correct sizes for viewport
  • ✅ Alt text from caption/overlay title is present
  • ✅ Placeholder background (#F5F7FA) shows while images load

🖱️ Hover & Interactions

  • ✅ Standard type: Caption fades in on hover with gradient overlay
  • ✅ Hover effect: Image lifts (-translateY-1) and shadow increases
  • ✅ Overlay type: Text cards are always visible, no hover interaction
  • ✅ Icons render correctly for overlay cards (Lucide React icons)
  • ✅ Transitions are smooth (0.3s duration)

📝 Content & Edge Cases

  • ✅ Comparison badges show with correct colors (teal for "Na", grey for "Voor")
  • ✅ Long caption text doesn't overflow container
  • ✅ Missing caption/overlay doesn't break layout
  • ✅ Overlay description truncates gracefully if too long
  • ✅ Icon selection renders all 6 icon options correctly

⚡ Performance

  • ✅ Images use Next.js Image component with automatic optimization
  • ✅ sizes attribute loads appropriate image size for viewport
  • ✅ No layout shift (CLS) when images load (aspect-ratio prevents)

7 Troubleshooting

Common issues and solutions:

Issue Cause Solution
Images not loading Image URL is undefined or media relation not populated Ensure images are populated in Payload query:
depth: 1 or manually populate with { relationTo: 'media', depth: 1 }
Grid not responsive on mobile Missing md: breakpoint prefix in Tailwind classes Use mobile-first approach:
grid-cols-1 md:grid-cols-2
Hover caption doesn't appear Caption text is missing or imageType is set to 'overlay' Check that caption1/caption2 fields have values AND imageType is 'standard'. Overlay type doesn't show hover captions.
Comparison badges missing Layout is not set to 'comparison' or badge text is empty Set layout: 'comparison' and ensure comparisonBadge1/comparisonBadge2 have text values (defaults: "Voor"/"Na")
Icons not rendering in overlay Lucide React not installed or icon name mismatch Install: npm install lucide-react
Check iconMap keys match select options (palette, code, megaphone, users, zap, shield)
Aspect ratio incorrect CSS aspect-ratio not supported in older browsers Fallback: Use padding-top hack for IE11:
padding-top: 75% (for 4:3)
Text overlay truncated Description text too long or card positioned off-screen Enforce maxLength in field config (200 chars) or use CSS line-clamp:
line-clamp-3
Layout shift when images load aspect-ratio not applied or Next.js Image missing width/height Use aspect-ratio CSS on wrapper + fill prop on Next.js Image (no explicit width/height needed)

Step 1: CSS Class Structure

Base class + Variant + Size (optional)

All buttons use the .btn base class, then add variant and size modifiers:

// Base button (always required) .btn { padding: 8px 18px; border-radius: var(--r-sm); font-family: var(--font); font-size: 12px; font-weight: 700; border: 1.5px solid transparent; cursor: pointer; display: inline-flex; align-items: center; gap: 6px; transition: all var(--dur); } // Color Variants (choose one) .btn-teal { background: var(--teal); color: #fff; } .btn-navy { background: var(--navy); color: #fff; } .btn-coral { background: var(--coral); color: #fff; } .btn-amber { background: linear-gradient(135deg, var(--amber), var(--amber-dark)); } .btn-green { background: var(--green); color: #fff; } .btn-outline-teal { background: none; border-color: var(--teal); color: var(--teal); } .btn-outline-neutral { background: none; border-color: var(--grey); color: var(--grey-dark); } .btn-ghost { background: none; border: none; color: var(--teal); } // Size Modifiers (optional) .btn-sm { padding: 5px 12px; font-size: 10px; } .btn-lg { padding: 11px 26px; font-size: 14px; }

Step 3: React/Next.js Component

Reusable Button component with TypeScript:

// components/ui/Button.tsx import React from 'react' type ButtonVariant = | 'teal' | 'navy' | 'coral' | 'amber' | 'green' | 'outline-teal' | 'outline-neutral' | 'ghost' type ButtonSize = 'sm' | 'md' | 'lg' interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> { variant?: ButtonVariant size?: ButtonSize icon?: React.ReactNode iconPosition?: 'left' | 'right' } export const Button: React.FC<ButtonProps> = ({ variant = 'teal', size = 'md', icon, iconPosition = 'left', className = '', children, ...props }) => { const classes = [ 'btn', `btn-${variant}`, size !== 'md' && `btn-${size}`, className, ] .filter(Boolean) .join(' ') return ( <button className={classes} {...props}> {icon && iconPosition === 'left' && icon} {children} {icon && iconPosition === 'right' && icon} </button> ) } // Usage: <Button variant="teal" icon={<ShoppingCart />}> In winkelwagen </Button> <Button variant="coral" size="sm" onClick={handleDelete}> Verwijderen </Button>
Step 4: Usage Guidelines

When to use each variant:

  • Teal: Primary CTAs (add to cart, checkout, save, submit)
  • Navy: Secondary actions (view details, more info)
  • Coral: Destructive actions (delete, cancel order, remove)
  • Amber: Premium/upsell actions (upgrade, PRO features)
  • Green: Confirmation actions (confirm, approve, accept)
  • Outline Teal: Secondary teal actions (view, details)
  • Outline Neutral: Tertiary actions (filter, options, settings)
  • Ghost: Minimal actions (skip, learn more, read more)

Size selection:

  • Small (sm): Compact spaces, tables, card actions
  • Default (md): Standard use, forms, modals
  • Large (lg): Hero CTAs, emphasis, landing pages
Step 5: Accessibility Best Practices
  • ✅ Use semantic <button> for actions, <a> for navigation
  • ✅ All buttons keyboard accessible (focus ring included)
  • ✅ Icon-only buttons need aria-label
  • ✅ Disabled state uses disabled attribute (not just CSS)
  • ✅ Loading state shows spinner + aria-busy="true"
  • ✅ Color contrast meets WCAG AA (4.5:1 minimum)
<!-- Icon-only button (needs aria-label) --> <button class="btn btn-teal" aria-label="In winkelwagen"> <i data-lucide="shopping-cart"></i> </button> <!-- Loading state --> <button class="btn btn-teal" aria-busy="true" disabled> <i data-lucide="loader" class="spin"></i> Bezig met laden... </button>

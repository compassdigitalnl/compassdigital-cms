5 Testing Checklist
  • Gradient Styles: Test all 3 background styles (dark navy gradient, light white, teal gradient). Verify teal glow appears only on dark style.
  • Button Layouts: Add 1 button, then 2 buttons. Test primary, secondary, ghost styles. Verify arrow icon appears only on primary buttons.
  • Variant Switching: Test centered (text center), split (text left, buttons right on desktop), full-width (edge-to-edge). Verify responsive stacking on mobile.
  • Text Contrast: On dark backgrounds, verify description has opacity 70%. On light backgrounds, verify text is navy/grey-dark with sufficient contrast.
  • Responsive Behavior: Test split layout stacks vertically on mobile. Buttons wrap to new line if too long. Title reduces from 32-36px to 28px on mobile.
  • Link Validation: Test internal links (/contact), external links (https://...), and hash links (#section). Verify buttons are keyboard accessible.
  • Accessibility: Verify focus states on buttons (teal glow ring). Test keyboard navigation (Tab key). Ensure sufficient color contrast (min 4.5:1).
6 Database Migration
SQL migrations/YYYYMMDDHHMMSS_add_cta_block.sql 
-- Add CTA block type to blocks table
ALTER TYPE block_type ADD VALUE IF NOT EXISTS 'cta';

-- Create cta_blocks table
CREATE TABLE IF NOT EXISTS cta_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  block_id UUID NOT NULL REFERENCES blocks(id) ON DELETE CASCADE,

  -- Content fields
  title TEXT NOT NULL,
  description TEXT,

  -- Design fields
  variant VARCHAR(20) NOT NULL DEFAULT 'centered'
    CHECK (variant IN ('centered', 'split', 'full-width')),
  style VARCHAR(20) NOT NULL DEFAULT 'dark'
    CHECK (style IN ('dark', 'light', 'gradient')),

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create junction table for buttons
CREATE TABLE IF NOT EXISTS cta_buttons (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cta_block_id UUID NOT NULL REFERENCES cta_blocks(id) ON DELETE CASCADE,
  label TEXT NOT NULL,
  link TEXT NOT NULL,
  style VARCHAR(20) NOT NULL DEFAULT 'primary'
    CHECK (style IN ('primary', 'secondary', 'ghost')),
  order_index INTEGER NOT NULL DEFAULT 0,

  CONSTRAINT unique_button_order UNIQUE (cta_block_id, order_index)
);

-- Create indexes
CREATE INDEX idx_cta_blocks_block_id ON cta_blocks(block_id);
CREATE INDEX idx_cta_blocks_variant ON cta_blocks(variant);
CREATE INDEX idx_cta_buttons_cta_block_id ON cta_buttons(cta_block_id);
CREATE INDEX idx_cta_buttons_order ON cta_buttons(cta_block_id, order_index);

-- Trigger for updated_at
CREATE OR REPLACE FUNCTION update_cta_blocks_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_cta_blocks_updated_at
  BEFORE UPDATE ON cta_blocks
  FOR EACH ROW EXECUTE FUNCTION update_cta_blocks_updated_at();

7 Troubleshooting Guide
Teal glow not visible on dark background
Cause: Glow positioned outside viewport or opacity too low.
Fix: Verify parent section has overflow-hidden. Check glow div has absolute positioning with negative top/right (-10px). Increase opacity from 0.3 to 0.4 if too subtle. Use radial-gradient with rgba(0,137,123,0.35).
Buttons not stacking on mobile
Cause: Missing flex-wrap or gap too large.
Fix: Add flex-wrap class to button container. Test with 2 long button labels. Consider stacking vertically (flex-col) on screens < 480px using responsive classes.
Split layout not working on desktop
Cause: Missing md:flex-row or md:justify-between classes.
Fix: Verify parent div has md:flex-row md:justify-between classes. Check Tailwind breakpoint is 768px (md:). Test at exact 768px width. Ensure text content has max-w-xl to prevent stretching.
Text contrast too low on gradient
Cause: Navy gradient too light or description opacity too high.
Fix: Darken gradient endpoint: from-navy to-[#06101B]. Reduce description opacity from text-white/70 to text-white/60. Add subtle text shadow for better readability.
Arrow icon not showing on primary button
Cause: SVG not rendering or conditional check failing.
Fix: Verify button.style === 'primary' condition. Check SVG has correct viewBox="0 0 24 24". Ensure h-4 w-4 classes apply. Test with stroke="currentColor" for color inheritance.
Light style border not visible
Cause: Border color too close to background or missing border class.
Fix: Verify light style has border border-grey classes. Check --grey color is #E8ECF1 (sufficient contrast with white). Test on different backgrounds.
Full-width variant constrained by container
Cause: Parent page layout has max-width constraint.
Fix: Remove max-w-4xl mx-auto when variant is full-width. Add w-full class. For true full-bleed, use negative margins: -mx-6 on section with px-6 on inner content.
Ghost button not visible on light background
Cause: text-teal-light (#26A69A) has low contrast on white.
Fix: Override ghost button color for light style: use text-teal (darker) instead of text-teal-light. Add hover:text-teal-dark for better visibility.

Usage Guidelines
  • Title: DM Serif Display font, 32-36px, keep under 60 characters for optimal readability
  • Description: Plain text (not rich text), 14-15px, opacity 70% on dark backgrounds
  • Buttons: Max 2 buttons. First button = primary action (most important). Use primary style for main CTA.
  • Variants: Centered (default, best for single CTA), Split (text/buttons side-by-side on desktop), Full-width (edge-to-edge for maximum impact)
  • Background Styles: Dark (navy gradient with teal glow, most common), Light (white with border), Gradient (full teal, use sparingly)
  • Mobile: All variants stack vertically on mobile. Title reduces to 28px. Buttons remain full-width on small screens.
  • Placement: Best used at end of pages, between major sections, or as conversion points in long content.

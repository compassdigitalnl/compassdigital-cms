Step 2: TypeScript Type Definitions

Create src/types/theme.ts for type safety:

// src/types/theme.ts export interface ThemeColors { // Primary Colors navy: string navyLight: string teal: string tealLight: string tealDark: string // Status Colors green: string coral: string amber: string blue: string purple: string // Neutral Colors white: string bg: string grey: string greyMid: string greyDark: string text: string } export interface Theme { id: string colors?: ThemeColors updatedAt: string createdAt: string }
Step 3: Usage in Next.js Components

Example: Fetch theme and generate CSS variables in your layout:

// app/layout.tsx import { getPayloadHMR } from '@payloadcms/next/utilities' import configPromise from '@payload-config' import { Theme } from '@/types/theme' export default async function RootLayout({ children }) { const payload = await getPayloadHMR({ config: configPromise }) // Fetch theme from Payload Global const theme = await payload.findGlobal({ slug: 'theme', }) as Theme // Generate CSS custom properties const cssVariables = theme.colors ? ` --navy: ${theme.colors.navy}; --navy-light: ${theme.colors.navyLight}; --teal: ${theme.colors.teal}; --teal-light: ${theme.colors.tealLight}; --teal-dark: ${theme.colors.tealDark}; --green: ${theme.colors.green}; --coral: ${theme.colors.coral}; --amber: ${theme.colors.amber}; --blue: ${theme.colors.blue}; --purple: ${theme.colors.purple}; --white: ${theme.colors.white}; --bg: ${theme.colors.bg}; --grey: ${theme.colors.grey}; --grey-mid: ${theme.colors.greyMid}; --grey-dark: ${theme.colors.greyDark}; --text: ${theme.colors.text}; ` : '' return ( <html lang="nl"> <head> {cssVariables && ( <style dangerouslySetInnerHTML={{ __html: `:root { ${cssVariables} }` }} /> )} </head> <body>{children}</body> </html> ) }

Step 5: Testing Checklist
1. Global exists: Navigate to /admin/globals/theme
2. All 16 color fields visible: 5 primary + 5 status + 6 neutrals
3. Default values set: All fields show correct hex values
4. Validation works: Try invalid hex (e.g. "red") → should show error
5. CSS variables generated: Inspect <html> in browser DevTools → see :root variables
6. Color changes reflect: Change teal to #FF0000 → save → check frontend buttons
7. TypeScript types work: No TS errors in components using theme
Step 6: Database Migration

⚠️ IMPORTANT: After creating Theme.ts, generate a database migration:

# Generate migration for new global npx payload migrate:create add_theme_global # Review generated migration file in src/migrations/ # It should contain CREATE TABLE for _theme_v table # Apply migration to database npx payload migrate # Verify in database # PostgreSQL: SELECT * FROM _theme_v; # Should show 1 row with default color values
Step 7: Common Issues & Solutions
Issue Solution
/admin/globals/theme shows 404 Check src/payload.config.ts → globals array includes Theme
Fields not showing in admin Restart dev server (npm run dev)
Colors not applying to frontend Check app/layout.tsx has CSS variable generation code
TypeScript errors on theme.colors Check src/types/theme.ts is imported correctly
Database error on save Run migration: npx payload migrate

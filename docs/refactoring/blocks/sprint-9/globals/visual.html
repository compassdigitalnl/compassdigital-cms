Visual Tokens Radius, shadows, z-index
Design Token
Border Radius
--r-sm: 8px Buttons, inputs, small cards
--r-md: 12px Cards, modals, panels (default)
--r-lg: 16px Large cards, hero sections
--r-xl: 20px Overlay panels (search, modals)
--r-full: 9999px Pills, avatars, badges
Box Shadows
--sh-sm 0 1px 3px rgba(10,22,40,0.06)

Subtle lift, default cards

--sh-md 0 4px 20px rgba(10,22,40,0.08)

Hover states, dropdowns

--sh-lg 0 8px 40px rgba(10,22,40,0.12)

Modals, overlays

--sh-xl 0 20px 60px rgba(10,22,40,0.16)

Floating panels, mega menus

Z-index Scale
Token Value Usage Examples
--z-dropdown 100 Dropdowns, tooltips Sort menu, filter dropdowns, product quick view
--z-sticky 200 Sticky headers, sidebars Main navigation, sticky sidebar, sticky CTA bar
--z-overlay 300 Search overlays, backdrops InstantSearch overlay, mobile menu backdrop
--z-modal 400 Modals, drawers QuickView modal, MiniCart flyout, address editor
--z-toast 500 Toast notifications AddToCart toast, system notifications, alerts
Z-index Management: Never use arbitrary z-index values. Only use these 5 tokens to ensure proper layering across all components. If you need a component "on top", determine which layer it belongs to conceptually.

Step 3 CSS Variable Generation in Next.js
src/app/layout.tsx

Fetch visual data from Payload and inject CSS custom properties into the DOM

// src/app/layout.tsx import { getPayloadHMR } from '@payloadcms/next/utilities' import config from '@payload-config' export default async function RootLayout({ children }) { const payload = await getPayloadHMR({ config }) // Fetch theme global const theme = await payload.findGlobal({ slug: 'theme', }) // Generate visual CSS variables const visual = theme?.visual || {} const visualVars = ` /* Border Radius */ --r-sm: ${visual.radiusSm || 8}px; --r-md: ${visual.radiusMd || 12}px; --r-lg: ${visual.radiusLg || 16}px; --r-xl: ${visual.radiusXl || 20}px; --r-full: ${visual.radiusFull || 9999}px; /* Box Shadows */ --sh-sm: ${visual.shadowSm || '0 1px 3px rgba(10, 22, 40, 0.06)'}; --sh-md: ${visual.shadowMd || '0 4px 20px rgba(10, 22, 40, 0.08)'}; --sh-lg: ${visual.shadowLg || '0 8px 40px rgba(10, 22, 40, 0.12)'}; --sh-xl: ${visual.shadowXl || '0 20px 60px rgba(10, 22, 40, 0.16)'}; /* Z-index Scale */ --z-dropdown: ${visual.zDropdown || 100}; --z-sticky: ${visual.zSticky || 200}; --z-overlay: ${visual.zOverlay || 300}; --z-modal: ${visual.zModal || 400}; --z-toast: ${visual.zToast || 500}; ` return ( <html lang="nl"> <head> <style dangerouslySetInnerHTML={{ __html: `:root { ${visualVars} }` }} /> </head> <body>{children}</body> </html> ) }

Step 8 Advanced Usage & Vertical Customization
Vertical-Specific Visual Overrides

Visual tokens CAN be customized per vertical for distinct aesthetic treatments

/* src/styles/verticals/beauty.css */ /* Beauty vertical: softer shadows, tighter radius for refined aesthetic */ [data-vertical="beauty"] { --r-sm: 6px; /* Tighter radius (8px → 6px) */ --r-md: 10px; /* More refined (12px → 10px) */ /* Softer shadows (lower opacity) */ --sh-sm: 0 1px 3px rgba(10, 22, 40, 0.04); /* 0.06 → 0.04 */ --sh-md: 0 4px 20px rgba(10, 22, 40, 0.06); /* 0.08 → 0.06 */ } /* src/styles/verticals/horeca.css */ /* Horeca vertical: warmer shadows, no changes to radius */ [data-vertical="horeca"] { /* Warmer shadow color (navy → warm brown) */ --sh-sm: 0 1px 3px rgba(44, 24, 16, 0.06); /* Matches --warm color */ --sh-md: 0 4px 20px rgba(44, 24, 16, 0.08); } /* src/styles/verticals/automotive.css */ /* Automotive vertical: sharper edges, stronger shadows for bold aesthetic */ [data-vertical="automotive"] { --r-sm: 4px; /* Sharper corners (8px → 4px) */ --r-md: 8px; /* More angular (12px → 8px) */ /* Stronger shadows for dramatic effect */ --sh-md: 0 4px 20px rgba(10, 22, 40, 0.12); /* 0.08 → 0.12 */ --sh-lg: 0 8px 40px rgba(10, 22, 40, 0.16); /* 0.12 → 0.16 */ }
Design Flexibility: Visual tokens are the most flexible design tokens. Radius affects perceived "softness" vs "sharpness", shadows affect depth perception, and both can be adjusted per vertical to match brand personality. The z-index scale should remain consistent across all verticals.
Animated Shadow & Radius Transitions

Smooth transitions between visual states for polished interactions

/* Card with animated shadow on hover */ .card { border-radius: var(--r-md); box-shadow: var(--sh-sm); transition: box-shadow 0.3s cubic-bezier(0.4, 0, 0.2, 1), transform 0.3s cubic-bezier(0.4, 0, 0.2, 1); } .card:hover { box-shadow: var(--sh-md); /* Elevates on hover */ transform: translateY(-2px); /* Subtle lift */ } /* Button with shadow press effect */ .button { border-radius: var(--r-sm); box-shadow: var(--sh-sm); transition: all 0.15s ease; } .button:active { box-shadow: none; /* Flatten on press */ transform: translateY(1px); /* Press down */ } /* Modal with staggered entrance animation */ .modal { border-radius: var(--r-xl); box-shadow: var(--sh-lg); z-index: var(--z-modal); animation: modalEnter 0.3s cubic-bezier(0.4, 0, 0.2, 1); } @keyframes modalEnter { from { opacity: 0; transform: scale(0.95); box-shadow: none; } to { opacity: 1; transform: scale(1); box-shadow: var(--sh-lg); } }
Z-index Debugging Utility

DevTools helper to visualize z-index stacking across all layers

// src/utils/debugZIndex.ts // Add this to your browser DevTools console for z-index debugging export function debugZIndex() { const elements = Array.from(document.querySelectorAll('*')) const zIndexMap = new Map() elements.forEach(el => { const zIndex = window.getComputedStyle(el).zIndex if (zIndex !== 'auto') { const layer = parseInt(zIndex) if (!zIndexMap.has(layer)) { zIndexMap.set(layer, []) } zIndexMap.get(layer).push({ element: el, tag: el.tagName, class: el.className, }) } }) console.log('🔍 Z-index Stack Analysis:') Array.from(zIndexMap.keys()) .sort((a, b) => b - a) .forEach(layer => { const items = zIndexMap.get(layer) console.log(`\nLayer ${layer}:`, items.length, 'elements') items.forEach(item => { console.log(` - <${item.tag}> .${item.class}`) }) }) // Expected layers: // 500: Toasts (topmost) // 400: Modals, drawers // 300: Overlays, backdrops // 200: Sticky headers // 100: Dropdowns, tooltips } // Usage in browser console: // > debugZIndex()

Step 2: TypeScript Type Definitions

Create src/types/blocks.ts for type safety:

// src/types/blocks.ts import { Media } from '@/payload-types' export interface Logo { image: string | Media name: string link?: string id?: string } export interface LogoBarBlock { blockType: 'logobar' id?: string title?: string logos: Logo[] autoScroll?: boolean variant: 'light' | 'white' | 'dark' } // Type guard to check if a block is a LogoBarBlock export function isLogoBarBlock(block: any): block is LogoBarBlock { return block?.blockType === 'logobar' } // Helper to extract image URL from Media object or string export function getLogoImageUrl(image: string | Media): string { if (typeof image === 'string') return image return image?.url || '' }
Step 3: React Component Implementation

Create src/branches/blocks/shared/LogoBar/Component.tsx to render the block:

// src/branches/blocks/shared/LogoBar/Component.tsx import React from 'react' import Image from 'next/image' import { LogoBarBlock, getLogoImageUrl } from '@/types/blocks' import { cn } from '@/lib/utils' interface LogoBarProps { block: LogoBarBlock } export const LogoBar: React.FC<LogoBarProps> = ({ block }) => { const { title, logos, autoScroll, variant = 'light' } = block // Background variant classes const backgroundClass = { light: 'bg-grey-light', white: 'bg-white', dark: 'bg-gradient-to-br from-navy to-navy-light', }[variant] const titleColorClass = variant === 'dark' ? 'text-white/40' : 'text-grey-mid' const logoCardClass = variant === 'dark' ? 'bg-white/8 border border-white/10' : 'bg-white border border-grey' // Duplicate logos for infinite scroll effect if autoScroll enabled const displayLogos = autoScroll ? [...logos, ...logos] : logos return ( <section className={cn('py-12 md:py-16', backgroundClass)}> <div className="container"> {title && ( <p className={cn( 'text-center text-xs font-bold uppercase tracking-wider mb-6', titleColorClass )}> {title} </p> )} <div className={cn( 'flex items-center gap-8 max-w-5xl mx-auto overflow-x-auto py-2', autoScroll ? 'justify-start animate-scroll-horizontal' : 'justify-between' )} style={autoScroll ? { animation: 'scroll-logos 20s linear infinite', } : {}} > {displayLogos.map((logo, index) => { const imageUrl = getLogoImageUrl(logo.image) const LogoWrapper = logo.link ? 'a' : 'div' return ( <LogoWrapper key={`${logo.name}-${index}`} href={logo.link || undefined} target={logo.link ? '_blank' : undefined} rel={logo.link ? 'noopener noreferrer' : undefined} className={cn( 'flex items-center justify-center rounded-lg transition-all duration-200', 'grayscale opacity-50 hover:grayscale-0 hover:opacity-100 hover:scale-105', 'w-[120px] min-w-[120px] h-[60px] flex-shrink-0', logoCardClass, logo.link && 'cursor-pointer' )} > {imageUrl ? ( <Image src={imageUrl} alt={logo.name} width={120} height={60} className="max-w-[90%] max-h-[90%] object-contain" /> ) : ( <span className="font-bold text-sm text-navy"> {logo.name} </span> )} </LogoWrapper> ) })} </div> </div> </section> ) }

CSS Animation for Auto-Scroll:

Add to your global CSS or Tailwind config:

// tailwind.config.ts export default { theme: { extend: { keyframes: { 'scroll-logos': { '0%': { transform: 'translateX(0)' }, '100%': { transform: 'translateX(-50%)' }, }, }, animation: { 'scroll-horizontal': 'scroll-logos 20s linear infinite', }, }, }, } // Or in globals.css: @keyframes scroll-logos { 0% { transform: translateX(0); } 100% { transform: translateX(-50%); } } .animate-scroll-horizontal { animation: scroll-logos 20s linear infinite; } .animate-scroll-horizontal:hover { animation-play-state: paused; }

Step 5: Testing Checklist
1. Block appears in page builder: Navigate to Pages → Edit → Blocks → Add Block → "Logo Bar" is visible
2. Logo upload works: Click "Add Logo" → Upload image → verify image appears in preview
3. Validation works: Try adding <4 logos (should show error), try invalid URL (should show error)
4. Grayscale hover effect: On frontend, hover over logos → verify they colorize and scale
5. Auto-scroll animation: Enable auto-scroll checkbox → verify logos scroll horizontally, pause on hover
6. Background variants: Test all 3 variants (light/white/dark) → verify correct backgrounds and logo card styles
7. Mobile responsive: Resize browser to mobile width → verify horizontal scroll works, logos maintain size
8. Links work: Add link to logo → click on frontend → verify opens in new tab with proper rel attributes
9. TypeScript compilation: Run npm run build → no TypeScript errors

Step 7: Common Issues & Solutions
Issue Solution
Logo images not displaying Check getLogoImageUrl() helper returns correct URL. Verify Media collection configured correctly. Check image uploaded to /media folder.
Grayscale filter not working Ensure Tailwind CSS has filter and grayscale utilities enabled. Check browser support for CSS filters (IE11 not supported).
Auto-scroll animation not smooth Verify keyframes in tailwind.config.ts. Check logos are duplicated (doubled) when autoScroll is true. Ensure animation-play-state: paused on hover.
Mobile horizontal scroll not working Add overflow-x: auto to container. Ensure logos have flex-shrink: 0 and fixed width. Test on real mobile device, not just browser resize.
Links not opening in new tab Check target="_blank" and rel="noopener noreferrer" are set on <a> tag. Verify link field has valid URL.
Database error: logos array not saving Run migration: npx payload migrate. Check pages_blocks_logos table exists with correct foreign key relationship to pages_blocks.
Variant styles not applying Check Tailwind classes exist: bg-grey-light, bg-navy, from-navy, to-navy-light. Verify backgroundClass mapping in Component.tsx.
TypeScript error: Media type not found Import Media from @/payload-types (auto-generated). Run npx payload generate:types to regenerate types after schema changes.

Best Practices & Usage Tips
Logo Image Guidelines:
  • Recommended size: 240x120px (2:1 aspect ratio)
  • Format: PNG or SVG with transparent background
  • File size: Keep under 50KB for fast loading
  • Color: Upload color versions — grayscale filter applied automatically
When to Use Auto-Scroll:
  • 8+ logos: Creates continuous infinite scroll effect
  • Space constraints: Fits many logos in limited horizontal space
  • Trade shows/events: Eye-catching movement draws attention
  • Don't use: For 3-6 logos (static display is cleaner)
Variant Selection Guide:
  • Light: Default choice, works with most page layouts
  • White: Clean minimal look, good after content blocks
  • Dark: High contrast, use before/after hero sections
Accessibility Considerations:
  • Always fill in "Company Name" field for proper alt text
  • Auto-scroll pauses on hover for keyboard navigation
  • Logo links open in new tab with rel="noopener noreferrer"
  • Consider adding aria-label to section for screen readers
💡 Pro Tip:

For maximum trust-building impact, use 6-8 recognizable logos arranged in order of brand recognition (strongest first/last). Add title like "Trusted by industry leaders" for extra credibility.

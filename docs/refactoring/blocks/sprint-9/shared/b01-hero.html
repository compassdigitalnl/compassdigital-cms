Usage Notes & Best Practices

  • Variant Selection: Use default for full-width impact pages, split for feature announcements with imagery, centered for short conversion-focused CTAs.
  • Title Typography: Always uses DM Serif Display for visual hierarchy. Keep titles under 60 characters for readability.
  • Background Gradients: Default navy gradient includes teal glow pseudo-element (top-right, 200-280px circle) for brand consistency across all hero variants.
  • Button Array: Supports up to 3 buttons. Primary button should be first (most important action). Use ghost style for tertiary actions.
  • Accessibility: Title is always H1 (one per page). Ensure color contrast ratio ≥ 4.5:1 for text on gradient backgrounds.
  • Performance: Background images should be optimized (WebP format, max 1920px wide, ~200KB). Use Payload's image optimization.
  • Mobile Behavior: Split layout automatically stacks at <768px. Reduce font sizes: title 28px, body 14px on mobile.
  • CMS Integration: Rich text description supports all Lexical editor features (bold, italic, links, lists). Sanitize user input.

5 Testing Checklist

Comprehensive testing checklist covering all variants, responsive behavior, and accessibility requirements.

  • Variant Rendering: Verify all 3 variants (default, split, centered) render correctly with proper layout, alignment, and spacing at desktop/tablet/mobile breakpoints.
  • Background Styles: Test gradient (with teal glow pseudo-element), image (with overlay), and solid color backgrounds. Confirm text contrast meets WCAG 4.5:1 ratio.
  • Button Array: Add 1, 2, and 3 buttons with different styles (primary, secondary, ghost). Verify proper gap spacing, hover states, and click behavior.
  • Rich Text Description: Test Lexical editor features (bold, italic, links, lists, paragraphs). Confirm serializeLexical outputs valid React components.
  • Mobile Responsiveness: Verify split layout stacks vertically on mobile. Check font sizes reduce correctly (title: 28px, body: 14px). Test button wrapping with flex-wrap.
  • Image Optimization: Confirm Next.js Image component loads background images with priority flag. Test WebP format support, lazy loading disabled for hero.
  • Accessibility: Verify title uses H1 tag (only one per page). Test keyboard navigation for buttons. Check screen reader announces subtitle as supporting text.

7 Troubleshooting Guide
Background image not displaying
Cause: Next.js Image component requires width/height or fill prop, or image URL is incorrect.
Fix: Verify backgroundImage.url exists and is absolute (https://...). Ensure Image component has fill prop with object-cover class. Check next.config.js has domain whitelisted in images.domains array.
Gradient glow effect not visible
Cause: Pseudo-element positioned outside viewport or z-index too low.
Fix: Confirm parent section has position: relative and overflow: hidden. Verify glow div has position: absolute with negative top/right values (-40px). Check radial-gradient uses rgba(0,137,123,0.12) for correct opacity.
Rich text description renders as [object Object]
Cause: Lexical JSON passed directly to React without serialization.
Fix: Import serializeLexical utility from @/utilities/serializeLexical. Call serializeLexical({ nodes: description }) to convert Lexical JSON to React components. Ensure description is not null before passing.
Buttons overlapping on mobile
Cause: Button container missing flex-wrap or gap too large.
Fix: Add flex-wrap class to button container div. Reduce gap from gap-4 to gap-3 on mobile. Test with 3 long button labels ("Start Free Trial Today"). Consider stacking vertically (flex-col) on screens < 480px.
Split layout not stacking on mobile
Cause: Grid columns not responsive or md: breakpoint not applying.
Fix: Verify parent div has grid md:grid-cols-2 classes. Check Tailwind config includes all breakpoints. Test at exact 768px width. Ensure no min-width CSS overriding grid behavior.
Text contrast too low on gradient background
Cause: Navy gradient too light or text opacity too high.
Fix: Darken gradient: from-[#0A1628] to-[#06101B] (darker endpoint). Reduce description opacity from 0.8 to 0.7. Add text-shadow: 0 2px 4px rgba(0,0,0,0.3) for better readability. Test with WebAIM contrast checker.
Subtitle not showing teal color
Cause: Parent text-white class overriding subtitle color, or Tailwind purging class.
Fix: Use !important: text-[#26A69A]! or override with inline style. Ensure text-[#26A69A] is whitelisted in tailwind.config.js safelist. Check subtitle element not inheriting color: currentColor.
Hero block not taking full width
Cause: Parent container has max-width constraint or padding issues.
Fix: Add w-full class to section element. Remove any max-w-* classes on hero container. Check parent page layout doesn't have restrictive container. For full-bleed hero, use negative margins: -mx-6 on section with px-6 on inner content.

Usage Guidelines
  • Title: Always H1, DM Serif Display font, 36-48px, keep under 60 characters
  • Subtitle: 11-12px, uppercase, teal color (#26A69A), optional overline
  • Description: Rich text (Lexical), 14-15px, opacity 70-80% on dark backgrounds
  • Buttons: Max 3 buttons, primary first (most important action), 12px gap
  • Variants: Default (centered), Split (2-col on desktop), Centered (compact CTA)
  • Background: Gradient (navy + teal glow), Image (with overlay), Solid color
  • Mobile: Stack split layout, reduce title to 28px, body to 14px, maintain contrast

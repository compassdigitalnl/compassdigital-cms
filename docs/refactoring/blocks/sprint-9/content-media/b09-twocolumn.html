B9 Two Column Block
Neutral

Two text columns with rich text support. Flexible split ratios: 50/50, 60/40, or 40/60. Both columns support full Lexical editor features.

Main Content (60%)

The 40/60 split reverses the emphasis, putting the wider column on the right. This is useful when you want the main content to flow on the right side while keeping contextual information on the left.

Common use cases include:

  • Documentation with left-side table of contents
  • Product pages with specifications sidebar
  • Case studies with metadata panel
  • Tutorials with navigation steps

Step 2: TypeScript Type Definitions

Create src/types/blocks.ts and add the TwoColumnBlock interface:

// src/types/blocks.ts import { SerializedEditorState } from '@payloadcms/richtext-lexical/lexical' export interface TwoColumnBlock { blockType: 'twocolumn' id?: string split: '50-50' | '60-40' | '40-60' columnOne: SerializedEditorState columnTwo: SerializedEditorState } // Type guard to check if a block is a TwoColumnBlock export function isTwoColumnBlock(block: any): block is TwoColumnBlock { return block?.blockType === 'twocolumn' } // Helper type for split ratio mapping export type ColumnSplit = TwoColumnBlock['split']
Step 3: React Component Implementation

Create src/branches/blocks/shared/TwoColumn/Component.tsx to render the block:

// src/branches/blocks/shared/TwoColumn/Component.tsx import React from 'react' import { RichText } from '@payloadcms/richtext-lexical/react' import { TwoColumnBlock } from '@/types/blocks' import { cn } from '@/lib/utils' interface TwoColumnProps { block: TwoColumnBlock } export const TwoColumn: React.FC<TwoColumnProps> = ({ block }) => { const { split, columnOne, columnTwo } = block // Map split ratio to Tailwind grid classes const gridClass = { '50-50': 'grid-cols-1 md:grid-cols-2', '60-40': 'grid-cols-1 md:grid-cols-[60fr_40fr]', '40-60': 'grid-cols-1 md:grid-cols-[40fr_60fr]', }[split] return ( <section className="py-12 md:py-16"> <div className="container"> <div className={cn('grid gap-8 md:gap-12', gridClass)}> {/* Column One */} <div className="prose prose-slate max-w-none prose-headings:font-display prose-headings:text-navy prose-a:text-teal prose-a:no-underline hover:prose-a:underline"> <RichText data={columnOne} /> </div> {/* Column Two */} <div className="prose prose-slate max-w-none prose-headings:font-display prose-headings:text-navy prose-a:text-teal prose-a:no-underline hover:prose-a:underline"> <RichText data={columnTwo} /> </div> </div> </div> </section> ) }

Alternative: Custom Column Wrapper

If you need more control over individual column styling, you can wrap each column in a custom div with specific classes:

// Alternative implementation with custom column wrappers export const TwoColumn: React.FC<TwoColumnProps> = ({ block }) => { const { split, columnOne, columnTwo } = block const gridClass = { '50-50': 'grid-cols-1 md:grid-cols-2', '60-40': 'grid-cols-1 md:grid-cols-[60fr_40fr]', '40-60': 'grid-cols-1 md:grid-cols-[40fr_60fr]', }[split] // Different styling for sidebar columns (40% width) const isSidebarLeft = split === '40-60' const isSidebarRight = split === '60-40' return ( <section className="py-12 md:py-16"> <div className="container"> <div className={cn('grid gap-8 md:gap-12', gridClass)}> {/* Column One with conditional sidebar styling */} <div className={cn( "prose prose-slate max-w-none", isSidebarLeft && "prose-sm", // Smaller text for sidebar )}> <RichText data={columnOne} /> </div> {/* Column Two with conditional sidebar styling */} <div className={cn( "prose prose-slate max-w-none", isSidebarRight && "prose-sm", // Smaller text for sidebar )}> <RichText data={columnTwo} /> </div> </div> </div> </section> ) }

Step 5: Testing Checklist
1. Block appears in page builder: Navigate to Pages → Edit → Blocks → Add Block → "Two Column Block" is visible
2. All split options work: Test 50/50, 60/40, and 40/60 splits → verify grid classes applied correctly
3. Lexical editors load: Both column editors initialize with toolbar features
4. Rich text features work: Test headings, lists, links, formatting in both columns
5. Responsive stacking: On mobile (<768px), columns stack vertically with proper gap
6. Content saves and persists: Save page → reload admin → both columns retain content
7. TypeScript compilation: Run npm run build → no TypeScript errors
Step 6: Database Migration

⚠️ IMPORTANT: After creating the Two Column block, generate a database migration:

# Generate migration for TwoColumn block npx payload migrate:create add_twocolumn_block # Review generated migration file in src/migrations/ # It should contain ALTER TABLE statements for pages_blocks table # Adding columns for split, columnOne, and columnTwo # Expected schema changes: # - pages_blocks.split (type: varchar) — enum: '50-50', '60-40', '40-60' # - pages_blocks.column_one (type: jsonb) — stores serialized Lexical state # - pages_blocks.column_two (type: jsonb) — stores serialized Lexical state # Apply migration to database npx payload migrate # Verify in database (PostgreSQL example) psql -d your_database -c "SELECT column_name, data_type FROM information_schema.columns WHERE table_name = 'pages_blocks' AND column_name IN ('split', 'column_one', 'column_two');" # Expected output: # column_name | data_type # --------------+----------- # split | varchar # column_one | jsonb # column_two | jsonb
Step 7: Common Issues & Solutions
Issue Solution
Grid columns not splitting correctly Check Tailwind config: ensure @tailwindcss/typography plugin is installed and grid-cols-[60fr_40fr] syntax is supported (Tailwind 3.0+)
Columns stack on desktop instead of side-by-side Verify md: breakpoint prefix is present: md:grid-cols-2. Check viewport width is >768px.
Lexical editors not loading in admin Check imports: @payloadcms/richtext-lexical installed? Run npm install @payloadcms/richtext-lexical
Content not rendering on frontend Verify <RichText data={columnOne} /> component imported from @payloadcms/richtext-lexical/react (not the config package)
Split ratio not applying Check grid class mapping logic: ensure all 3 split values ('50-50', '60-40', '40-60') are handled in the switch/object mapping
Prose styles not working Install @tailwindcss/typography: npm install -D @tailwindcss/typography and add to plugins: [require('@tailwindcss/typography')]
Database error on save Run migration: npx payload migrate → check pages_blocks table has split, column_one, column_two columns
TypeScript error: SerializedEditorState Import from correct path: import { SerializedEditorState } from '@payloadcms/richtext-lexical/lexical'
Gap between columns too large/small Adjust gap in grid className: gap-8 md:gap-12 → change to gap-6 md:gap-10 or desired spacing
Step 8: Register Block in Payload Config

Add the Two Column block to your Pages collection's blocks field:

// src/collections/Pages.ts import { CollectionConfig } from 'payload/types' import { TwoColumn } from '@/branches/blocks/shared/TwoColumn/config' // Import other blocks... export const Pages: CollectionConfig = { slug: 'pages', fields: [ { name: 'title', type: 'text', required: true, }, { name: 'slug', type: 'text', required: true, unique: true, }, { name: 'blocks', type: 'blocks', label: 'Page Blocks', blocks: [ TwoColumn, // ← Add TwoColumn block here // Hero, // Features, // Content, // etc... ], }, ], }

Next Step: Add the block component to your page renderer:

// app/(frontend)/[...slug]/page.tsx import { TwoColumn } from '@/branches/blocks/shared/TwoColumn/Component' export default function Page({ page }) { return ( <main> {page.blocks?.map((block, index) => { switch (block.blockType) { case 'twocolumn': return <TwoColumn key={index} block={block} /> // Other blocks... case 'hero': return <Hero key={index} block={block} /> case 'content': return <Content key={index} block={block} /> default: return null } })} </main> ) }

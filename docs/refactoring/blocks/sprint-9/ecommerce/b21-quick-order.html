Step 2: TypeScript Type Definitions

Create type-safe interfaces for the QuickOrder block:

// src/branches/blocks/ecommerce/cart/QuickOrder/types.ts export interface QuickOrderBlock { blockType: 'quickorder' subtitle?: string title: string description?: string defaultRows: number maxRows: number minAutocompleteChars: number allowCsvUpload: boolean uploadHelpText?: string maxCsvFileSize?: number showInfoBox: boolean infoTitle?: string infoIcon?: string infoText?: string } export interface QuickOrderRow { sku: string productName?: string quantity: number price?: number } export interface QuickOrderProps { block: QuickOrderBlock }
Step 3: Usage in Next.js Components

React component implementation met SKU autocomplete en CSV parsing:

// src/branches/blocks/ecommerce/cart/QuickOrder/Component.tsx 'use client' import { useState } from 'react' import { QuickOrderProps, QuickOrderRow } from './types' export function QuickOrder({ block }: QuickOrderProps) { const [rows, setRows] = useState<QuickOrderRow[]>( Array(block.defaultRows).fill({ sku: '', quantity: 1 }) ) // SKU Autocomplete handler const handleSkuChange = async (index: number, sku: string) => { if (sku.length >= block.minAutocompleteChars) { // Fetch product suggestions from API const suggestions = await fetch(`/api/products/search?sku=${sku}`) // Show autocomplete dropdown } } // CSV Upload handler const handleCsvUpload = (file: File) => { const reader = new FileReader() reader.onload = (e) => { const text = e.target.result const lines = text.split('\n') const parsed = lines.map(line => { const [sku, qty] = line.split(',') return { sku: sku.trim(), quantity: parseInt(qty) || 1 } }) setRows(parsed) } reader.readAsText(file) } // Add to cart handler const handleSubmit = async () => { const filledRows = rows.filter(r => r.sku) await fetch('/api/cart/bulk-add', { method: 'POST', body: JSON.stringify({ items: filledRows }), }) } return ( <section className="quick-order-block"> <div className="container"> {block.subtitle && ( <p className="subtitle">{block.subtitle}</p> )} <h2 className="title">{block.title}</h2> {block.description && ( <p className="description">{block.description}</p> )} {block.allowCsvUpload && ( <div className="csv-upload"> <input type="file" accept=".csv" onChange={(e) => handleCsvUpload(e.target.files[0])} /> <p>{block.uploadHelpText}</p> </div> )} <table className="quick-order-table"> {/* Table rows implementation */} </table> <button onClick={handleSubmit} className="submit-btn"> Toevoegen aan winkelwagen </button> {block.showInfoBox && ( <div className="info-box"> <h4>{block.infoIcon} {block.infoTitle}</h4> <p>{block.infoText}</p> </div> )} </div> </section> ) }
Step 4: Admin Panel Structure

When you add this block to a Page's layout field, you will see:

📦 Quick Order Block
▸ Content Fields
- subtitle: text → "B2B Snelbestellen"
- title: text (required) → "Bulk Order"
- description: textarea → instructies
▸ Table Configuration (row met 3 velden)
- Default Aantal Rijen: number (1-20) → default: 5
- Maximum Rijen: number (5-100) → default: 50
- Min Chars Autocomplete: number (1-5) → default: 2
▸ CSV Upload Settings
- ☑ Enable CSV Upload → default: true
- Als enabled: uploadHelpText (textarea)
- Als enabled: maxCsvFileSize (number 1-10 MB)
▸ Info Box (conditional)
- ☑ Show Info/Tips Box
- Als checked: infoTitle + infoIcon (row)
- Als checked: infoText (textarea)
Step 5: Testing Checklist
1. Block beschikbaar: Navigate to Pages → Edit → Add block → "Quick Order" verschijnt in lijst
2. Table rows rendering: Default 5 lege rijen zichtbaar, "Rij toevoegen" knop werkt
3. SKU autocomplete: Type 2+ chars in SKU field → autocomplete dropdown verschijnt met suggesties
4. CSV upload: Upload valid CSV → rows vullen automatisch met SKU + quantity
5. Quantity steppers: + en − buttons werken, manual input accepteert alleen numbers
6. Order summary: Real-time update van totaal aantal producten, stuks, en geschatte waarde
7. Add to cart: Submit button disabled als geen filled rows → na submit producten in cart

Step 7: Common Issues & Solutions
Issue Solution
Block niet zichtbaar in admin Check payload.config.ts → blocks array in Pages collection → moet QuickOrder bevatten
Autocomplete werkt niet Check API endpoint /api/products/search bestaat → return format: [{ sku, name, price }]
CSV upload faalt Check file size limit → max maxCsvFileSize MB → verify CSV format (SKU,Quantity per line)
Add to cart niet werkend Check /api/cart/bulk-add endpoint → accepts array of { sku, quantity } objects
Conditional fields niet zichtbaar Check admin.condition → enable checkboxes in admin → save → conditional fields verschijnen
TypeScript errors op row state Regenerate types: npx payload generate:types → import QuickOrderRow type
Table rows maximum niet werkend Check frontend validation → rows.length >= block.maxRows → disable "Rij toevoegen" button

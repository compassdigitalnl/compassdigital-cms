Usage Guidelines
  • Image: 180px height, gradient placeholder or actual blog thumbnail
  • Category Badge: Top-left overlay with dark backdrop blur
  • Title: DM Serif Display, 18px, navy color
  • Excerpt: 13px, grey-dark, max 2-3 lines with ellipsis
  • Meta: Border-top separator, date + read-more link
  • Grid: 3 columns → 2 @900px → 1 @640px

Step 2: TypeScript Type Definitions

Create src/types/blocks.ts for type safety:

// src/types/blocks.ts import { Post } from './payload-types' export interface BlogPreviewBlock { blockType: 'blogpreview' id?: string title?: string description?: string columns: '2' | '3' posts: string[] | Post[] // Relationship can be populated or not showExcerpt: boolean showReadTime: boolean showCategory: boolean } // Type guard to check if a block is a BlogPreviewBlock export function isBlogPreviewBlock(block: any): block is BlogPreviewBlock { return block?.blockType === 'blogpreview' } // Helper to check if posts are populated export function arePostsPopulated( posts: string[] | Post[] ): posts is Post[] { return posts.length > 0 && typeof posts[0] === 'object' } // Calculate estimated read time from content export function calculateReadTime(content: string): number { const wordsPerMinute = 200 const wordCount = content.split(/\s+/).length return Math.ceil(wordCount / wordsPerMinute) }

Step 4: Admin Panel Structure

When editors navigate to /admin/pages and add a BlogPreview block, they will see:

Field Type Default Description
title Text (60% width) — Optional section heading
columns Select (40% width) "3" 2 or 3 columns layout
description Textarea — Optional subtitle below heading
posts Relationship — Select 2-6 blog posts (required)
showExcerpt Checkbox true Show post excerpt below title
showReadTime Checkbox false Show estimated reading time
showCategory Checkbox true Show category badge overlay

Admin UI Flow:

  1. Editor clicks "+ Add Block" in Page Builder
  2. Selects "Blog Preview Block" from dropdown
  3. Fills optional title/description
  4. Selects 2-6 published posts via relationship picker
  5. Configures display options (checkboxes)
  6. Saves and publishes page

Step 6: Database Migration

BlogPreview block uses relationship fields which require database tables. Generate migration:

# Generate migration for BlogPreview block npx payload migrate:create add_blogpreview_block # Expected migration contents: # - Creates pages_blocks_blogpreview table # - Columns: id, title, description, columns, showExcerpt, showReadTime, showCategory # - Creates pages_blocks_blogpreview_posts_rels table for relationship # - Foreign keys: pages_blocks_blogpreview.id, posts.id # Verify migration file cat src/migrations/XXXXXX_add_blogpreview_block.ts # Expected SQL operations: # CREATE TABLE pages_blocks_blogpreview ( # id SERIAL PRIMARY KEY, # _order INTEGER, # _parent_id INTEGER REFERENCES pages(id), # title VARCHAR(255), # description TEXT, # columns VARCHAR(10) DEFAULT '3', # show_excerpt BOOLEAN DEFAULT true, # show_read_time BOOLEAN DEFAULT false, # show_category BOOLEAN DEFAULT true # ); # Run migration npx payload migrate # Verify tables exist npx payload migrate:status

🚨 CRITICAL: Migration Required!

Without running migrations, the BlogPreview block will:

  • ❌ Work in development (SQLite auto-migrates)
  • ❌ FAIL in production (PostgreSQL requires explicit migrations)
  • ❌ Cause "relation does not exist" database errors
  • ❌ Block users from saving pages with this block

Always run migrations before deploying! See DATABASE_MIGRATION_GUIDE.md for details.

Step 7: Troubleshooting Guide

Common issues and solutions:

Issue Cause Solution
Block not visible in admin Block not registered in payload.config.ts Add BlogPreview to blocks array in Pages collection (see Step 8)
Posts dropdown empty No published posts or wrong relationTo value 1) Publish at least 2 posts in /admin/posts
2) Verify relationTo: 'posts' matches collection slug
Posts show as IDs, not objects Relationship not populated in query Add depth: 1 or higher to page query:
const page = await payload.findByID({ collection: 'pages', id, depth: 1 })
Thumbnails don't display Missing upload field in Posts or image URLs incorrect 1) Verify Posts collection has thumbnail upload field
2) Check Next.js image domains in next.config.js
TypeScript errors on Post type Payload types not generated or outdated Run npx payload generate:types to regenerate payload-types.ts
Read time shows 0 or NaN Post content field is empty or wrong type 1) Ensure posts have content
2) Verify calculateReadTime receives string, not richText object
Grid doesn't collapse on mobile Tailwind responsive classes not applied Check gridClass mapping includes mobile-first classes (e.g., grid-cols-1 md:grid-cols-3)
Database error: "relation does not exist" Migration not run on production database Run npx payload migrate on production server (see Step 6)

Debugging Tips:

  • Check Payload admin console for validation errors when saving
  • Inspect browser DevTools Network tab for failed API requests
  • Use console.log(block) in Component to verify data structure
  • Test with minimum 2 posts (minRows validation)
  • Verify Posts collection exists and has required fields (title, slug, excerpt, publishedAt, thumbnail, category)

Register the BlogPreview block in your Payload config and Pages collection:

// src/payload.config.ts import { buildConfig } from 'payload/config' import { BlogPreview } from './branches/blocks/shared/BlogPreview/config' // ... other imports export default buildConfig({ collections: [ { slug: 'pages', fields: [ { name: 'blocks', type: 'blocks', blocks: [ BlogPreview, // ← Add here // ... other blocks (Hero, Features, CTA, etc.) ], }, // ... other page fields ], }, // ... other collections ], })
// app/[...slug]/page.tsx — Render blocks on frontend import { BlogPreview } from '@/branches/blocks/shared/BlogPreview/Component' import { isBlogPreviewBlock } from '@/types/blocks' export default async function Page({ params }) { const page = await payload.findByID({ collection: 'pages', id: params.slug, depth: 2, // ← Populate posts relationship! }) return ( <div> {page.blocks?.map((block) => { if (isBlogPreviewBlock(block)) { return <BlogPreview key={block.id} block={block} /> } // ... other block types })} </div> ) }

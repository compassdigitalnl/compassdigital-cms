Neutral

Step 2: TypeScript Type Definitions

Create src/types/blocks.ts for type safety:

// src/types/blocks.ts export interface SpacerBlock { blockType: 'spacer' id?: string size: 'sm' | 'md' | 'lg' | 'xl' showDivider?: boolean } // Type guard to check if a block is a SpacerBlock export function isSpacerBlock(block: any): block is SpacerBlock { return block?.blockType === 'spacer' }
Step 3: React Component Implementation

Create src/branches/blocks/shared/Spacer/Component.tsx to render the block:

// src/branches/blocks/shared/Spacer/Component.tsx import React from 'react' import { SpacerBlock } from '@/types/blocks' import { cn } from '@/lib/utils' interface SpacerProps { block: SpacerBlock } export const Spacer: React.FC<SpacerProps> = ({ block }) => { const { size, showDivider } = block // Map size to Tailwind height classes // Desktop: sm=24px, md=48px, lg=80px, xl=120px // Mobile: sm=24px, md=48px, lg=64px, xl=80px (reduced) const heightClass = { sm: 'h-6', // 24px (no mobile change) md: 'h-12', // 48px (no mobile change) lg: 'h-20 md:h-[80px]', // 80px desktop, 64px mobile xl: 'h-20 md:h-[120px]',// 120px desktop, 80px mobile }[size] return ( <div className={cn('relative', heightClass)} aria-hidden="true"> {showDivider && ( <div className="absolute inset-0 flex items-center"> <div className="w-full border-t border-grey" /> </div> )} </div> ) }

Alternative: Custom CSS Classes

If you prefer more explicit control over spacing, create custom CSS classes:

// src/app/globals.css (or your main CSS file) /* Spacer Block Sizes */ .spacer-sm { height: 24px; } .spacer-md { height: 48px; } .spacer-lg { height: 80px; } .spacer-xl { height: 120px; } /* Mobile responsiveness */ @media (max-width: 768px) { .spacer-lg { height: 64px; } .spacer-xl { height: 80px; } } /* Divider line */ .spacer-divider { position: relative; display: flex; align-items: center; } .spacer-divider::before { content: ''; position: absolute; left: 0; right: 0; top: 50%; height: 1px; background: var(--grey); }

Step 5: Usage Guidelines & Best Practices
Size When to Use Example
sm Minimal spacing between tightly related sections Between a paragraph and an infobox within the same content block
md Standard spacing between distinct content blocks (default) Between a testimonials section and a features section
lg Large gap between major page sections Between main content and footer CTA section
xl Extra-large gap for dramatic separation Between hero section and first content section (landing pages)
💡
Pro Tips
  • Default to md: Medium spacing works for 80% of use cases — start here
  • Use dividers sparingly: Only when you need visual separation (e.g., between chapters)
  • Mobile-first: Large spacers automatically reduce on mobile to prevent excessive whitespace
  • Alternative to margin: Spacer blocks are more flexible than component margins in page builders
  • Accessibility: Spacers use aria-hidden="true" to hide from screen readers
Step 6: Testing Checklist
1. Block appears in page builder: Navigate to Pages → Edit → Blocks → Add Block → "Spacer Block" is visible
2. All 4 sizes render correctly: Test sm/md/lg/xl → verify heights match expected values (24/48/80/120px)
3. Mobile responsiveness works: Resize browser → lg reduces to 64px, xl reduces to 80px on mobile
4. Divider line toggles: Enable "Show Divider Line" → horizontal line appears centered in spacer
5. Between different block types: Test spacer between Hero → Content → CTA → correct spacing applied
6. Default value works: Add new Spacer block → defaults to "md" size, no divider
7. Visual consistency: Spacer heights feel proportional on both desktop and mobile devices

Step 8: Common Issues & Solutions
Issue Solution
Spacer height not applied Check Tailwind classes → ensure height classes (h-6, h-12, etc.) are correct and included in safelist if using dynamic classes
Mobile heights not reducing Verify responsive classes → h-20 md:h-[80px] pattern ensures mobile-first approach with desktop override
Divider line not showing Check conditional rendering → showDivider && (...) → ensure boolean field saves correctly in database
Spacing inconsistent across pages Ensure all spacers use the same size system → avoid mixing Tailwind classes with custom pixel values
Database error on save Run migration: npx payload migrate → verify pages_blocks table has size and show_divider columns
TypeScript error on size prop Check type definition → ensure size is typed as union 'sm' | 'md' | 'lg' | 'xl' not string
Divider line too thick/thin Adjust CSS → border-t uses 1px by default, can customize with border-t-2 or custom CSS
Spacer collapsing (no height) Ensure explicit height classes → avoid relying on content height, spacers must have fixed height values

// src/collections/Pages.ts import { CollectionConfig } from 'payload/types' import { Spacer } from '@/branches/blocks/shared/Spacer/config' // Import other blocks... export const Pages: CollectionConfig = { slug: 'pages', fields: [ { name: 'title', type: 'text', required: true, }, { name: 'slug', type: 'text', required: true, unique: true, }, { name: 'blocks', type: 'blocks', label: 'Page Blocks', blocks: [ Spacer, // ← Add Spacer block here // Hero, // Content, // CTA, // etc... ], }, ], }

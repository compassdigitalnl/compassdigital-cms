7 Troubleshooting Guide
Featured card not scaling correctly
Cause: scale-105 class conflicts with grid layout or other transforms.
Fix: Ensure parent grid has items-stretch removed if using scale. Add transform-gpu class for better performance. Check that no parent element has overflow: hidden which clips the scaled card.
Cannot add more than 4 plans in admin
Cause: maxRows: 4 constraint in Payload config, plus database trigger.
Fix: This is intentional. Design supports 2-4 plans max. If more plans needed, consider splitting into multiple Pricing blocks or redesigning to use a comparison table format instead.
Feature checkmarks not displaying
Cause: Unicode ✓ character not rendering in font, or CSS ::before not applying.
Fix: Ensure font supports Unicode checkmark (U+2713). Alternatively, replace with inline SVG icon or use a custom icon component. Verify feature.included boolean is correctly passed to component.
Button gradient not showing
Cause: Tailwind gradient classes not included in build, or buttonVariant not 'gradient'.
Fix: Check buttonVariant === 'gradient' in plan data. Ensure Tailwind config includes gradient utilities. Verify bg-gradient-to-r from-[#00897B] to-[#26A69A] classes are not purged. Add to safelist if needed.
Grid not responsive on tablet
Cause: Tailwind breakpoint md: not triggering at correct viewport width.
Fix: Verify Tailwind md breakpoint is 768px (default). Check browser DevTools to see which classes apply at tablet width. For layout: 'auto', grid should show 2 columns at md breakpoint. Ensure no CSS override conflicts.
Badge text overflowing on small cards
Cause: Long badge text (e.g., "Most Popular Choice") too wide for card.
Fix: Limit badge text to 12-15 characters max. Add text-ellipsis and max-w-full to badge span. Or use nowrap with smaller font size (9px) and adjust padding. Consider vertical badge placement on narrow cards.
Hover animation laggy on mobile
Cause: CSS transforms trigger repaints on low-end devices, enableHoverEffect also affects mobile.
Fix: Disable hover effects on touch devices using @media (hover: hover) CSS query. Add will-change-transform only on :hover, not default state. Reduce animation complexity: use translateY only, no scale/shadow on mobile.
Multiple featured plans showing
Cause: Admin allows multiple plans to have featured: true checked.
Fix: Add admin validation to ensure only one plan can be featured. Implement custom validate function in Payload config that checks sibling rows. Or add database constraint: CREATE UNIQUE INDEX ON pricing_plans (pricing_block_id) WHERE featured = true. UI should auto-uncheck other plans when one is featured.
Pricing text not inverting on dark background
Cause: isDark prop not correctly passed from server to client component.
Fix: In Component.tsx, ensure isDark = backgroundColor === 'navy' calculates correctly. Pass isDark as prop to PricingClient. In client component, verify conditional classes use isDark ? 'text-white' : 'text-[#0A1628]' pattern. Check no hardcoded text colors override dynamic classes.

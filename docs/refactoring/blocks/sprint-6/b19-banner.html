5 Testing Checklist
  • Variant Colors: Test all 3 variants (announcement/promo/warning). Verify gradient backgrounds, text color contrast (WCAG AA minimum 4.5:1). Test in dark mode if applicable.
  • Dismissal: Click X button, verify banner disappears. Check localStorage for dismissal key. Reload page - banner should stay hidden. Change dismissalKey in admin - banner should reappear.
  • Date Range: Set showFrom to future date - banner hidden. Set showUntil to past date - banner hidden. Set valid date range - banner visible. Test timezone handling.
  • Sticky Behavior: Enable sticky, scroll page - banner stays at top with z-index 100. Test with existing sticky header - verify z-index stacking. Disable sticky - banner scrolls with page.
  • Links: Test link with internal URL (/page), external URL (https://...), hash link (#section). Test "Open in new tab" - verify target="_blank" and rel="noopener noreferrer". Test without link - verify no empty space.
  • Icons: Test each icon option (bell, gift, alert-triangle, info, zap, star). Verify icon color matches text. Test "None" - no icon shown. Test icon hidden on mobile.
  • Mobile Responsive: Test at 320px, 375px, 414px widths. Verify vertical stack layout. Verify icon hidden, dismiss button absolute positioned. Test long message wrapping. Test link wraps to new line.
  • Accessibility: Test keyboard navigation (Tab to dismiss button, Enter to dismiss). Test screen reader (message announced, dismiss button labeled). Test high contrast mode. Verify focus ring visible on dismiss button.
  • Performance: Test with multiple banners on one page. Verify no layout shift on dismissal. Test localStorage cleanup (old keys removed). Test with slow network - verify no flash of banner before dismissal check.
  • Edge Cases: Test very long message (150+ chars) - verify truncation or scroll. Test message with HTML special chars (<, >, &) - verify proper escaping. Test without message - verify validation error in admin.

CREATE TABLE IF NOT EXISTS banner_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  block_id UUID NOT NULL REFERENCES blocks(id) ON DELETE CASCADE,

  variant VARCHAR(20) NOT NULL DEFAULT 'announcement'
    CHECK (variant IN ('announcement', 'promo', 'warning')),
  message TEXT NOT NULL,
  icon VARCHAR(30) DEFAULT 'bell'
    CHECK (icon IN ('bell', 'gift', 'alert-triangle', 'info', 'zap', 'star', 'none')),

  -- Link fields (nullable group)
  link_text VARCHAR(100),
  link_url TEXT,
  link_new_tab BOOLEAN DEFAULT false,

  -- Behavior fields
  dismissible BOOLEAN DEFAULT true,
  dismissal_key VARCHAR(100),
  sticky BOOLEAN DEFAULT false,
  show_from TIMESTAMPTZ,
  show_until TIMESTAMPTZ,

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),

  -- Constraints
  CONSTRAINT link_complete CHECK (
    (link_text IS NULL AND link_url IS NULL) OR
    (link_text IS NOT NULL AND link_url IS NOT NULL)
  ),
  CONSTRAINT dismissal_key_required CHECK (
    dismissible = false OR dismissal_key IS NOT NULL
  ),
  CONSTRAINT date_range_valid CHECK (
    show_from IS NULL OR show_until IS NULL OR show_from < show_until
  )
);

7 Troubleshooting Guide
Banner doesn't dismiss on click
Cause: localStorage not accessible (private browsing) or dismissalKey missing.
Fix: Add try-catch around localStorage access. Fallback to sessionStorage. Verify dismissalKey is set in admin. Test dismissible checkbox is enabled.
Dismissed banner reappears on refresh
Cause: dismissalKey changed or localStorage cleared.
Fix: Check dismissalKey hasn't changed in admin. Test localStorage persistence across page loads. Verify no code clearing localStorage on init. Use browser DevTools → Application → Local Storage to debug.
Banner not sticky on scroll
Cause: Sticky class not applied or z-index conflict.
Fix: Verify "sticky" checkbox enabled in admin. Check CSS: position: sticky; top: 0; z-index: 100. Test parent container has no overflow: hidden. Inspect with DevTools to verify computed position.
Banner overlaps existing sticky header
Cause: Z-index stacking issue or both elements sticky at top: 0.
Fix: If header should be above: set header z-index: 200, banner z-index: 100. If banner should be above: swap z-indexes. Or: render banner inside header component with relative positioning.
Icon not showing
Cause: Lucide icon not imported or icon name mismatch.
Fix: Verify lucide-react installed: npm list lucide-react. Check icon name matches Lucide docs (kebab-case). Test icon prop not "none". Import all icons: import * as Icons from 'lucide-react'. Fallback: render bell icon if icon not found.
Link not clickable
Cause: Link URL missing or dismiss button overlapping.
Fix: Verify both link.text and link.url are set. Test z-index on link vs dismiss button. Check pointer-events not disabled. Inspect with DevTools to verify <a> element rendered. Test hover state changes cursor to pointer.
Mobile layout broken
Cause: Mobile styles not applying or viewport too narrow.
Fix: Test @media (max-width: 640px) styles apply in DevTools. Verify flex-direction: column on .banner-content. Check dismiss button absolute positioning. Test with actual device, not just DevTools emulation. Verify message wraps (no white-space: nowrap).
Banner causes layout shift on page load
Cause: Banner rendered after initial paint or height not reserved.
Fix: Check dismissal logic runs synchronously before render. Use CSS: min-height or skeleton placeholder. Measure Cumulative Layout Shift (CLS) in Lighthouse. Consider server-side dismissal check if user logged in.
Date range not working
Cause: Date comparison issue or timezone mismatch.
Fix: Verify showFrom/showUntil are ISO strings. Use new Date() for comparison. Test with dates in past, present, future. Check server vs client timezone handling. Log comparison values: console.log(now, showFrom, showUntil). Test with real dates, not "test" strings.
Banner shows when it shouldn't (or vice versa)
Cause: Visibility logic error or useEffect dependency issues.
Fix: Add debug logs in useEffect. Verify all conditions: date range, dismissal, isVisible state. Test each condition independently. Check useEffect dependencies include all used state/props. Verify no race condition between date check and dismissal check.

Usage Guidelines
  • Announcement (Navy): General updates, new features, blog posts, company news. Neutral tone, informational. Example: "New dashboard available! Check out the updates →"
  • Promo (Teal): Sales, discounts, limited offers, free shipping. Positive tone, action-oriented. Example: "Summer Sale! 30% off all items. Shop now →"
  • Warning (Amber): Maintenance notices, service interruptions, important policy changes. Urgent but not alarming. Example: "Scheduled maintenance on Sunday 2-6 AM. Learn more →"
  • Message Length: Keep under 100 characters for single-line display. Max 150 characters before mobile wrapping. Test on 320px viewport. Prioritize clarity over creativity.
  • Dismissal Keys: Use format: "banner-{type}-{date}" (e.g., "banner-promo-2024-03"). Change key to re-show banner to all users. Never reuse keys for different messages.
  • Sticky Banners: Only use for critical messages (security alerts, service outages). Never use for promotions. Test with existing sticky elements (header, chat widget). Verify mobile UX doesn't block content.
  • Date Ranges: Always set showUntil for time-sensitive messages (sales, events). Use showFrom for scheduled announcements (release dates). Test timezone handling for global audiences.
  • Icons: Use icons that reinforce message meaning. Bell = general update, Gift = promotion, Alert Triangle = warning, Zap = urgent/flash, Star = featured/new. Avoid decorative icons.
  • Links: Always include link for more info (landing page, blog post, help doc). Use action-oriented text ("Shop now", "Learn more", "See details"). Test link works before publishing.
  • Multiple Banners: Only show ONE banner at a time. Use priority system if multiple active. Test: promo banners expire, fall back to announcement banner, fall back to no banner.
  • Accessibility: Ensure 4.5:1 contrast ratio (white text on colored backgrounds). Provide descriptive link text. Don't rely solely on color to convey meaning. Test with screen readers.
  • Performance: Banner is client component for dismissal. Consider server-side dismissal check for logged-in users. Minimize localStorage writes. Clean up old dismissal keys periodically.

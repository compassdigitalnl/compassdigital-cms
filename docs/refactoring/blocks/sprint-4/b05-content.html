Primary

Rich text content area powered by Payload Lexical editor. Supports headings, lists, links, images, code blocks. 3 width variants: narrow (640px), wide (900px), full.

Over Compass Digital

Compass Digital is een multi-tenant e-commerce platform dat speciaal is ontworpen voor bedrijven die meerdere webshops willen beheren vanuit één centrale locatie. Met ondersteuning voor 10 verschillende branches en meer dan 156 componenten, biedt Compass Digital een complete oplossing voor moderne e-commerce.

  • Multi-tenant architectuur met complete data isolatie
  • 10 pre-built industry themes (medisch, horeca, bouw, etc.)
  • 156+ production-ready componenten

Step 1: Payload Block Configuration

Create src/branches/blocks/shared/Content/config.ts with the following configuration:

// src/branches/blocks/shared/Content/config.ts import { Block } from 'payload/types' import { lexicalEditor, HeadingFeature, BoldFeature, ItalicFeature, LinkFeature, UnorderedListFeature, OrderedListFeature, BlockquoteFeature, InlineCodeFeature, CodeBlockFeature, ParagraphFeature, UnderlineFeature, StrikethroughFeature, } from '@payloadcms/richtext-lexical' export const Content: Block = { slug: 'content', labels: { singular: 'Content Block', plural: 'Content Blocks', }, fields: [ { name: 'content', type: 'richText', label: 'Content', required: true, editor: lexicalEditor({ features: ({ defaultFeatures }) => [ ParagraphFeature(), HeadingFeature({ enabledHeadingSizes: ['h2', 'h3', 'h4', 'h5', 'h6'], }), BoldFeature(), ItalicFeature(), UnderlineFeature(), StrikethroughFeature(), LinkFeature({ fields: [ { name: 'rel', label: 'Rel Attribute', type: 'select', options: [ { label: 'noopener', value: 'noopener' }, { label: 'noreferrer', value: 'noreferrer' }, { label: 'nofollow', value: 'nofollow' }, ], hasMany: true, }, ], }), UnorderedListFeature(), OrderedListFeature(), BlockquoteFeature(), InlineCodeFeature(), CodeBlockFeature(), ], }), admin: { description: 'Rich text editor with support for headings, lists, links, formatting, and code blocks.', }, }, { name: 'maxWidth', type: 'select', label: 'Max Width', defaultValue: 'narrow', options: [ { label: 'Narrow (640px)', value: 'narrow', }, { label: 'Wide (900px)', value: 'wide', }, { label: 'Full (100%)', value: 'full', }, ], admin: { description: 'Controls the maximum width of the content container. Narrow for blog posts, wide for documentation, full for tables/galleries.', }, }, ], }
Step 2: TypeScript Type Definitions

Create src/types/blocks.ts for type safety:

// src/types/blocks.ts import { SerializedEditorState } from '@payloadcms/richtext-lexical/lexical' export interface ContentBlock { blockType: 'content' id?: string content: SerializedEditorState maxWidth: 'narrow' | 'wide' | 'full' } // Type guard to check if a block is a ContentBlock export function isContentBlock(block: any): block is ContentBlock { return block?.blockType === 'content' }
Step 3: React Component Implementation

Create src/branches/blocks/shared/Content/Component.tsx to render the block:

// src/branches/blocks/shared/Content/Component.tsx import React from 'react' import { RichText } from '@payloadcms/richtext-lexical/react' import { ContentBlock } from '@/types/blocks' import { cn } from '@/lib/utils' interface ContentProps { block: ContentBlock } export const Content: React.FC<ContentProps> = ({ block }) => { const { content, maxWidth } = block // Map maxWidth to Tailwind classes const maxWidthClass = { narrow: 'max-w-2xl', // 640px wide: 'max-w-4xl', // 900px full: 'max-w-full', // 100% }[maxWidth] return ( <section className="py-12 md:py-16"> <div className="container"> <div className={cn('mx-auto', maxWidthClass)}> <div className="prose prose-slate max-w-none prose-headings:font-display prose-headings:text-navy prose-a:text-teal prose-a:no-underline hover:prose-a:underline"> <RichText data={content} /> </div> </div> </div> </section> ) }

Step 5: Testing Checklist
1. Block appears in page builder: Navigate to Pages → Edit → Blocks → Add Block → "Content Block" is visible
2. Rich text editor loads: Click "Add Content Block" → Lexical editor initializes with toolbar
3. All Lexical features work: Test bold, italic, headings (H2-H6), lists (UL/OL), links, blockquotes, code
4. Max width variants render: Test all 3 variants (narrow/wide/full) → verify max-width CSS classes applied
5. Content saves and persists: Save page → reload admin → content still present and editable
6. Preview matches frontend: Click "Preview" → styling matches production (prose, colors, spacing)
7. TypeScript compilation: Run npm run build → no TypeScript errors
Step 6: Database Migration

⚠️ IMPORTANT: After creating the Content block, generate a database migration:

# Generate migration for Content block npx payload migrate:create add_content_block # Review generated migration file in src/migrations/ # It should contain ALTER TABLE statements for pages_blocks table # Adding columns for content (jsonb) and maxWidth (varchar) # Expected schema changes: # - pages_blocks.content (type: jsonb) — stores serialized Lexical state # - pages_blocks.max_width (type: varchar) — enum: 'narrow', 'wide', 'full' # Apply migration to database npx payload migrate # Verify in database (PostgreSQL example) psql -d your_database -c "SELECT column_name, data_type FROM information_schema.columns WHERE table_name = 'pages_blocks' AND column_name IN ('content', 'max_width');" # Expected output: # column_name | data_type # -------------+----------- # content | jsonb # max_width | varchar
Step 7: Common Issues & Solutions
Issue Solution
Lexical editor not loading Check imports: @payloadcms/richtext-lexical installed? Run npm install @payloadcms/richtext-lexical
Content not rendering on frontend Verify <RichText data={content} /> component imported from @payloadcms/richtext-lexical/react
Max width not applied Check className mapping in Component.tsx → ensure Tailwind classes (max-w-2xl, max-w-4xl) are correct
Prose styles not working Install @tailwindcss/typography: npm install -D @tailwindcss/typography and add to plugins: [require('@tailwindcss/typography')]
Database error on save Run migration: npx payload migrate → check pages_blocks table has content and max_width columns
TypeScript error: SerializedEditorState Import from correct path: import { SerializedEditorState } from '@payloadcms/richtext-lexical/lexical'
Toolbar features missing Check features array in lexicalEditor config → ensure all desired features imported and added
Step 8: Register Block in Payload Config

Add the Content block to your Pages collection's blocks field:

// src/collections/Pages.ts import { CollectionConfig } from 'payload/types' import { Content } from '@/branches/blocks/shared/Content/config' // Import other blocks... export const Pages: CollectionConfig = { slug: 'pages', fields: [ { name: 'title', type: 'text', required: true, }, { name: 'slug', type: 'text', required: true, unique: true, }, { name: 'blocks', type: 'blocks', label: 'Page Blocks', blocks: [ Content, // ← Add Content block here // Hero, // Features, // CTA, // etc... ], }, ], }

Next Step: Add the block component to your page renderer:

// app/(frontend)/[...slug]/page.tsx import { Content } from '@/branches/blocks/shared/Content/Component' export default function Page({ page }) { return ( <main> {page.blocks?.map((block, index) => { switch (block.blockType) { case 'content': return <Content key={index} block={block} /> // Other blocks... case 'hero': return <Hero key={index} block={block} /> default: return null } })} </main> ) }

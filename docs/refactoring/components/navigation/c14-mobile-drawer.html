C14-MobileDrawer

Mobile hamburger menu with nested accordion navigation, contact info, and toggles

STEP 1: Component Overview

C14-MobileDrawer is the mobile navigation drawer that appears from the left side when users tap the hamburger menu icon on mobile devices (<768px). It provides full site navigation with nested accordion menus up to 3 levels deep, contact information, and B2B/B2C + language toggles.

Key features:

  • 3-section layout: Header (logo + close), scrollable body (categories), footer (contact + toggles)
  • Nested accordions: Up to 3 levels deep (L1 → L2 → L3) with expand/collapse arrows
  • Smooth animations: Transform translateX(-100%) with backdrop blur effect
  • Contact quick access: Phone and email links in footer
  • Inline toggles: B2B/B2C price toggle and language switcher in footer
  • Smart hierarchy: Visual differentiation per level (padding, background, icon size)

Mobile-only component: This drawer is hidden on desktop (≥768px). Desktop uses C14-MegaNavigation instead.

STEP 2: When to Use

Use C14-MobileDrawer when:

  • Building mobile navigation for viewports <768px
  • You need nested category navigation (product categories, service types, content sections)
  • You want to combine navigation with utility links (account, orders, favorites)
  • You need quick access to contact methods (phone, email)
  • You want users to switch B2B/B2C pricing or language without leaving the menu

Don't use when:

  • Building desktop navigation (≥768px) — use C14-MegaNavigation instead
  • You only have 1-2 top-level items — use a simple mobile menu instead
  • You need search functionality in the drawer — integrate C14-SearchBar separately

STEP 5: Layout Rules

Overall Structure

  • Fixed position: position: fixed; top: 0; left: 0; bottom: 0;
  • Width: 320px (max 85vw for very small screens)
  • Flexbox column: display: flex; flex-direction: column;
  • Header: flex-shrink: 0 (fixed height)
  • Body: flex: 1; overflow-y: auto; (scrollable, takes remaining space)
  • Footer: flex-shrink: 0 (fixed height)

Animation

  • Closed state: transform: translateX(-100%)
  • Open state: transform: translateX(0)
  • Transition: 0.35s cubic-bezier(0.4,0,0.2,1)
  • Backdrop: Fades in/out with opacity transition

STEP 6: Accessibility

Keyboard Navigation

  • ESC key: Closes the drawer (add keydown event listener)
  • Tab navigation: All interactive elements (items, buttons, links) must be focusable
  • Focus trap: When drawer is open, focus should stay within drawer (prevent tabbing to content behind)
  • Focus management: When drawer opens, focus close button; when closes, return focus to hamburger

ARIA Attributes

Screen Reader Support

  • Add aria-label to all icon-only buttons (close, toggle buttons)
  • Use aria-expanded to indicate accordion state (true/false)
  • Use aria-controls to connect accordion triggers to their submenus
  • Add role="navigation" to drawer container
  • Section labels should use aria-label or be real headings (<h2>)

STEP 7: Implementation Notes

Integration with Header

The mobile drawer is triggered by a hamburger button in C14-MegaNavigation. Ensure proper integration:

<!-- In mobile header (<768px) -->
<button class="mobile-menu-btn"
        onclick="openDrawer()"
        aria-label="Open navigation menu"
        aria-expanded="false"
        aria-controls="mobileDrawer">
  <i data-lucide="menu"></i>
</button>

Body Scroll Lock

When drawer is open, prevent scrolling of main page content:

function openDrawer() {
  document.getElementById('mobileDrawer').classList.add('open');
  document.getElementById('drawerBackdrop').classList.add('open');
  document.body.style.overflow = 'hidden'; // Lock scroll
}

function closeDrawer() {
  document.getElementById('mobileDrawer').classList.remove('open');
  document.getElementById('drawerBackdrop').classList.remove('open');
  document.body.style.overflow = ''; // Unlock scroll
}

Accordion State Management

Only one L1 section should be open at a time (optional, for cleaner UX):

function toggleSub(item) {
  const isOpen = item.classList.contains('open');
  const sub = item.nextElementSibling;

  // Optional: Close all other L1 accordions
  if (!isOpen && item.parentElement.classList.contains('md-body')) {
    const allL1Items = document.querySelectorAll('.md-body > .md-item');
    allL1Items.forEach(i => {
      i.classList.remove('open');
      if (i.nextElementSibling) i.nextElementSibling.classList.remove('open');
    });
  }

  // Toggle current item
  item.classList.toggle('open');
  if (sub) sub.classList.toggle('open');

  // Update ARIA
  const isNowOpen = item.classList.contains('open');
  item.setAttribute('aria-expanded', isNowOpen);
}

Persistence

Save user preferences to localStorage:

function switchPrice(btn, type) {
  // Remove active from all buttons in this toggle group
  btn.parentElement.querySelectorAll('button').forEach(b => b.classList.remove('active'));
  btn.classList.add('active');

  // Save to localStorage
  localStorage.setItem('priceType', type);

  // Dispatch event for other components to listen
  window.dispatchEvent(new CustomEvent('priceTypeChanged', { detail: { type } }));
}

function switchLang(btn, lang) {
  btn.parentElement.querySelectorAll('button').forEach(b => b.classList.remove('active'));
  btn.classList.add('active');

  localStorage.setItem('language', lang);
  window.dispatchEvent(new CustomEvent('languageChanged', { detail: { lang } }));
}

Responsive Behavior

/* Hide drawer trigger on desktop */
@media (min-width: 768px) {
  .mobile-menu-btn {
    display: none;
  }

  .mobile-drawer,
  .mobile-drawer-backdrop {
    display: none; /* Or just don't render on desktop */
  }
}

Performance

  • Use CSS transforms: translateX() is GPU-accelerated, smoother than left animation
  • Lazy-load submenus: For very deep navigation, only render visible submenus (use display: none instead of v-if)
  • Debounce scroll: If adding scroll-based effects, debounce scroll events

STEP 8: Variations & Related Components

Variations

This component can be adapted for different use cases:

1. E-commerce Drawer

  • Add search bar at top of .md-body (integrate C14-SearchBar)
  • Add "Recent searches" section above categories
  • Add promo banner in footer (above contact)

2. SaaS/Dashboard Drawer

  • Remove product categories, use app navigation instead
  • Add user profile card in header (avatar + name + role)
  • Add notification count badges to menu items
  • Remove B2B/B2C toggle, add dark mode toggle instead

3. Content/Blog Drawer

  • Replace product categories with content categories (Articles, Guides, Videos)
  • Add social links in footer
  • Remove price toggle, keep only language switcher

4. Multi-brand Drawer

  • Add brand/site switcher at top of drawer (dropdown)
  • Each brand has different logo, colors, categories
  • Use data attributes to theme drawer: data-brand="plastimed"

Common Patterns

Pattern 1: Drawer + Search Overlay

Mobile header has both hamburger (navigation) and search icon:

  • Hamburger opens .mobile-drawer from left
  • Search icon opens .is-overlay (C1) full-screen search
  • Both have separate backdrops, same z-index priority

Pattern 2: Drawer with Cart Badge

Show cart count in drawer footer (not just in header):

<div class="md-footer">
  <a href="/cart" class="md-cart-link">
    <i data-lucide="shopping-cart"></i>
    <span>Winkelwagen</span>
    <span class="cart-count">3</span>
  </a>
  <!-- Contact info below -->
</div>

Pattern 3: Nested Mega Drawer

For very complex navigation, use a "drill-down" pattern instead of accordions:

  • L1 item click slides in a new full-screen panel (L2 items)
  • Add "Back" button at top of L2 panel
  • Breadcrumb trail shows: Home > Diagnostiek > Bloeddrukmeters
  • More mobile-friendly for deep hierarchies (avoids nested scrolling)

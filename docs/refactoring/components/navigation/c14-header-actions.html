📋 STEP 1: COMPONENT OVERVIEW

Component: C14 - Header Actions

Parent: C14-MegaNavigation (Header section)

Category: Navigation / Actions

Complexiteit: Low (button group with badges)

Purpose:

Action buttons in header right section: account access, cart, and optional utilities (bestellijsten, favorieten, etc.).

Structure:

  • Icon Buttons: 42x42px square buttons (bestellijsten, user, etc.)
  • Separator: 1px vertical line (optional, desktop-only)
  • Cart Button: Primary CTA with cart count badge (navy bg)

Elements:

  • Bestellijsten: Clipboard-list icon (desktop-only)
  • User Account: User icon (always visible)
  • Separator: Visual divider (desktop-only)
  • Shopping Cart: Cart icon + count badge (primary action)

🎯 STEP 2: WHEN TO USE

Ideaal voor:

  • E-commerce websites (cart is core functionality)
  • B2B platforms (bestellijsten = order lists)
  • Sites requiring user authentication (account button)
  • Desktop-first layouts (full button set)

Common Action Buttons:

  • E-commerce: Favorieten, Vergelijk, Cart, Account
  • B2B: Bestellijsten, Offertes, Cart, Account
  • SaaS: Notificaties, Help, Settings, Account
  • Content: Saved Articles, Bookmarks, Profile

Responsive Behavior:

  • Desktop (>768px): Full button set with separators
  • Mobile (<768px): Essential buttons only (user + cart)
  • Optional buttons marked desktop-only hide on mobile

Dynamic Cart Count:

// Update cart count dynamically
function updateCartCount(count) {
  const badge = document.querySelector('.cart-count');

  if (count > 0) {
    badge.textContent = count > 99 ? '99+' : count;
    badge.style.display = 'flex';
  } else {
    badge.style.display = 'none';
  }

  // Animate badge on count change
  badge.style.transform = 'scale(1.2)';
  setTimeout(() => {
    badge.style.transform = 'scale(1)';
  }, 200);
}

// Listen for cart updates
document.addEventListener('cartupdate', (e) => {
  updateCartCount(e.detail.count);
});

  </button>

Focus Management:

  • Visible focus ring via :focus-visible
  • Hover state distinct from focus state
  • Badge count announced in button aria-label

Screen Reader Support:

  • Icons marked aria-hidden="true"
  • Buttons have descriptive aria-label
  • Cart count included in aria-label: "Winkelwagen (3 items)"
  • Cart count changes announced via live region (optional)

💡 STEP 7: IMPLEMENTATION NOTES

Cart Count Animation:

// Animate cart count on add-to-cart
function animateCartAdd(newCount) {
  const cartBtn = document.querySelector('.h-btn-cart');
  const badge = cartBtn.querySelector('.cart-count');

  // Update count
  badge.textContent = newCount;

  // Pulse animation
  cartBtn.style.transform = 'scale(1.1)';
  badge.style.transform = 'scale(1.3)';

  setTimeout(() => {
    cartBtn.style.transform = 'scale(1)';
    badge.style.transform = 'scale(1)';
  }, 300);

  // Update aria-label
  cartBtn.setAttribute('aria-label', `Winkelwagen (${newCount} items)`);
}

User Authentication State:

// Show different button based on login state
const userBtn = document.querySelector('.h-btn[title="Mijn account"]');

if (user.isLoggedIn) {
  // Show user avatar or initials
  userBtn.innerHTML = `
    <div class="user-avatar">${user.initials}</div>
  `;
  userBtn.title = user.name;
} else {
  // Show login icon
  userBtn.innerHTML = `<i data-lucide="user"></i>`;
  userBtn.title = 'Inloggen';
}

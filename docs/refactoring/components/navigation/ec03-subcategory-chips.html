📋 STEP 1: Component Overview

Component ID: EC03

Component Name: SubcategoryChips

Category: Navigation

Source File: /ecommerce-flows/ef1-shop-archive.html (lines 44-62)

Target Folder: /refactoring/components/navigation/

Description

Horizontal scrollable navigation component displaying subcategories as pill-style chips. Appears directly below CategoryHero on shop archive pages. Active chip uses teal accent colors, inactive chips use grey. Each chip can optionally display an icon and product count.

Key Features

  • ✅ Horizontal scroll: Overflow-x auto for many subcategories
  • ✅ Active state: Teal background + border for current subcategory
  • ✅ Inactive state: Grey background + border, hover effect
  • ✅ Product counts: Optional count display in grey (e.g., "(87)")
  • ✅ Icons: Optional Lucide icons (14px)
  • ✅ Pill shape: 100px border-radius for rounded ends
  • ✅ Keyboard navigation: Tab-focusable with visible focus ring

🎯 STEP 2: When to Use

Use this component when:

  • ✅ Below CategoryHero on shop archive pages
  • ✅ Filtering products by subcategory type
  • ✅ Displaying material variants (Nitril, Latex, Vinyl)
  • ✅ Size/type filters (Small, Medium, Large)
  • ✅ Collection filters (New, Sale, Bestsellers)
  • ✅ Any horizontal navigation with 3-8 items

Design Patterns:

  • First chip: Often "Alle [category]" with icon, shows all items
  • Subsequent chips: Specific subcategories with counts
  • Active indicator: Only ONE chip active at a time
  • Count format: Parentheses with number (e.g., "(87)")
  • Icon usage: Optional, typically only on first "All" chip

Don't use this component for:

  • ❌ Vertical sidebar navigation (use C21: FilterSidebar)
  • ❌ More than 10-12 chips (becomes overwhelming)
  • ❌ Multi-select filters (use checkbox filters in sidebar)
  • ❌ Breadcrumb navigation (use C19: Breadcrumbs)

Related to:

  • C21: FilterSidebar — Vertical filter panel (left side)
  • Active filter chips — Removable filter tags (different styling)

Auto-Calculate Counts

// Hook to calculate product count per subcategory
{
  slug: 'categories',
  hooks: {
    afterRead: [
      async ({ doc, req }) => {
        const productCount = await req.payload.count({
          collection: 'products',
          where: {
            category: {
              equals: doc.id
            }
          }
        })

        return {
          ...doc,
          productCount: productCount.totalDocs
        }
      }
    ]
  }
}

💻 STEP 7: Implementation Notes

React/Next.js Component Example

'use client'

import { LucideIcon } from 'lucide-react'
import * as Icons from 'lucide-react'
import Link from 'next/link'
import { useRef, useEffect } from 'react'

interface SubcategoryChip {
  id: string
  label: string
  slug: string
  icon?: string
  count?: number
  href: string
}

interface SubcategoryChipsProps {
  chips: SubcategoryChip[]
  activeSlug: string
}

export function SubcategoryChips({ chips, activeSlug }: SubcategoryChipsProps) {
  const activeRef = useRef<HTMLAnchorElement>(null)

  // Scroll active chip into view on mount
  useEffect(() => {
    if (activeRef.current) {
      activeRef.current.scrollIntoView({
        behavior: 'smooth',
        block: 'nearest',
        inline: 'center'
      })
    }
  }, [activeSlug])

  return (
    <nav className="subcategory-chips" aria-label="Subcategorieën">
      <div className="subcategory-chips__list" role="list">
        {chips.map((chip) => {
          const isActive = chip.slug === activeSlug
          const IconComponent = chip.icon
            ? (Icons[chip.icon as keyof typeof Icons] as LucideIcon)
            : null

          return (
            <Link
              key={chip.id}
              href={chip.href}
              ref={isActive ? activeRef : null}
              className={`subcategory-chip ${
                isActive ? 'subcategory-chip--active' : 'subcategory-chip--inactive'
              }`}
              aria-current={isActive ? 'page' : undefined}
              role="listitem"
            >
              {IconComponent && <IconComponent size={14} aria-hidden="true" />}
              {chip.label}
              {chip.count !== undefined && (
                <span
                  className="subcategory-chip__count"
                  aria-label={`${chip.count} producten`}
                >
                  ({chip.count})
                </span>
              )}
            </Link>
          )
        })}
      </div>
    </nav>
  )
}

Server Component Usage

// app/shop/[category]/page.tsx
import { SubcategoryChips } from '@/components/navigation/SubcategoryChips'
import { getPayloadClient } from '@/lib/payload'

export default async function CategoryPage({
  params
}: {
  params: { category: string }
}) {
  const payload = await getPayloadClient()

  // Get parent category
  const category = await payload.findByID({
    collection: 'categories',
    id: params.category
  })

  // Get subcategories
  const subcategories = await payload.find({
    collection: 'categories',
    where: {
      parent: {
        equals: category.id
      }
    }
  })

  // Transform to chips
  const chips = [
    {
      id: 'all',
      label: `Alle ${category.name.toLowerCase()}`,
      slug: 'all',
      icon: category.icon,
      href: `/shop/${category.slug}`
    },
    ...subcategories.docs.map(sub => ({
      id: sub.id,
      label: sub.name,
      slug: sub.slug,
      count: sub.productCount,
      href: `/shop/${category.slug}/${sub.slug}`
    }))
  ]

  return (
    <>
      <CategoryHero category={category} />
      <SubcategoryChips chips={chips} activeSlug="all" />
      {/* Rest of page */}
    </>
  )
}

Important Implementation Details

  • ✅ Auto-scroll active chip: Use scrollIntoView() on mount to center active chip
  • ✅ Dynamic icon loading: Load Lucide icons dynamically by name
  • ✅ Optional counts: Hide count if count === undefined
  • ✅ First chip pattern: "Alle [category]" with parent icon, links to main category page
  • ✅ SEO-friendly: Use Next.js <Link> for server-side navigation

C15-Footer

Complete site footer with branding, navigation columns, contact info, trust badges, and legal links

STEP 1: Component Overview

C15-Footer is the site-wide footer component that appears at the bottom of every page. It provides comprehensive navigation, contact information, trust signals, and legal compliance links in a structured 3-section layout.

Key features:

  • 3-section structure: Top (brand + nav), middle (trust badges), bottom (copyright + legal)
  • 4-column grid: Brand (1.5fr) + 3 navigation columns (1fr each)
  • Navy background: Consistent with C14-Topbar and dark sections
  • Teal accents: Logo highlight, social icons hover, link hover states
  • Trust signals: Certifications, payment methods, shipping info with green checkmarks
  • Fully responsive: 4-col → 2-col → 1-col on mobile
  • 20px border radius: Rounded corners for modern card-like appearance

Foundation component: The footer is one of the two anchor components (with C14-MegaNavigation) that appear on every page and define the site's visual identity.

STEP 2: When to Use

Use C15-Footer when:

  • Building the main site layout (appears on every page)
  • You need comprehensive site navigation in the footer
  • You want to display trust signals and certifications
  • You need prominent contact information
  • Legal compliance links must be accessible (GDPR, terms, privacy)
  • Social media links should be available site-wide

Don't use when:

  • Building a minimal landing page (use simplified footer instead)
  • Creating checkout/payment flows (use simplified footer to reduce distractions)
  • Building a single-page application without traditional page structure

STEP 5: Layout Rules

Grid Structure

The footer uses a 4-column grid in the top section:

  • Column 1 (Brand): 1.5fr — wider for logo, tagline, and social icons
  • Column 2 (Categories): 1fr — product/category navigation
  • Column 3 (Service): 1fr — customer service links
  • Column 4 (Contact): 1fr — contact information

Responsive Breakpoints

Breakpoint Grid Changes
>900px 1.5fr 1fr 1fr 1fr Desktop: full 4-column layout
640px - 900px 1fr 1fr Tablet: 2 columns, brand spans first column
<640px 1fr Mobile: single column, reduced padding, stacked legal links

Spacing

  • Top section padding: 48px top, 40px horizontal, 32px bottom
  • Column gap: 36px (desktop), 32px (tablet), 24px (mobile)
  • Mid section padding: 20px vertical, 40px horizontal
  • Bottom section padding: 16px vertical, 40px horizontal
  • Link spacing: 8px vertical margin between links
  • Social icon gap: 6px between icons

Borders

  • Top border between sections: 1px solid rgba(255,255,255,0.06)
  • Very subtle, maintains visual hierarchy without harsh dividers

    <nav class="sf-col" aria-label="Product categories">
      <h4>Categorieën</h4>
      <ul class="sf-links">...</ul>
    </nav>

    <nav class="sf-col" aria-label="Customer service">
      <h4>Klantenservice</h4>

    </nav>

    <address class="sf-col">
      <h4>Contact</h4>
      <!-- Use <address> for contact info -->
    </address>
  </div>
</footer>

ARIA Attributes

  • role="contentinfo" on .site-footer
  • aria-label on each navigation section to identify purpose
  • <address> element for contact information
  • aria-label on social icon buttons (e.g., "LinkedIn", "Instagram")

Keyboard Navigation

  • All links and social icons are keyboard focusable
  • Focus ring visible on all interactive elements
  • Tab order follows visual hierarchy: brand → categories → service → contact → trust → legal

Focus Styling

STEP 7: Implementation Notes

Logo Accent Pattern

The logo uses an <em> tag for the accent color:

<div class="sf-logo">compass<em>design</em></div>
/* Result: compass in white, design in teal-light */

Social Icons

Social icons use a semi-transparent background with hover transition to teal:

  • Default: rgba(255,255,255,0.06) background + border
  • Hover: Teal background, white icon
  • Size: 36x36px square, 8px border radius
  • Icon size: 16x16px

Link Styling

All footer links follow this pattern:

.sf-links a {
  color: rgba(255,255,255,0.4);  /* Low opacity for hierarchy */
  transition: color 0.12s;        /* Fast transition */
}

.sf-links a:hover {
  color: var(--teal-light);       /* Teal accent on hover */
}

Trust Badges Layout

Trust badges use flexbox with wrapping for responsive behavior:

.sf-mid {
  display: flex;
  gap: 24px;
  flex-wrap: wrap;  /* Wraps on narrow viewports */
}

Responsive Strategy

Desktop (>900px):

  • 4-column grid (1.5fr 1fr 1fr 1fr)
  • All content visible side-by-side
  • Legal links in single row

Tablet (640-900px):

  • 2-column grid (1fr 1fr)
  • Brand column spans first position
  • Navigation columns wrap below

Mobile (<640px):

  • 1-column stack
  • Reduced padding (32px 24px)
  • Legal links stack vertically with 8px gap
  • Copyright and legal center-aligned

Integration with Newsletter

The footer often appears directly below C16-Newsletter component:

<!-- Newsletter above footer -->
<div class="nl-banner">...</div>

<!-- Footer below, add margin-top -->
<footer class="site-footer" style="margin-top: 32px;">
  ...
</footer>

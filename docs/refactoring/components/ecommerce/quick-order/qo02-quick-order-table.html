QO02: QuickOrderTable

Component Identity: Multi-row data table for B2B quick order entry with product search, quantity input, and bulk add to cart

Category: E-commerce / B2B / Forms

Source File: ef5-quick-order.html (lines 38-110)

STEP 1: Component Overview

Purpose

The QuickOrderTable is a data table that allows B2B users to quickly add multiple products to their cart by entering SKUs or product names. It displays product autocomplete results, quantity inputs, prices, and totals in a structured grid layout.

Key Features

  • 5-column grid: Product | Quantity | Price | Total | Delete
  • Product autocomplete: Search by SKU or name, fills in price automatically
  • Staffel hints: Shows volume discount opportunities (amber text)
  • Dynamic total: Real-time calculation as quantities change
  • Add row button: Dynamically add more input rows
  • Bulk add: Add all valid rows to cart at once
  • Empty state: Shows placeholder row when no products added yet

Structure

The table consists of 3 sections:

  • Header: Column labels (grey bg, uppercase, 12px)
  • Body: Product rows (white bg, 20px vertical padding)
  • Footer: "Add row" button + Total summary + Bulk add button (grey bg)

STEP 2: When to Use

Use This Component When:

  • Building a B2B quick order page
  • Users need to add 2+ products at once (bulk entry)
  • Users know SKUs/article numbers (repeat orders)
  • You want to enable CSV import (pairs with QO04)

Don't Use This Component When:

  • B2C single product ordering (use ProductCard + Add to Cart instead)
  • Users browse visually (use ProductGrid instead)
  • Mobile-first flows (this is desktop-optimized)

Content Guidelines

  • Column headers: Keep labels short (max 3 words)
  • Placeholder text: "Zoek op artikelnummer of naam..." (descriptive)
  • Empty price: Show "—" when no product selected
  • Staffel hints: "Nog X voor €Y (−Z%)" format

STEP 5: Layout Rules

Grid Structure

  • Column 1 (Product): 1fr (flexible, takes remaining space)
  • Column 2 (Quantity): 140px fixed width
  • Column 3 (Price): 140px fixed width
  • Column 4 (Total): 180px fixed width (right-aligned)
  • Column 5 (Delete): 48px fixed width (icon button)

Row Anatomy

  • Product cell: Input field + SKU display (or staffel hint)
  • Quantity cell: Number input (centered text, monospace)
  • Price cell: Read-only text display (monospace)
  • Total cell: Calculated value (right-aligned, bold)
  • Delete cell: Icon button (40×40px, 8px radius)

Footer Layout

  • Left: "Add row" button
  • Right: Total summary + Bulk add button (horizontal flex, 24px gap)

STEP 6: Accessibility

Semantic HTML

  • Use semantic table structure: <table>, <thead>, <tbody>, <tfoot>
  • Or use role="table", role="row", role="cell" if using divs
  • Column headers should use <th> or role="columnheader"

ARIA Attributes

  • aria-label="Quick order table" on container
  • aria-label="Verwijder product" on delete buttons
  • aria-live="polite" on total amount (announces changes to screen readers)
  • aria-describedby linking quantity input to staffel hint

Keyboard Navigation

  • Tab through: Product input → Quantity input → Delete button → Next row
  • Enter key in product input triggers autocomplete selection
  • Escape key closes autocomplete dropdown
  • Arrow keys navigate autocomplete results

Form Validation

  • Product input: Required if quantity > 0
  • Quantity input: Min value = 1, show error if 0 or negative
  • Bulk add button: Disabled if no valid products (opacity 0.5)

STEP 7: Implementation Notes

React/Next.js Component

'use client'

import { useState } from 'react'
import { Plus, ShoppingCart, Trash2, TrendingDown } from 'lucide-react'

export function QuickOrderTable() {
  const [rows, setRows] = useState([
    { id: '1', query: '', quantity: 0, price: 0, total: 0 },
  ])

  const addRow = () => {
    setRows([...rows, {
      id: Date.now().toString(),
      query: '',
      quantity: 0,
      price: 0,
      total: 0
    }])
  }

  const deleteRow = (id: string) => {
    setRows(rows.filter(r => r.id !== id))
  }

  const updateRow = (id: string, updates: Partial<QuickOrderRow>) => {
    setRows(rows.map(r => r.id === id ? { ...r, ...updates } : r))
  }

  const totalAmount = rows.reduce((sum, r) => sum + r.total, 0)
  const productCount = rows.filter(r => r.productId).length

  return (
    <div className="quick-order-table">
      {/* Header */}
      <div className="quick-order-table-header">
        <div>Artikelnummer of naam</div>
        <div>Aantal</div>
        <div>Prijs/stuk</div>
        <div className="align-right">Totaal</div>
        <div></div>
      </div>

      {/* Body */}
      <div className="quick-order-table-body">
        {rows.map(row => (
          <QuickOrderRow
            key={row.id}
            row={row}
            onUpdate={updateRow}
            onDelete={deleteRow}
          />
        ))}
      </div>

      {/* Footer */}
      <div className="quick-order-table-footer">
        <button onClick={addRow} className="add-row-button">
          <Plus /> Rij toevoegen
        </button>
        <div className="table-footer-summary">
          <div className="table-footer-total">
            <div className="table-footer-total-label">
              Totaal ({productCount} producten)
            </div>
            <div className="table-footer-total-value">
              € {totalAmount.toFixed(2)}
            </div>
          </div>
          <button
            className="bulk-add-button"
            disabled={productCount === 0}
          >
            <ShoppingCart /> Toevoegen aan winkelwagen
          </button>
        </div>
      </div>
    </div>
  )
}

Performance Optimization

  • Use React.memo for QuickOrderRow to prevent unnecessary re-renders
  • Debounce product search (300ms) to reduce API calls
  • Virtualize rows if >50 products (use react-window)
  • Lazy load autocomplete dropdown component

Important: This component is typically used with QO03 (QuickOrderRow) as child component. See QO03 for detailed row implementation.

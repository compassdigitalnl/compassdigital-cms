QO01: QuickOrderHeader

Component Identity: Quick Order page header with title, description, and action buttons (CSV upload, bestellijst access)

Category: E-commerce / B2B

Source File: ef5-quick-order.html (lines 14-36)

STEP 1: Component Overview

Purpose

The QuickOrderHeader serves as the page header for the B2B Quick Order (Snelbestellen) flow. It provides context about the functionality and quick access to utility features like CSV upload and saved order lists (bestellijsten).

Key Features

  • Zap icon: 28×28px teal icon indicating speed/efficiency
  • Title + Description: Clear hierarchy (28px title + 14px description)
  • Action buttons: 2 buttons by default (CSV upload + Use bestellijst)
  • Flexible actions: Supports 1-3 action buttons
  • Responsive layout: Stacks vertically on mobile (<768px)

Usage Context

Use at the top of Quick Order pages to orient B2B users and provide quick access to bulk ordering utilities. The header should be the first element after the main navigation.

STEP 2: When to Use

Use This Component When:

  • Building a B2B quick order page (snelbestelformulier)
  • You need to provide CSV upload functionality
  • Users have saved order lists (bestellijsten) available
  • You want to explain the quick order functionality upfront

Don't Use This Component When:

  • Building B2C flows (use standard page headers instead)
  • The page doesn't have bulk ordering functionality
  • You need a hero section (this is a utility header, not marketing)

Content Guidelines

  • Title: Keep it short (1-2 words). Examples: "Snelbestellen", "Quick Order", "Bulk Entry"
  • Description: Explain the functionality in 1 sentence (max 100 characters)
  • Button labels: Use action verbs: "CSV uploaden", "Gebruik bestellijst", "Export Excel"

  • Alignment: justify-content: space-between + align-items: center
  • Padding: 32px (top/bottom) × 40px (left/right) on desktop, 24px all sides on mobile
  • Background: White card with 1px grey border, 14px border-radius

Content Section (Left)

  • Display: flex-direction: column
  • Gap: 8px between title and description
  • Icon + Title: display: flex, 12px gap, vertically centered

Actions Section (Right)

  • Display: flex (horizontal row)
  • Gap: 12px between buttons
  • Mobile: Full-width stacked buttons (flex-direction: column)

Responsive Breakpoints

  • <768px: Stack vertically (content top, actions bottom), reduce title to 24px, full-width buttons

STEP 6: Accessibility

Semantic HTML

  • Use <header role="banner"> for the container
  • Use <h1> for the title (or appropriate heading level based on page structure)
  • Use <p> for the description
  • Use <button type="button"> for action buttons

ARIA Attributes

  • aria-hidden="true" on all decorative icons
  • Button labels should be descriptive (already are: "CSV uploaden", "Gebruik bestellijst")

Keyboard Navigation

  • All buttons must be keyboard focusable (native <button> elements)
  • Focus ring: box-shadow: 0 0 0 3px var(--teal-glow)
  • Tab order: Title → Button 1 → Button 2 → etc.

Color Contrast

  • Title (#0A1628 on white): 16.1:1 ✓ (WCAG AAA)
  • Description (#64748B on white): 6.8:1 ✓ (WCAG AAA)
  • Button text (navy/teal on backgrounds): 4.5:1+ ✓ (WCAG AA)

STEP 7: Implementation Notes

React/Next.js Component

'use client'

import { Zap, Upload, List } from 'lucide-react'

interface QuickOrderHeaderAction {
  id: string
  label: string
  icon: string
  variant: 'secondary' | 'teal'
  onClick?: () => void
}

interface QuickOrderHeaderProps {
  title?: string
  description?: string
  actions: QuickOrderHeaderAction[]
}

export function QuickOrderHeader({
  title = 'Snelbestellen',
  description = 'Voer meerdere artikelen tegelijk in voor een snelle bestelling',
  actions,
}: QuickOrderHeaderProps) {
  const iconMap = {
    upload: Upload,
    list: List,
    zap: Zap,
  }

  return (
    <header className="quick-order-header" role="banner">
      <div className="quick-order-header-content">
        <h1 className="quick-order-header-title">
          <Zap className="quick-order-header-icon" aria-hidden="true" />
          {title}
        </h1>
        <p className="quick-order-header-description">{description}</p>
      </div>
      <div className="quick-order-header-actions">
        {actions.map((action) => {
          const Icon = iconMap[action.icon as keyof typeof iconMap]
          return (
            <button
              key={action.id}
              type="button"
              className={`quick-order-action-button ${action.variant}`}
              onClick={action.onClick}
            >
              {Icon && <Icon aria-hidden="true" />}
              {action.label}
            </button>
          )
        })}
      </div>
    </header>
  )
}

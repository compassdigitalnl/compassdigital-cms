STEP 1: Component Overview

Component ID: C23

Component Name: Quantity Stepper

Category: E-commerce / Form Controls

Priority: HIGH (core shopping functionality)

Description

A numeric input control with increment/decrement buttons, used for selecting product quantities throughout the e-commerce flow. Features three size variants to accommodate different contexts (compact product cards, standard cart items, prominent hero sections) and optional rounded styling.

Key Features

  • 3 size variants: Small (30×36px), Medium (36×40px), Large (42×48px)
  • Monospace input: JetBrains Mono for clear number display
  • Min/max validation: Prevents invalid quantities (min 1, max 999)
  • Keyboard accessible: Arrow keys, Tab navigation, direct input
  • Visual feedback: Hover states, focus rings, disabled state
  • Touch-friendly: Button targets meet 48px accessibility standard (large variant)
  • Optional rounded variant: Pill-style borders for modern aesthetic

STEP 2: When to Use

Primary Use Cases

  • Product cards: Quick-add functionality on shop archive pages (use small variant)
  • Cart line items: Adjust quantities in shopping cart (use medium variant)
  • Product detail pages: Main quantity selector above add-to-cart button (use large variant)
  • Quick order forms: Bulk ordering interfaces (use medium variant)
  • Staffel calculator: Volume pricing quantity input (use medium/large variant)
  • Bundle builder: Configure multi-product bundles (use small/medium variant)

When NOT to Use

  • Single quantity only: If product is always qty=1, omit stepper and use direct ATC
  • Large ranges (100+): Use dropdown or direct input field instead
  • Decimal quantities: Component designed for integers only (use custom input for weights/volumes)
  • Non-transactional counts: Use standard number input or select for filters, settings, etc.

STEP 7: Implementation Notes

JavaScript Logic

Vanilla JavaScript (Standalone)

function stepQty(btn, delta) {
  const input = btn.parentElement.querySelector('input');
  let val = parseInt(input.value) || 1;
  const min = parseInt(input.min) || 1;
  const max = parseInt(input.max) || 999;

  val = Math.max(min, Math.min(max, val + delta));
  input.value = val;

  // Update ARIA
  input.setAttribute('aria-valuenow', val);

  // Trigger change event for frameworks
  input.dispatchEvent(new Event('change', { bubbles: true }));

  // Optional: Announce to screen readers
  announceQty(val);
}

function announceQty(val) {
  const liveRegion = document.getElementById('qty-announce');
  if (liveRegion) {
    liveRegion.textContent = `Quantity: ${val}`;
  }
}

React Component Example

import React, { useState } from 'react';

interface QuantityStepperProps {
  value: number;
  onChange: (newValue: number) => void;
  min?: number;
  max?: number;
  size?: 'sm' | 'md' | 'lg';
  rounded?: boolean;
  disabled?: boolean;
  ariaLabel?: string;
}

export const QuantityStepper: React.FC<QuantityStepperProps> = ({
  value,
  onChange,
  min = 1,
  max = 999,
  size = 'md',
  rounded = false,
  disabled = false,
  ariaLabel = 'Quantity',
}) => {
  const handleStep = (delta: number) => {
    const newValue = Math.max(min, Math.min(max, value + delta));
    onChange(newValue);
  };

  const handleInputChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const newValue = parseInt(e.target.value) || min;
    onChange(Math.max(min, Math.min(max, newValue)));
  };

  return (
    <div className={`qty-stepper qty-${size} ${rounded ? 'qty-rounded' : ''}`}>
      <button
        onClick={() => handleStep(-1)}
        disabled={disabled || value <= min}
        aria-label="Decrease quantity"
      >
        
      </button>
      <input
        type="number"
        value={value}
        onChange={handleInputChange}
        min={min}
        max={max}
        disabled={disabled}
        aria-label={ariaLabel}
        aria-valuemin={min}
        aria-valuemax={max}
        aria-valuenow={value}
      />
      <button
        onClick={() => handleStep(1)}
        disabled={disabled || value >= max}
        aria-label="Increase quantity"
      >
        +
      </button>
    </div>
  );
};

import { QuantityStepper } from '@/components/ui/QuantityStepper';

export function ProductQuickAdd({ product }) {
  const [qty, setQty] = useState(1);

        onClick={handleAddToCart}
        disabled={isAdding || product.stock < 1}
        className="add-to-cart"
      >
        {isAdding ? '...' : '=Ň'}
      </button>
    </div>
  );
}

Validation & Edge Cases

  • Empty input: Reset to min value on blur
  • Non-numeric input: Sanitize with parseInt() + fallback to min
  • Exceeds max: Clamp to max value (show warning if stock-limited)
  • Below min: Clamp to min value (usually 1)
  • Decimal input: Round to nearest integer (no decimal quantities)
  • Stock limits: Set max attribute to available stock, disable increment when reached

Performance Considerations

  • Debounce onChange: If triggering API calls (cart updates), debounce by 300-500ms
  • Optimistic UI: Update input value immediately, sync with backend asynchronously
  • Event delegation: Use single event listener on parent for multiple steppers

Testing Checklist

  •  Increment button increases value by 1
  •  Decrement button decreases value by 1
  •  Cannot decrement below min (button disabled)
  •  Cannot increment above max (button disabled)
  •  Direct input accepts valid numbers
  •  Invalid input (text, negative, decimal) is sanitized
  •  Keyboard arrows work (up/down)
  •  Tab navigation focuses input
  •  Focus ring visible and accessible
  •  Disabled state prevents interaction
  •  Screen reader announces changes
  •  Touch targets e48px on mobile

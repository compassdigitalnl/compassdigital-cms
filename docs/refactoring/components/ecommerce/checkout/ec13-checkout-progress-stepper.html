Purpose

4-step horizontal progress indicator specifically designed for the checkout flow. Shows user's current position in the checkout process (Gegevens → Verzending → Betaling → Bevestiging) with clear visual states for completed, active, and pending steps.

Visual Pattern

  • Horizontal layout with 4 evenly-spaced step circles
  • Connection lines between steps (grey background, green fill for completed)
  • Step circles (40px diameter) with 3px white border
  • 3 states: completed (green ✓), active (teal number), pending (grey number)
  • Step labels below circles (12px, 600 weight)
  • Max-width 600px, centered on page

Key Features

  • ✅ Visual progress tracking (connection line fills left-to-right)
  • ✅ 3 distinct states (completed, active, pending)
  • ✅ Green checkmark for completed steps (success feedback)
  • ✅ Stronger shadow on active step (visual emphasis)
  • ✅ Smooth transitions between states
  • ✅ Responsive design (hides labels on mobile)
  • ✅ Color-coded: green (done), teal (current), grey (todo)
  • ✅ Fixed 4 steps (checkout-specific, not configurable)

STEP 2: When to Use

✅ USE this component when:

  • ✅ Checkout flow — show progress through 4 steps
  • ✅ Multi-step forms — indicate current position
  • ✅ Onboarding flows — guide users through setup
  • ✅ Wizard interfaces — sequential data entry
  • ✅ Order tracking — show fulfillment stages (adapt states)

❌ DON'T use this component when:

  • ❌ Single-page checkout — no steps to track
  • ❌ Non-linear flows — users can jump between steps
  • ❌ Loading indicators — use spinner or skeleton instead
  • ❌ Generic progress (0-100%) — use progress bar instead
  • ❌ More than 5 steps — too crowded, consider vertical stepper

Integration Points

  • Page location: Top of checkout page, above form content
  • Step 1 (Gegevens): Personal info + delivery address (EC14)
  • Step 2 (Verzending): Shipping method selection (EC11)
  • Step 3 (Betaling): Payment method selection (EC12)
  • Step 4 (Bevestiging): Order review + T&C acceptance
  • Related components:

STEP 7: Implementation Notes

React/Next.js Component

'use client';

import { Check } from 'lucide-react';

interface Step {
  id: number;
  label: string;
}

const DEFAULT_STEPS: Step[] = [
  { id: 1, label: 'Gegevens' },
  { id: 2, label: 'Verzending' },
  { id: 3, label: 'Betaling' },
  { id: 4, label: 'Bevestiging' },
];

export function CheckoutProgressStepper({
  currentStep,
  steps = DEFAULT_STEPS,
  onStepClick,
  className = '',
}: {
  currentStep: number;
  steps?: Step[];
  onStepClick?: (step: number) => void;
  className?: string;
}) {
  // Calculate progress line fill width
  const progressWidth = ((currentStep - 1) / (steps.length - 1)) * 100;

  // Determine step state
  const getStepState = (stepId: number) => {
    if (stepId < currentStep) return 'completed';
    if (stepId === currentStep) return 'active';
    return 'pending';
  };

  return (
    <nav
      aria-label="Checkout progress"
      className={`checkout-progress ${className}`}
    >
      {/* Progress line */}
      <div className="progress-line">
        <div
          className="progress-line-fill"
          style={{ width: `${progressWidth}%` }}
        />
      </div>

      {/* Steps */}
      {steps.map((step) => {
        const state = getStepState(step.id);
        const isClickable = onStepClick && state === 'completed';

        return (
          <div
            key={step.id}
            className={`step ${state}`}
            aria-current={state === 'active' ? 'step' : undefined}
            onClick={() => isClickable && onStepClick(step.id)}
            style={{ cursor: isClickable ? 'pointer' : 'default' }}
          >
            <div className="step-circle">
              {state === 'completed' ? (
                <Check size={20} />
              ) : (
                step.id
              )}
            </div>
            <div className="step-label">{step.label}</div>
          </div>
        );
      })}
    </nav>
  );
}

State Management in Checkout Flow

// Checkout page with step tracking

'use client';

import { useState } from 'react';
import { CheckoutProgressStepper } from '@/components/ui-elements/ec13-checkout-progress-stepper';

export default function CheckoutPage() {
  const [currentStep, setCurrentStep] = useState(1);

  const handleStepComplete = () => {
    if (currentStep < 4) {
      setCurrentStep(currentStep + 1);
    }
  };

  const handleStepBack = () => {
    if (currentStep > 1) {
      setCurrentStep(currentStep - 1);
    }
  };

  return (
    <div className="checkout-container">
      <CheckoutProgressStepper
        currentStep={currentStep}
        onStepClick={setCurrentStep} // Allow navigation to completed steps
      />

      {currentStep === 1 && <AddressForm onNext={handleStepComplete} />}
      {currentStep === 2 && <ShippingSelection onNext={handleStepComplete} onBack={handleStepBack} />}
      {currentStep === 3 && <PaymentSelection onNext={handleStepComplete} onBack={handleStepBack} />}
      {currentStep === 4 && <OrderReview onBack={handleStepBack} />}
    </div>
  );
}

URL-based Step Tracking

// Using Next.js router for step tracking (better for back button)

import { useRouter, useSearchParams } from 'next/navigation';

export default function CheckoutPage() {
  const router = useRouter();
  const searchParams = useSearchParams();
  const currentStep = parseInt(searchParams.get('step') || '1');

  const goToStep = (step: number) => {
    router.push(`/checkout?step=${step}`);
  };

  return (
    <CheckoutProgressStepper
      currentStep={currentStep}
      onStepClick={goToStep}
    />
  );
}

🖼️ PD01: ProductGallery — Complete Documentation

Product image gallery with main image, thumbnail navigation, badge overlays, zoom, and lightbox functionality.
Version: 1.0 | Last updated: 25 Februari 2026 | Status: ✅ Production Ready

STEP 1: Component Overview

Purpose

PD01: ProductGallery is the primary visual component on product detail pages. It displays the main product image in a large, prominent format with thumbnail navigation for multiple views. The gallery supports badge overlays (Sale, New, Pro), zoom-on-hover for detail inspection, and lightbox modal for full-screen viewing. On mobile, it transforms into a swipeable carousel.

Key Features

  • Main Image: 1:1 aspect ratio (600×600px typical), 20px border-radius, gradient background
  • Thumbnail Navigation: 5 thumbnails in horizontal row (or vertical column), active state with teal border + glow
  • Badge Overlays: Positioned top-left (Sale/New/Pro) or top-right (Config options)
  • Sticky Positioning: Gallery sticks to viewport (top: 100px) on desktop while user scrolls product info
  • Zoom on Hover: Desktop-only, subtle scale transform (1.05x) for detail inspection
  • Lightbox Modal: Full-screen image viewer with prev/next navigation, ESC to close
  • Mobile Carousel: Swipeable on touch devices, scroll-snap for smooth navigation
  • Responsive: Horizontal thumbnails → 4 columns on mobile, vertical variant → horizontal scroll on mobile

Component Hierarchy

.product-gallery (sticky container)
├── .main-img (1:1 aspect ratio)
│   ├── .img-badge (badge-new | badge-sale | badge-pro)
│   ├── .badge-config (top-right overlay)
│   └── img (product image)
└── .thumb-row (horizontal) OR .thumb-column (vertical)
    └── .thumb × 5 (active state on first)

STEP 2: When to Use This Component

✅ Use ProductGallery When:

  • Product Detail Pages: Primary placement on left side of 2-column layout
  • Quick View Modals (C5): Compact variant with fewer thumbnails (3-4)
  • Variable Products: Gallery images update when user selects different variant (color/style)
  • Product Comparison: Show 2-4 galleries side-by-side

❌ Don't Use ProductGallery For:

  • Product Cards: Use single thumbnail image instead (EC01 pattern)
  • Cart Items: Use small square thumbnail (EC06: CartLineItem)
  • Blog/Content Images: Use standard <figure> with optional lightbox
  • General Image Galleries: This is product-specific with e-commerce features (badges, variants)

Related Components:

  • PD02: ProductMeta — Product info shown to the right of gallery
  • C18: ProductBadges — Badge variants used as overlays (integrated here)
  • C5: QuickView — Lightbox modal that includes compact gallery
  • VP01-VP07: Variant Selectors — Selecting variant triggers gallery image update

interface ProductGalleryProps {
  // Images
  images: Array<{

  enableZoom?: boolean;               // Zoom on hover (default: true)
  enableLightbox?: boolean;           // Lightbox on click (default: true)

  aspectRatio?: number;               // Main image ratio (default: 1 = square)
  borderRadius?: number;              // Main image radius in px (default: 20)
  thumbnailCount?: number;            // Max thumbnails shown (default: 5)

  // Callbacks
  onImageChange?: (index: number) => void;
  onLightboxOpen?: (index: number) => void;
  onLightboxClose?: () => void;

  // Optional customization
  className?: string;
  stickyOffset?: number;              // Top offset for sticky (default: 100px)
}

Sticky Positioning

On desktop, the gallery uses position: sticky; top: 100px; to stay visible while the user scrolls through product info. This creates a "sticky sidebar" effect.

Why 100px offset? Accounts for fixed header (72px) + spacing (28px). Adjust stickyOffset prop to match your header height.

Responsive Behavior

Breakpoint Layout Changes
Desktop (≥1024px) • 2-column grid (gallery left, info right)
• Sticky positioning enabled
• 5 thumbnails visible
• Zoom on hover
Tablet (768-1023px) • Still 2-column but narrower
• Sticky still active
• Zoom disabled
Mobile (≤640px) • Single column (gallery → info)
• Sticky disabled (position: static)
• 4 thumbnails visible
• Swipe-enabled carousel
• Vertical variant → horizontal scroll thumbs

ARIA Attributes

  • role="region" + aria-label on gallery container
  • role="button" on main image (clickable for lightbox)
  • aria-label on main image ("Klik om te vergroten")
  • aria-pressed on thumbnail buttons (true for active)
  • aria-hidden="true" on decorative badges
  • aria-label on thumbnails ("Afbeelding X van Y: [description]")
  • Lightbox: role="dialog", aria-modal="true", aria-labelledby

Keyboard Navigation

  • Main image: Focusable (tabindex="0"), activates with Enter/Space
  • Thumbnails: Focusable buttons, left/right arrow keys to navigate
  • Lightbox: ESC to close, left/right arrows for prev/next, Tab traps focus inside modal
  • Focus ring: Visible teal outline (3px glow) on all interactive elements

// Example: Announce image change
function switchImage(index) {
  // Update visual state
  setActiveImage(index);

  // Update aria-pressed on thumbnails
  thumbnails.forEach((thumb, i) => {
    thumb.setAttribute('aria-pressed', i === index);
  });

  // Announce to screen reader
  const announcement = `Afbeelding ${index + 1} van ${images.length} geselecteerd`;
  announceToSR(announcement);
}

// Focus trap in lightbox
useEffect(() => {
  if (lightboxOpen) {
    const focusableElements = lightbox.querySelectorAll('button, [href], [tabindex]:not([tabindex="-1"])');
    const firstElement = focusableElements[0];
    const lastElement = focusableElements[focusableElements.length - 1];

    firstElement.focus();

    document.addEventListener('keydown', (e) => {
      if (e.key === 'Tab') {
        if (e.shiftKey && document.activeElement === firstElement) {
          e.preventDefault();
          lastElement.focus();
        } else if (!e.shiftKey && document.activeElement === lastElement) {
          e.preventDefault();
          firstElement.focus();
        }
      }
    });
  }
}, [lightboxOpen]);

Mobile Swipe Support

// Add touch swipe detection on mobile
useEffect(() => {
  let touchStartX = 0;
  let touchEndX = 0;

  const handleTouchStart = (e: TouchEvent) => {
    touchStartX = e.changedTouches[0].screenX;
  };

  const handleTouchEnd = (e: TouchEvent) => {
    touchEndX = e.changedTouches[0].screenX;
    handleSwipe();
  };

  const handleSwipe = () => {
    const swipeThreshold = 50; // Min distance for swipe

    if (touchEndX < touchStartX - swipeThreshold) {
      // Swipe left → next image
      handleImageSwitch(activeIndex < images.length - 1 ? activeIndex + 1 : 0);
    }

    if (touchEndX > touchStartX + swipeThreshold) {
      // Swipe right → prev image
      handleImageSwitch(activeIndex > 0 ? activeIndex - 1 : images.length - 1);
    }
  };

  const mainImg = document.querySelector('.main-img');
  if (mainImg && window.innerWidth < 640) {
    mainImg.addEventListener('touchstart', handleTouchStart as any);
    mainImg.addEventListener('touchend', handleTouchEnd as any);

    return () => {
      mainImg.removeEventListener('touchstart', handleTouchStart as any);
      mainImg.removeEventListener('touchend', handleTouchEnd as any);
    };
  }
}, [activeIndex, images.length]);

Performance Optimization

  • Image loading: Use Next.js Image component with priority on main image, loading="lazy" on thumbnails
  • Thumbnail sizes: Generate 100×100px thumbnails server-side, don't load full images
  • Lightbox preload: Preload next/prev images in lightbox for instant navigation
  • Sticky performance: Use CSS position: sticky (no JavaScript)
  • Zoom transform: Use CSS transform (GPU-accelerated), not width/height

Common Pitfalls

  • ❌ Large images: Loading 5× full-resolution images kills performance — use thumbnails!
  • ❌ Missing alt text: Every image needs descriptive alt text for accessibility
  • ❌ No keyboard nav: Lightbox must support arrow keys + ESC to close
  • ❌ Focus trap broken: Tab key must cycle only within lightbox when open
  • ❌ Body scroll leak: Prevent <body> scroll when lightbox is open
  • ❌ Sticky too high: Gallery disappears under header — adjust top offset
  • ❌ Mobile zoom: Disable zoom on mobile (not useful on small screens)

STEP 1: Component Overview

Component ID: PD03

Name: ProductTabs

Category: Product Detail / Content Organization

Version: 1.0.0

Description

Tabbed content navigation for product detail pages. Organizes product information into logical sections (description, specifications, reviews, FAQ) with horizontal tabs on desktop and accordion layout on mobile.

Key Features

  • ✓ Horizontal tab navigation with active state (teal underline + text)
  • ✓ Tab content panels with smooth fade-in animation
  • ✓ Badge support (review counts, notification indicators)
  • ✓ ARIA tabs pattern (full accessibility support)
  • ✓ Keyboard navigation (arrow keys, Home/End)
  • ✓ Mobile accordion mode (<768px)
  • ✓ URL hash support for deep linking (#tab-reviews)
  • ✓ Lazy loading support (load content on tab activation)
  • ✓ 3 style variants: Default (underline), Pills (rounded), Boxed (with background)

STEP 2: When to Use This Component

✅ Use ProductTabs when:

  • Building product detail pages with multiple content sections (description, specs, reviews, FAQ)
  • Organizing long-form product information without overwhelming the page
  • Creating service pages with multiple informational sections
  • Displaying user-generated content (reviews, Q&A) alongside product details
  • Building comparison tables or technical specifications that need progressive disclosure
  • Implementing documentation or knowledge base layouts

❌ Don't use ProductTabs when:

  • Content is short enough to display all at once (use simple sections instead)
  • User needs to compare content across tabs simultaneously (use split layout)
  • Content has strict hierarchical navigation (use sidebar nav instead)
  • Building multi-step forms (use ProgressSteps/Wizard instead)

Related Components

  • PD04: ProductSpecsTable — Use inside "Specificaties" tab for structured attribute display
  • C10: ReviewWidget — Use inside "Reviews" tab for review display with ratings
  • C25: ProgressSteps — Alternative for multi-step processes (checkout, forms)
  • C24: AccountSidebar — Alternative for vertical navigation in account sections

STEP 6: Accessibility (WCAG 2.1 Level AA)

ARIA Tabs Pattern

Implements the complete W3C ARIA Tabs Design Pattern:

  • role="tablist" on tab navigation container
  • role="tab" on each tab button
  • role="tabpanel" on each content panel
  • aria-selected="true|false" on tab buttons (active state)
  • aria-controls="panel-id" on tabs (links to panel)
  • aria-labelledby="tab-id" on panels (links back to tab)
  • id attributes on all tabs and panels for ARIA relationships

Keyboard Navigation

Key Action
Tab Move focus into/out of tab list
→ (Right Arrow) Activate next tab (wraps to first)
← (Left Arrow) Activate previous tab (wraps to last)
Home Activate first tab
End Activate last tab
Space / Enter Activate focused tab (if not auto-activated)

Focus Management

  • Focus ring: box-shadow: 0 0 0 3px var(--teal-glow) on tab buttons
  • Visible focus: Focus ring appears on all interactive elements (tabs, accordion headers)
  • Focus order: Tab buttons are in DOM order (left-to-right)
  • Focus trap: None (users can Tab out of tab list freely)

Screen Reader Support

  • Screen readers announce: "Beschrijving, tab, 1 of 4, selected"
  • Badge counts are included in tab label (e.g., "Reviews 214")
  • Panel content is announced when tab is activated
  • Mobile accordion: Headers announce as buttons with expanded/collapsed state

Color Contrast (WCAG AA)

  • Inactive tabs: #94A3B8 on #FAFBFC = 4.51:1 ✅
  • Active tabs: #00897B on #FAFBFC = 5.12:1 ✅
  • Tab content: #64748B on #FAFBFC = 6.24:1 ✅
  • Headings: #0A1628 on #FAFBFC = 12.89:1 ✅

Touch Targets (Mobile)

  • Minimum size: 40×40px (tab buttons and accordion headers)
  • Spacing: 4px gap between tap targets (prevents mis-taps)
  • Active area: Full button area (not just text) is clickable

  if (!isLoaded) {

React/Next.js Implementation

'use client';

import { useState, useEffect } from 'react';
import { useSearchParams, useRouter } from 'next/navigation';

export function ProductTabs({ tabs, defaultTab }) {
  const [activeTab, setActiveTab] = useState(defaultTab || tabs[0].id);
  const searchParams = useSearchParams();
  const router = useRouter();

  // URL hash navigation
  useEffect(() => {
    const hash = window.location.hash.substring(1);
    if (hash && tabs.find(t => t.id === hash)) {
      setActiveTab(hash);
    }
  }, [searchParams]);

  const handleTabClick = (tabId) => {
    setActiveTab(tabId);
    router.push(`#${tabId}`, { scroll: false });
  };

  return (
    <div className="tabs-section">
      <div className="tabs-nav" role="tablist">
        {tabs.map(tab => (
          <button
            key={tab.id}
            className={`tab-btn ${activeTab === tab.id ? 'active' : ''}`}
            role="tab"
            aria-selected={activeTab === tab.id}
            aria-controls={`panel-${tab.id}`}
            id={`tab-${tab.id}`}
            onClick={() => handleTabClick(tab.id)}
          >
            {tab.label}
            {tab.badge && <span className="tab-badge">{tab.badge}</span>}
          </button>
        ))}
      </div>

      {tabs.map(tab => (
        <div
          key={tab.id}
          className={`tab-panel ${activeTab === tab.id ? 'active' : ''}`}
          role="tabpanel"
          id={`panel-${tab.id}`}
          aria-labelledby={`tab-${tab.id}`}
        >
          {tab.content}
        </div>
      ))}
    </div>
  );
}

Performance Optimization

  • Lazy load heavy content: Reviews, technical specs, embedded videos
  • CSS animations: Use transforms (GPU-accelerated) for fade-in effects
  • Debounce resize: Only update mobile accordion on final resize event
  • Preload critical tabs: Load "Beschrijving" content server-side, lazy load others

Common Pitfalls

  • ❌ Missing ARIA attributes: Tabs won't work with screen readers
  • ❌ No keyboard nav: Users can't navigate with arrow keys (required by ARIA pattern)
  • ❌ Hiding inactive panels with display:none: Breaks keyboard focus order (use visibility or opacity instead)
  • ❌ Fixed tab button widths: Content may overflow or wrap unexpectedly
  • ❌ Not testing mobile accordion: Transitions may break on iOS Safari

Browser Support

  • ✅ Chrome/Edge 90+
  • ✅ Firefox 88+
  • ✅ Safari 14+ (iOS 14+)
  • ✅ All modern mobile browsers
  • ⚠️ IE11: Requires polyfills for Array.from(), classList

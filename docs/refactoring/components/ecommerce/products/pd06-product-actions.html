STEP 1: Component Overview

Purpose

PD06: ProductActions is the primary conversion component on product pages. It combines three critical purchase actions in one cohesive row: (1) quantity selection, (2) add to cart CTA, and (3) wishlist toggle. This component is the culmination of the product page flow — everything above it (gallery, meta, variants) leads to this moment where the user commits to purchasing.

Key Features

  • Quantity Stepper: −/+ buttons with numeric input, enforces min/max limits
  • Add to Cart Button: Large teal CTA with shopping cart icon, shows current price
  • Wishlist Button: Heart icon toggle for saving products, coral hover state
  • Stock Validation: Disables buttons when out of stock, shows warnings when low
  • Loading State: Spinner animation during API calls
  • Success Feedback: Green state + icon change when item added
  • Dynamic Pricing: Price updates based on quantity × unit price (for variable products)
  • Responsive Layout: Stacks on mobile (<640px), single row on desktop

STEP 2: When to Use This Component

✅ Use ProductActions When:

  • Product Detail Pages: Primary placement below variant selectors + config summary
  • Quick View Modals (C5): Use compact variant (44px height)
  • Sticky Add-to-Cart Bar (C6): Mobile sticky bar at bottom of screen
  • Product Configurators: Below final configuration summary

❌ Don't Use ProductActions For:

STEP 4: Props & Configuration

React/Next.js Component Props

interface ProductActionsProps {
  // Product data
  product: {
    id: string;                       // Product ID
    price: number;                    // Unit price in cents (e.g., 15995 = €159.95)
    stock: {
      status: 'ok' | 'low' | 'out';  // Stock availability
      quantity?: number;              // Current stock count (for validation)
    };
  };

  // Optional configuration
  minQuantity?: number;               // Min qty (default: 1)
  maxQuantity?: number;               // Max qty (default: 99, or stock quantity)
  initialQuantity?: number;           // Starting qty (default: 1)
  showTotalPrice?: boolean;           // Show total price below (default: true)
  variant?: 'default' | 'compact';    // Size variant (default: 'default')

  // Callbacks
  onAddToCart?: (productId: string, quantity: number) => Promise;
  onWishlistToggle?: (productId: string, isActive: boolean) => void;
  onQuantityChange?: (quantity: number) => void;

  // Optional customization
  addButtonText?: string;             // Custom CTA text (default: "In winkelwagen")
  wishlistEnabled?: boolean;          // Show/hide wishlist button (default: true)
  className?: string;                 // Additional CSS classes
}

State Management

const [quantity, setQuantity] = useState(initialQuantity || 1);
const [isWishlisted, setIsWishlisted] = useState(false);
const [isLoading, setIsLoading] = useState(false);
const [isSuccess, setIsSuccess] = useState(false);

// Quantity validation
const handleIncrement = () => {
  const max = maxQuantity || product.stock.quantity || 99;
  if (quantity < max) {
    setQuantity(q => q + 1);
    onQuantityChange?.(quantity + 1);
  }
};

const handleDecrement = () => {
  if (quantity > minQuantity) {
    setQuantity(q => q - 1);
    onQuantityChange?.(quantity - 1);
  }
};

// Add to cart with loading state
const handleAddToCart = async () => {
  setIsLoading(true);
  try {
    await onAddToCart(product.id, quantity);
    setIsSuccess(true);
    setTimeout(() => setIsSuccess(false), 2000);
  } catch (error) {
    console.error('Failed to add to cart', error);
  } finally {
    setIsLoading(false);
  }
};

// Wishlist toggle
const handleWishlistToggle = () => {
  const newState = !isWishlisted;
  setIsWishlisted(newState);
  onWishlistToggle?.(product.id, newState);
};

ARIA Attributes

  • role="group" on purchase row and qty stepper
  • aria-label on all buttons (descriptive action text)
  • aria-pressed on wishlist button (toggle state)
  • aria-hidden="true" on decorative icons
  • aria-live="polite" on stock warnings (announces changes)
  • disabled attribute when stock is out or qty limits reached

Keyboard Navigation

  • Tab order: − button → input → + button → Add to cart → Wishlist
  • − button: Activates with Enter/Space
  • + button: Activates with Enter/Space
  • Qty input: Read-only, no direct editing (prevents invalid values)
  • Add to cart: Activates with Enter/Space
  • Wishlist: Activates with Enter/Space, announces state change

function announceToSR(message) {
  const liveRegion = document.querySelector('[aria-live="polite"]');
  liveRegion.textContent = message;
  setTimeout(() => liveRegion.textContent = '', 1000);
}

STEP 7: Implementation Notes

Stock Validation Logic

// Determine max quantity based on stock
const getMaxQuantity = (product) => {
  if (product.stock.status === 'out') return 0;
  if (product.stock.status === 'low' && product.stock.quantity) {
    return product.stock.quantity;
  }
  return 99; // Default max for 'ok' status
};

// Validate before adding to cart
const handleAddToCart = async () => {
  // Check stock availability
  if (product.stock.status === 'out') {
    showError('Dit product is tijdelijk uitverkocht');
    return;
  }

  // Check quantity vs stock
  const max = getMaxQuantity(product);
  if (quantity > max) {
    showError(`Nog maar ${max} stuks beschikbaar`);
    setQuantity(max);
    return;
  }

  // Proceed with add to cart
  setIsLoading(true);
  await onAddToCart(product.id, quantity);
  setIsLoading(false);
  setIsSuccess(true);
};

Dynamic Price Calculation

// For variable products, recalculate total when qty or variant changes
const calculateTotalPrice = (basePrice, selectedVariants, quantity) => {
  let unitPrice = basePrice;

  // Add variant price modifiers
  selectedVariants.forEach(variant => {
    if (variant.priceModifier) {
      unitPrice += variant.priceModifier; // e.g., +1000 (= +€10)
    }
  });

  // Apply volume pricing (staffel)
  const volumeDiscount = getVolumeDiscount(unitPrice, quantity);
  if (volumeDiscount) {
    unitPrice = volumeDiscount.unitPrice;
  }

  return unitPrice * quantity;
};

// Format price for display
const formatPrice = (cents) => {
  return `€ ${(cents / 100).toFixed(2).replace('.', ',')}`;
};

// Update button text with current price
useEffect(() => {
  const total = calculateTotalPrice(product.price, selectedVariants, quantity);
  setButtonText(`In winkelwagen — ${formatPrice(total)}`);
}, [quantity, selectedVariants]);

Wishlist Integration

const handleWishlistToggle = async () => {
  const newState = !isWishlisted;
  setIsWishlisted(newState);

  // Optimistic update (instant visual feedback)
  try {
    if (newState) {
      // Add to wishlist
      await fetch('/api/wishlist/add', {
        method: 'POST',
        body: JSON.stringify({ productId: product.id })
      });
      showToast({ type: 'success', message: 'Toegevoegd aan verlanglijst' });
    } else {
      // Remove from wishlist
      await fetch('/api/wishlist/remove', {
        method: 'DELETE',
        body: JSON.stringify({ productId: product.id })
      });
      showToast({ type: 'info', message: 'Verwijderd van verlanglijst' });
    }
  } catch (error) {
    // Revert on error
    setIsWishlisted(!newState);
    showError('Er ging iets mis');
  }
};

Common Pitfalls

  • ❌ No stock validation: Always check stock before adding to cart (server-side too!)
  • ❌ Editable qty input: Keep input readonly to prevent invalid values (use buttons only)
  • ❌ No loading state: Users will double-click if no visual feedback → duplicate cart items
  • ❌ Missing price updates: For variable products, recalculate price on variant/qty change
  • ❌ No error handling: API can fail — always show error message to user
  • ❌ Blocking UI during load: Disable only the add button, not qty/wishlist
  • ❌ No success feedback: Show toast + open minicart so user knows item was added

📋 Component Documentation

STEP 1: Component Overview

Component ID: PD04

Name: ProductSpecsTable

Category: Product Detail / Data Display

Version: 1.0.0

Description

Two-column specifications table for displaying product attributes in a structured format. Supports collapsible groups, icon-enriched labels, copy-to-clipboard functionality, and responsive mobile layout. Designed for use inside the "Specificaties" tab of product detail pages.

Key Features

  • ✓ 2-column grid layout (label + value) with responsive stacking
  • ✓ Collapsible groups with smooth expand/collapse animation
  • ✓ Icon support per attribute (Lucide icons, 16px)
  • ✓ Copy-to-clipboard for SKU/EAN/codes (hover-revealed button)
  • ✓ Badge indicators (certified, warning, info)
  • ✓ Alternating row colors (zebra striping)
  • ✓ Highlight support for key specifications
  • ✓ Link support for related values (brands, categories)
  • ✓ List value support (multiple items in one row)
  • ✓ 3 variants: Default (grouped), Compact (smaller), Simple (flat)

Design Source

File: NOT IN PROTOTYPE (built from scratch)

Design System: Compass Design System tokens applied

Inspiration: Amazon product details, Apple tech specs tables

STEP 2: When to Use This Component

✅ Use ProductSpecsTable when:

  • Displaying structured product attributes (SKU, dimensions, materials, certifications)
  • Building the "Specificaties" tab inside PD03: ProductTabs
  • Presenting technical specifications for electronics, medical devices, or industrial products
  • Showing compliance data (CE marks, ISO certifications, safety warnings)
  • Organizing large sets of product metadata (10+ attributes)
  • Providing copyable reference codes (SKU, EAN, part numbers)

❌ Don't use ProductSpecsTable when:

  • Displaying tabular data (use standard <table> with thead/tbody instead)
  • Showing comparison data across multiple products (use comparison table component)
  • Presenting narrative product information (use plain text paragraphs in Description tab)
  • Building forms or editable data (use form components instead)

Related Components

  • PD03: ProductTabs — ProductSpecsTable goes inside "Specificaties" tab
  • PD02: ProductMeta — Displays high-level product info (price, rating, stock) above tabs
  • EC04: StockIndicator — Can be embedded in specs table for stock status
  • C18: ProductBadges — Certification badges (CE, ISO) can be used in specs values

STEP 5: Layout Rules

Grid System

  • Layout: CSS Grid with 2 columns (grid-template-columns: 200px 1fr)
  • Label column: Fixed 200px width (160px in compact variant)
  • Value column: Flexible (1fr) to fill remaining space
  • Mobile: Stacks to 1 column (grid-template-columns: 1fr) on <768px

Group Spacing

  • Between groups: 24px margin-bottom
  • Group header: 12px vertical padding, 2px border-bottom (grey)
  • Header → Table: 12px margin-top

Row Styling

  • Cell padding: 12px vertical, 16px horizontal (desktop)
  • Row border: 1px solid var(--grey) at bottom
  • Zebra striping: Even rows have light grey background (rgba(245,247,250,0.5))
  • Hover state: All rows get grey-light background on hover

Label Cell

  • Layout: Flex row with 8px gap (icon + text)
  • Icon size: 16px (flex-shrink: 0 to prevent squishing)
  • Font: 13px, 600 weight, grey-dark color
  • Border: 1px solid var(--grey) on right side (desktop only)

Value Cell

  • Layout: Flex row with space-between (value + copy button)
  • Font: 14px, 400 weight, navy color
  • Mono variant: JetBrains Mono, 13px, 500 weight, teal color
  • Copy button: Opacity 0 by default, opacity 1 on row hover (always visible on mobile)

Responsive Breakpoints

// 2. Copy to Clipboard
async function copyToClipboard(text, button) {
  try {
    await navigator.clipboard.writeText(text);
    button.classList.add('copied');
    button.innerHTML = '✓ Gekopieerd!';

    setTimeout(() => {
      button.classList.remove('copied');
      button.innerHTML = 'Kopiëren';
    }, 2000);
  } catch (err) {
    console.error('Copy failed:', err);
  }
}

React/Next.js Implementation

'use client';

import { useState } from 'react';
import * as Icons from 'lucide-react';

export function ProductSpecsTable({ groups, variant = 'default' }) {
  const [collapsedGroups, setCollapsedGroups] = useState<Set<string>>(new Set());
  const [copiedValue, setCopiedValue] = useState<string | null>(null);

  const toggleGroup = (groupId: string) => {
    setCollapsedGroups(prev => {
      const next = new Set(prev);
      if (next.has(groupId)) {
        next.delete(groupId);
      } else {
        next.add(groupId);
      }
      return next;
    });
  };

  const copyToClipboard = async (text: string, id: string) => {
    await navigator.clipboard.writeText(text);
    setCopiedValue(id);
    setTimeout(() => setCopiedValue(null), 2000);
  };

  return (
    <div className={`specs-table-wrapper ${variant}`}>
      {groups.map(group => (
        <div
          key={group.id}
          className={`specs-group ${collapsedGroups.has(group.id) ? 'collapsed' : ''}`}
        >
          <div
            className="specs-group-header"
            onClick={() => toggleGroup(group.id)}
          >
            <div className="specs-group-title">
              {group.icon && <Icon name={group.icon} className="specs-group-icon" />}
              <span>{group.title}</span>
            </div>
            <Icons.ChevronDown className="specs-group-toggle" />
          </div>

          <div className="specs-table">
            {group.specs.map((spec, i) => (
              <div key={i} className={`specs-row ${spec.highlight ? 'highlight' : ''}`}>
                <div className="specs-label">
                  {spec.icon && <Icon name={spec.icon} className="specs-label-icon" />}
                  <span>{spec.label}</span>
                </div>
                <div className={`specs-value ${spec.type === 'mono' ? 'mono' : ''}`}>
                  {spec.value}
                  {spec.copyable && (
                    <button
                      className={`specs-copy-btn ${copiedValue === spec.label ? 'copied' : ''}`}
                      onClick={() => copyToClipboard(spec.value, spec.label)}
                    >
                      <Icons.Copy className="specs-copy-icon" />
                      {copiedValue === spec.label ? 'Gekopieerd!' : 'Kopiëren'}
                    </button>
                  )}
                </div>
              </div>
            ))}
          </div>
        </div>
      ))}
    </div>
  );
}

Common Pitfalls

  • ❌ Using <table> element: Harder to style responsively (use CSS Grid instead)
  • ❌ Not testing mobile layout: Stacked layout may break with long values
  • ❌ Copy button always visible: Clutters desktop layout (use hover-reveal)
  • ❌ No clipboard API fallback: Older browsers may not support navigator.clipboard
  • ❌ Missing Lucide.createIcons(): Icons won't render after dynamic updates

Browser Support

  • ✅ Chrome/Edge 90+
  • ✅ Firefox 88+
  • ✅ Safari 14+ (iOS 14+)
  • ✅ All modern mobile browsers
  • ⚠️ Clipboard API: Requires HTTPS (or localhost)
  • ⚠️ IE11: Requires polyfills for CSS Grid, clipboard API

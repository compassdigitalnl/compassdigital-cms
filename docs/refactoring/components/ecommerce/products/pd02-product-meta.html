📋 PD02: ProductMeta — Complete Documentation

Product detail page header component with category, title, rating, description, price, and stock status.
Version: 1.0 | Last updated: 25 Februari 2026 | Status: ✅ Production Ready

STEP 1: Component Overview

Purpose

PD02: ProductMeta is the primary information block displayed at the top of every product detail page. It communicates essential product information at a glance: what the product is (title + category), how much it costs (price + discount), what others think of it (rating + review count), and whether it's available (stock status).

Key Features

  • Category Breadcrumb: Uppercase teal label (e.g., "Schoenen · Custom")
  • Product Title: Large, bold H1 (28px, 800 weight) with excellent readability
  • Star Rating: Visual ★★★★★ display with numeric score (e.g., "4.8") and clickable review count
  • Short Description: 2-3 line product summary in readable grey text
  • Stock Indicator: 3 states (in stock/low stock/out of stock) with color-coded dots
  • Price Display: Current price (large, bold), optional original price (strikethrough), discount percentage badge
  • Price Note: Small grey text for BTW, shipping info, volume pricing hints
  • Trust Badges: Optional inline badges (free shipping, warranty, certifications)

STEP 2: When to Use This Component

✅ Use ProductMeta When:

  • Product Detail Pages: Every single product page requires this header
  • Quick View Modals: Condensed version in lightbox product previews
  • Product Comparison: Show meta info side-by-side for 2-4 products
  • Admin/CMS Previews: Display product info in backend interface

❌ Don't Use ProductMeta For:

  • Product Cards: Use EC01: ProductCard instead (compact grid/list layout)
  • Cart Items: Use EC06: CartLineItem (shows selected variant + quantity)
  • Search Results: Use EC01 with list variant for shop/archive pages
  • Related Products: Use carousel with EC01 cards, not full meta

Related Components:

  • EC01: ProductCard — Compact card for grids/lists (uses subset of this styling)
  • EC04: StockIndicator — Reusable stock status badge (integrated here)
  • C18: ProductBadges — Sale/New/Pro badges (shown in gallery, not meta)
  • C10: ReviewWidget — Full review section (rating here links to it)
  • VP01-VP07: Variant selectors (placed below ProductMeta)
  • PD06: ProductActions — Add to cart + wishlist (placed below variants)

React/Next.js Component Props

interface ProductMetaProps {
  // Required
  product: {
    title: string;                    // Product name (H1)
    price: number;                    // Current price in cents (e.g., 15995 = €159.95)
    category: string;                 // Category breadcrumb (e.g., "Schoenen · Custom")

    // Optional
    shortDescription?: string;        // 2-3 line product summary
    priceOriginal?: number;           // Original price (if on sale)
    priceNote?: string;               // BTW/shipping info below price

    // Rating
    rating?: {
      value: number;                  // Average rating (0-5, e.g., 4.8)

      reviewsUrl?: string;            // Link to reviews section (default: "#reviews")
    };

    // Stock
    stock?: {
      status: 'ok' | 'low' | 'out';  // Stock level
      quantity?: number;              // Current stock count (for "low" status)
      message?: string;               // Custom stock message
      expectedDate?: string;          // Restock date (for "out" status)
    };

    // Trust badges
    trustBadges?: Array<{
      icon: string;                   // Lucide icon name
      label: string;                  // Badge text
    }>;
  };

  // Optional styling
  className?: string;                 // Additional CSS classes
  showTrustBadges?: boolean;          // Show/hide trust badges (default: true)
  variant?: 'default' | 'compact';    // Layout variant
}

Dynamic Price Calculation

For variable products (products with multiple variants at different prices), the price should update dynamically when the user selects a variant. Use JavaScript to calculate: 

// Example: Update price when color swatch is selected
function updateProductPrice(basePrice, selectedVariants) {
  let totalPrice = basePrice;

  // Add variant price modifiers
  selectedVariants.forEach(variant => {
    if (variant.priceModifier) {
      totalPrice += variant.priceModifier;  // e.g., +€10 for premium color
    }
  });

  // Update price display
  const priceElement = document.querySelector('.price-current');
  priceElement.textContent = formatPrice(totalPrice);

  // If variable product with range, show "Vanaf €X"
  if (hasMultiplePricePoints) {
    priceElement.textContent = `Vanaf ${formatPrice(lowestPrice)}`;
  }
}

function formatPrice(cents) {
  return `€ ${(cents / 100).toFixed(2).replace('.', ',')}`;
}

ARIA Attributes

  • aria-label on category div ("Product category")
  • aria-hidden="true" on decorative star emojis (screen readers use numeric value instead)

  • role="region" + aria-label on price display
  • aria-label on price elements for clarity ("Huidige prijs", "Oorspronkelijke prijs")

Screen Reader Announcements

When stock status changes (e.g., via AJAX after variant selection):

function updateStockStatus(status, quantity) {
  const indicator = document.querySelector('.stock-indicator');

  // Update visual state
  indicator.className = `stock-indicator ${status}`;

  // Update text content (announces via aria-live="polite")
  if (status === 'ok') {
    indicator.innerHTML = '<span class="stock-dot"></span> Op voorraad';
  } else if (status === 'low') {
    indicator.innerHTML = `<span class="stock-dot"></span> Nog ${quantity} stuks op voorraad`;
  } else if (status === 'out') {
    indicator.innerHTML = '<span class="stock-dot"></span> Tijdelijk uitverkocht';
  }

  // Screen reader will announce: "Nog 12 stuks op voorraad"
}

    return { className: status, message: '' };

Performance Optimization

  • Fonts: Preload Plus Jakarta Sans 800 and DM Sans 400 (most used weights)
  • Icons: Load Lucide icons lazily or use static SVGs for trust badges
  • Price updates: Debounce dynamic price calculations (300ms) when variants change
  • Hydration: Mark as client component only if interactive (rating link is fine as static)
  • Images: If trust badges use images instead of icons, lazy load them

Common Pitfalls

  • ❌ Multiple H1s: Only use ONE H1 per page (the product title)
  • ❌ Hardcoded prices: Always calculate from cents to avoid floating-point errors
  • ❌ Missing aria-labels: Screen readers need context for ratings and prices
  • ❌ No stock updates: If stock changes via AJAX, update aria-live region
  • ❌ Wrong discount calculation: Round percentage to nearest integer, don't show decimals
  • ❌ Broken review link: Always provide fallback (#reviews) if reviewsUrl is missing

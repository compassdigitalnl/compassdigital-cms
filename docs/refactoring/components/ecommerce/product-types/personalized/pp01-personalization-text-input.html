📋 STEP 1: Component Overview

PP01: PersonalizationTextInput is a text input component for product personalization. Features real-time character counting, validation, helper text, and optional/required states. Perfect for engraving, monogramming, custom messages, and any text-based product customization.

Key Features

  • ✅ Character Counter: Live counter showing current/max characters (e.g., "5/12")
  • ✅ Color-Coded States: Grey (default), teal (focused), amber (near limit), coral (at limit/error)
  • ✅ Validation: Real-time validation with error messages
  • ✅ Helper Text: Instructions + price (free or paid)
  • ✅ Optional/Required: Visual indicators for field importance
  • ✅ Icon Prefix: Optional icon on left side of input
  • ✅ Textarea Variant: For longer messages (200+ characters)
  • ✅ Responsive: 2-col → 1-col on mobile

Use Cases

  • ✍️ Engraving (names, dates, messages)
  • 🎁 Gift messages & greeting cards
  • 👕 Custom text on products (t-shirts, mugs, jewelry)
  • 🏷️ Monogramming & initials
  • 📝 Custom labels & dedications
  • 💌 Personal notes & inscriptions

🎯 STEP 2: When to Use This Component

✅ Use VariantTextInput When:

  • Text personalization needed — Products allow custom text (engraving, printing)
  • Character limits apply — Production constraints (12 chars for engraving, 200 for cards)
  • Validation required — Certain characters forbidden or minimum length needed
  • Price varies — Personalization is free or costs extra (show pricing)
  • Optional or required — Some products require personalization, others don't
  • Multiple fields — Left/right product, first/last name, line 1/2/3

❌ Don't Use When:

  • Predefined options only — Use VP03 (Dropdown) or VP04 (Image Radio) for preset choices
  • No character limit — Use standard form inputs if no validation needed
  • Complex configuration — Use PC01-PC08 (Product Configurator) for multi-step wizards
  • File upload needed — Use PP04 (PersonalizationImageUpload) for custom images/logos

Character Counter Logic

function updateCounter(input: HTMLInputElement) { const current = input.value.length; const max = input.maxLength; const counter = input.parentElement.querySelector('.cf-counter span'); const counterDiv = input.parentElement.querySelector('.cf-counter'); if (counter) { counter.textContent = current.toString(); } if (counterDiv) { counterDiv.classList.remove('near-limit', 'at-limit'); const percentage = (current / max) * 100; if (percentage >= 100) { counterDiv.classList.add('at-limit'); } else if (percentage >= 75) { counterDiv.classList.add('near-limit'); } } }

Default Values

  • Columns: 2 (auto-fit, 240px min per field)
  • Optional: true (show "Optioneel" badge)
  • MaxLength: 12 (common engraving limit)
  • Required: false (optional by default)
  • Price: 0 (free personalization)

Keyboard Navigation

  • Tab: Focus moves between input fields
  • Typing: Updates character counter in real-time
  • Backspace/Delete: Decreases counter
  • Paste: Validates max length (browser handles truncation)

Screen Reader Support

  • Labels: Explicit <label for="..."> association
  • Required fields: aria-required="true" + visual asterisk with aria-label
  • Counter: aria-live="polite" announces changes (not too disruptive)
  • Error messages: aria-invalid="true" + aria-describedby="error-id"
  • Helper text: Linked via aria-describedby

Form Validation

// Client-side validation example function validateField(input: HTMLInputElement): boolean { const value = input.value; const minLength = 3; // Example minimum const errorDiv = input.parentElement.nextElementSibling; if (input.required && value.length < minLength) { input.classList.add('error'); input.setAttribute('aria-invalid', 'true'); if (errorDiv) { errorDiv.textContent = `Minimaal ${minLength} karakters vereist`; errorDiv.classList.add('cf-error'); } return false; } input.classList.remove('error'); input.setAttribute('aria-invalid', 'false'); return true; }

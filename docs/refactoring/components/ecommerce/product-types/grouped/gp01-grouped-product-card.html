📦 STEP 1: Component Overview

Component: GP01: GroupedProductCard

Purpose: Vertical product card with checkbox for multi-select in grouped product grid view. Allows individual product selection and purchase from a related product set.

Key Features:

  • White card with 1.5px grey border, 14px border-radius
  • Checkbox top-left (22×22px, 6px radius) for multi-select
  • Selected state: teal border + light teal background (rgba 0.02)
  • Product icon/emoji (64×64px, grey background)
  • Badges (Popular, New) with purple/teal variants
  • Product name (16px, 700 weight) + SKU (mono, 11px grey)
  • Price: current + old price (strikethrough) optional
  • Stock indicator: green (in-stock), amber (low-stock), coral (out-of-stock)
  • Individual add-to-cart button (outline style, full-width)
  • Out-of-stock state: greyscale filter + disabled checkbox + disabled button
  • Hover: teal border + shadow + translateY(-2px)
  • Keyboard accessible (checkbox tab-focusable, Space to toggle)

Visual Description: Vertical card with checkbox in top-left corner for multi-selection. Product displayed with icon, name, price, and stock status. Button at bottom for individual add-to-cart. Selected cards get teal accent and light background.

📋 STEP 2: When to Use

Primary Use Cases:

  • Grouped Product Sets: Display related products that can be purchased individually or together
  • Multi-Select Shopping: Allow customers to select multiple items before bulk add-to-cart
  • Complete Sets with Discounts: Show individual items in a set (e.g., 6-piece tool kit)
  • Product Bundles: Alternative to Bundle Builder when items are optional
  • Grid View Mode: Desktop/tablet view with 2-4 column grid

When NOT to Use:

  • ❌ For list view (use GP02: GroupedProductRow instead)
  • ❌ For single-product pages (use EC01: ProductCard)
  • ❌ For forced bundles (use Bundle Builder instead)
  • ❌ For subscription products (use SP04: SubscriptionProductCard)
  • ❌ For variant selection (use Variable Product components)

STEP 1: Component Overview

Component: MixMatchProgressCounter

Category: Product Types → Mix & Match

Purpose: Visual progress indicator showing how many items have been selected out of the required total for a Mix & Match box. Motivates completion and provides immediate feedback on selection status.

Key Features:

  • Horizontal progress bar with animated fill (teal → green when complete)
  • Large numeric counter displaying current/max items (e.g., "3 / 6")
  • Current count highlighted in teal (or green when complete)
  • Descriptive label ("items geselecteerd")
  • Smooth 0.3s animation when progress changes
  • Responsive: horizontal layout on desktop, vertical stack on mobile

Visual Description:

A white card with rounded corners contains three elements: a progress bar (left, takes most space), a bold numeric counter (middle), and a small text label (right). The progress bar shows a teal fill representing completion percentage. When the box is complete, the fill turns green and the counter number becomes green.

STEP 2: When to Use

Use this component when:

  • Mix & Match product pages — Primary use case for build-your-own boxes with fixed item counts
  • Bundle builders — When users must select exactly X items from available options
  • Gift box configurators — "Choose 8 products for your gift set"
  • Sample packs — "Select 5 samples to try" promotions
  • Subscription box customization — Monthly boxes where users pick their items
  • Multi-step forms — As a secondary indicator in addition to step numbers

Do NOT use when:

  • No fixed item count required (use simple counter instead)
  • Minimum/maximum range (e.g., "3-6 items") — progress bar implies exact target
  • Percentage-based progress unrelated to item count (use ProgressSteps C25 instead)
  • Shopping cart — this is for pre-purchase customization, not cart totals

ARIA Attributes:

  • role="progressbar" on .box-progress — Identifies as progress indicator
  • aria-valuenow — Current value (e.g., 3)
  • aria-valuemin — Minimum value (always 0)
  • aria-valuemax — Maximum value (e.g., 6)
  • aria-label — Full description: "3 of 6 items selected"
  • aria-hidden="true" on visual elements (bar, counter, label) — Prevents redundant screen reader announcements

Keyboard Navigation:

  • No direct keyboard interaction (informational only)
  • Focus moves naturally past component to next interactive element

Screen Reader Announcements:

  • On mount: "Progress bar, 3 of 6 items selected"
  • On update: Use ARIA live region to announce changes: aria-live="polite"
  • Example: When current changes from 3→4: "4 of 6 items selected"
  • On complete: "6 of 6 items selected, complete"

STEP 7: Implementation Notes

React/Next.js Component Example:

import React, { useMemo } from 'react';

interface MixMatchProgressCounterProps {
  current: number;
  max: number;
  label?: string;
  showLabel?: boolean;
  variant?: 'default' | 'compact';
  className?: string;
  onChange?: (current: number, max: number, percentage: number) => void;
}

export const MixMatchProgressCounter: React.FC<MixMatchProgressCounterProps> = ({
  current,
  max,
  label = 'items geselecteerd',
  showLabel = true,
  variant = 'default',
  className = '',
  onChange,
}) => {
  // Calculate percentage
  const percentage = useMemo(() => {
    const pct = Math.min(100, Math.max(0, (current / max) * 100));
    onChange?.(current, max, pct);
    return pct;
  }, [current, max, onChange]);

  const isComplete = current >= max;

  return (
    <div
      className={`box-progress ${className}`}
      role="progressbar"
      aria-valuenow={current}
      aria-valuemin={0}
      aria-valuemax={max}
      aria-label={`${current} of ${max} items selected${isComplete ? ', complete' : ''}`}
      aria-live="polite"
      aria-atomic="true"
    >
      <div className="bp-bar">
        <div
          className={`bp-fill ${isComplete ? 'full' : ''}`}
          style={{ width: `${percentage}%` }}
          aria-hidden="true"
        />
      </div>
      <div className={`bp-count ${isComplete ? 'complete' : ''}`} aria-hidden="true">
        <em>{current}</em> / {max}
      </div>
      {showLabel && (
        <div className="bp-label" aria-hidden="true">
          {label}
        </div>
      )}
    </div>
  );
};

State Management:

// Example with Zustand store
interface MixMatchStore {
  selectedItems: Product[];
  maxItems: number;
  addItem: (product: Product) => void;
  removeItem: (productId: string) => void;
  isComplete: () => boolean;
}

const useMixMatchStore = create<MixMatchStore>((set, get) => ({
  selectedItems: [],
  maxItems: 6,

  addItem: (product) => {
    const { selectedItems, maxItems } = get();
    if (selectedItems.length < maxItems) {
      set({ selectedItems: [...selectedItems, product] });
    }
  },

  removeItem: (productId) => {
    set((state) => ({
      selectedItems: state.selectedItems.filter(p => p.id !== productId),
    }));
  },

  isComplete: () => {
    const { selectedItems, maxItems } = get();
    return selectedItems.length >= maxItems;
  },
}));

// Usage in component
function MixMatchPage() {
  const { selectedItems, maxItems } = useMixMatchStore();

  return (
    <MixMatchProgressCounter
      current={selectedItems.length}
      max={maxItems}
    />
  );
}

Performance Optimization:

  • Memoization: Use useMemo for percentage calculation to avoid re-renders
  • CSS transitions: Hardware-accelerated transform instead of width for smoother animation (optional optimization)
  • Debounce onChange: If onChange triggers expensive operations, debounce by 200ms
  • CSS containment: Add contain: layout style; to .box-progress for render isolation

Edge Cases:

  • current > max: Clamp percentage to 100%, show complete state
  • current < 0: Clamp percentage to 0%, show empty state
  • max = 0: Return early, show error state or hide component (invalid config)
  • Fractional values: Round current/max to nearest integer for display
  • Very large max: If max > 999, use abbreviated format (e.g., "1k")
  • Removing items: Ensure fill animates backwards smoothly (same 0.3s transition)

Browser Compatibility:

  • Modern browsers: Full support (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
  • Flexbox: Use autoprefixer for older browsers
  • Border-radius: Fully supported, no fallback needed
  • CSS transitions: Fully supported, degrades gracefully (instant change if unsupported)
  • ARIA progressbar: Supported in all modern screen readers (NVDA, JAWS, VoiceOver)

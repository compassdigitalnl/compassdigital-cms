📦 STEP 1: Component Overview

Component: SP01: SubscriptionFilterTabs

Purpose: Pill-style category filter tabs for subscription product archives. Allows users to filter between "Alle", "Persoonlijk", and "Cadeau" subscription types.

Key Features:

  • Pill-shaped buttons (100px border-radius) with icons and labels
  • Product count badges per category
  • Active state: teal background + white text
  • Inactive state: white background + grey border + grey text
  • Hover state: teal border + teal text (inactive tabs)
  • Flex-wrap for responsive stacking
  • Icon support (Lucide icons at 14px)

Visual Description: Horizontal row of pill-shaped filter buttons with icons, labels, and count badges. Active tab has teal background, inactive tabs are white with grey borders.

📋 STEP 2: When to Use

Primary Use Cases:

  • Subscription Product Archives: Filter between subscription types (Personal vs Gift subscriptions)
  • Magazine/Wine Subscription Catalogs: Distinguish between self-purchase and gift subscriptions
  • 2-3 Category Filters: When you have a small number of mutually exclusive categories (max 5 tabs)
  • Quick Visual Filtering: When users need to quickly switch between subscription types

When NOT to Use:

  • ❌ For more than 5 categories (use dropdown or sidebar filter instead)
  • ❌ For multi-select filters (use checkboxes in sidebar - see SP03)
  • ❌ For binary toggle (use toggle switch - see SP02)
  • ❌ For complex filtering with multiple dimensions (use FilterSidebar)

💻 STEP 7: Implementation Notes

React/Next.js Component:

'use client';

import { useState } from 'react';
import { Grid, User, Gift } from 'lucide-react';

interface Tab {
  id: string;
  label: string;
  icon: React.ReactNode;
  count: number;
}

export const SubscriptionFilterTabs = ({
  tabs,
  activeId,
  onChange,
  showCounts = true
}) => {
  return (
    <div className="filter-tabs" role="tablist" aria-label="Subscription filters">
      {tabs.map((tab) => (
        <button
          key={tab.id}
          className={`filter-tab ${activeId === tab.id ? 'active' : ''}`}
          role="tab"
          aria-selected={activeId === tab.id}
          aria-controls={`products-${tab.id}`}
          onClick={() => onChange(tab.id)}
        >
          {tab.icon}
          {tab.label}
          {showCounts && (
            <span className="filter-count" aria-label={`${tab.count} producten`}>
              {tab.count}
            </span>
          )}
        </button>
      ))}
    </div>
  );
};

Usage Example:

const [activeTab, setActiveTab] = useState('all');

const tabs = [
  { id: 'all', label: 'Alle', icon: <Grid size={14} />, count: 6 },
  { id: 'personal', label: 'Persoonlijk', icon: <User size={14} />, count: 3 },
  { id: 'gift', label: 'Cadeau', icon: <Gift size={14} />, count: 3 },
];

<SubscriptionFilterTabs
  tabs={tabs}
  activeId={activeTab}
  onChange={setActiveTab}
/>

State Management:

  • Store active tab ID in component state or URL query params
  • Filter product list based on active tab
  • Update URL with ?type=persoonlijk for shareable links
  • Persist user preference in LocalStorage (optional)

Performance Optimization:

  • Memoize tab list if derived from large dataset
  • Use React.memo() on tab component if re-renders are expensive
  • Lazy load product grid after tab switch (debounce 100ms)

Edge Cases:

  • Zero products in category: Show count as "0" or hide tab entirely
  • Dynamic counts: Update counts when products are added/removed
  • Loading state: Disable tabs during product fetch with cursor not-allowed
  • Error state: Show fallback if categories fail to load

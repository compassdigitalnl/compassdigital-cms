📦 STEP 1: Component Overview

Component: SP04: SubscriptionProductCard

Purpose: Vertical product card for subscription products in grid view. Displays subscription type, pricing tiers, features, and add-to-cart action.

Key Features:

  • White card with 1.5px grey border, 14px border-radius
  • Badge system: POPULAIR (purple), Persoonlijk (grey), Cadeau (coral)
  • Emoji icon (48px): 👤 (Persoonlijk) or 🎁 (Cadeau)
  • Title + subtitle (edition count + type)
  • Tier indicator (e.g., "3 edities per abonnement")
  • Pricing: current price + old price (strikethrough) + period + savings badge (green)
  • Feature list with checkmarks (2-3 items, flexes to fill space)
  • Stock status indicator (green dot + "Beschikbaar")
  • Full-width teal add-to-cart button
  • Hover: teal border + shadow + translateY(-2px)

Visual Description: Vertical card with badges at top, emoji icon, title, pricing breakdown, feature checkmarks, and action button at bottom. Cards align consistently in grid using flexbox with margin-top: auto pattern.

📋 STEP 2: When to Use

Primary Use Cases:

  • Subscription Product Archives: Display magazine/wine subscriptions in grid layout
  • Product Category Pages: 3-column grid of subscription options
  • Comparison Shopping: Side-by-side subscription tiers
  • Desktop Browsing: Grid view optimized for desktop/tablet

When NOT to Use:

  • ❌ For list view (use SP05: SubscriptionProductRow instead)
  • ❌ For detailed product pages (use full product page layout)
  • ❌ For non-subscription products (use EC01: ProductCard)
  • ❌ On mobile with many features (switch to list view for better scanability)

  };
  features: string[];        // Array of feature descriptions
  inStock: boolean;
  onAddToCart: () => void;
}

📐 STEP 5: Layout Rules

Card Structure:

  • Flexbox column layout: display: flex; flex-direction: column;
  • Content flows vertically: badges → icon → title → subtitle → pricing → features → bottom section
  • Features list gets flex: 1 to fill available space
  • Bottom section uses margin-top: auto to stick to bottom
  • Min-height on features (80px) ensures consistent button alignment across cards with different feature counts

Grid Integration:

  • Card works in any grid: display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 20px;
  • Min-width recommendation: 280px
  • Max-width recommendation: 360px
  • Typical grid layouts: 4-col (desktop) → 3-col (tablet) → 2-col (mobile landscape) → 1-col (mobile portrait)

Responsive Behavior:

  • >1200px: 4-column grid
  • 900-1200px: 3-column grid
  • 640-900px: 2-column grid
  • <640px: 1-column grid OR switch to SP05 list view

Spacing Rules:

  • Between cards: 20px gap (both row and column)
  • Internal padding: 20px on all sides
  • Vertical rhythm: 12px between most sections, 16px before features/bottom

💻 STEP 7: Implementation Notes

React/Next.js Component Example:

import { ShoppingCart, Check, TrendingDown, Package, Star, User, Gift } from 'lucide-react';

export function SubscriptionProductCard({
  id,
  title,
  subtitle,
  type,
  icon,
  badges = [],
  tier,
  pricing,
  features,
  inStock,
  onAddToCart
}: SubscriptionProductCardProps) {
  const typeIcon = type === 'personal' ? '👤' : '🎁';
  const savings = pricing.old
    ? Math.round(((pricing.old - pricing.current) / pricing.old) * 100)
    : null;

  return (
    <article
      className="product-card"
      aria-labelledby={`product-title-${id}`}
      aria-describedby={`product-desc-${id}`}
    >
      {badges.length > 0 && (
        <div className="product-badges" role="group" aria-label="Product badges">
          {badges.map((badge, i) => (
            <span key={i} className={`badge ${badge.variant}`}>
              {badge.icon && <Icon name={badge.icon} size={11} />}
              {badge.label}
            </span>
          ))}
        </div>
      )}

      <div className="product-type-icon">{icon || typeIcon}</div>

      <h3 id={`product-title-${id}`} className="product-title">{title}</h3>
      <p id={`product-desc-${id}`} className="product-subtitle">{subtitle}</p>

      <div className="product-pricing">
        <div className="product-tier">
          <Package size={14} />
          {tier.label}
        </div>

        <div className="price-row">
          <span className="price-current">€{pricing.current.toFixed(2)}</span>
          {pricing.old && (
            <span className="price-old">€{pricing.old.toFixed(2)}</span>
          )}
        </div>
        <div className="price-period">{pricing.period}</div>

        {savings && (
          <span className="price-savings">
            <TrendingDown size={11} />
            Bespaar {savings}%
          </span>
        )}
      </div>

      <ul className="product-features" aria-label="Subscription features">
        {features.map((feature, i) => (
          <li key={i}>
            <Check size={14} />
            {feature}
          </li>
        ))}
      </ul>

      <div className="product-bottom">
        <div className="product-status" role="status" aria-live="polite">
          <span className="status-dot" aria-hidden="true"></span>
          {inStock ? 'Beschikbaar' : 'Tijdelijk uitverkocht'}
        </div>

        <button
          className="product-cta"
          onClick={onAddToCart}
          disabled={!inStock}
          aria-label={`Voeg ${title} toe aan winkelwagen`}
        >
          <ShoppingCart size={16} />
          Voeg toe aan winkelwagen
        </button>
      </div>
    </article>
  );
}

Data Fetching (Payload CMS):

// Server Component (Next.js App Router)
import { getPayload } from 'payload';
import config from '@payload-config';

export async function SubscriptionProductGrid({ categoryId }: { categoryId?: string }) {
  const payload = await getPayload({ config });

  const products = await payload.find({
    collection: 'subscriptionProducts',
    where: categoryId ? { category: { equals: categoryId } } : {},
    sort: 'isPopular',
    limit: 12,
  });

  return (
    <div className="demo-grid">
      {products.docs.map(product => (
        <SubscriptionProductCard
          key={product.id}
          id={product.id}
          title={product.title}
          subtitle={`${product.editionCount} nummers - ${product.type === 'personal' ? 'Persoonlijk' : 'Cadeau'}`}
          type={product.type}
          icon={product.icon}
          badges={[
            ...(product.isPopular ? [{ label: 'Populair', variant: 'popular', icon: 'star' }] : []),
            {
              label: product.type === 'personal' ? 'Persoonlijk' : 'Cadeau',
              variant: product.type === 'personal' ? 'persoonlijk' : 'cadeau',
              icon: product.type === 'personal' ? 'user' : 'gift'
            }
          ]}
          tier={{
            count: product.editionCount,
            label: `${product.editionCount} edities per abonnement`
          }}
          pricing={{
            current: product.price,
            old: product.oldPrice,
            period: 'per abonnement',
            savings: product.oldPrice ? Math.round(((product.oldPrice - product.price) / product.oldPrice) * 100) : undefined
          }}
          features={product.features}
          inStock={product.inStock}
          onAddToCart={() => handleAddToCart(product.id)}
        />
      ))}
    </div>
  );
}

Performance Optimizations:

  • Use next/image for product type icons if using images instead of emoji
  • Lazy load cards below the fold with loading="lazy"
  • Memoize card components if used in dynamic filter contexts: React.memo(SubscriptionProductCard)
  • Virtual scrolling for large product catalogs (100+ items) using react-window
  • CSS containment: contain: layout style paint; for scroll performance

State Management:

  • Add-to-cart action should trigger global cart state update (Zustand/Redux)
  • Stock status should update in real-time via WebSocket or polling
  • Optimistic UI: show loading state on button during add-to-cart
  • Toast notification on successful add (reuse C3: AddToCartToast)

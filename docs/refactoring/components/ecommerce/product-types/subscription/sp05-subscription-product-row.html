📦 STEP 1: Component Overview

Component: SP05: SubscriptionProductRow

Purpose: Horizontal product row for subscription products in list view. Displays subscription information across 5 columns for efficient scanning and comparison.

Key Features:

  • White card with 1.5px grey border, 14px border-radius
  • 5-column grid layout: 60px 1fr auto auto auto
  • 2-row layout: badges on row 1, edition count on row 2
  • Column 1: Emoji icon (60×60px, grey bg, 8px radius) — spans both rows
  • Column 2: Type badges (row 1) + edition count (row 2) — flexible width
  • Column 3: Tier + price + savings badge — spans both rows, auto width
  • Column 4: Feature checkmarks (2-3 items) — spans both rows, auto width
  • Column 5: Add-to-cart button — spans both rows, auto width
  • Hover: teal border + shadow + translateY(-1px)
  • Responsive: collapses to 3-col (tablet) → 1-col (mobile)

Visual Description: Horizontal product row with icon on far left, product info spread across middle columns, and action button on far right. All content vertically centered using grid alignment. Optimized for desktop list views and comparison shopping.

📋 STEP 2: When to Use

Primary Use Cases:

  • List View Mode: When user toggles from grid to list view (C22: SortDropdown)
  • Comparison Shopping: Comparing subscription tiers side-by-side
  • Desktop Browsing: Wide screens where horizontal layout is more efficient
  • Feature Comparison: When users need to scan features across multiple products
  • High Product Counts: More products visible per screen vs grid view

When NOT to Use:

  • ❌ For mobile/narrow screens (use SP04: SubscriptionProductCard instead)
  • ❌ For detailed product pages (use full product page layout)
  • ❌ For non-subscription products (use EC01: ProductCard list variant)
  • ❌ When visual appeal is more important than efficiency (grid view better)
  • ❌ For products with many features (4+) that won't fit in column 4

📐 STEP 5: Layout Rules

Grid Structure:

  • 5-column grid: grid-template-columns: 60px 1fr auto auto auto;
  • 2 rows: grid-template-rows: auto auto;
  • Vertical centering: align-items: center;
  • Column gap: 20px, row gap: 6px
  • Most columns span both rows with grid-row: 1 / 3;
  • Column 2 uses both rows separately (badges on row 1, edition count on row 2)

Column Behavior:

  • Column 1 (60px fixed): Icon, always same width, spans both rows
  • Column 2 (1fr flexible): Takes remaining space after columns 3-5 sized, split into 2 rows
  • Column 3 (auto): Pricing, fits content width, spans both rows
  • Column 4 (auto): Features, fits content width, spans both rows
  • Column 5 (auto): Button, fits content width, spans both rows

Responsive Breakpoints:

  • >1024px: Full 5-column layout
  • 768-1024px: 3-column layout (icon, info, pricing/button stacked)
  • <768px: 1-column layout (all elements stacked) OR switch to SP04 grid view

Spacing Rules:

  • Between rows: 12px gap
  • Internal padding: 16px 20px (reduced vs grid view for density)
  • Column gap: 20px (consistent horizontal spacing)
  • Row gap: 6px (tight vertical spacing within column 2)

💻 STEP 7: Implementation Notes

React/Next.js Component Example:

import { ShoppingCart, Check, TrendingDown, Package, Star, User, Gift } from 'lucide-react';

export function SubscriptionProductRow({
  id,
  title,
  subtitle,
  type,
  icon,
  badges = [],
  tier,
  pricing,
  features,
  inStock,
  onAddToCart
}: SubscriptionProductRowProps) {
  const typeIcon = type === 'personal' ? '👤' : '🎁';
  const savings = pricing.old
    ? Math.round(((pricing.old - pricing.current) / pricing.old) * 100)
    : null;

  const ariaLabel = `${subtitle} - ${tier.label} voor €${pricing.current.toFixed(2)}`;

  return (
    <article className="product-row" aria-label={ariaLabel}>
      <div className="product-type-icon" aria-hidden="true">
        {icon || typeIcon}
      </div>

      {badges.length > 0 && (
        <div className="product-badges" role="group" aria-label="Product badges">
          {badges.map((badge, i) => (
            <span key={i} className={`badge ${badge.variant}`} role={badge.variant === 'popular' ? 'status' : undefined}>
              {badge.icon && <Icon name={badge.icon} size={11} aria-hidden="true" />}
              {badge.label}
            </span>
          ))}
        </div>
      )}

      <div className="product-subtitle">{subtitle}</div>

      <div className="product-pricing">
        <div className="product-tier" aria-label={tier.label}>
          <Package size={12} aria-hidden="true" />
          {tier.count} edities
        </div>

        <div className="price-row">
          <span
            className="price-current"
            aria-label={`Huidige prijs ${pricing.current.toFixed(2).replace('.', ' euro ')}`}
          >
            €{pricing.current.toFixed(2)}
          </span>
          {pricing.old && (
            <span
              className="price-old"
              aria-label={`Oude prijs ${pricing.old.toFixed(2).replace('.', ' euro ')}`}
            >
              €{pricing.old.toFixed(2)}
            </span>
          )}
        </div>

        {savings && (
          <span className="price-savings" role="status">
            <TrendingDown size={10} aria-hidden="true" />
            Bespaar {savings}%
          </span>
        )}
      </div>

      <ul className="product-features" aria-label="Subscription features">
        {features.slice(0, 3).map((feature, i) => (
          <li key={i}>
            <Check size={13} aria-hidden="true" />
            {feature}
          </li>
        ))}
      </ul>

      <div className="product-bottom">
        <button
          className="product-cta"
          onClick={onAddToCart}
          disabled={!inStock}
          aria-label={`Voeg ${subtitle} toe aan winkelwagen`}
        >
          <ShoppingCart size={15} />
          Voeg toe
        </button>
      </div>
    </article>
  );
}

Performance Optimizations:

  • Use React.memo(SubscriptionProductRow) to prevent re-renders during filtering
  • Virtual scrolling for large lists (50+ items) using react-window
  • CSS containment: contain: layout style paint;
  • Lazy load rows below the fold
  • Optimize grid recalculation with will-change: transform; on hover

Feature Count Guidelines:

  • Max 3 features recommended for list view (more creates layout issues)
  • If product has 4+ features, either:
    1. Prioritize top 3 most important features
    2. Show "3 edities per jaar" + "Automatisch verlengd" + "+2 meer" link
    3. Or recommend grid view for feature-heavy products
  • Feature text should be concise (max 25 characters)

🔗 STEP 8: Variations & Related Components

Component Variants:

  • Default (Desktop): Full 5-column layout as documented above
  • Tablet (3-col): Icon + info column + stacked pricing/button
  • Mobile (1-col): All elements stacked OR switch to SP04 grid view
  • Compact: Reduce padding to 12px 16px for ultra-dense lists
  • Out of Stock: Greyscale filter + opacity 0.6 + disabled button
  • Clickable Row: Entire row clickable to open QuickView modal, cursor pointer

Responsive Strategy:

  • Desktop (>1024px): List view preferred (more products visible)
  • Tablet (768-1024px): List view acceptable with 3-col layout
  • Mobile (<768px): Force switch to grid view (SP04) — better UX
  • View mode toggle (C22) should auto-switch to grid on mobile
  • Or: hide list view button entirely on mobile breakpoints

Related Components:

  • SP04: SubscriptionProductCard — Vertical grid view variant
  • SP01: SubscriptionFilterTabs — Category filter tabs
  • SP02: SubscriptionFilterToggle — Binary toggle switch
  • SP03: SubscriptionFilterSidebar — Multi-select filters
  • C22: SortDropdown — Sort options + grid/list view toggle ⭐ KEY INTEGRATION
  • C18: ProductBadges — Badge system (11 variants)
  • EC01: ProductCard — Standard product list variant
  • C5: QuickView Modal — Opens on row click for quick product details
  • C3: AddToCartToast — Toast notification on add-to-cart
  • C20: Pagination — Pagination for long product lists

Layout Combinations:

  • List Archive Pattern: SP01/SP02/SP03 (filters) + C22 (sort + toggle) + SP05 list + C20 (pagination)
  • Filtered List: SP03 sidebar (280px) + SP05 list (flex: 1)
  • Comparison Table: Multiple SP05 rows stacked for side-by-side comparison
  • Hybrid View: Featured products in SP04 grid (top) + full catalog in SP05 list (below)

Testing Checklist:

  • ✅ 5 columns align correctly with varying content lengths
  • ✅ Column 2 (1fr) takes remaining space properly
  • ✅ Badges + edition count align correctly in 2-row layout
  • ✅ All span-both-rows columns vertically centered
  • ✅ Hover states work (border, shadow, translateY)
  • ✅ Pricing displays correctly with/without old price
  • ✅ Features display correctly (1, 2, 3 items)
  • ✅ Button stays right-aligned in column 5
  • ✅ Responsive collapse to 3-col (tablet) works smoothly
  • ✅ Responsive collapse to 1-col (mobile) OR auto-switch to grid
  • ✅ Keyboard navigation works (tab to button, enter to add)
  • ✅ Screen reader announces full row info correctly
  • ✅ View mode toggle (C22) switches between SP04/SP05 correctly

Edge Cases:

  • Long edition count text (2-line clamp in column 2)
  • No old price (hide strikethrough, no savings badge)
  • No badges (hide badges container, edition count takes full row 1)
  • 1 feature only (still displays correctly in column 4)
  • 4+ features (truncate to 3, or show "+X more")
  • Very long feature text (wrap to 2 lines max, then ellipsis)
  • Out of stock (disable button, greyscale row)
  • Narrow screens (1024-1200px) — text wrapping in column 2/4

Comparison: SP04 vs SP05

  • SP04 (Grid View): Better for visual browsing, fewer products per screen, mobile-friendly
  • SP05 (List View): Better for comparison, more products per screen, desktop-optimized
  • When to use SP04: Mobile, tablet, visual appeal, marketing pages
  • When to use SP05: Desktop, comparison shopping, high product counts, efficiency
  • Best practice: Offer both via C22 toggle, default to grid on mobile, list on desktop

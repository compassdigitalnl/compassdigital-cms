📦 STEP 1: Component Overview

Component: SP03: SubscriptionFilterSidebar

Purpose: Sidebar filter panel with multiple checkbox dimensions for subscription products. Allows multi-select filtering by Type, Edition Count, and Availability.

Key Features:

  • White card sidebar (280px width, sticky positioning)
  • Teal sliders icon + "Filters" title
  • Multiple filter sections with uppercase labels
  • Checkboxes with product counts per option
  • Multi-select support (checkboxes vs radio buttons)
  • Clean checkbox styling with teal accent color
  • Filter counts update dynamically

Visual Description: Vertical sidebar with grouped checkbox filters. Each section has an uppercase label, and each checkbox row shows the filter name + product count.

📋 STEP 2: When to Use

Primary Use Cases:

  • Multi-Dimension Filtering: When you need to filter by 2+ dimensions simultaneously (Type + Editions + Availability)
  • Complex Product Catalogs: Large subscription archives with many filtering options
  • Multi-Select Filters: When users can select multiple options per dimension (e.g., show 3 AND 6 edition subscriptions)
  • Desktop Layouts: Side-by-side layout with filters on left, products on right

When NOT to Use:

  • ❌ For single-dimension filtering (use SP01 tabs or SP02 toggle)
  • ❌ On mobile without collapse/drawer (takes up too much space)
  • ❌ For binary choices (use SP02 toggle switch)
  • ❌ When you have < 5 total filter options (use inline filters instead)

💻 STEP 7: Implementation Notes

React/Next.js Component:

'use client';

import { Sliders } from 'lucide-react';

export const SubscriptionFilterSidebar = ({ sections, onChange }) => {
  return (
    <aside className="filter-sidebar" role="complementary" aria-label="Filters">
      <h3>
        <Sliders size={18} aria-hidden="true" />
        Filters
      </h3>

      {sections.map((section) => (
        <div key={section.id} className="filter-section">
          <div className="filter-section-title">{section.title}</div>

          {section.options.map((option) => (
            <div key={option.id} className="filter-checkbox">
              <input
                type="checkbox"
                id={`filter-${option.id}`}
                checked={option.checked}
                onChange={(e) => onChange(section.id, option.id, e.target.checked)}
              />
              <label htmlFor={`filter-${option.id}`}>
                {option.label}
                <span className="filter-checkbox-count" aria-label={`${option.count} producten`}>
                  {option.count}
                </span>
              </label>
            </div>
          ))}
        </div>
      ))}
    </aside>
  );
};

Usage Example:

const [filters, setFilters] = useState({
  type: { personal: true, gift: true },
  editions: { '3': true, '6': true, '12': true },
  stock: { available: true }
});

const handleFilterChange = (sectionId, optionId, checked) => {
  setFilters(prev => ({
    ...prev,
    [sectionId]: {
      ...prev[sectionId],
      [optionId]: checked
    }
  }));
};

const sections = [
  {
    id: 'type',
    title: 'Type',
    options: [
      { id: 'personal', label: 'Persoonlijk', count: 3, checked: filters.type.personal },
      { id: 'gift', label: 'Cadeau', count: 3, checked: filters.type.gift }
    ]
  },
  // More sections...
];

<SubscriptionFilterSidebar sections={sections} onChange={handleFilterChange} />

State Management:

  • Store filter state in component state or URL query params
  • Filter product list based on active checkboxes
  • Update URL with ?type=personal,gift&editions=3,6 for shareable links
  • Persist user preferences in LocalStorage

Performance Optimization:

  • Debounce filter changes (200ms) to avoid excessive re-renders
  • Memoize filtered product list with useMemo
  • Use React.memo() on filter sections if expensive

Edge Cases:

  • All unchecked: Show empty state or re-check at least one filter
  • Zero products: Grey out checkbox or show "0" count
  • Dynamic counts: Update counts when products change
  • Loading state: Disable checkboxes during product fetch

📋 STEP 1: Component Overview

Component Name: VariantDropdownSelector (VP03)

Category: Product Types → Variable Product

Purpose: Dropdown selection for product variants with complex options (icons, descriptions, prices). Used when visual swatches (VP01) or simple buttons (VP02) are insufficient.

Key Features

  • ✅ Trigger Button — 48px height, full-width, displays selected option with icon
  • ✅ Dropdown Menu — Overlaid panel (max-height 280px) with scrollable items
  • ✅ Icon + Name + Description — Each item shows emoji/icon (28×28px), option name, and detailed description
  • ✅ Price Indicator — "Incl." (green) or "+ €X" (black) shown per option
  • ✅ Selected State — Teal background + checkmark icon in menu
  • ✅ Keyboard Accessible — Arrow keys for navigation, Enter to select
  • ✅ Click-Outside-to-Close — Menu closes when clicking outside dropdown
  • ✅ Smooth Transitions — Border color, arrow rotation, menu appearance

Visual Description

The component consists of a trigger button showing the selected option (icon + name + description) and a chevron-down arrow. When clicked, a dropdown menu appears below with a list of options. Each option displays an icon (emoji or Lucide icon), a bold name, a grey description line, and a price indicator. The selected option has a light teal background and a teal checkmark circle on the right.

Usage Context

Use this component for variants that need more context than a simple button or swatch can provide. Ideal for material selection, finish types, sole types, or any option where customers benefit from seeing detailed descriptions and multiple attributes per choice.

🎯 STEP 2: When to Use

Primary Use Cases

  1. Material Selection — Cotton, Merino Wool, Polyester (with care/benefit descriptions)
  2. Finish/Coating — Matte, Glossy, Satin (with visual characteristics explained)
  3. Technical Specs — Sole types, lens types, battery capacities (with detailed specs)
  4. Service Levels — Standard, Express, Premium (with delivery time + features)
  5. Subscription Tiers — Basic, Pro, Enterprise (with feature lists)

When NOT to Use

  • ❌ Visual variants — Use VP01 (color swatches) for colors/patterns
  • ❌ Simple options — Use VP02 (size buttons) for sizes (S/M/L) without extra context
  • ❌ Binary choices — Use toggle switch or radio buttons for yes/no options
  • ❌ More than 12 options — Consider categorization or search functionality

Guidelines

  • Always include a clear icon per option (helps visual scanning)
  • Use emojis for casual/fun products, Lucide icons for professional/B2B products
  • Keep descriptions to 1 line (~40 chars) — if longer, truncate with ellipsis
  • Show "Incl." in green for options included in base price (not "+€0")
  • Order options logically: most popular first, or price ascending
  • For identical prices, order alphabetically by name

⚙️ STEP 4: Props & Configuration

TypeScript Interface

interface DropdownOption {
  id: string
  name: string              // Display name (e.g., "ComfortFlex", "Merino Wool")
  description: string       // Short description (e.g., "Dagelijks gebruik · EVA schuim")
  icon: string              // Emoji or Lucide icon name
  iconBg?: string           // Background color for icon container (e.g., "var(--teal-glow)")
  price?: number            // Price adjustment (0 = included, >0 = additional cost)
  value: string             // Value for form submission
  available?: boolean       // Stock availability (default: true)
  featured?: boolean        // Show "Populair" badge (optional)
}

interface VariantDropdownSelectorProps {
  label?: string                          // Default: "Opties"
  icon?: string                           // Lucide icon name for label, default: "layers"
  options: DropdownOption[]               // Array of dropdown options
  selected: string                        // ID of selected option
  onSelect: (option: DropdownOption) => void  // Selection handler
  placeholder?: string                    // Placeholder when none selected
  showTypeBadge?: boolean                 // Show "Dropdown" badge, default: true
  disabled?: boolean                      // Disable dropdown, default: false
  required?: boolean                      // Mark as required field, default: false
  maxHeight?: number                      // Max dropdown height in px, default: 280
}

♿ STEP 6: Accessibility

ARIA Attributes

  • Trigger: role="button", aria-haspopup="listbox", aria-expanded="true/false"
  • Menu: role="listbox", aria-labelledby="label-id"
  • Item: role="option", aria-selected="true/false"

Keyboard Navigation

  • Tab: Focus trigger button
  • Enter/Space: Open dropdown (when trigger focused)
  • Arrow Down/Up: Navigate options (when menu open)
  • Enter: Select focused option and close menu
  • Escape: Close menu without selecting

Screen Reader Support

  • Trigger announces: "Zooltype, ComfortFlex, Dagelijks gebruik, button, collapsed"
  • Menu announces: "Listbox, 4 options"
  • Item announces: "ComfortFlex, Dagelijks gebruik, Incl., option, selected"

💻 STEP 7: Implementation Notes

See VP01 and VP02 for detailed React examples. This component follows the same patterns with dropdown-specific logic (open/close state, click-outside handling, keyboard navigation).

Click-Outside-to-Close Pattern

useEffect(() => {
  const handleClickOutside = (e: MouseEvent) => {
    if (!dropdownRef.current?.contains(e.target as Node)) {
      setIsOpen(false)
    }
  }

  document.addEventListener('click', handleClickOutside)
  return () => document.removeEventListener('click', handleClickOutside)
}, [])

📋 STEP 1: Component Overview

Component Name: VariantSizeSelector (VP02)

Category: Product Types → Variable Product

Purpose: Size selection interface for variable products (clothing, shoes, accessories), with real-time stock status indicators per size.

Key Features

  • ✅ Size Radio Buttons — 52×44px buttons with size labels (38, 39, S, M, L, etc.)
  • ✅ Stock Indicators — Color-coded dots (green/amber/red) in top-right corner
  • ✅ Out of Stock State — Greyed out + strikethrough + not clickable
  • ✅ Selected State — Teal background + white text + teal glow shadow
  • ✅ Size Chart Link — Optional "Maattabel" link to open size guide modal
  • ✅ Flexible Width — Auto-width for numeric sizes (52px), wider for text (70-90px)
  • ✅ Keyboard Accessible — Arrow keys for navigation, Enter/Space to select
  • ✅ Responsive — Flex-wrap layout adapts to container width

Visual Description

The component displays a horizontal row of size buttons, each showing a size label (number or text). Available sizes have a small colored dot in the top-right corner indicating stock level: green (in stock), amber (low stock), or red (very low stock). Out-of-stock sizes are shown with reduced opacity and strikethrough text. The selected size button has a teal background with white text and a subtle glow shadow.

Stock Status Colors

  • Green dot: ≥10 units in stock (good availability)
  • Amber dot: 3-9 units in stock (low stock, buy soon)
  • Red dot: 1-2 units in stock (very low, last chance)
  • No button (or OOS state): 0 units (out of stock)

Usage Context

Use this component on product pages for clothing, footwear, or any product with standardized size variants. Ideal for 6-12 size options. For products with many sizes (15+), consider grouping or using a dropdown.

🎯 STEP 2: When to Use

Primary Use Cases

  1. Apparel — T-shirts, jeans, dresses (XS, S, M, L, XL, XXL)
  2. Footwear — Shoes, boots, sandals (EU: 36-47, US: 6-13, UK: 3-12)
  3. Accessories — Gloves, hats, belts (One Size, S/M/L, numeric)
  4. Home Goods — Bedding, curtains (Single, Double, King, Queen)
  5. Custom Products — Any product with discrete size options

When NOT to Use

  • ❌ Continuous dimensions — Use slider or number input for custom measurements (e.g., "Enter waist in cm")
  • ❌ Complex size charts — Products with length × width × height need multi-input form
  • ❌ More than 15 sizes — Use dropdown selector (VP03) for better UX
  • ❌ Non-size variants — This component is specifically for size; use VP01 (color swatches) or VP03 (dropdown) for other attributes

Guidelines

  • Always provide a "Maattabel" (size chart) link for clothing/shoes — reduces returns
  • Order sizes logically: XS → S → M → L → XL → XXL (not alphabetically!)
  • Show out-of-stock sizes but disable them (don't hide) — helps users see full range
  • Use stock dots for B2C; hide for B2B (business customers get stock numbers via API)
  • For shoes: specify system (EU, US, UK) in label to avoid confusion
  • For unisex products: link to both men's and women's size charts

⚙️ STEP 4: Props & Configuration

TypeScript Interface

interface Size {
  id: string
  label: string              // Display label (e.g., "43", "M", "Large")
  value: string              // Value for form submission
  stock: number              // Stock quantity (0 = OOS, 1-2 = red, 3-9 = amber, ≥10 = green)
  sku?: string               // Variant SKU for inventory tracking
  priceAdjustment?: number   // Price change for this size (rare, but possible)
}

interface VariantSizeSelectorProps {
  label?: string                    // Default: "Maat"
  icon?: string                     // Lucide icon name, default: "ruler"
  sizes: Size[]                     // Array of size options
  selected: string                  // ID of selected size
  onSelect: (size: Size) => void    // Selection handler
  sizeChartUrl?: string             // Link to size chart/guide
  showStockDots?: boolean           // Show stock indicators, default: true
  showTypeBadge?: boolean           // Show "Radio" badge, default: true
  buttonWidth?: 'auto' | number     // Auto-width or fixed (e.g., 60)
  required?: boolean                // Show required indicator (*), default: false
  disabled?: boolean                // Disable all sizes, default: false
}

State Management

  • Parent component manages selected size ID (controlled component)
  • onSelect callback receives full size object for price/stock updates
  • Stock updates: Use WebSocket or polling to refresh stock in real-time
  • Auto-select: If selected size goes OOS, auto-select first available size

API Integration (Payload CMS)

// Fetch product with size variants + real-time stock
const product = await payload.findByID({
  collection: 'products',
  id: productId,
})

// Transform to component props
const sizes = product.sizeVariants.map(variant => ({
  id: variant.id,
  label: variant.label,
  value: variant.value,
  stock: variant.stock,
  sku: variant.sku,
}))

// Default selection: first available size
const defaultSelected = sizes.find(s => s.stock > 0)?.id || sizes[0].id

Performance Optimization

  • Memoize size buttons: Wrap in React.memo to prevent re-renders
  • Debounce stock updates: If using real-time stock, debounce updates (500ms)
  • Lazy load size chart: Only fetch size chart PDF when user clicks link

Edge Cases & Error Handling

  1. All sizes out of stock: Show "Tijdelijk niet beschikbaar" message, disable add-to-cart
  2. Selected size becomes unavailable: Auto-select first available, show toast notification
  3. Missing stock data: Assume stock = 0 (hide stock dots, disable button)
  4. Invalid size label: Fallback to "Maat {index}"

🔀 STEP 8: Variations & Related Components

Component Variants

1. No Stock Dots (Simple Mode)

Use case: B2B customers, digital products, or pre-order items

  • Set showStockDots: false
  • All sizes shown without stock indicators
  • Stock managed via backend validation on add-to-cart

2. Fixed-Width Buttons

Use case: Consistent visual alignment for mixed numeric/text sizes

  • Set buttonWidth: 60 (or any fixed px value)
  • All buttons same width, centered text

3. Two-Row Layout (Many Sizes)

Use case: Products with 12+ sizes (e.g., shoes with both men's and women's)

  • Use CSS Grid: grid-template-columns: repeat(auto-fill, 52px)
  • Group logically: "Men's Sizes" header + row, "Women's Sizes" header + row

Related Components

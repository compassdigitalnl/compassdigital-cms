📋 STEP 1: Component Overview

VP04: VariantImageRadio is a visual variant selector that uses image cards as radio buttons. Each option displays an image/icon, name, and price modifier. Perfect for options where visual representation helps decision-making.

Key Features

  • ✅ Visual Selection: Large image cards (80×80px) with emoji or actual product images
  • ✅ Single Selection: Radio button behavior (one active at a time)
  • ✅ Price Display: Shows included or surcharge pricing per option
  • ✅ Active State: Teal border + checkmark badge + background glow
  • ✅ Stock Indicators: Optional badges showing availability
  • ✅ Disabled State: Greyscale + no pointer events for out-of-stock
  • ✅ Responsive Grid: Auto-fit layout (4-col → 2-col on mobile)
  • ✅ Hover Effects: Border color change + subtle scale-up on image

Use Cases

  • 🎀 Closure types (laces, velcro, zip, buckle)
  • 📐 Pattern variations (solid, striped, geometric, abstract)
  • ✨ Surface finish (matte, glossy, metallic, textured)
  • 🎨 Color families (when swatches aren't detailed enough)
  • 🔧 Component styles (handle types, knob designs, frame styles)
  • 📦 Packaging options (standard box, gift wrap, eco-packaging)

🎯 STEP 2: When to Use This Component

✅ Use VariantImageRadio When:

  • Visual clarity is essential — Options are best understood through images (e.g., closure types, patterns)
  • Small option set — 3-6 options (more than 8 becomes overwhelming)
  • Price varies per option — Need to show surcharges clearly (+€5, +€10)
  • Single selection required — Only one option can be active (radio behavior)
  • Desktop + mobile friendly — Grid layout works well on all screen sizes

❌ Don't Use When:

  • Too many options — 10+ options → use VP03 (Dropdown) instead
  • Multi-select needed — Use VP05 (Checkbox Addons) for multi-selection
  • Simple color choice — Use VP01 (Color Swatches) for color variants
  • Size selection — Use VP02 (Size Selector) for standard sizing (S/M/L or numerical)
  • Text-only options — Images add no value → use dropdown or buttons

⚙️ STEP 4: Props & Configuration

TypeScript Interface

interface ImageRadioOption { id: string; name: string; image: string; // URL or emoji imageBackground?: string; // Background color for image price: number; // 0 for included, positive for surcharge priceLabel?: string; // Override (e.g., "Incl.", "+€5") stockStatus?: 'in-stock' | 'limited' | 'out-of-stock'; stockLabel?: string; // Override badge text disabled?: boolean; } interface VariantImageRadioProps { label: string; options: ImageRadioOption[]; selected: string; // ID of selected option onChange: (id: string) => void; required?: boolean; columns?: 2 | 3 | 4; // Grid columns (default: auto-fit) showCheckmark?: boolean; // Show checkmark badge (default: true) showStockBadges?: boolean; // Show stock badges (default: false) }

Payload CMS Schema

{ name: 'closureType', type: 'select', label: 'Vetersluiting', required: true, options: [ { label: 'Klassiek', value: 'classic', image: '/images/variants/closure-classic.jpg', price: 0 }, { label: 'Speed Lace', value: 'speedlace', image: '/images/variants/closure-speedlace.jpg', price: 5.00 }, { label: 'Magnetisch', value: 'magnetic', image: '/images/variants/closure-magnetic.jpg', price: 12.00 }, { label: 'Slip-on', value: 'slipon', image: '/images/variants/closure-slipon.jpg', price: 8.00 } ], admin: { components: { Field: VariantImageRadio } } }

Default Values

  • Columns: auto-fit (responsive, 140px minimum per card)
  • Checkmark: Shown (can be hidden with showCheckmark: false)
  • Stock Badges: Hidden (enable with showStockBadges: true)
  • Image Size: 80×80px (desktop), 60×60px (mobile)
  • Minimum Height: 140px (desktop), 120px (mobile)

📐 STEP 5: Layout Rules

Grid Behavior

  • Desktop (>640px): grid-template-columns: repeat(auto-fit, minmax(140px, 1fr))
  • Mobile (≤640px): grid-template-columns: repeat(2, 1fr) (always 2 columns)
  • Gap: 12px (desktop), 8px (mobile)
  • Fixed Columns: Override with inline style grid-template-columns: repeat(3, 1fr) for 3-column layout

Card Structure

<div class="img-radio [selected] [disabled]"> <!-- Optional: Stock Badge (top-left) --> <div class="ir-stock [in-stock|out-of-stock]">Op voorraad</div> <!-- Optional: Checkmark Badge (top-right) --> <div class="ir-checkmark"><i data-lucide="check"></i></div> <!-- Image/Icon (80×80px) --> <div class="ir-img" style="background:#E8F5E9;"> 🎀 <!-- OR: <img src="..." alt="..."> --> </div> <!-- Option Name --> <div class="ir-name">Klassiek</div> <!-- Price --> <div class="ir-price [included|surcharge]">Incl.</div> </div>

Keyboard Navigation

  • Tab: Focus moves between image radio cards
  • Space/Enter: Select focused option
  • Arrow Keys: Navigate between options (recommended for radio groups)
  • Disabled items: Skip in tab order (tabindex="-1")

Focus Management

Screen Reader Support

  • Role: role="radio" on each card
  • Container: Wrapper with role="radiogroup" and aria-labelledby
  • Announcements: "Klassiek, included, selected" or "Speed Lace, plus 5 euros, not selected"
  • Stock status: Include in aria-describedby (e.g., "Out of stock")

Image Handling

  • Emoji placeholder: Use during development (as shown in demos)
  • Production images: Replace with actual product photos
  • Image optimization: Use next/image with 160×160px dimensions (2x for retina)
  • Lazy loading: Enable for performance (loading="lazy")
  • Alt text: Always include descriptive alt text for images

Performance Considerations

  • Image size: Keep under 20KB per image (WebP format recommended)
  • Lazy loading: Load images only when visible (use Intersection Observer)
  • CSS containment: Add contain: layout to cards for better rendering performance
  • Debouncing: Debounce onChange calls if triggering heavy calculations

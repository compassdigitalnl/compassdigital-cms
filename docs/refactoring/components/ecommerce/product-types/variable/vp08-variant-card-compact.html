VP08: Variant Card Compact

Variant Selection Card — Compact Grid View

Compact product variant card for multi-select grids. Designed for variable products (same product, multiple sizes/colors), this card shows variant info, stock status, price, and quantity stepper. Checkbox selection automatically sets quantity to 1. Used in inline (50/50) and archive (full-width grid) layouts.

STEP 1: Component Overview

Purpose

Compact card for selecting product variants in multi-select grid layouts. Enables customers to select multiple sizes/colors/variants of the same product and add them all to cart simultaneously.

Key Features

  • Checkbox Selection: Visual checkbox in top-left corner indicates selection state
  • Variant Display: Large variant name (e.g., "60x90") + full description below
  • Price Display: Monospace price + "per stuk" label for clarity
  • Stock Indicator: 3 states (in-stock green, low-stock amber, pre-order purple)
  • Quantity Stepper: Standard stepper (40×44px buttons, 60px input) matching pt1-single-product.html
  • Hover Effects: Border color change, shadow elevation, translateY animation
  • Selected State: Teal border + light teal background tint

Visual Description

White card with 12px padding, 10px border-radius, 1.5px grey border. Checkbox (18×18px) in top-left. Vertically stacked content: variant name (18px, 800 weight), description (11px grey), price (16px mono), stock indicator (10px with colored bg), quantity stepper (standard 44px height). On hover: teal border + shadow + lift. On select: teal border + light teal bg.

STEP 2: When to Use

Primary Use Cases

  • Variable Products with Multiple Variants: Products like yoga mats (multiple sizes), T-shirts (sizes/colors), supplements (package sizes)
  • Bulk B2B Orders: B2B customers ordering multiple sizes/variants of the same product simultaneously
  • Inline Product Pages: 50/50 layout next to product image (desktop), stacked on mobile
  • Archive/Category Pages: Full-width grid showing all variants for quick selection
  • Quick Order Flows: Streamlined ordering where customers know exactly what variants they need

When NOT to Use

  • ❌ Single Variant Selection: Use VP01-VP07 (color swatches, size buttons, dropdowns) for single-select
  • ❌ Different Products: Use GP01-GP07 (grouped products) for bundling different products together
  • ❌ Complex Configurations: Use PC01-PC08 (product configurator) for step-by-step customization
  • ❌ Mix & Match Packs: Use MM01-MM07 for "build your own pack" patterns (e.g., choose 6 from 20)

Responsive Behavior

  • Desktop (≥1024px): 6 columns (explicit), 10px gap, full card content
  • Tablet (640-1023px): 3-4 columns (auto-fill), reduce font sizes slightly
  • Mobile (<640px): 2 columns, hide "per stuk" label, reduce padding to 10px

STEP 7: Implementation Notes

React/Next.js Component

'use client'; import { useState } from 'react'; import { Check } from 'lucide-react'; interface VariantCardCompactProps { variantId: string; variantName: string; variantFullName: string; price: number; stockStatus: 'in-stock' | 'low-stock' | 'pre-order'; stockQuantity?: number; selected: boolean; quantity: number; onSelect: (id: string) => void; onQuantityChange: (id: string, qty: number) => void; } export default function VariantCardCompact({ variantId, variantName, variantFullName, price, stockStatus, stockQuantity, selected, quantity, onSelect, onQuantityChange }: VariantCardCompactProps) { const handleCardClick = () => { if (!selected) { onSelect(variantId); onQuantityChange(variantId, 1); // Auto-set to 1 } else { onSelect(variantId); onQuantityChange(variantId, 0); // Deselect } }; const handleQuantityChange = (delta: number) => { const newQty = Math.max(0, Math.min(999, quantity + delta)); onQuantityChange(variantId, newQty); // Auto-select if incrementing from 0 if (newQty > 0 && !selected) { onSelect(variantId); } // Auto-deselect if decremented to 0 if (newQty === 0 && selected) { onSelect(variantId); } }; const stockConfig = { 'in-stock': { label: 'Op voorraad', class: 'in-stock' }, 'low-stock': { label: `Nog ${stockQuantity} stuks`, class: 'low-stock' }, 'pre-order': { label: 'Pre-order', class: 'pre-order' } }; const stock = stockConfig[stockStatus]; return ( <div className={`variant-card-compact ${selected ? 'selected' : ''}`} onClick={handleCardClick} role="checkbox" aria-checked={selected} tabIndex={0} onKeyDown={(e) => { if (e.key === ' ' || e.key === 'Enter') { e.preventDefault(); handleCardClick(); } }} > <div className={`variant-checkbox ${selected ? 'checked' : ''}`}> {selected && <Check size={12} />} </div> <div className="variant-name">{variantName}</div> <div className="variant-fullname">{variantFullName}</div> <div className="variant-price"> <span className="variant-price-value">€ {price.toFixed(2)}</span> <span className="variant-price-note">per stuk</span> </div> <div className={`stock-indicator ${stock.class}`}> <span className="stock-dot"></span> {stock.label} </div> <div className="qty-stepper" onClick={(e) => e.stopPropagation()}> <button className="qty-btn minus" onClick={() => handleQuantityChange(-1)} disabled={quantity === 0} aria-label="Verminder aantal" > − </button> <input type="number" className="qty-input" value={quantity} onChange={(e) => onQuantityChange(variantId, parseInt(e.target.value) || 0)} min={0} max={999} aria-label="Aantal" /> <button className="qty-btn plus" onClick={() => handleQuantityChange(1)} aria-label="Verhoog aantal" > + </button> </div> </div> ); }

State Management Pattern

// Parent component managing multiple variants const [selectedVariants, setSelectedVariants] = useState<Set<string>>(new Set()); const [quantities, setQuantities] = useState<Record<string, number>>({}); const handleSelect = (variantId: string) => { setSelectedVariants(prev => { const updated = new Set(prev); if (updated.has(variantId)) { updated.delete(variantId); } else { updated.add(variantId); } return updated; }); }; const handleQuantityChange = (variantId: string, qty: number) => { setQuantities(prev => ({ ...prev, [variantId]: qty })); }; // Calculate totals const total = Array.from(selectedVariants).reduce((sum, id) => { const variant = variants.find(v => v.id === id); const qty = quantities[id] || 0; return sum + (variant?.price || 0) * qty; }, 0);

Performance Optimization

  • Memoization: Use React.memo for cards (only re-render if props change)
  • Virtualization: Use react-window for grids with 50+ variants
  • Debounce Quantity Input: Debounce input changes by 300ms to reduce re-renders
  • CSS Containment: Add contain: layout style paint to cards for browser optimization

Edge Cases

  • Stock Limit: Disable stepper + button if quantity reaches stock limit
  • Out of Stock: Grey out card, disable all interactions, show "Uitverkocht" badge
  • Price Changes: Update price display when volume pricing tier changes
  • Mobile Touch: Increase hit area to min 44×44px for touch targets

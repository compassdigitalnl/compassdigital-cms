📋 STEP 1: Component Overview

VP05: VariantCheckboxAddons is a multi-select addon system where users can choose zero, one, or multiple optional extras. Each addon displays a checkbox, icon, name, description, and price. Perfect for cross-sells, warranty extensions, services, and customization options.

Key Features

  • ✅ Multi-Select: Users can select 0-N options (checkbox behavior)
  • ✅ Rich Content: Icon + name + description + price per addon
  • ✅ Visual Feedback: Selected state with teal styling + checkmark
  • ✅ Price Display: Shows surcharge (+€X) or "Gratis" for free options
  • ✅ Optional Badges: "Populair", "Aanbevolen", "Nieuw" labels
  • ✅ Total Summary: Optional counter + total price of selected addons
  • ✅ Disabled State: Greyscale for unavailable/out-of-stock addons
  • ✅ Responsive Grid: 2-col → 1-col on mobile

Use Cases

  • 🧴 Product protection (coating, insurance, warranty)
  • 🎁 Packaging & gifting options (gift wrap, gift card, message)
  • 🔧 Accessories & parts (extra laces, replacement parts, tools)
  • 📦 Shipping upgrades (express, insurance, tracking)
  • 🛡️ Service extensions (installation, setup, training)
  • ⏱️ Production speed (express manufacturing, rush order)

🎯 STEP 2: When to Use This Component

✅ Use VariantCheckboxAddons When:

  • Optional extras — Products have add-ons users may or may not want
  • Multi-select needed — Users can select multiple options (unlike radio buttons)
  • Price varies per addon — Each option has its own price modifier
  • Rich descriptions needed — Each addon needs explanation (icon + name + description)
  • Cross-sell opportunities — Promote complementary products/services
  • 2-8 options — Enough to show grid, not so many it overwhelms (9+ use collapsible sections)

❌ Don't Use When:

  • Single selection only — Use VP02 (Size), VP03 (Dropdown), or VP04 (Image Radio) instead
  • Required selection — Checkboxes imply optional; use radio buttons for required choices
  • Simple yes/no toggle — Use a single checkbox or toggle switch
  • Too many options (10+) — Use collapsible sections or categorized lists
  • No price variation — If all options are free, use simpler checkbox list

Alternative Components

Scenario Use Instead
Single selection required VP02: VariantSizeSelector or VP04: VariantImageRadio
Complex dropdown with many options VP03: VariantDropdownSelector
Simple yes/no option Single checkbox or toggle switch
Grouped products (buy together) GP01-GP07: Grouped Products components

⚙️ STEP 4: Props & Configuration

TypeScript Interface

interface CheckboxAddon { id: string; name: string; description: string; icon: string; // Emoji or image URL price: number; // Surcharge amount (0 for free) priceLabel?: string; // Override (e.g., "Gratis", "+ €12,50") badge?: 'popular' | 'recommended' | 'new' | string; badgeLabel?: string; // Override badge text disabled?: boolean; selected?: boolean; // Default selected state } interface VariantCheckboxAddonsProps { label: string; description?: string; // Optional helper text addons: CheckboxAddon[]; selected: string[]; // Array of selected addon IDs onChange: (ids: string[]) => void; showTotal?: boolean; // Show total summary (default: false) columns?: 1 | 2 | 3; // Grid columns (default: 2) optional?: boolean; // Show "Optioneel" badge (default: true) }

Payload CMS Schema

{ name: 'productAddons', type: 'array', label: 'Extra opties', fields: [ { name: 'addon', type: 'select', label: 'Add-on', options: [ { label: 'Beschermende coating', value: 'coating', description: 'Water- en vuilafstotende nano-coating', price: 12.50, icon: '🧴' }, { label: 'Premium verpakking', value: 'premium-box', description: 'Houten geschenkdoos met zijdepapier', price: 9.95, icon: '🎁' }, // ... more addons ] } ] }

Default Values

  • Columns: 2 (auto-fill, min 280px per card)
  • Show Total: false (hide total summary)
  • Optional: true (show "Optioneel" badge)
  • Selected: [] (empty array - no addons selected by default)

💻 STEP 7: Implementation Notes

React/Next.js Component

export function VariantCheckboxAddons({ label, addons, selected, onChange, showTotal = false, columns = 2 }: VariantCheckboxAddonsProps) { const gridStyle = { gridTemplateColumns: columns === 1 ? '1fr' : `repeat(auto-fill, minmax(280px, 1fr))` }; const toggleAddon = (id: string) => { const newSelected = selected.includes(id) ? selected.filter(i => i !== id) : [...selected, id]; onChange(newSelected); }; const totalPrice = addons .filter(a => selected.includes(a.id)) .reduce((sum, a) => sum + a.price, 0); return ( <div className="variant-section"> <div className="var-header"> <div className="var-label"> <PlusCircle size={14} /> {label} </div> </div> <div className="addon-grid" style={gridStyle}> {addons.map(addon => ( <div key={addon.id} className={`addon ${selected.includes(addon.id) ? 'selected' : ''} ${addon.disabled ? 'disabled' : ''}`} onClick={() => !addon.disabled && toggleAddon(addon.id)} role="checkbox" aria-checked={selected.includes(addon.id)} tabIndex={addon.disabled ? -1 : 0} > {addon.badge && ( <div className={`addon-badge ${addon.badge}`}> {addon.badgeLabel || addon.badge} </div> )} <div className="addon-check"><Check size={12} /></div> <div className="addon-icon">{addon.icon}</div> <div className="addon-info"> <div className="addon-name">{addon.name}</div> <div className="addon-desc">{addon.description}</div> </div> <div className="addon-price"> {addon.priceLabel || (addon.price === 0 ? 'Gratis' : `+ €${addon.price.toFixed(2)}`)} </div> </div> ))} </div> {showTotal && selected.length > 0 && ( <div className="addon-total" aria-live="polite"> <div className="addon-total-label"> Geselecteerde extra's ({selected.length}): </div> <div className="addon-total-price"> <span className="currency">€</span> {totalPrice.toFixed(2)} </div> </div> )} </div> ); }

Performance Considerations

  • Memoization: Use React.memo for individual addon cards to prevent unnecessary re-renders
  • Debouncing: Debounce price calculations if onChange triggers expensive operations
  • Lazy rendering: For long lists (10+), use virtualization (react-window)
  • Image optimization: Use next/image for icon images with proper dimensions

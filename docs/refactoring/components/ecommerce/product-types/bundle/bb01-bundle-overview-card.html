BB01: Bundle Overview Card

Pre-configured product bundle cards with pricing, savings, and popularity indicators

STEP 1: Component Overview

Purpose: Display pre-made product bundles that users can select instead of building their own custom bundle. Shows bundle composition, pricing, and savings at a glance.

Key Features

  • Product Icons: Visual representation of included products via emoji/icons (2-5 products)
  • Bundle Name & Description: Clear title and brief description of what's included
  • Pricing Display: Current price + old price (strike-through) + savings badge
  • Popular Badge: "Populairst" label for best-selling bundles
  • Hover States: Border color change, elevation, smooth transitions
  • Click to Select: Entire card is clickable to select the bundle

Usage Context

Use this component on bundle builder pages, product upsell sections, or checkout flows to showcase pre-configured packages that offer convenience and savings compared to individual purchases.

Visual Description

White card with centered content layout. Product icons displayed in a horizontal row at top, followed by bundle name, description, pricing (with old price strike-through), and green savings badge. Popular bundles show a teal banner across the top.

STEP 2: When to Use

✅ Primary Use Cases

  1. Bundle Builder Pages: Show curated package options alongside custom bundle builder
  2. Upsell Sections: Suggest popular bundles during product browsing or checkout
  3. Pricing Pages: Display tiered bundle offerings (Starter, Pro, Enterprise)
  4. Category Pages: Feature themed product sets (e.g., "Complete Design Kit", "Developer Essentials")
  5. Cart Recommendations: Suggest completing a bundle when user has 1-2 matching items

❌ When NOT to Use

  • Single Products: Use standard product cards (EC01) instead
  • Subscription Plans: Use SubscriptionPlanCard (SP01) for recurring billing
  • Custom Bundles: Use BundleProductCard (BB02) for user-built bundles
  • Service Packages: Use a different card design for non-product bundles
💡 Pro Tip: Limit to 3-4 bundle options per section to avoid choice paralysis. Use the "popular" badge on your best-selling bundle to guide decision-making.

STEP 7: Implementation Notes

React/Next.js Component Example

import React from 'react';
import { Package } from 'lucide-react';

interface BundleOverviewCardProps {
  id: string;
  name: string;
  description: string;
  productIcons: string[];
  price: number;
  originalPrice: number;
  savings?: number;
  showPercentage?: boolean;
  isPopular?: boolean;
  size?: 'compact' | 'default' | 'large';
  onClick?: () => void;
  className?: string;
}

export const BundleOverviewCard: React.FC<BundleOverviewCardProps> = ({
  id,
  name,
  description,
  productIcons,
  price,
  originalPrice,
  savings = originalPrice - price,
  showPercentage = true,
  isPopular = false,
  size = 'default',
  onClick,
  className = ''
}) => {
  const discountPercentage = Math.round((savings / originalPrice) * 100);

  const handleClick = () => {
    onClick?.();
  };

  const handleKeyDown = (e: React.KeyboardEvent) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      onClick?.();
    }
  };

  return (
    <article
      className={`bundle-overview-card ${isPopular ? 'popular' : ''} ${size} ${className}`}
      role="button"
      tabIndex={0}
      onClick={handleClick}
      onKeyDown={handleKeyDown}
      aria-label={`Select ${name} bundle. ${description}. Current price ${price} euros, was ${originalPrice} euros. Save ${savings} euros.`}
    >
      {/* Product Icons */}
      <div className="bundle-icons" aria-hidden="true">
        {productIcons.map((icon, index) => (
          <div key={index} className="bundle-emoji">
            {icon}
          </div>
        ))}
      </div>

      {/* Bundle Info */}
      <h3 className="bundle-name">{name}</h3>
      <p className="bundle-desc">{description}</p>

      {/* Pricing */}
      <div className="bundle-pricing">
        <span className="bundle-price" aria-label={`Current price ${price} euros`}>
          €{price}
        </span>
        <del className="bundle-old-price" aria-label={`Original price ${originalPrice} euros`}>
          €{originalPrice}
        </del>
      </div>

      {/* Savings Badge */}
      <span className="bundle-save" aria-label={`You save ${savings} euros`}>
        Bespaar €{savings}
        {showPercentage && ` (${discountPercentage}%)`}
      </span>
    </article>
  );
};

Performance Optimization

  • Lazy Loading: Load bundle images only when visible (IntersectionObserver)
  • Memoization: Memoize savings calculation if recalculated frequently
  • CSS Containment: Add contain: layout style paint; for better rendering performance
  • Bundle Size: Use SVG icons instead of emoji for smaller payloads (optional)

Edge Cases & Error Handling

  • Missing Products: Show placeholder icon if product icon is undefined
  • Invalid Pricing: Validate originalPrice > price, show error if not
  • Too Many Icons: If >5 products, show first 4 + "+X more" indicator
  • Zero Savings: Don't show savings badge if savings <= 0
  • Long Names: Truncate bundle name at 40 characters with ellipsis

Browser Compatibility

  • Grid Layout: Supported in all modern browsers (IE11 needs fallback)
  • CSS Custom Properties: Supported (IE11 needs PostCSS processing)
  • Flexbox: Fully supported
  • Emoji Support: May render differently across OS (use SVG for consistency)

Migration Notes

From Old Bundle Cards: If migrating from a previous bundle card design, ensure:
  • Update pricing to use price and originalPrice props (not discountedPrice)
  • Replace custom badge logic with isPopular boolean
  • Migrate product icons to emoji/string array format
  • Test keyboard navigation (may not have existed in old version)
⚠️ Breaking Change: The products prop now expects an array of icon strings, not full product objects. Use a transform function to extract icons from product data before passing to this component.

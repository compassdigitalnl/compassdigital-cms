STEP 1: Component Overview

Component Name: BB06: BundleProgressBar

Category: Product Types › Bundle Builder

Version: 1.0.0

Last Updated: 25 Februari 2026

Purpose

The BundleProgressBar component visualizes the user's progress toward completing their bundle. It displays the number of items selected, a visual progress bar, and the currently active discount percentage. This component motivates users to complete their bundle and reinforces the value proposition through real-time visual feedback.

Key Features

  • Visual Progress: Horizontal bar with teal gradient fill showing completion percentage
  • Item Counter: Monospace display (e.g., "3/5 producten") in teal color
  • Active Discount Badge: Green pill showing current discount (e.g., "−20% korting actief")
  • Smooth Animation: 0.4s cubic-bezier transition on progress bar width changes
  • Complete State: Green gradient when bundle is fully populated
  • Compact Layout: Horizontal flex with responsive mobile stacking
  • White Card Design: Standalone card with border for visual separation

Visual Hierarchy

  1. Progress Bar Fill: Teal gradient (most visually prominent)
  2. Item Count: Teal monospace text (second most prominent)
  3. Discount Badge: Green background (third, motivational element)
  4. Label: Navy text (fourth, contextual label)

Progress Calculation

Progress % = (Current Items / Max Items) × 100

Example:
3 items selected, 5 max → 60% progress
5 items selected, 5 max → 100% progress (complete)

STEP 2: When to Use

Primary Use Cases

  • Bundle Builder Top: Main use case — displays at top of bundle sidebar as persistent progress indicator
  • Multi-step Forms: Shows completion progress in stepped forms or wizards
  • Goal Tracking: Visualizes progress toward a specific item count goal
  • Sticky Header: Fixed at top on mobile during product selection
  • Onboarding Flows: Tracks user progress through setup steps

When NOT to Use

  • Checkout Steps: Use C25 (ProgressSteps) for discrete step indicators
  • Single Item Selection: Don't show progress bar for non-bundle products
  • Unknown Max Items: Progress bar requires a known maximum (denominator)
  • Complex Workflows: Use dedicated wizard component for branching flows

Recommended Pairings

  • BB02: BundleProductCard — Selectable products (each selection updates progress)
  • BB03: BundleItemRow — Selected products list (appears below this component)
  • BB04: BundleDiscountTiers — Alternative progress indicator (can replace or complement)
  • BB05: BundleTotalCalculator — Total calculator (appears at bottom, this at top)

Placement Guidelines

  • Bundle Sidebar: Very top, above item list (optimal position)
  • Fixed Header (Mobile): Sticky at top during scrolling on mobile
  • Modal Header: Top of modal when bundle builder in overlay
  • Cart Summary: Top of cart sidebar showing bundle progress

Progress Calculation

function calculateProgress(current: number, max: number): number {
  if (max === 0) return 0;
  return Math.min(100, (current / max) * 100);
}

// Usage

STEP 7: Implementation Notes

React/Next.js Component Example

'use client';

import { Tag, BadgeCheck } from 'lucide-react';
import { useMemo } from 'react';

interface BundleProgressBarProps {
  currentItems: number;
  maxItems: number;
  discount?: number;
  label?: string;
  showSavingsBadge?: boolean;
  size?: 'compact' | 'default';
}

export function BundleProgressBar({
  currentItems,
  maxItems,
  discount = 0,
  label = 'Je bundel',
  showSavingsBadge = true,
  size = 'default',
}: BundleProgressBarProps) {
  // Calculate progress percentage
  const progressPercentage = useMemo(() => {
    if (maxItems === 0) return 0;
    return Math.min(100, (currentItems / maxItems) * 100);
  }, [currentItems, maxItems]);

  // Determine if bundle is complete
  const isComplete = currentItems >= maxItems;

  // Generate savings badge text
  const savingsBadgeText = useMemo(() => {
    if (discount === 0) return null;
    if (isComplete) return `−${discount}% max korting!`;
    return `−${discount}% korting actief`;
  }, [discount, isComplete]);

  return (
    <div
      className={`progress ${isComplete ? 'complete' : ''} ${size !== 'default' ? size : ''}`}
      role="region"
      aria-label="Bundle progress"
    >
      <div className="prog-label" id="prog-label">
        {label}
      </div>

      <div
        className="prog-bar"
        role="progressbar"
        aria-labelledby="prog-label"
        aria-valuenow={currentItems}
        aria-valuemin={0}
        aria-valuemax={maxItems}
        aria-valuetext={`${currentItems} van ${maxItems} items geselecteerd`}
      >
        <div
          className="prog-fill"
          style={{ width: `${progressPercentage}%` }}
        />
      </div>

      <div className="prog-count" aria-live="polite">
        {currentItems}/{maxItems} producten
      </div>

      {showSavingsBadge && discount > 0 && (
        <div className="prog-savings" role="status" aria-live="polite">
          {isComplete ? (
            <BadgeCheck size={13} aria-hidden="true" />
          ) : (
            <Tag size={13} aria-hidden="true" />
          )}
          {savingsBadgeText}
        </div>
      )}
    </div>
  );
}

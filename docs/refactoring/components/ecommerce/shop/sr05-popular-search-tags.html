Purpose:
PopularSearchTags displays trending or frequently searched queries as clickable pill tags. It helps users discover popular products/categories and reduces search friction by suggesting common terms. Data can be sourced from analytics (real trending searches) or curated manually (strategic merchandising).

Key Features:

  • White card with trending-up icon header
  • Flex-wrapped pill tags (100px border-radius)
  • Teal text on light teal background (default)
  • Hover: solid teal background + white text
  • Small font (12px) for compact display
  • Responsive wrapping (2-3 tags per row typically)
  • Supports 3-10 tags (5-8 recommended)
  • Analytics-driven or manually curated

When to Use:

  • ✅ Search results page sidebar
  • ✅ Shop archive sidebar (below filters)
  • ✅ Homepage sidebar (discovery)
  • ✅ Empty search state (suggestions)
  • ✅ Below RecentSearchesSidebar (SR04)
  • ❌ Don't show if no analytics data available
  • ❌ Don't use for navigation (use CategoryNav instead)
  • ❌ Don't show too many tags (max 10)

STEP 2: When to Use

Use Cases:

  • Trending Products: Seasonal items (e.g., "mondmaskers" during flu season)
  • Popular Categories: Most-searched categories ("handschoenen", "spuiten")
  • Strategic Merchandising: Promote specific products/brands
  • Discovery: Help users find what they didn't know they needed
  • Empty State: Show suggestions when no recent search history

Data Sources:

  • Analytics: Top searches from past 7/30 days (real trending)
  • Manual Curation: Merchandising team selects strategic queries
  • Hybrid: 70% analytics + 30% curated (balance discovery + strategy)
  • Personalized: Based on user's browsing category (future feature)

Anti-patterns:

  • ❌ Too many tags (≥15) causing visual clutter
  • ❌ Outdated tags (e.g., "COVID test" after pandemic)
  • ❌ Misspelled queries from analytics (clean data first)
  • ❌ Sensitive/inappropriate search terms
  • ❌ Duplicate tags (e.g., "handschoenen" + "handschoen")

Guidelines:

  • Show 5-8 tags typically (max 10)
  • Update tags weekly/monthly based on data
  • Order by search volume (most popular first)
  • Normalize queries (lowercase, trim, dedupe)
  • Filter out brand names if licensing prohibits
  • Track click-through rate per tag

Analytics Query (Server-Side):

// Example: Get top searches from past 30 days
async function getPopularSearches(limit = 8): Promise {
  const thirtyDaysAgo = new Date()
  thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30)

  const searches = await db.searchLogs.aggregate([
    {
      $match: {
        timestamp: { $gte: thirtyDaysAgo },
        resultCount: { $gt: 0 } // Only successful searches
      }
    },
    {
      $group: {
        _id: { $toLower: '$query' }, // Normalize
        count: { $sum: 1 }
      }
    },
    {
      $sort: { count: -1 }
    },
    {
      $limit: limit
    }
  ])

  return searches.map(s => ({
    query: s._id,
    searchVolume: s.count,
    isCurated: false
  }))
}

Hybrid Strategy:

async function getHybridTags(limit = 8): Promise {
  const analyticsTags = await getPopularSearches(Math.floor(limit * 0.7))
  const curatedTags = await payload.findGlobal({ slug: 'popularSearches' })

  // Merge: curated first, then analytics (dedupe)
  const merged = [
    ...curatedTags.tags.map(t => ({ ...t, isCurated: true })),
    ...analyticsTags.filter(a =>
      !curatedTags.tags.some(c => c.query.toLowerCase() === a.query.toLowerCase())
    )
  ]

  return merged.slice(0, limit)
}

STEP 7: Implementation Notes

React/Next.js Component:

'use client'
import { TrendingUp, Fire, Zap, Star } from 'lucide-react'
import Link from 'next/link'

const iconMap = {
  'trending-up': TrendingUp,
  'fire': Fire,
  'zap': Zap,
  'star': Star
}

export function PopularSearchTags({
  tags,
  maxTags = 8,
  title = 'Populaire zoekopdrachten',
  icon = 'trending-up',
  onTagClick
}: PopularSearchTagsProps) {
  const Icon = iconMap[icon] || TrendingUp

  if (tags.length === 0) return null

  return (
    <aside className="popular-search-tags" aria-label="Popular searches">
      <div className="popular-search-header">
        <Icon className="popular-search-icon" />
        <h3 className="popular-search-title">{title}</h3>
      </div>
      <div className="popular-search-tags-list">
        {tags.slice(0, maxTags).map((tag, index) => (
          <Link
            key={tag.query}
            href={`/search?q=${encodeURIComponent(tag.query)}`}
            className="popular-search-tag"
            onClick={() => onTagClick?.(tag.query)}
          >
            {tag.query}
          </Link>
        ))}
      </div>
    </aside>
  )
}

Analytics Tracking:

function trackPopularTagClick(query: string, position: number, isCurated: boolean) {
  gtag('event', 'popular_tag_click', {
    query,
    position,
    source: isCurated ? 'curated' : 'analytics',
    event_category: 'Search',
    event_label: 'Popular Search Widget'
  })

  // Track CTR per tag for optimization
  fetch('/api/analytics/tag-click', {
    method: 'POST',
    body: JSON.stringify({ query, position })
  })
}

Tag Normalization:

function normalizeTag(query: string): string {
  return query
    .toLowerCase()
    .trim()
    .replace(/\s+/g, ' ')        // Collapse multiple spaces
    .replace(/[^a-z0-9\s-]/gi, '') // Remove special chars
}

function deduplicateTags(tags: PopularTag[]): PopularTag[] {
  const seen = new Set()
  return tags.filter(tag => {
    const normalized = normalizeTag(tag.query)
    if (seen.has(normalized)) return false
    seen.add(normalized)
    return true
  })
}

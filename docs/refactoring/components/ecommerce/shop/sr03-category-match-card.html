📋 Component Documentation

STEP 1: Component Overview

Component ID: SR03

Component Name: CategoryMatchCard

Category: E-commerce / Search

Priority: 🟡 Medium

Source File: ef6-search-results.html

Output Location: /refactoring/components/ecommerce/sr03-category-match-card.html

Purpose:
CategoryMatchCard provides quick filter shortcuts for search results. When a search query matches categories, brands, or content types, these cards appear above product results as one-click filters to narrow the search scope. They improve search UX by surfacing high-level matches before detailed product listings.

Key Features:

  • Compact horizontal card layout (icon + label + count)
  • Teal icon (18px) for visual categorization
  • Two-line text: bold label + grey count
  • Hover state: teal border + light teal background
  • 6 icon variants: layers, tag, file-text, package, store, sparkles
  • Responsive wrapping (flex-wrap)
  • Clickable link (navigates to filtered results)
  • Keyboard accessible with focus ring

When to Use:

  • ✅ Search results page with category/brand matches
  • ✅ Query matches content types (articles, products, bundles)
  • ✅ Multiple entity types found (category + brand + content)
  • ✅ Quick filter shortcuts above main results
  • ❌ Don't show if only product results (no high-level matches)
  • ❌ Don't use for primary navigation (use FilterSidebar instead)
  • ❌ Don't show more than 6 cards (information overload)

STEP 2: When to Use

Use Cases:

  • Category Match: Search "handschoenen" → show "Categorie: Handschoenen (87 producten)"
  • Brand Match: Search "sempermed" → show "Merk: Sempermed (23 producten)"
  • Content Match: Search "latex allergy" → show "Kennisbank artikelen (5 artikelen)"
  • Material Match: Search "nitril" → show "Materiaal: Nitril (78 producten)"
  • Bundle Match: Search "starter kit" → show "Bundel: Starter Kit (1 product)"
  • Vendor Match: Search specific vendor name → show "Leverancier: X (56 producten)"

Anti-patterns:

  • ❌ Showing cards with 0 results (misleading)
  • ❌ Duplicate information (category card + sidebar filter both active)
  • ❌ Too many cards (≥7) causing visual clutter
  • ❌ Generic labels without specificity ("Products", "Items")
  • ❌ Cards that don't actually filter results when clicked

Guidelines:

  • Show max 6 cards, prioritize by match relevance
  • Display count for context (87 producten, 5 artikelen)
  • Use singular/plural correctly based on count
  • Order: Category → Brand → Material → Vendor → Content → Bundle
  • Hide section if no high-level matches found
  • Track clicks for search analytics

Payload CMS Integration:

// Server-side search logic
async function generateCategoryMatches(query: string): Promise {
  const matches: CategoryMatch[] = []

  // Category match
  const category = await payload.find({
    collection: 'categories',
    where: { name: { contains: query } },
    limit: 1
  })
  if (category.docs.length > 0) {
    const count = await getProductCountForCategory(category.docs[0].id)
    matches.push({
      id: category.docs[0].id,
      type: 'category',
      label: `Categorie: ${category.docs[0].name}`,
      count,
      countUnit: 'producten',
      url: `/shop?category=${category.docs[0].slug}`
    })
  }

  // Brand match
  const brand = await payload.find({
    collection: 'brands',
    where: { name: { contains: query } },
    limit: 1
  })
  if (brand.docs.length > 0) {
    const count = await getProductCountForBrand(brand.docs[0].id)
    matches.push({
      id: brand.docs[0].id,
      type: 'brand',
      label: `Merk: ${brand.docs[0].name}`,
      count,
      countUnit: 'producten',
      url: `/shop?brand=${brand.docs[0].slug}`
    })
  }

  // Content match
  const articles = await payload.find({
    collection: 'posts',
    where: {
      or: [
        { title: { contains: query } },
        { content: { contains: query } }
      ]
    },
    limit: 100
  })
  if (articles.totalDocs > 0) {
    matches.push({
      id: 'content',
      type: 'content',
      label: 'Kennisbank artikelen',
      count: articles.totalDocs,
      countUnit: 'artikelen',
      url: `/kennisbank?q=${encodeURIComponent(query)}`
    })
  }

  return matches.slice(0, 6) // Max 6 cards
}

STEP 7: Implementation Notes

React/Next.js Component:

'use client'
import { Layers, Tag, FileText, Package, Store, Sparkles } from 'lucide-react'
import Link from 'next/link'

const iconMap = {
  category: Layers,
  brand: Tag,
  content: FileText,
  bundle: Package,
  vendor: Store,
  material: Sparkles
}

export function CategoryMatchCard({ match, onClick }: CategoryMatchCardProps) {
  const Icon = iconMap[match.type]

  return (
    <Link
      href={match.url}
      className="category-match-card"
      onClick={() => onClick?.(match)}
      aria-label={`Filter naar ${match.label}, ${match.count} ${match.countUnit}`}
    >
      <Icon className="category-match-icon" />
      <div className="category-match-content">
        <div className="category-match-label">{match.label}</div>
        <div className="category-match-count">
          {match.count} {match.countUnit || 'items'}
        </div>
      </div>
    </Link>
  )
}

export function CategoryMatches({ matches, maxCards = 6 }: CategoryMatchesProps) {
  if (matches.length === 0) return null

  return (
    <div className="category-matches-container">
      {matches.slice(0, maxCards).map(match => (
        <CategoryMatchCard key={match.id} match={match} />
      ))}
    </div>
  )
}

Pluralization Helper:

function pluralize(count: number, singular: string, plural: string): string {
  return count === 1 ? singular : plural
}

// Usage:
const countUnit = pluralize(match.count, 'product', 'producten')
const countUnit = pluralize(match.count, 'artikel', 'artikelen')
const countUnit = pluralize(match.count, 'bundel', 'bundels')

Analytics Tracking:

function trackCategoryMatchClick(match: CategoryMatch, originalQuery: string) {
  gtag('event', 'category_match_click', {
    match_type: match.type,
    match_label: match.label,
    result_count: match.count,
    original_query: originalQuery,
    event_category: 'Search',
    event_label: 'Quick Filter'
  })
}

URL Generation Pattern:

function generateMatchURL(match: CategoryMatch, baseQuery: string): string {
  const params = new URLSearchParams({ q: baseQuery })

  switch (match.type) {
    case 'category':
      params.set('category', match.id)
      return `/shop?${params}`
    case 'brand':
      params.set('brand', match.id)
      return `/shop?${params}`
    case 'content':
      return `/kennisbank?${params}`
    case 'bundle':
      params.set('type', 'bundle')
      return `/shop?${params}`
    // ... etc
  }
}

Performance Optimization:

  • Generate matches server-side to avoid client-side DB queries
  • Cache category/brand counts (update daily)
  • Limit to 6 cards max to prevent DOM bloat
  • Use Link component for prefetching (Next.js)

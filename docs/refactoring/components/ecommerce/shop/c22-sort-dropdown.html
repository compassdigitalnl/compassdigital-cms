STEP 1: Component Overview

C22: SortDropdown is a dual-function toolbar component that combines product sorting with view mode toggling. It provides users with control over how products are ordered and displayed in shop archives.

Component Composition

  • Sort Dropdown: Native <select> element with custom chevron arrow styling
  • View Toggle: Button group for switching between grid and list layouts
  • Result Count: Optional text showing filtered product count

Key Features

  • Custom-styled native select dropdown (accessibility + mobile-friendly)
  • Grid/List view toggle with icon indicators
  • Teal accent for focus states and active view
  • Responsive layout (stacks on mobile)
  • 3 size variants: sm, md (default), lg
  • LocalStorage persistence for user preference

Best Practice: The sort dropdown uses a native <select> element instead of a custom dropdown for better accessibility, keyboard navigation, and mobile support. The custom arrow is applied via CSS background-image.

STEP 2: When to Use

Primary Use Cases

  • Product Archives: Shop category pages, search results, brand pages
  • Content Archives: Blog listings, article archives, knowledge base
  • Directory Listings: Vendor directories, team member grids, location listings
  • Search Results: Any paginated result set requiring sorting

When NOT to Use

  • Single Product Page: No sorting needed for individual items
  • Cart/Checkout: Fixed order preferred for transaction flow
  • Static Content: Pages without dynamic filtering/sorting
  • <5 Items: Sorting unnecessary for very small lists

Typical Placement

  • Between filters sidebar and product grid
  • Right-aligned (sort + view toggle) with left-aligned result count
  • Above pagination, below active filter tags
  • Sticky on scroll for long product lists (optional)

Mobile Consideration: On screens <640px, the toolbar stacks vertically with the sort dropdown taking full width. The view toggle shrinks but remains visible.

// Helper function
function announceToScreenReader(message) {
  const announcement = document.createElement('div');
  announcement.setAttribute('role', 'status');
  announcement.setAttribute('aria-live', 'polite');
  announcement.className = 'sr-only'; // Visually hidden
  announcement.textContent = message;
  document.body.appendChild(announcement);
  setTimeout(() => announcement.remove(), 1000);
}

STEP 7: Implementation Notes

React/TypeScript Component

import React, { useState, useEffect } from 'react';

interface SortOption {
  value: string;
  label: string;
}

interface SortDropdownProps {
  options: SortOption[];
  defaultValue?: string;
  defaultView?: 'grid' | 'list';
  size?: 'sm' | 'md' | 'lg';
  onSortChange?: (value: string) => void;
  onViewChange?: (view: 'grid' | 'list') => void;
  persistPreference?: boolean;
}

export function SortDropdown({
  options,
  defaultValue = 'relevance',
  defaultView = 'grid',
  size = 'md',
  onSortChange,
  onViewChange,
  persistPreference = true
}: SortDropdownProps) {
  const [sortValue, setSortValue] = useState(defaultValue);
  const [viewMode, setViewMode] = useState(defaultView);

  // Load saved preferences
  useEffect(() => {
    if (persistPreference) {
      const saved = localStorage.getItem('shop-preferences');
      if (saved) {
        const { sort, view } = JSON.parse(saved);
        if (sort) setSortValue(sort);
        if (view) setViewMode(view);
      }
    }
  }, [persistPreference]);

  // Save preferences on change
  const savePreference = (sort: string, view: string) => {
    if (persistPreference) {
      localStorage.setItem('shop-preferences', JSON.stringify({ sort, view }));
    }
  };

  const handleSortChange = (e: React.ChangeEvent<HTMLSelectElement>) => {
    const newValue = e.target.value;
    setSortValue(newValue);
    savePreference(newValue, viewMode);
    onSortChange?.(newValue);
  };

  const handleViewChange = (newView: 'grid' | 'list') => {
    setViewMode(newView);
    savePreference(sortValue, newView);
    onViewChange?.(newView);
  };

  return (
    <div className="shop-toolbar-right">
      <select
        className={`sort-select ${size !== 'md' ? size : ''}`}
        value={sortValue}
        onChange={handleSortChange}
        aria-label="Sorteer producten"
      >
        {options.map(opt => (
          <option key={opt.value} value={opt.value}>
            {opt.label}
          </option>
        ))}
      </select>

      <div className={`view-toggle ${size !== 'md' ? size : ''}`} role="group" aria-label="Weergave modus">
        <button
          className={`view-btn ${viewMode === 'grid' ? 'active' : ''}`}
          onClick={() => handleViewChange('grid')}
          aria-label="Grid weergave"
          aria-pressed={viewMode === 'grid'}
        >
          <i data-lucide="grid-3x3" aria-hidden="true"></i>
        </button>
        <button
          className={`view-btn ${viewMode === 'list' ? 'active' : ''}`}
          onClick={() => handleViewChange('list')}
          aria-label="Lijst weergave"
          aria-pressed={viewMode === 'list'}
        >
          <i data-lucide="list" aria-hidden="true"></i>
        </button>
      </div>
    </div>
  );
}

Custom Arrow SVG

The chevron arrow is applied via CSS background-image using a data URI:

background-image: url("data:image/svg+xml,%3Csvg width='12' height='8' viewBox='0 0 12 8' fill='none' xmlns='http://www.w3.org/2000/svg'%3E%3Cpath d='M1 1.5L6 6.5L11 1.5' stroke='%2394A3B8' stroke-width='2' stroke-linecap='round'/%3E%3C/svg%3E");
background-repeat: no-repeat;
background-position: right 12px center;

Note: The stroke color %2394A3B8 is URL-encoded --grey-mid (#94A3B8).

URL State Sync (Optional)

For shareable URLs with sort/view state:

// Read from URL params
const params = new URLSearchParams(window.location.search);
const initialSort = params.get('sort') || 'relevance';
const initialView = params.get('view') || 'grid';

// Update URL on change
function updateURL(sort: string, view: string) {
  const url = new URL(window.location.href);
  url.searchParams.set('sort', sort);
  url.searchParams.set('view', view);
  window.history.replaceState({}, '', url.toString());
}

Testing Checklist

  •  Sort dropdown opens and closes correctly
  •  Selected sort option displays in dropdown
  •  View toggle switches between grid and list
  •  Active view button has teal background + icon
  •  Focus states show teal ring on all elements
  •  Keyboard navigation works (Tab, Enter, arrows)
  •  localStorage saves user preferences
  •  Responsive: stacks on mobile (<640px)
  •  Disabled state prevents interaction
  •  Screen reader announces sort/view changes
  •  Product grid updates when view changes
  •  All size variants (sm, md, lg) render correctly

Performance Tip: When sorting/filtering causes a re-render of the product grid, use CSS content-visibility: auto on product cards to improve rendering performance for large lists (>50 items).

STEP 1: Component Overview

Component ID: SR04

Component Name: RecentSearchesSidebar

Category: Navigation / Search

Priority: 🟢 Low

Source File: ef6-search-results.html

Purpose:
RecentSearchesSidebar displays the user's recent search queries in a sidebar widget, allowing quick re-execution of past searches. It improves UX by reducing repeated typing and surfaces search patterns for analytics. Stored in localStorage/cookies, it persists across sessions.

Key Features:

  • White card with clock icon header
  • Vertical list of recent queries (3-5 items typically)
  • Clickable links to re-execute searches
  • Hover state: light grey bg + teal text
  • Text overflow ellipsis for long queries
  • "Geschiedenis wissen" (Clear history) button
  • Empty state when no history
  • LocalStorage persistence

When to Use:

  • ✅ Search results page sidebar
  • ✅ Shop archive sidebar (below filters)
  • ✅ User has performed ≥1 search
  • ✅ Desktop layout with sidebar space
  • ❌ Don't show on first-time visitors (no history)
  • ❌ Don't show on mobile (limited space)
  • ❌ Don't persist sensitive/private search terms

STEP 2: When to Use

Use Cases:

  • Repeat Searches: User searches "handschoenen nitril" multiple times over days
  • Comparison Shopping: User searches related terms ("latex", "vinyl", "nitril")
  • Quick Access: Medical professional repeatedly searching common products
  • Session Continuity: User returns days later, sees previous search context
  • Analytics Insight: Track what users search for repeatedly

Anti-patterns:

  • ❌ Storing sensitive terms (patient names, personal medical info)
  • ❌ Showing failed searches (0 results) without context
  • ❌ Duplicate entries for identical queries
  • ❌ Infinite history (max 5-10 items)
  • ❌ No privacy controls (should allow clearing)

Guidelines:

  • Store max 5-10 recent searches (FIFO queue)
  • Remove duplicates (keep most recent)
  • Expire entries after 30 days
  • Provide "Clear history" button
  • Respect DNT (Do Not Track) header
  • Use sessionStorage for incognito mode detection

STEP 4: Props & Configuration

TypeScript Interface:

interface RecentSearch {
  query: string
  timestamp: number          // Unix timestamp
  resultCount?: number       // Optional: track if search was successful
}

interface RecentSearchesSidebarProps {
  searches: RecentSearch[]
  maxItems?: number          // Default: 5
  onSearchClick?: (query: string) => void
  onClearHistory?: () => void
  emptyMessage?: string
}

LocalStorage Pattern:

const STORAGE_KEY = 'recentSearches'
const MAX_ITEMS = 5
const EXPIRY_DAYS = 30

function saveSearch(query: string): void {
  const searches = getRecentSearches()

  // Remove duplicates (keep most recent)
  const filtered = searches.filter(s => s.query !== query)

  // Add new search at beginning
  filtered.unshift({
    query,
    timestamp: Date.now(),
    resultCount: undefined // Set after results load
  })

  // Trim to max items
  const trimmed = filtered.slice(0, MAX_ITEMS)

  localStorage.setItem(STORAGE_KEY, JSON.stringify(trimmed))
}

function getRecentSearches(): RecentSearch[] {
  const stored = localStorage.getItem(STORAGE_KEY)
  if (!stored) return []

  const searches = JSON.parse(stored)
  const now = Date.now()
  const expiryMs = EXPIRY_DAYS * 24 * 60 * 60 * 1000

  // Filter expired entries
  return searches.filter(s => now - s.timestamp < expiryMs)
}

function clearHistory(): void {
  localStorage.removeItem(STORAGE_KEY)
}

Privacy-Aware Implementation:

function shouldTrackSearchHistory(): boolean {
  // Check DNT header
  if (navigator.doNotTrack === '1') return false

  // Check user preferences
  const consent = localStorage.getItem('cookieConsent')
  if (consent !== 'analytics') return false

  // Use sessionStorage in incognito mode
  try {
    localStorage.setItem('test', 'test')
    localStorage.removeItem('test')
    return true
  } catch {
    // Incognito mode - use sessionStorage instead
    return false
  }
}

STEP 7: Implementation Notes

React/Next.js Component:

'use client'
import { Clock } from 'lucide-react'
import Link from 'next/link'
import { useState, useEffect } from 'react'

export function RecentSearchesSidebar({
  maxItems = 5,
  onSearchClick,
  onClearHistory,
  emptyMessage = 'Je hebt nog geen zoekopdrachten uitgevoerd.'
}: RecentSearchesSidebarProps) {
  const [searches, setSearches] = useState([])

  useEffect(() => {
    setSearches(getRecentSearches())
  }, [])

  const handleClear = () => {
    clearHistory()
    setSearches([])
    onClearHistory?.()
  }

  if (searches.length === 0) {
    return (
      <aside className="recent-searches-sidebar" aria-label="Recent searches">
        <div className="recent-searches-header">
          <Clock className="recent-searches-icon" />
          <h3 className="recent-searches-title">Recent gezocht</h3>
        </div>
        <div className="recent-searches-empty">{emptyMessage}</div>
      </aside>
    )
  }

  return (
    <aside className="recent-searches-sidebar" aria-label="Recent searches">
      <div className="recent-searches-header">
        <Clock className="recent-searches-icon" />
        <h3 className="recent-searches-title">Recent gezocht</h3>
      </div>
      <div className="recent-searches-list">
        {searches.slice(0, maxItems).map((search, index) => (
          <Link
            key={`${search.query}-${index}`}
            href={`/search?q=${encodeURIComponent(search.query)}`}
            className="recent-search-item"
            onClick={() => onSearchClick?.(search.query)}
          >
            {search.query}
          </Link>
        ))}
      </div>
      <button
        className="recent-searches-clear"
        onClick={handleClear}
        aria-label="Clear search history"
      >
        Geschiedenis wissen
      </button>
    </aside>
  )
}

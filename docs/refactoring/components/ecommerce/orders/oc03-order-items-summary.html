STEP 1: Component Overview

Purpose

The OrderItemsSummary component displays a compact list of products included in an order. It appears on the order confirmation page below the order details cards (OC02), providing customers with a quick visual reference of what they purchased. Each item shows a thumbnail, product name, variant details (size/color), quantity, and line price.

Component Identity

  • Name: OrderItemsSummary
  • ID: OC03
  • Category: E-commerce / Order Display
  • Type: Read-only list display
  • Priority: High (Phase 2)

Usage Context

Primary use is on the order confirmation page (post-checkout). Also used in order detail pages in customer account area, and can be adapted for email confirmation templates (compact variant). Not suitable for cart flows (use EC06 CartLineItem instead).

Key Features

  • Clean horizontal row layout with thumbnail (60×60px), product info, and price
  • Grey header bar with product count (e.g., "2 producten")
  • Flexible metadata display (size, color, quantity) with icons
  • Subtle hover effect (light teal background)
  • Collapsible variant for long order lists
  • Compact variant for email templates (smaller thumbnails and text)
  • Empty state for edge cases
  • Responsive: stacks price below on mobile

STEP 2: When to Use This Component

Primary Use Cases

  1. Order Confirmation Page: Display purchased items after checkout success
  2. Order History (Account Area): Show order contents in order detail view
  3. Email Confirmation: Compact variant in order confirmation emails
  4. Admin Order View: Quick overview of order contents
  5. Invoice Pages: List items on printable invoices

When NOT to Use

  • Cart page: Use EC06 CartLineItem instead (includes qty stepper, remove button)
  • Checkout review: Use compact variant of EC06 CartLineItem
  • Product pages: Not suitable for product listings
  • Search results: Use EC01 ProductCard

Content Guidelines

Product Names: Display full product name, truncate to 2 lines on mobile. Avoid abbreviations.

Metadata: Show relevant variants (size, color, qty) with icons. Keep to 2-3 metadata items max per product. Quantity is always shown.

Prices: Show line total (price × quantity), not unit price. Format as "€ XX,XX" with space after €. Use Plus Jakarta Sans 800 weight for visual hierarchy.

Thumbnails: 60×60px (default), 48×48px (compact). Use placeholder emoji or actual product images with light grey background.

Variants Selection Guide

  • Default: Order confirmation page, order history (2-10 items)
  • Collapsible: Long orders (10+ items) to save vertical space
  • Compact: Email templates, mobile, sidebar widgets
  • Empty: Error handling (order sync issues, cancelled orders)

STEP 7: Implementation Notes

React/Next.js Component

import { Package, PackageX, Ruler, Hash, Palette, Droplet } from 'lucide-react'
import Image from 'next/image'

const metadataIconMap = {
  ruler: Ruler,
  hash: Hash,
  palette: Palette,
  droplet: Droplet,
}

export default function OrderItemsSummary({
  items,
  variant = 'default',
  title = 'Bestelde producten',
  showCount = true,
  emptyState,
}: OrderItemsSummaryProps) {
  const [isCollapsed, setIsCollapsed] = useState(false)

  const formatPrice = (cents: number) => {
    return `€ ${(cents / 100).toFixed(2).replace('.', ',')}`
  }

  if (items.length === 0) {
    return (
      <section className="order-items-summary" aria-label="Order items summary">
        <div className="order-items-header">
          <h2 className="order-items-title">
            <Package size={20} aria-hidden="true" />
            {title}
            {showCount && <span className="order-items-count">0 producten</span>}
          </h2>
        </div>
        <div className="order-items-empty">
          <PackageX size={48} />
          <div className="order-items-empty-title">{emptyState?.title}</div>
          <div className="order-items-empty-text">{emptyState?.description}</div>
        </div>
      </section>
    )
  }

  const isCollapsible = variant === 'collapsible'

  return (
    <section
      className={`order-items-summary ${variant} ${isCollapsed ? 'collapsed' : ''}`}
      aria-label="Order items summary"
    >
      <div
        className="order-items-header"
        {...(isCollapsible && {
          role: 'button',
          'aria-expanded': !isCollapsed,
          onClick: () => setIsCollapsed(!isCollapsed),
        })}
      >
        <h2 className="order-items-title">
          <Package size={20} aria-hidden="true" />
          {title}
          {showCount && (
            <span className="order-items-count" aria-label={`${items.length} products`}>
              {items.length} {items.length === 1 ? 'product' : 'producten'}
            </span>
          )}
          {isCollapsible && (
            <div className="order-items-toggle">
              <ChevronDown size={20} />
            </div>
          )}
        </h2>
      </div>
      <ul className="order-items-list">
        {items.map((item) => (
          <li key={item.id} className="order-item">
            <div className="order-item-image">
              {item.image ? (
                <Image src={item.image} alt={item.name} width={60} height={60} />
              ) : (
                item.imagePlaceholder || '📦'
              )}
            </div>
            <div className="order-item-info">
              <h3 className="order-item-name">{item.name}</h3>
              {item.metadata && (
                <dl className="order-item-meta">
                  {item.metadata.map((meta, i) => {
                    const Icon = metadataIconMap[meta.icon as keyof typeof metadataIconMap]
                    return (
                      <div key={i} className="order-item-meta-item">
                        {Icon && <Icon size={12} aria-hidden="true" />}
                        <span>{meta.label}</span>
                      </div>
                    )
                  })}
                </dl>
              )}
            </div>
            <div
              className="order-item-price"
              aria-label={`Line total ${formatPrice(item.price)}`}
            >
              {formatPrice(item.price)}
            </div>
          </li>
        ))}
      </ul>
    </section>
  )
}

Payload CMS Data Transformation

// Transform Payload order data to component props
function transformOrderItems(order: PayloadOrder): OrderItem[] {
  return order.items.map((item) => {
    const product = item.product as Product
    const variant = item.variant as ProductVariant | undefined

    // Build metadata array
    const metadata = []

    if (variant?.attributes?.size) {
      metadata.push({ icon: 'ruler', label: `Maat: ${variant.attributes.size}` })
    }
    if (variant?.attributes?.color) {
      metadata.push({ icon: 'palette', label: `Kleur: ${variant.attributes.color}` })
    }
    metadata.push({ icon: 'hash', label: `Aantal: ${item.quantity}` })

    return {
      id: item.id,
      name: item.snapshot?.name || product.name,
      image: item.snapshot?.image || product.images?.[0]?.url,
      price: item.price,  // Line total in cents (already calculated)
      quantity: item.quantity,
      metadata,
    }
  })
}

Collapsible Toggle Function

function toggleCollapse(elementId: string) {
  const element = document.getElementById(elementId)
  if (element) {
    element.classList.toggle('collapsed')

    // Update aria-expanded
    const header = element.querySelector('[aria-expanded]')
    if (header) {
      const isExpanded = header.getAttribute('aria-expanded') === 'true'
      header.setAttribute('aria-expanded', String(!isExpanded))
    }
  }
}

STEP 8: Variations & Related Components

Component Variations

1. Default Variant

Standard order confirmation layout. 60×60px thumbnails, 16px font price, full padding. Best for 2-10 items.

2. Collapsible Variant

Adds clickable header with chevron icon. Collapses item list on click. Use for long orders (10+ items) to save vertical space. Header stays visible when collapsed.

<OrderItemsSummary variant="collapsible" items={items} />

3. Compact Variant

Smaller thumbnails (48×48px), reduced font sizes (13px name, 14px price), less padding. Designed for email templates, mobile sidebars, or space-constrained layouts.

<OrderItemsSummary variant="compact" items={items} />

4. Empty State

Displays centered icon + message when items array is empty. Useful for error handling, cancelled orders, or data sync issues.

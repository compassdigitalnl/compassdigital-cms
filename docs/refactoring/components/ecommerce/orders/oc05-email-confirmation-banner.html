STEP 1: Component Overview

Purpose

The EmailConfirmationBanner component displays a reassuring info message at the bottom of the order confirmation page, confirming that a confirmation email has been (or will be) sent to the customer's email address. It uses a blue background (info variant by default) with a mail icon and emphasizes the email address in bold.

Component Identity

  • Name: EmailConfirmationBanner
  • ID: OC05
  • Category: UI Elements / Notification Banner
  • Type: Read-only informational display
  • Priority: High (Phase 2 - completes Order Confirmation flow)

Usage Context

Appears as the last component on the order confirmation page, below the NextStepsCTA (OC04). Reassures customers that they'll receive email confirmation. Also usable for other confirmation scenarios (contact forms, subscriptions, account creation).

Key Features

  • Horizontal flex layout: icon (20×20px) + text + optional close button
  • 4 color variants: info (blue), success (green), warning (amber), error (coral)
  • Bold email address highlighting (<strong>)
  • Optional inline links (resend email, support contact)
  • Optional close button (dismiss banner)
  • Compact variant for smaller contexts
  • Responsive: stacks icon/text vertically on mobile

STEP 2: When to Use This Component

Primary Use Cases

  1. Order Confirmation: Confirm email sent with order details + invoice
  2. Account Registration: Confirm verification email sent
  3. Password Reset: Confirm reset link sent
  4. Contact Form Submission: Confirm receipt notification sent
  5. Subscription Confirmation: Confirm welcome email sent
  6. Quote Request: Confirm quote request received and email sent

When NOT to Use

  • Real-time errors: Use C17 ToastSystem for transient errors
  • Form validation errors: Use inline field errors
  • Empty states: Use dedicated empty state components
  • Multi-step flows: Use EC13 CheckoutProgressStepper

Content Guidelines

Message Structure: Start with clear action statement ("Je ontvangt..."), include timeframe ("binnen 5 minuten"), emphasize email address in <strong>, optionally add helpful links.

Email Address: Always show the customer's email address in bold to confirm correct address. Format as <strong>email@example.com</strong>.

Tone: Reassuring and helpful. Use "je" (informal Dutch) for B2C, "u" (formal) for B2B contexts.

Timeframe: If email is delayed (async sending), mention realistic timeframe (5-15 min). For instant sends, use "direct" or omit timeframe.

Variant Selection Guide

  • Info (blue): Default for confirmation messages (90% of cases)
  • Success (green): Email successfully sent (real-time confirmation)
  • Warning (amber): Email delayed or check spam folder reminder
  • Error (coral): Email sending failed, show support contact

STEP 7: Implementation Notes

React/Next.js Component

import { Mail, CheckCircle, AlertTriangle, XCircle, X } from 'lucide-react'

const iconMap = {
  info: Mail,
  success: CheckCircle,
  warning: AlertTriangle,
  error: XCircle,
}

export default function EmailConfirmationBanner({
  message,
  email,
  variant = 'info',
  icon,
  showClose = false,
  onClose,
  compact = false,
  link,
}: EmailConfirmationBannerProps) {
  const [isVisible, setIsVisible] = useState(true)

  const Icon = icon ? Icons[icon] : iconMap[variant]
  const iconSize = compact ? 16 : 20

  const handleClose = () => {
    setIsVisible(false)
    onClose?.()
  }

  if (!isVisible) return null

  return (
    <aside
      role="status"
      aria-live="polite"
      className={`email-confirmation-banner ${variant} ${compact ? 'compact' : ''}`}
    >
      <div className="banner-icon" aria-hidden="true">
        <Icon size={iconSize} />
      </div>
      <div className="banner-text">
        {typeof message === 'string' ? (
          <span dangerouslySetInnerHTML={{ __html: message }} />
        ) : (
          message
        )}
        {email && !message.includes(email) && (
          <> op <strong>{email}</strong></>
        )}
        {link && (
          <> <a href={link.href} onClick={link.onClick}>{link.text}</a></>
        )}
      </div>
      {showClose && (
        <button
          className="banner-close"
          onClick={handleClose}
          aria-label="Sluit melding"
        >
          <X size={16} />
        </button>
      )}
    </aside>
  )
}

Dynamic Variant Based on Email Status

function getEmailBannerProps(emailStatus: EmailStatus, email: string) {
  switch (emailStatus) {
    case 'sent':
      return {
        message: `Bevestigingsmail verzonden naar <strong>${email}</strong>`,
        variant: 'success' as const,
        icon: 'check-circle',
      }

    case 'pending':
      return {
        message: `Je ontvangt binnen 5 minuten een bevestigingsmail op <strong>${email}</strong>`,
        variant: 'info' as const,
        icon: 'mail',
      }

    case 'delayed':
      return {
        message: `<strong>Let op:</strong> Het kan tot 15 minuten duren voordat de bevestigingsmail arriveert. Check ook je spam folder.`,
        variant: 'warning' as const,
        icon: 'alert-triangle',
      }

    case 'failed':
      return {
        message: `<strong>Email kon niet verzonden worden.</strong> Neem contact op met support@example.com`,
        variant: 'error' as const,
        icon: 'x-circle',
        link: {
          text: 'Contacteer support',
          href: '/contact',
        },
      }

    default:
      return {
        message: `Orderbevestiging wordt verzonden naar <strong>${email}</strong>`,
        variant: 'info' as const,
      }
  }
}

Resend Email Handler

async function handleResendEmail(orderId: string) {
  try {
    const response = await fetch(`/api/orders/${orderId}/resend-email`, {
      method: 'POST',
    })

    if (response.ok) {
      // Show success toast
      toast.success('Bevestigingsmail opnieuw verzonden!')
    } else {
      toast.error('Kon email niet verzenden. Probeer het opnieuw.')
    }
  } catch (error) {
    console.error('Resend email error:', error)
    toast.error('Er ging iets mis.')
  }
}

<EmailConfirmationBanner
  message="Bevestigingsmail verzonden"
  email={order.customerEmail}
  link={{
    text: 'Mail niet ontvangen?',
    onClick: () => handleResendEmail(order.id),
  }}
/>

LocalStorage Persistence (Dismiss State)

function useDismissedBanner(bannerId: string) {
  const [isDismissed, setIsDismissed] = useState(() => {
    return localStorage.getItem(`banner_dismissed_${bannerId}`) === 'true'
  })

  const dismiss = () => {
    localStorage.setItem(`banner_dismissed_${bannerId}`, 'true')
    setIsDismissed(true)
  }

  return { isDismissed, dismiss }
}

// In component:
const { isDismissed, dismiss } = useDismissedBanner('email-confirmation')

if (isDismissed) return null

return (
  <EmailConfirmationBanner
    {...props}
    showClose={true}
    onClose={dismiss}
  />
)

STEP 8: Variations & Related Components

Component Variations

1. Info Variant (Default)

Blue background + mail icon. Standard for email confirmation messages. Use for 90% of cases.

2. Success Variant

Green background + check-circle icon. Use when email has been successfully sent (real-time confirmation via webhook).

3. Warning Variant

Amber background + alert-triangle icon. Use for delayed emails or spam folder reminders.

4. Error Variant

Coral background + x-circle icon. Use when email sending failed. Include support contact link.

5. Compact Variant

Smaller padding (12px×16px), smaller icon (16px), smaller text (13px). Use for sidebars or email templates.

6. With Close Button

Add dismissible close button (X icon). Use for non-critical banners that can be hidden.

7. With Link

Include inline link for "Resend email" or "Contact support". Use underlined text for clarity.

Related Components

OC04: NextStepsCTA

  • Relationship: Appears directly above OC05
  • Usage: Action buttons, then OC05 as final reassurance
  • Spacing: 24px gap between OC04 and OC05

C17: ToastSystem

  • Relationship: Different use case (transient vs. persistent)
  • Difference: C17 auto-dismisses, OC05 stays visible
  • Usage: C17 for immediate feedback, OC05 for ongoing status

InfoBox (EC09-like)

  • Relationship: Similar banner pattern but different context
  • Difference: InfoBox for general info, OC05 for email confirmation specifically

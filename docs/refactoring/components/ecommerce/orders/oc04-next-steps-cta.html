STEP 1: Component Overview

Purpose

The NextStepsCTA component displays a grid of actionable buttons on the order confirmation page, guiding customers to logical next steps after completing a purchase. The primary button (track order) uses a teal gradient with shadow, while secondary buttons use white backgrounds with grey borders. Each button features a centered icon above text.

Component Identity

  • Name: NextStepsCTA
  • ID: OC04
  • Category: UI Elements / CTA Buttons
  • Type: Interactive button group
  • Priority: High (Phase 2)

Usage Context

Appears on the order confirmation page below the order items summary (OC03). Provides quick access to common post-purchase actions. Also usable in email confirmation templates (compact variant) and account order history pages.

Key Features

  • 3-column grid layout (default), adapts to 1-column on mobile
  • Primary button with teal gradient + shadow (track order)
  • Secondary buttons with white bg + grey border (download, shop)
  • Centered icon (28×28px) above text (14px, weight 700)
  • Hover animation: lift (-2px) + enhanced shadow
  • Optional badge overlay (e.g., "Nieuw" for new features)
  • Compact variant for email templates
  • Icon-only variant (horizontal layout)
  • Supports 2-4 buttons (3 is optimal)

STEP 2: When to Use This Component

Primary Use Cases

  1. Order Confirmation Page: Guide customers to next actions after checkout
  2. Order Detail (Account Area): Show order-specific actions (track, reorder, cancel)
  3. Email Confirmation: Compact variant with 2-3 key actions
  4. Success Pages: Any post-action success page (subscription, contact form, etc.)

When NOT to Use

  • Product pages: Use standard add-to-cart button pattern
  • Checkout flow: Use checkout-specific buttons (place order, continue)
  • Forms: Use form submit buttons, not CTA blocks
  • Navigation: Use C14 MegaNav or regular nav links

Content Guidelines

Button Text: Use action verbs (Track, Download, Bekijk). Keep to 2-3 words max. Avoid generic "Click here" text.

Icon Selection: Choose icons that match action semantics (map-pin for track, download for PDF, shopping-bag for continue shopping). Use Lucide icon set for consistency.

Primary Button: Reserve for the most important action (usually track order or view details). Only one primary per group.

Button Count: 3 buttons is optimal for desktop (equal-width grid). 2-4 is acceptable. 5+ buttons should use different layout pattern.

Action Priority Guidelines

  1. Primary (Teal gradient): Track order, View details, Download invoice (single most important)
  2. Secondary (White): Download invoice, Reorder, Continue shopping, Customer service
  3. Tertiary (Navy): Alternative primary if teal doesn't fit brand (B2B contexts)
  4. Destructive (Coral): Cancel order, Report problem (use sparingly)

STEP 7: Implementation Notes

React/Next.js Component

import Link from 'next/link'
import * as Icons from 'lucide-react'

export default function NextStepsCTA({ actions, variant = 'default', columns }: NextStepsCTAProps) {
  const gridColumns = columns || actions.length
  const containerClass = `next-steps-cta ${variant}`

  return (
    <section
      aria-label="Next steps"
      className={containerClass}
      style={{ gridTemplateColumns: `repeat(${gridColumns}, 1fr)` }}
    >
      {actions.map((action) => {
        const Icon = Icons[action.icon as keyof typeof Icons]
        const Component = action.href ? Link : 'button'

        return (
          <Component
            key={action.id}
            href={action.href || '#'}
            className={`next-step-button ${action.variant}`}
            onClick={action.onClick}
            aria-label={action.label}
            {...(action.href?.startsWith('http') && {
              target: '_blank',
              rel: 'noopener noreferrer',
            })}
            {...(action.href?.endsWith('.pdf') && { download: true })}
          >
            {action.badge && (
              <span className="next-step-button-badge">
                {action.badge.text}
              </span>
            )}
            {Icon && <Icon size={28} aria-hidden="true" />}
            <span className="next-step-button-text">{action.label}</span>
          </Component>
        )
      })}
    </section>
  )
}

Dynamic Action Generation (B2B)

function generateOrderActions(order: PayloadOrder, user: User) {
  const actions = []

  // Track order (always available)
  actions.push({
    id: 'track',
    label: 'Track bestelling',
    icon: 'map-pin',
    variant: 'primary',
    href: `/account/orders/${order.id}/track`,
  })

  // Download invoice (available after payment)
  if (order.paymentStatus === 'paid') {
    actions.push({
      id: 'invoice',
      label: 'Download factuur',
      icon: 'download',
      variant: 'secondary',
      href: `/api/orders/${order.id}/invoice`,
    })
  }

  // Reorder (B2B only)
  if (user.role === 'b2b') {
    actions.push({
      id: 'reorder',
      label: 'Bestel opnieuw',
      icon: 'copy',
      variant: 'secondary',
      onClick: () => reorderItems(order.items),
    })
  }

  // Continue shopping
  actions.push({
    id: 'shop',
    label: 'Verder winkelen',
    icon: 'shopping-bag',
    variant: 'secondary',
    href: '/shop',
  })

  return actions
}

Analytics Tracking

function handleActionClick(action: NextStepAction, order: PayloadOrder) {
  // Track with Google Analytics
  if (window.gtag) {
    window.gtag('event', 'next_step_action', {
      action_id: action.id,
      action_label: action.label,
      order_id: order.id,
      order_value: order.total / 100,
    })
  }

  // Track with custom analytics
  analytics.track('Order Confirmation Action', {
    actionId: action.id,
    actionLabel: action.label,
    orderId: order.id,
    timestamp: new Date().toISOString(),
  })

  // Execute action
  if (action.onClick) {
    action.onClick()
  } else if (action.href) {
    window.location.href = action.href
  }
}

STEP 8: Variations & Related Components

Component Variations

1. Default Variant (3 Buttons)

Standard 3-column grid. Primary teal button + 2 secondary white buttons. Best for order confirmation pages.

2. Compact Variant

Smaller padding (20px), smaller icons (24px), smaller text (13px). Use for email templates or space-constrained layouts.

<NextStepsCTA variant="compact" actions={actions} />

3. Icon-Only Variant (Horizontal)

Horizontal layout with icon + text side-by-side. More compact vertically. Good for narrow sidebars.

<NextStepsCTA variant="icon-only" actions={actions} />

4. 4-Button Layout (B2B)

4 equal-width columns. Add reorder or custom action. Automatically stacks on mobile.

<NextStepsCTA actions={b2bActions} columns={4} />

5. With Badge

Add "Nieuw" badge to highlight new features (e.g., review/rating feature).

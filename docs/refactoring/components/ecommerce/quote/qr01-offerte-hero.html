📋 Component Documentation

STEP 1: Component Overview

Component ID: QR01

Component Name: OfferteHero

Category: UI Elements / Hero

Priority: 🟢 Low

Source File: ef7-offerte.html

Output Location: /refactoring/components/ui-elements/qr01-offerte-hero.html

Purpose:
OfferteHero is a B2B-focused hero section for quote request pages. It features a dark navy gradient background with a decorative teal glow, a prominent badge (e.g., "B2B SERVICE"), and clear messaging about the quote process. It sets the professional tone for B2B transactions and differentiates quote requests from standard checkout flows.

Key Features:

  • Dark gradient background (navy → navy-light, 135deg)
  • Decorative teal radial glow (top-right, semi-transparent)
  • B2B badge pill (teal bg, uppercase, 11px)
  • Large title (32px, 800 weight, white)
  • Description text (15px, 50% white opacity, max-width 560px)
  • Optional icon in badge
  • Responsive: smaller padding and font sizes on mobile
  • Relative positioning for content stacking

When to Use:

  • ✅ Quote request (offerte) pages
  • ✅ B2B-specific flows (wholesale, bulk orders)
  • ✅ Project inquiry forms
  • ✅ Custom pricing request pages
  • ❌ Don't use for B2C checkout (use regular hero instead)
  • ❌ Don't use for product detail pages (too heavy)
  • ❌ Don't use for general content pages

STEP 2: When to Use

Use Cases:

  • B2B Quote Requests: Large orders (500+ units) requiring custom pricing
  • Project Inquiries: Construction/medical projects with specific requirements
  • Wholesale Orders: Bulk purchases for resellers/distributors
  • Custom Products: Made-to-order or customized medical supplies
  • Tender Submissions: Government/hospital procurement processes

Anti-patterns:

  • ❌ Using for regular B2C product pages (confusing messaging)
  • ❌ Multiple heroes on same page (visual overload)
  • ❌ Too much text in description (≥3 sentences max)
  • ❌ Missing badge (loses B2B differentiation)
  • ❌ Using bright colors (conflicts with dark theme)

Guidelines:

  • Keep title concise (2-4 words)
  • Description should explain value proposition (1-2 sentences)
  • Badge text uppercase (e.g., "B2B SERVICE", "WHOLESALE")
  • Mention response time if applicable ("binnen 24 uur")
  • Use professional tone (no exclamation marks unless strategic)
  • Ensure sufficient contrast for text readability

STEP 5: Layout Rules

Component Anatomy:

┌────────────────────────────────────────────────┐
│  ╭─ Teal glow (decorative, top-right) ─╮      │
│  │                                       │      │
│  │  [B2B SERVICE]                        │      │
│  │  Offerte aanvragen                    │      │
│  │  Vul het formulier in en ontvang...   │      │
│  ╰───────────────────────────────────────╯      │
└────────────────────────────────────────────────┘

Container:
- Background: linear-gradient(135deg, #0A1628, #121F33)
- Padding: 32px 40px
- Border-radius: 14px
- Position: relative (for absolute glow positioning)

Glow (::before pseudo-element):
- Position: absolute, top: -30%, right: -15%
- Size: 400px × 400px circle
- Background: radial-gradient teal glow
- Z-index: 0 (below content)

Content:
- Position: relative, z-index: 1 (above glow)
- Badge → Title → Description (vertical stack)

Positioning Strategy:

  • Container: position: relative; overflow: hidden; (clips glow)
  • Glow: position: absolute; positioned top-right with negative margins
  • Content: position: relative; z-index: 1; ensures text appears above glow

Responsive Behavior:

  • Desktop (≥768px): Full padding (32px/40px), 32px title, 400px glow
  • Tablet (480-768px): Reduced padding (24px/24px), 26px title, 280px glow
  • Mobile (<480px): Minimal padding (24px/16px), 22px title, glow repositioned
  • Description max-width ensures readable line length on all screens

STEP 6: Accessibility

Semantic HTML:

  • Use <header> or <section> as container
  • Use <h1> for title (page title)
  • Use <p> for description
  • Use <div> for badge (presentational)

ARIA Attributes:

  • aria-label="B2B quote request" (optional, on container)
  • role="banner" (if using <section> instead of <header>)

Color Contrast:

  • Title (white) on navy gradient: 16.1:1 (WCAG AAA ✓)
  • Description (50% white) on navy: 8.1:1 (WCAG AAA ✓)
  • Badge text (#26A69A) on teal bg: 4.5:1 (WCAG AA ✓)

Screen Reader Considerations:

  • Badge announced before title (natural reading order)
  • Glow pseudo-element not announced (decorative only)
  • Icon in badge decorative (aria-hidden="true" via Lucide)

STEP 7: Implementation Notes

React/Next.js Component:

'use client'
import { Briefcase, Users, Building, Package } from 'lucide-react'

const iconMap = {
  briefcase: Briefcase,
  users: Users,
  building: Building,
  package: Package
}

export function OfferteHero({
  badge,
  badgeIcon,
  title,
  description,
  responseTime,
  maxWidth = 560,
  ariaLabel
}: OfferteHeroProps) {
  const Icon = badgeIcon ? iconMap[badgeIcon] : null

  const fullDescription = responseTime
    ? `${description} Ontvang binnen ${responseTime} een offerte.`
    : description

  return (
    <header className="offerte-hero" aria-label={ariaLabel}>
      <div className="offerte-hero-content">
        <div className="offerte-badge">
          {Icon && <Icon className="offerte-badge-icon" />}
          {badge}
        </div>
        <h1 className="offerte-title">{title}</h1>
        <p className="offerte-description" style={{ maxWidth }}>
          {fullDescription}
        </p>
      </div>
    </header>
  )
}

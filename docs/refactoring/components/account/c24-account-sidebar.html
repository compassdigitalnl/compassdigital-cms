STEP 1: Component Overview

Purpose: The Account Sidebar provides user identification and navigation for account management pages. It consists of a user profile card showing avatar, name, company, and membership date, followed by a navigation menu with icons, labels, and optional notification badges.

Property Value
Component ID C24
Category Account Components
Width 280px (fixed, desktop) / 100% (mobile)
Source File plastimed-my-account.html (lines 230-250)
Dependencies Lucide Icons

Key Features:

  • User profile card with gradient avatar (initials)
  • Company name display (B2B) or hidden (B2C)
  • Membership date with calendar icon
  • Icon-based navigation menu
  • Active state with teal background + left border
  • Badge counts for pending items (coral)
  • Special logout link styling (coral)
  • Hover states on all nav items
  • Fully keyboard accessible
STEP 2: When to Use

Use the Account Sidebar when:

  •  Building account dashboard layouts
  •  Creating order history pages
  •  Displaying user settings/profile pages
  •  Showing saved addresses, wishlists, or favorites
  •  Building subscription or membership management
  •  Any multi-section account area requiring navigation

Don't use when:

  • L User is not logged in (show login prompt instead)
  • L On mobile viewports (use hamburger menu or tabs)
  • L For public-facing navigation (use C14 MegaNav)
  • L For single-page account forms (no navigation needed)

Common Use Cases:

  • B2B Accounts: Show company name, display all sections (orders, invoices, repeat orders, bestellijsten)
  • B2C Accounts: Hide company field, simplify menu (orders, favorites, addresses, settings)
  • Dashboard Pages: "Dashboard" link active, shows overview of account activity
  • Order Management: "Bestellingen" active, badge shows pending/processing orders
  • Settings Pages: "Accountinstellingen" active, user edits profile/password/preferences

STEP 5: Layout Rules

Layout Pattern: The sidebar is a fixed-width (280px) column positioned on the left side of account pages. It uses a vertical stack with two main sections: user card (top) and navigation (below).

Structural Rules:

  • Container: .account-sidebar  max-width: 280px, width: 100%
  • User Card: White background, 16px border-radius, 24px padding, 16px margin-bottom
  • Avatar: 64×64px, centered above user info, teal gradient, 16px border-radius
  • User Info: Text-align center, name (16px bold) ’ company (13px grey) ’ member date (11px grey)
  • Navigation: White background, 16px border-radius, overflow hidden (for border-radius on first/last items)
  • Nav Links: Flexbox row, 14px padding vertical + 20px horizontal, 1px bottom border (except last)
  • Active State: 3px left border (teal) + teal background glow + teal text + bold weight

Responsive Behavior:

  • Desktop (e768px): Sidebar visible, 280px fixed width, positioned left in grid layout
  • Tablet/Mobile (<768px): Options:
    • Option A: Full-width sidebar at top of page (stack above main content)
    • Option B: Collapse to hamburger menu / drawer (recommended)
    • Option C: Convert to horizontal tabs (simple alternative)

STEP 7: Implementation Notes

React/Next.js Component:

import React from 'react';
import Link from 'next/link';
import { useRouter } from 'next/router';

interface AccountSidebarProps {
  user: {
    initials: string;
    name: string;
    company?: string;
    memberSince: string;
  };
  navItems: NavItem[];
  onLogout?: () => void;
}

export const AccountSidebar: React.FC<AccountSidebarProps> = ({
  user,
  navItems,
  onLogout
}) => {
  const router = useRouter();
  const isActive = (href: string) => router.pathname === href;

  return (
    <aside className="account-sidebar" role="complementary" aria-label="Account navigation">
      {/* User Card */}
      <div className="account-user" role="region" aria-label="User profile">
        <div className="user-avatar" aria-hidden="true">
          {user.initials}
        </div>
        <div className="user-name">{user.name}</div>
        {user.company && (
          <div className="user-company">{user.company}</div>
        )}
        <div className="user-since">
          <i data-lucide="calendar" aria-hidden="true" />
          Klant sinds {user.memberSince}
        </div>
      </div>

      {/* Navigation */}
      <nav className="account-nav" aria-label="Account menu">
        {navItems.map((item) => {
          if (item.isLogout) {
            return (
              <button
                key={item.id}
                onClick={onLogout}
                className="nav-link logout"
                type="button"
              >
                <i data-lucide={item.icon} aria-hidden="true" />
                {item.label}
              </button>
            );
          }

          const active = isActive(item.href || '');

          return (
            <Link
              key={item.id}
              href={item.href || '#'}
              className={`nav-link ${active ? 'active' : ''}`}
              aria-current={active ? 'page' : undefined}
            >
              <i data-lucide={item.icon} aria-hidden="true" />
              {item.label}
              {item.badge && item.badge > 0 && (
                <span
                  className="badge"
                  aria-label={`${item.badge} nieuwe ${item.label.toLowerCase()}`}
                >
                  {item.badge}
                </span>
              )}
            </Link>
          );
        })}
      </nav>
    </aside>
  );
};

Avatar Generation:

// Generate initials from name
function getInitials(name: string): string {
  return name
    .split(' ')
    .map(part => part[0])
    .join('')
    .toUpperCase()
    .slice(0, 2);
}

// Examples:
getInitials("Jan de Vries");      // "JV"
getInitials("Maria Koster");      // "MK"
getInitials("Peter");             // "P"

Date Formatting:

// Format membership date (Dutch locale)
function formatMemberSince(date: Date): string {
  const formatter = new Intl.DateTimeFormat('nl-NL', {
    month: 'long',
    year: 'numeric'
  });
  return formatter.format(date);
}

// Example: new Date('2019-03-15') ’ "maart 2019"

Logout Handler:

const handleLogout = async () => {
  try {
    // Clear auth token
    await fetch('/api/auth/logout', { method: 'POST' });

    // Clear local storage
    localStorage.removeItem('user');
    localStorage.removeItem('token');

    // Redirect to login
    router.push('/login');
  } catch (error) {
    console.error('Logout failed:', error);
  }
};

Badge Count Logic:

// Calculate badge counts from user data
const getBadgeCounts = (user: User) => ({
  orders: user.orders.filter(o =>
    o.status === 'processing' || o.status === 'pending'
  ).length,

  lists: user.bestellijsten?.length || 0,

  favorites: user.favorites?.length || 0,

  invoices: user.invoices.filter(i =>
    i.status === 'unpaid'
  ).length
});

Performance:

  • Sidebar is static (no complex state), renders once per page load
  • Icons loaded via Lucide CDN (cached)
  • No animations on initial render (fast paint)
  • Badge counts fetched server-side (no client loading states)
STEP 8: Variations & Related Components

Common Variations:

B2B Account Business/company accounts Show company name, include "Bedrijfsgegevens" nav item
B2C Account Personal consumer accounts Hide company field, simplify nav menu (remove B2B-specific items)
With Badges Show pending actions/notifications Add badge counts to relevant nav items (orders, invoices, etc.)
Minimal Menu Simplified account area (fewer sections) Reduce nav items to 4-6 core sections (dashboard, orders, favorites, settings, logout)
Mobile Drawer Mobile/tablet viewports Convert to slide-out drawer or collapsible accordion

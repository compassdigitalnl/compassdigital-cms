STEP 1: Component Overview

Component ID: SR02

Component Name: DidYouMeanBanner

Category: UI Elements / Feedback

Priority: 🟡 Medium

Source File: ef6-search-results.html

Output Location: /refactoring/components/ui-elements/sr02-did-you-mean-banner.html

Purpose:
DidYouMeanBanner displays search query correction suggestions when the user's search might contain typos or could be improved. It appears below the SearchQueryHeader and helps users quickly refine their search with suggested alternative queries.

Key Features:

  • Amber warning styling (non-intrusive, informational)
  • Info icon (16px) + suggestion text
  • Clickable suggestion links (amber, bold, underline)
  • Support for single or multiple suggestions
  • Optional close button (dismissible)
  • Slide-down animation on appearance
  • Alternative teal variant for related suggestions
  • Mobile-responsive text sizing

When to Use:

  • ✅ Search results with possible typos detected
  • ✅ Low result count with better alternative queries available
  • ✅ Common misspellings of product names/brands
  • ✅ Similar search queries with higher result counts
  • ❌ Don't show for exact matches with good result counts
  • ❌ Don't show multiple banners (max 1 per search)
  • ❌ Don't use for error states (use error banner instead)

STEP 2: When to Use

Use Cases:

  • Typo Detection: "handschoen" → suggest "handschoenen"
  • Spelling Correction: "nitrile" → suggest "nitril"
  • Synonym Suggestions: "wegwerp handschoenen" → suggest "disposable handschoenen"
  • Related Searches: Low results for "blauw nitril" → suggest "nitril blauw" or "nitril handschoenen blauw"
  • Popular Alternatives: Search for rare term → suggest more common equivalent

Anti-patterns:

  • ❌ Showing banner when results are good (≥10 results with exact match)
  • ❌ Suggesting unrelated queries
  • ❌ Too many suggestions (max 3 links)
  • ❌ Using for promotional content (use ProTipBanner instead)
  • ❌ Persistent banner after user clicks suggestion (should disappear)

Guidelines:

  • Show banner only if confidence score ≥70%
  • Limit to 1-3 suggestions max
  • Order suggestions by relevance/popularity
  • Use dismissible variant if banner is shown on every page load
  • Track clicks to improve suggestion algorithm
  • Consider user's search history for personalized suggestions

STEP 7: Implementation Notes

React/Next.js Component:

'use client'
import { Info, Lightbulb, X } from 'lucide-react'
import { useState } from 'react'

interface DidYouMeanBannerProps {
  message?: string
  suggestions: { text: string; query: string; url?: string }[]
  dismissible?: boolean
  variant?: 'info' | 'teal'
  onSuggestionClick?: (query: string) => void
}

export function DidYouMeanBanner({
  message = 'Bedoelde je misschien:',
  suggestions,
  dismissible = false,
  variant = 'info',
  onSuggestionClick
}: DidYouMeanBannerProps) {
  const [visible, setVisible] = useState(true)

  if (!visible || suggestions.length === 0) return null

  const Icon = variant === 'teal' ? Lightbulb : Info

  return (
    <aside
      role="status"
      aria-label="Zoek suggestie"
      className={`did-you-mean-banner ${variant}`}
    >
      <Icon className="did-you-mean-icon" />
      <span className="did-you-mean-text">
        {message}{' '}
        {suggestions.map((suggestion, index) => (
          <span key={suggestion.query}>
            <a
              href={suggestion.url || `/search?q=${encodeURIComponent(suggestion.query)}`}
              className="did-you-mean-link"
              onClick={() => onSuggestionClick?.(suggestion.query)}
            >
              {suggestion.text}
            </a>
            {index < suggestions.length - 2 && ', '}
            {index === suggestions.length - 2 && ' of '}
          </span>
        ))}?
      </span>
      {dismissible && (
        <button
          className="did-you-mean-close"
          onClick={() => setVisible(false)}
          aria-label="Sluit suggestie"
        >
          <X />
        </button>
      )}
    </aside>
  )
}

Search Algorithm Integration:

import Fuse from 'fuse.js'

// Generate suggestions based on catalog
function generateSuggestions(query: string, products: Product[]): DidYouMeanSuggestion[] {
  const fuse = new Fuse(products, {
    keys: ['name', 'brand', 'category'],
    threshold: 0.3, // Higher = more fuzzy
    includeScore: true
  })

  const results = fuse.search(query).slice(0, 3)

  return results.map(result => ({
    text: result.item.name,
    query: result.item.name,
    confidence: (1 - (result.score || 0)) * 100
  })).filter(s => s.confidence >= 70)
}

LocalStorage Persistence (Dismissible):

function useDismissedSuggestions() {
  const [dismissed, setDismissed] = useState<Set<string>>(() => {
    const stored = localStorage.getItem('dismissedSuggestions')
    return stored ? new Set(JSON.parse(stored)) : new Set()
  })

  const dismiss = (query: string) => {
    const updated = new Set(dismissed).add(query)
    setDismissed(updated)
    localStorage.setItem('dismissedSuggestions', JSON.stringify([...updated]))
  }

  return { dismissed, dismiss }
}

Analytics Tracking:

function trackSuggestionClick(originalQuery: string, suggestedQuery: string) {
  // Google Analytics 4
  gtag('event', 'search_suggestion_click', {
    original_query: originalQuery,
    suggested_query: suggestedQuery,
    event_category: 'Search',
    event_label: 'Did You Mean'
  })

  // Internal analytics
  fetch('/api/analytics/search-suggestion', {
    method: 'POST',
    body: JSON.stringify({ originalQuery, suggestedQuery })
  })
}

Performance Optimization:

  • Generate suggestions server-side to avoid client-side processing
  • Cache common misspellings and their corrections
  • Use debouncing if suggestions update during typing
  • Lazy load suggestion algorithm library (Fuse.js) only when needed

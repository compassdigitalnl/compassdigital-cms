Description: List of trust-building badges/signals (SSL, return policy, fast shipping, certifications) displayed near checkout CTAs to reduce cart abandonment and increase conversions.

Key Features:

  • Icon + text layout (16px teal icons)
  • Vertical list with 8px gaps
  • Top border separator
  • Common signals: SSL, return policy, shipping, support
  • Configurable via CMS
  • Multiple variants (compact, horizontal, card)

🎯 STEP 2: When to Use

Use this component when:

  • Below checkout CTA in EC07: OrderSummary
  • In mini cart flyout footer
  • On checkout page sidebar
  • On product pages near "Add to cart" button
  • In footer of quote request forms

Common Design Patterns:

  • Cart Sidebar: Below action buttons, above payment icons
  • Checkout: In order summary sidebar
  • Product Page: Horizontal variant below ATC button
  • Footer: Site-wide trust indicators

Do NOT use when:

  • In product listings (too much noise)
  • Above the fold on homepage (save space for content)
  • More than 4-5 signals (overwhelms user)

💻 STEP 7: Implementation Notes

React/Next.js Component

import * as Icons from 'lucide-react'

const defaultSignals = [
  { icon: 'shield-check', text: 'Veilig betalen met SSL' },
  { icon: 'rotate-ccw', text: '30 dagen retourrecht' },
  { icon: 'truck', text: 'Morgen in huis' }
]

export function TrustSignals({
  signals = defaultSignals,
  variant = 'default',
  className = ''
}: TrustSignalsProps) {

  return (
    <ul
      className={`trust-signals trust-signals--${variant} ${className}`}
      aria-label="Trust signals"
    >
      {signals.map((signal, index) => {
        const IconComponent = Icons[signal.icon as keyof typeof Icons] as React.FC<{
          size?: number
          'aria-hidden'?: boolean
        }>

        return (
          <li key={index} className="trust-signals__item">
            {IconComponent && (
              <IconComponent
                size={variant === 'compact' ? 14 : 16}
                aria-hidden="true"
              />
            )}
            {signal.text}
          </li>
        )
      })}
    </ul>
  )
}

Fetch from CMS

// Server component
export async function TrustSignalsFromCMS() {
  const settings = await payload.findGlobal({
    slug: 'site-settings'
  })

  const activeSignals = settings.trustSignals
    ?.filter(s => s.isActive)
    .map(s => ({ icon: s.icon, text: s.text }))

  return <TrustSignals signals={activeSignals} />
}

Important Implementation Details

  • Icon Mapping: Dynamically import from lucide-react using icon name strings
  • Semantic HTML: Use <ul> and <li> for list semantics
  • CMS Management: Store signals in global settings for easy updates
  • Limit Signals: Max 3-4 signals for best conversion (too many = diluted trust)

=╦ STEP 1: Component Overview

C25: ProgressSteps is a horizontal progress indicator component for multi-step flows like checkout processes, onboarding wizards, and multi-page forms. It displays numbered circles connected by horizontal lines, with three distinct states: done (green checkmark), active (teal), and pending (grey).

Key Features

  • 3 Visual States: Done (green bg + checkmark), Active (teal bg + number), Pending (grey border + number)
  • Connector Lines: 60px ū 2px lines between steps, turn green when completed
  • Numbered Circles: 32px diameter, font-size 13px, bold numbers
  • Step Labels: 13px below circles, color changes per state (green/navy/grey-mid)
  • Responsive: Hides labels on mobile (d640px), keeps only circles
  • Smooth Transitions: 0.3s cubic-bezier on all state changes
  • Flexible: Supports 3-5+ steps (max-width 600px container)

<» STEP 2: When to Use

Primary Use Cases

  • Checkout Flows: Cart Æ Checkout Æ Confirmation (3 steps)
  • Onboarding Wizards: Account Æ Company Æ Verification Æ Complete (4+ steps)
  • Multi-Page Forms: Personal Info Æ Address Æ Preferences Æ Review
  • Product Configurators: Base Æ Options Æ Personalization Æ Review Æ Checkout
  • Registration Flows: Email Æ Password Æ Profile Æ Verification

When NOT to Use

  • Single-Page Forms: Use section headings instead
  • Non-Linear Flows: Steps must be sequential (no jumping ahead)
  • Vertical Layouts: This is horizontal only (consider tabs for vertical)
  • Too Many Steps: 6+ steps may overflow on mobile, consider grouping

Best Practices

  • Show Current Position: Always have exactly 1 active step (teal)
  • Linear Progression: Users can't skip ahead to pending steps
  • Allow Back Navigation: Users can return to completed (done) steps
  • Clear Labels: Keep step names short (1-3 words), use nouns
  • Persistent Display: Keep visible throughout entire flow (sticky optional)

=ą STEP 5: Layout Rules

Container Behavior

  • White Background: .progress-bar has white bg (#FAFBFC)
  • Bottom Border: 1px solid var(--grey) separates from page content
  • Vertical Padding: 20px top/bottom for breathing room
  • Full Width: Container spans 100% width of viewport

Steps Layout

  • Centered: .steps is centered with margin: 0 auto
  • Max Width: 600px to prevent over-stretching on wide screens
  • Horizontal Flex: Steps laid out left-to-right with display: flex
  • Gap Zero: No gap between steps (lines handle spacing)

Step Sizing

  • Circle: Fixed 32px ū 32px with 2px border (total 36px ū 36px hit area)
  • Flex-shrink: Circles have flex-shrink: 0 to prevent squishing
  • Label: white-space: nowrap prevents wrapping
  • Gap: 10px between circle and label (vertical stack on mobile)

Connector Lines

  • Desktop: 60px ū 2px with 12px horizontal margin on each side
  • Mobile: 40px ū 2px with 8px horizontal margin
  • Flex-shrink: Lines have flex-shrink: 0 for consistent width
  • Alignment: Vertically centered with circles

Responsive Behavior

  • d640px: Hide labels (display: none), reduce line width/margins
  • Mobile Stack: Labels hidden, only circles + lines visible
  • Total Width Calc: 3 steps = (32px ū 3) + (40px ū 2) + (8px ū 4) = 208px min-width mobile
  • Overflow Handling: For 5+ steps, container may need horizontal scroll on mobile

Placement Guidelines

  • Position: Typically placed below header/nav, above main content
  • Sticky Option: Can be made sticky with position: sticky; top: 0; z-index: 200;
  • Isolation: Keep separate from page content with bottom border visual
  • Persistent: Should remain visible throughout entire multi-step flow

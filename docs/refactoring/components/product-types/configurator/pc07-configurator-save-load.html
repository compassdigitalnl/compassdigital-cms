📋 Component Documentation

STEP 1: Component Overview

PC07: ConfiguratorSaveLoad enables users to save, load, share, and export product configurations. Supports auto-save, manual save with naming, saved configurations list, share links, and multiple export formats.

Purpose: Prevent configuration loss, enable collaboration via sharing, allow users to compare multiple options, and facilitate purchase workflows by saving configurations for later checkout.

STEP 2: When to Use This Component

  • Multi-step configurators: Laptop builders, vehicle configurators, custom product designers
  • Long configuration sessions: Prevent data loss if user navigates away
  • Comparison shopping: Allow users to save multiple configurations to compare
  • B2B workflows: Share configurations with managers/procurement teams for approval
  • Consultation scenarios: Save configs for sales team follow-up
  • Quote requests: Attach saved configuration to quote/offerte form
  • Print/PDF export: Generate product specs for offline review

STEP 5: Layout Rules

  • Auto-save indicator: Top-right of header (next to action buttons)
  • Save modal: Centered overlay with backdrop blur, max-width 520px
  • Saved configs list: Full-width panel, border-bottom separators between cards
  • Config card layout: Header (name + meta + actions) → Specs tags → Footer (price + load button)
  • Share panel: Link input + copy button → Social share buttons below divider
  • Export options: 2-column grid (1-col on mobile), equal height cards
  • Empty state: Centered icon + text, 60px vertical padding

Responsive Behavior

  • Mobile (<640px): Hide spec tags, wrap meta items, stack share buttons
  • Tablet (640-900px): Full layout, reduce padding
  • Desktop (>900px): Optimal spacing, 2-col export grid

STEP 6: Accessibility (WCAG AA/AAA)

  • Modal focus trap: Keep focus within modal when open, restore focus on close
  • ESC to close: Modal dismissible via ESC key
  • Save status announcements: Use aria-live="polite" for save status changes
  • Button labels: All icon buttons have title or aria-label
  • Link copying: Announce "Link gekopieerd" to screen readers
  • Keyboard navigation: All interactive elements keyboard accessible (TAB, ENTER, SPACE)
  • Color contrast: All text meets WCAG AA (4.5:1 for small text, 3:1 for large)

STEP 7: Implementation Notes

React Component

'use client';

import { useState, useEffect } from 'react';
import { Save, Loader2, CheckCircle, AlertCircle } from 'lucide-react';

export function ConfiguratorSaveLoad({ config }: { config: ConfigurationData }) {
  const [saveStatus, setSaveStatus] = useState<AutoSaveStatus>({
    state: 'idle',
  });
  const [savedConfigs, setSavedConfigs] = useState<SavedConfiguration[]>([]);

  // Auto-save with debounce
  useEffect(() => {
    const timer = setTimeout(() => {
      autoSaveConfig();
    }, 2000);  // 2 seconds after last change

    return () => clearTimeout(timer);
  }, [config]);

  async function autoSaveConfig() {
    setSaveStatus({ state: 'saving' });

    try {
      // Save to localStorage or API
      const saved = {
        id: crypto.randomUUID(),
        config,
        updatedAt: new Date(),
      };

      localStorage.setItem('autosave', JSON.stringify(saved));

      setSaveStatus({
        state: 'saved',
        lastSaved: new Date(),
      });

      // Reset to idle after 3 seconds
      setTimeout(() => {
        setSaveStatus({ state: 'idle' });
      }, 3000);
    } catch (error) {
      setSaveStatus({
        state: 'error',
        error: error.message,
      });
    }
  }

  async function saveConfigWithName(name: string, notes?: string) {
    const newConfig: SavedConfiguration = {
      id: crypto.randomUUID(),
      name,
      notes,
      config,
      createdAt: new Date(),
      updatedAt: new Date(),
      price: calculateTotal(config),
      productName: config.model,
      productModel: config.model,
      specs: buildSpecsList(config),
    };

    // Save to API
    await fetch('/api/configurations', {
      method: 'POST',
      body: JSON.stringify(newConfig),
    });

    // Update local state
    setSavedConfigs([newConfig, ...savedConfigs]);
  }

  function generateShareLink(configId: string): string {
    return `${window.location.origin}/config/${configId}`;
  }

  async function copyShareLink(link: string) {
    await navigator.clipboard.writeText(link);
    // Show toast notification
  }

  return (
    <div>
      {/* Auto-save status */}
      {saveStatus.state !== 'idle' && (
        <div
          className={`autosave-status ${saveStatus.state}`}
          role="status"
          aria-live="polite"
        >
          {saveStatus.state === 'saving' && (
            <>
              <Loader2 />
              <span>Opslaan...</span>
            </>
          )}
          {saveStatus.state === 'saved' && (
            <>
              <CheckCircle />
              <span>Opgeslagen</span>
            </>
          )}
          {saveStatus.state === 'error' && (
            <>
              <AlertCircle />
              <span>Niet opgeslagen</span>
            </>
          )}
        </div>
      )}

      {/* Saved configurations list */}
      {/* ... */}
    </div>
  );
}

STEP 8: Variations & Related Components

Auto-save Variants

  • Idle: No indicator shown
  • Saving: Amber pill with loader-2 spinner + "Opslaan..." text
  • Saved: Green pill with check-circle icon + "Opgeslagen" text (auto-hides after 3s)
  • Error: Coral pill with alert-circle icon + "Niet opgeslagen" text (persists until retry)

Export Formats

  • PDF: Branded document with product specs, pricing breakdown, company logo
  • Print: Browser print dialog with optimized layout
  • E-mail: Send configuration summary to user's email
  • JSON: Raw data export for API integration

Related Components

  • PC01: ConfiguratorHeader — Contains save button that triggers PC07 modal
  • PC05: ConfiguratorPriceBreakdown — Price/spec data used in saved configs
  • C17: ToastSystem — Show success/error notifications for save/load actions
  • QR01-QR05: Quote Request — Attach saved configuration to quote form

Best Practices

  • Auto-save frequency: 2 seconds after last change (debounce)
  • Conflict resolution: If auto-save fails, prompt user to save manually with name
  • Storage limits: Limit to 10 saved configs per user (free tier), unlimited for Pro
  • Share link expiry: Share links expire after 30 days
  • Version history: Store config snapshots for "undo" functionality
  • Duplicate prevention: Check for identical configs before saving
  • Cross-device sync: Save to database (not localStorage) for multi-device access
  • Guest users: Store in localStorage + prompt to create account to save permanently

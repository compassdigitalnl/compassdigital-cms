PC01: Configurator Step Indicator

Progress bar showing completed, active, and pending configuration steps

Step 1: Component Overview

What is PC01?

PC01 (Configurator Step Indicator) is a horizontal progress bar that visualizes the user's position in a multi-step product configuration flow. It shows which steps are completed (checkmarks), which step is currently active (highlighted), and which steps are still pending (greyed out).

UX Pattern: This component follows the "stepped progression" pattern seen in e-commerce configurators (Apple, Dell, Coolblue) where users build complex products option-by-option across multiple screens.

Step 2: When to Use

Use PC01 when:

  • Building product configurators (laptops, bikes, furniture, etc.)
  • Multi-step forms with 3-7 distinct stages (optimal: 5 steps)
  • Users need to see their progress through a sequential workflow
  • Linear navigation (can't skip ahead without completing previous steps)
  • Each step takes 30+ seconds to complete (worthwhile to show progress)

Don't use when:

  • Single-page configuration (all options visible at once)
  • Non-linear workflows (users can skip around freely)
  • Fewer than 3 steps (overkill for simple flows)
  • More than 7 steps (consider grouping steps into phases)

Step 6: Accessibility

WCAG 2.1 AA Compliance

  • Use role="progressbar" on container
  • aria-valuenow="3" (current step index)
  • aria-valuemin="1" and aria-valuemax="5"
  • aria-label="Step 3 of 5: Specs"
  • Each step has aria-current="step" for active step

📋 PC02: ConfiguratorStepNav

Component Type: Product Type Component — Configurator Navigation
Category: Progress Indicator / Wizard Navigation
Complexity: Medium

💡 Purpose: Progress indicator for multi-step product configuration flows. Shows users where they are in the customization process and allows navigation between completed steps.

STEP 1: Component Overview

What is ConfiguratorStepNav?

ConfiguratorStepNav is a horizontal (or vertical) progress indicator that visualizes the user's position in a multi-step product configuration process. It shows completed steps with checkmarks, the current step with a teal highlight, and upcoming steps in grey.

Key Features

  • Visual progress tracking: Numbered circles with checkmarks for completed steps
  • State indicators: 4 states (done, active, pending, error) with distinct colors
  • Connection lines: Visual connectors between steps (grey for pending, teal for completed)
  • Clickable navigation: Optional clickable steps to jump back to previous stages
  • Flexible step count: Supports 3-7 steps (3-step minimal, 5-step standard, 7-step complex)
  • Responsive design: Hides labels on mobile, shows vertical variant in sidebars
  • Error indication: Red state for validation failures
  • Accessibility: ARIA attributes, keyboard navigation, screen reader support

Visual Hierarchy

  1. Step dots (primary): 36px circles with numbers or checkmarks — most prominent
  2. Connection lines (secondary): 2px lines connecting steps — visual guide
  3. Step labels (tertiary): 12px text below dots — contextual information

STEP 2: When to Use This Component

Primary Use Cases

  • Product configurators: Multi-step customization flows (laptops, cars, furniture)
  • Personalization wizards: Custom product creation (engraving, printing, embroidery)
  • Bundle builders: Build-your-own product bundles
  • Quote requests: Multi-step B2B quote forms
  • Complex forms: Any multi-step form with validation

When NOT to Use

  • Single-step forms: Use simple form without progress indicator
  • Linear checkout: Use CheckoutProgressStepper (EC13) instead — optimized for checkout flow
  • 2-step flows: Too simple — use tabs or accordion instead
  • 8+ steps: Too complex — consider grouping steps or simplifying flow

⚠️ Important: ConfiguratorStepNav is for PRODUCT CONFIGURATION flows. For CHECKOUT flows, use EC13: CheckoutProgressStepper instead (optimized for cart → shipping → payment → confirmation).

// State object
const configuratorState = {
  currentStep: 3,
  completedSteps: [1, 2],
  stepData: {
    1: { model: 'ProBook Ultra 16"' },
    2: { color: 'Space Grey' },
    3: { processor: null, ram: null, storage: null }, // In progress
  },
  errors: {},
};

// Update step
function goToStep(stepNumber) {
  // Validate current step before proceeding
  if (!validateStep(configuratorState.currentStep)) {
    setStepError(configuratorState.currentStep, 'Vul alle vereiste velden in');
    return;
  }

  // Update state
  configuratorState.currentStep = stepNumber;
  if (stepNumber > configuratorState.currentStep) {
    configuratorState.completedSteps.push(configuratorState.currentStep);
  }

  // Re-render step nav
  renderStepNav();
}

// Render function
function renderStepNav() {
  const steps = [
    { id: 'model', label: 'Model', number: 1 },
    { id: 'color', label: 'Kleur', number: 2 },
    { id: 'specs', label: 'Specs', number: 3 },
    { id: 'extras', label: "Extra's", number: 4 },
    { id: 'overview', label: 'Overzicht', number: 5 },
  ];

  steps.forEach((step, index) => {
    const stepEl = document.querySelector(`.stp:nth-child(${index + 1})`);

    // Determine state
    let state = 'pending';
    if (configuratorState.completedSteps.includes(step.number)) {
      state = 'done';
    } else if (step.number === configuratorState.currentStep) {
      state = configuratorState.errors[step.number] ? 'error' : 'active';
    }

    // Update classes
    stepEl.className = 'stp ' + state;
    if (state === 'done') stepEl.classList.add('clickable');

    // Update dot content
    const dot = stepEl.querySelector('.stp-dot');
    if (state === 'done') {
      dot.innerHTML = '<i data-lucide="check"></i>';
    } else if (state === 'error') {
      dot.innerHTML = '<i data-lucide="alert-circle"></i>';
    } else {
      dot.textContent = step.number;
    }
  });

  // Re-initialize Lucide icons
  lucide.createIcons();
}

React/Next.js Component

// components/ConfiguratorStepNav.tsx
import React from 'react';
import { Check, AlertCircle } from 'lucide-react';

interface Step {
  id: string;
  label: string;
  number: number;
  state: 'done' | 'active' | 'pending' | 'error';
  href?: string;
  onClick?: () => void;
}

interface ConfiguratorStepNavProps {
  steps: Step[];
  currentStep: number;
  variant?: 'default' | 'compact' | 'vertical';
  clickableSteps?: boolean;
  onStepClick?: (stepId: string, stepNumber: number) => void;
}

export function ConfiguratorStepNav({
  steps,
  currentStep,
  variant = 'default',
  clickableSteps = false,
  onStepClick,
}: ConfiguratorStepNavProps) {

  const handleStepClick = (step: Step) => {
    if (!clickableSteps || step.state === 'pending') return;
    if (onStepClick) onStepClick(step.id, step.number);
  };

  return (
    <nav
      className={`step-bar ${variant}`}
      role="navigation"
      aria-label="Configuratie voortgang"
    >
      <ol className="step-row" role="list">
        {steps.map((step, index) => (
          <li
            key={step.id}
            className={`stp ${step.state} ${clickableSteps && step.state === 'done' ? 'clickable' : ''}`}
            role="listitem"
          >
            {index < steps.length - 1 && (
              <div className="stp-line" aria-hidden="true" />
            )}

            {clickableSteps && step.state === 'done' ? (
              <button
                className="stp-dot"
                onClick={() => handleStepClick(step)}
                aria-label={`Stap ${step.number}: ${step.label} - Voltooid`}
                aria-current={step.number === currentStep ? 'step' : 'false'}
              >
                {step.state === 'done' && <Check size={15} />}
                {step.state === 'error' && <AlertCircle size={15} />}
                {(step.state === 'active' || step.state === 'pending') && step.number}
              </button>
            ) : (
              <div
                className="stp-dot"
                aria-label={`Stap ${step.number}: ${step.label} - ${
                  step.state === 'done' ? 'Voltooid' :
                  step.state === 'active' ? 'Huidige stap' :
                  step.state === 'error' ? 'Fout' :
                  'Nog niet bereikt'
                }`}
                aria-current={step.state === 'active' ? 'step' : 'false'}
                aria-disabled={step.state === 'pending'}
              >
                {step.state === 'done' && <Check size={15} />}
                {step.state === 'error' && <AlertCircle size={15} />}
                {(step.state === 'active' || step.state === 'pending') && step.number}
              </div>
            )}

            <div className="stp-label">{step.label}</div>
          </li>
        ))}
      </ol>

      {/* Screen reader live region */}
      <div className="sr-only" role="status" aria-live="polite" aria-atomic="true">
        Nu op stap {currentStep} van {steps.length}: {steps[currentStep - 1]?.label}
      </div>
    </nav>
  );
}

State Management (React Hook)

// hooks/useConfiguratorSteps.ts
import { useState, useCallback } from 'react';

interface StepData {
  [key: string]: any;
}

export function useConfiguratorSteps(totalSteps: number) {
  const [currentStep, setCurrentStep] = useState(1);
  const [completedSteps, setCompletedSteps] = useState<number[]>([]);
  const [stepData, setStepData] = useState<Record<number, StepData>>({});
  const [stepErrors, setStepErrors] = useState<Record<number, string>>({});

  const goToStep = useCallback((stepNumber: number) => {
    // Validate before moving forward
    if (stepNumber > currentStep) {
      // Add validation logic here
      const isValid = true; // Replace with actual validation
      if (!isValid) {
        setStepErrors(prev => ({ ...prev, [currentStep]: 'Vul alle vereiste velden in' }));
        return false;
      }
    }

    // Mark current step as completed if moving forward
    if (stepNumber > currentStep && !completedSteps.includes(currentStep)) {
      setCompletedSteps(prev => [...prev, currentStep]);
    }

    setCurrentStep(stepNumber);
    return true;
  }, [currentStep, completedSteps]);

  const updateStepData = useCallback((stepNumber: number, data: StepData) => {
    setStepData(prev => ({ ...prev, [stepNumber]: data }));
    // Clear errors for this step
    setStepErrors(prev => {
      const newErrors = { ...prev };
      delete newErrors[stepNumber];
      return newErrors;
    });
  }, []);

  const nextStep = useCallback(() => {
    if (currentStep < totalSteps) {
      return goToStep(currentStep + 1);
    }
    return false;
  }, [currentStep, totalSteps, goToStep]);

  const prevStep = useCallback(() => {
    if (currentStep > 1) {
      setCurrentStep(currentStep - 1);
      return true;
    }
    return false;
  }, [currentStep]);

  const resetSteps = useCallback(() => {
    setCurrentStep(1);
    setCompletedSteps([]);
    setStepData({});
    setStepErrors({});
  }, []);

  return {
    currentStep,
    completedSteps,
    stepData,
    stepErrors,
    goToStep,
    nextStep,
    prevStep,
    updateStepData,
    resetSteps,
  };
}

Design Patterns

Pattern: Linear Progression

  • Users must complete steps in order (can't skip ahead)
  • Can navigate back to edit previous steps
  • Best for: Dependent configuration options (e.g., RAM depends on processor choice)

Pattern: Free Navigation

  • Users can jump to any step at any time
  • Validation happens on final "Add to cart" click
  • Best for: Independent options (color, accessories, engraving)

Pattern: Progressive Disclosure

  • Show only current step's content, hide others
  • Reduces cognitive load, focuses attention
  • Best for: Complex configurators with many options

  • ✓ Steps render correctly (3-7 step counts)
  • ✓ State changes work (done → active → pending → error)
  • ✓ Clickable steps navigate correctly
  • ✓ Connection lines color correctly based on state
  • ✓ Mobile: Labels hidden, dots remain visible
  • ✓ Vertical variant: Labels always visible

  • ✓ ARIA attributes correct (aria-current, aria-label)
  • ✓ Error state shows red color and alert icon
  • ✓ Hover effects work on clickable steps
  • ✓ Icons render correctly (Lucide initialization)

Common Pitfalls

  • ❌ Too many steps: 8+ steps overwhelm users — consider grouping or simplifying
  • ❌ Unclear labels: Use action verbs ("Kies kleur") not nouns ("Kleur")
  • ❌ Allowing forward skips: Don't let users skip required steps
  • ❌ No validation feedback: Always show error state when validation fails
  • ❌ Mobile labels hidden: On vertical variant, always show labels (essential context)
  • ❌ Missing ARIA: Step nav is navigation — must have role="navigation"

💡 Best Practice: Keep configurators to 3-5 steps max. If you need more, group related options into single steps (e.g., "Specs" = processor + RAM + storage in one step).

Real-World Examples

  • Apple Mac Studio: 3 steps (Model → Chip → Storage)
  • Nike By You: 7 steps (various customization options)
  • Dell PC Builder: 5 steps (Processor → Memory → Storage → Display → Accessories)
  • IKEA Kitchen Planner: 4 steps (Layout → Cabinets → Appliances → Summary)

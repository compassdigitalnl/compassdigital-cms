📋 Component Documentation

STEP 1: Component Overview

PC06: ConfiguratorValidation provides real-time validation feedback for product configurator forms with error messages, success states, warnings, validation summaries, and progress indicators.

Purpose: Guide users through complex multi-step configurator flows by providing clear, immediate feedback on required fields, input validation, and configuration completeness. Prevents user frustration and reduces form abandonment.

STEP 2: When to Use This Component

  • Multi-step configurators: Laptop/computer builders, vehicle configurators, custom product builders
  • Form validation: Contact forms, checkout, registration, quote requests
  • Required field indicators: Mark mandatory vs optional fields
  • Real-time feedback: Email validation, KVK lookup, promo code verification
  • Progress tracking: Show configuration completeness percentage
  • Error prevention: Warn before incompatible option selections
  • Success confirmation: Auto-filled fields from external APIs

STEP 5: Layout Rules

  • Validation summary: Place at top of form section, before first field
  • Field order: Label → Input → Message (error/success/warning/hint)
  • Required indicator: Immediately after label text (red asterisk)
  • Optional indicator: After label in grey "(optioneel)"
  • Error message priority: Show error messages ABOVE success/hint
  • Icon size: 14px for messages, 18-22px for summary banners
  • Validation indicator: Absolute positioned right inside input (right: 16px)
  • Character counter: Below textarea, right-aligned
  • Step error inline: Between step nav and form content
  • Validation progress: Fixed below header or at form start

Responsive Behavior

  • Mobile (<640px): Hide close button on validation summary, reduce icon sizes
  • Tablet (640-900px): All features visible
  • Desktop (>900px): Full layout with tooltips

STEP 6: Accessibility (WCAG AA/AAA)

  • ARIA labels: All inputs have associated labels via <label> or aria-label
  • Error announcements: Use role="alert" for error messages to trigger screen reader
  • Required fields: Mark with aria-required="true" + visual asterisk
  • Invalid state: Use aria-invalid="true" on error inputs
  • Describedby: Link error messages to inputs via aria-describedby
  • Color contrast:
    • Error text (#FF6B6B on white): 4.5:1 ✓
    • Success text (#00C853 on white): 4.5:1 ✓
    • Warning text (#F59E0B on white): 3.5:1 (large text only)
  • Focus management: Move focus to first error field on validation fail
  • Keyboard navigation: All interactive elements keyboard accessible
  • Live regions: Real-time validation uses aria-live="polite"

STEP 7: Implementation Notes

React/Next.js Component

'use client';

import { useState, useEffect } from 'react';
import { AlertCircle, CheckCircle, AlertTriangle, Info, Loader2 } from 'lucide-react';

export function ValidatedField({
  label,
  required,
  value,
  onChange,
  validator
}: FormFieldProps) {
  const [state, setState] = useState<'default' | 'error' | 'success' | 'loading'>('default');
  const [message, setMessage] = useState<string>('');

  // Real-time validation with debounce
  useEffect(() => {
    if (!validator || !value) return;

    const timer = setTimeout(async () => {
      setState('loading');
      const result = await validator(value);
      setState(result.valid ? 'success' : 'error');
      setMessage(result.message || '');
    }, 500);  // 500ms debounce

    return () => clearTimeout(timer);
  }, [value, validator]);

  return (
    <div className="form-field">
      <label className="field-label">
        {label}
        {required && <span className="field-required">*</span>}
      </label>

      <div className="field-with-icon">
        <input
          type="text"
          className={`field-input ${state}`}
          value={value}
          onChange={(e) => onChange(e.target.value)}
          aria-required={required}
          aria-invalid={state === 'error'}
          aria-describedby={message ? 'field-message' : undefined}
        />

        {state === 'loading' && (
          <div className="validation-indicator loading">
            <Loader2 />
          </div>
        )}
        {state === 'success' && (
          <div className="validation-indicator success">
            <CheckCircle />
          </div>
        )}
        {state === 'error' && (
          <div className="validation-indicator error">
            <AlertCircle />
          </div>
        )}
      </div>

      {message && (
        <span
          id="field-message"
          className={`field-${state}`}
          role="alert"
        >
          {state === 'error' && <AlertCircle />}
          {state === 'success' && <CheckCircle />}
          {message}
        </span>
      )}
    </div>
  );
}

Validation Patterns

// Email validation
const validateEmail = async (email: string): Promise<ValidationResult> => {
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  if (!regex.test(email)) {
    return {
      valid: false,
      state: 'error',
      message: 'Vul een geldig e-mailadres in (bijv. naam@organisatie.nl)'
    };
  }
  return { valid: true, state: 'success' };
};

// KVK number validation (with API lookup)
const validateKVK = async (kvk: string): Promise<ValidationResult> => {
  if (kvk.length !== 8 || !/^\d+$/.test(kvk)) {
    return {
      valid: false,
      state: 'error',
      message: 'KVK-nummer moet 8 cijfers zijn'
    };
  }

  // API call to verify KVK
  const response = await fetch(`/api/kvk/${kvk}`);
  const data = await response.json();

  if (data.found) {
    return {
      valid: true,
      state: 'success',
      message: `Bedrijf gevonden: ${data.companyName}`
    };
  }

  return {
    valid: false,
    state: 'error',
    message: 'KVK-nummer niet gevonden'
  };
};

// Character limit validation
const validateLength = (text: string, maxLength: number): ValidationResult => {
  if (text.length > maxLength) {
    return {
      valid: false,
      state: 'error',
      message: `Maximaal ${maxLength} tekens toegestaan`
    };
  }

  if (text.length > maxLength * 0.9) {
    return {
      valid: true,
      state: 'warning',
      message: `Nog ${maxLength - text.length} tekens over`
    };
  }

  return { valid: true, state: 'success' };
};

  },

STEP 8: Variations & Related Components

Validation Summary Variants

  • Error (default): Coral border, alert-circle icon, blocks form submission
  • Warning: Amber border, alert-triangle icon, allows submission with confirmation
  • Info: Blue border, info icon, helpful guidance (not blocking)

Field State Variants

  • Default: Grey border, grey-light background, no message
  • Focus: Teal border, white background, teal glow (4px shadow)
  • Error: Coral border, coral-light background, error message with alert-circle icon
  • Success: Green border, green-light background, success message with check-circle icon
  • Warning: Amber border, amber-light background, warning message with alert-triangle icon
  • Loading: Default styling + spinning loader-2 icon (right-positioned)

Best Practices

  • Validate on blur: Check fields when user leaves input (not on every keystroke)
  • Debounce real-time: Wait 500ms after typing stops before validating (API calls)
  • Clear on fix: Remove error message immediately when user corrects input
  • Focus first error: Scroll to and focus first invalid field on submit attempt
  • Summary at top: Show validation summary above form, not at bottom
  • Specific messages: "Vul een geldig e-mailadres in" > "Ongeldige invoer"
  • Positive feedback: Show success states for auto-filled/verified fields
  • Progressive disclosure: Only show validation after first interaction, not on page load
  • Save progress: Allow saving incomplete forms (don't block with validation)

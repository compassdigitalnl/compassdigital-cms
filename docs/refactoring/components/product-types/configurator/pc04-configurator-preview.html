📋 PC04: ConfiguratorPreview

Component Type: Product Type Component — Configurator Live Preview
Category: Visual Display / Product Showcase
Complexity: Medium-High

💡 Purpose: Live visual preview of the configured product that updates in real-time as users make selections. Shows current color, configuration summary, and provides rotation/fullscreen controls for detailed viewing.

STEP 1: Component Overview

What is ConfiguratorPreview?

ConfiguratorPreview is a sticky visual display component that shows the currently configured product. It updates dynamically as users select options (color, specs, extras), providing immediate visual feedback. The component features a dark gradient background, decorative glow effects, and interactive controls for rotation and fullscreen viewing.

Key Features

  • Sticky positioning: Stays visible while scrolling through configurator options (desktop only)
  • Dark gradient background: Navy gradient with decorative teal glow effect
  • Product display: Large emoji (120px) or 3D model with drop shadow
  • Color indicator: Top-left overlay showing current color selection
  • Rotation controls: Rotate and fullscreen buttons (bottom-right)
  • Configuration summary: Product name + selected options (bottom info bar)
  • Fullscreen modal: Expanded view for detailed product inspection
  • Smooth animations: 360° rotation animation, hover effects
  • Responsive design: Mobile-optimized (non-sticky, smaller size)

Component Anatomy

  1. Preview Card: White container with 20px border-radius, sticky positioning
  2. Visual Area: Dark gradient bg (320px height) with decorative glow
  3. Product Display: Emoji (120px) or 3D model with drop shadow
  4. Color Indicator: Top-left badge (dot + color name, translucent bg)
  5. Rotation Controls: Bottom-right buttons (rotate + fullscreen icons)
  6. Info Bar: White bar at bottom (product name + config summary)

STEP 2: When to Use This Component

Primary Use Cases

  • Product configurators: Visual products (laptops, phones, cars, furniture)
  • Multi-step customization: When color/material selection affects appearance
  • High-value products: Products where visual preview influences purchase decision
  • Build-your-own flows: Custom computers, vehicles, furniture assemblies

When NOT to Use

  • Non-visual products: Services, subscriptions, digital goods (use pricing card instead)
  • Single-variant products: No customization options (use regular product image)
  • Mobile-first flows: Limited screen space (use compact variant or skip preview)
  • Text-heavy products: Books, courses (image preview not useful)

Component Pairing

ConfiguratorPreview works best when combined with:

  • PC03: ConfiguratorOptionSelector — User selects options → preview updates
  • PC05: ConfiguratorPriceBreakdown — Both components sticky on desktop (left: preview, right: pricing)
  • PC02: ConfiguratorStepNav — Shows progress, preview shows current state

Configuration Summary Format

The configSummary string follows this pattern:

  • Separator: Middle dot ( · ) with spaces
  • Order: Color → Processor → RAM → Storage → Extras
  • Example: "Space Grey · M4 Pro · 36GB · 512GB · AppleCare+"
  • Max length: ~60 characters (truncate with ellipsis on small screens)

STEP 7: Implementation Notes

React/Next.js Component

// components/ConfiguratorPreview.tsx
import React, { useState } from 'react';
import { RotateCcw, Maximize2, X } from 'lucide-react';

interface ConfiguratorPreviewProps {
  productImage: string | React.ReactNode;
  productName: string;
  configSummary: string;
  currentColor?: { name: string; value: string };
  variant?: 'default' | 'compact' | 'large';
  sticky?: boolean;
  stickyTop?: string;
  onRotate?: () => void;
  onFullscreen?: () => void;
  showRotateButton?: boolean;
  showFullscreenButton?: boolean;
}

export function ConfiguratorPreview({
  productImage,
  productName,
  configSummary,
  currentColor,
  variant = 'default',
  sticky = true,
  stickyTop = '88px',
  onRotate,
  onFullscreen,
  showRotateButton = true,
  showFullscreenButton = true,
}: ConfiguratorPreviewProps) {
  const [isRotating, setIsRotating] = useState(false);
  const [fullscreenOpen, setFullscreenOpen] = useState(false);

  const handleRotate = () => {
    setIsRotating(true);
    setTimeout(() => setIsRotating(false), 600);
    onRotate?.();
  };

  const handleFullscreen = () => {
    setFullscreenOpen(true);
    onFullscreen?.();
  };

  const closeFullscreen = () => {
    setFullscreenOpen(false);
  };

  return (
    <>
      <div
        className={`preview-card ${variant}`}
        style={{ position: sticky ? 'sticky' : 'static', top: stickyTop }}
        role="region"
        aria-label="Live product preview"
      >
        <div className="preview-visual" aria-live="polite">
          {currentColor && (
            <div className="preview-color-indicator" aria-label={`Current color: ${currentColor.name}`}>
              <div className="pci-dot" style={{ background: currentColor.value }} aria-hidden="true" />
              <span>{currentColor.name}</span>
            </div>
          )}

          <div
            className={`preview-product ${isRotating ? 'rotating' : ''}`}
            role="img"
            aria-label={`Configured ${productName} in ${currentColor?.name || 'default'} color`}
          >
            {productImage}
          </div>

          <div className="preview-rotate" role="toolbar" aria-label="Preview controls">
            {showRotateButton && (
              <button
                className="pr-btn"
                aria-label="Rotate product view"
                onClick={handleRotate}
              >
                <RotateCcw size={16} />
              </button>
            )}
            {showFullscreenButton && (
              <button
                className="pr-btn"
                aria-label="View in fullscreen"
                onClick={handleFullscreen}
              >
                <Maximize2 size={16} />
              </button>
            )}
          </div>
        </div>

        <div className="preview-bar">
          <div>
            <div className="pvb-name" aria-label="Product name">{productName}</div>
            <div className="pvb-config" aria-label="Current configuration">{configSummary}</div>
          </div>
        </div>
      </div>

      {/* Fullscreen Modal */}
      {fullscreenOpen && (
        <div className="preview-modal active" onClick={closeFullscreen}>
          <div className="preview-modal-content" onClick={(e) => e.stopPropagation()}>
            <button className="preview-modal-close" onClick={closeFullscreen} aria-label="Close fullscreen">
              <X size={20} />
            </button>
            <div className="preview-modal-product">{productImage}</div>
          </div>
        </div>
      )}
    </>
  );
}

Dynamic Configuration Updates

// Parent component managing configurator state
function ProductConfigurator() {
  const [config, setConfig] = useState({
    color: { name: 'Space Grey', value: '#2D3436' },
    processor: 'M4 Pro',
    ram: '36GB',
    storage: '512GB',
    extras: [],
  });

  const configSummary = useMemo(() => {
    const parts = [
      config.color.name,
      config.processor,
      config.ram,
      config.storage,
      ...config.extras,
    ];
    return parts.join(' · ');
  }, [config]);

  return (
    <div className="configurator-layout">
      <ConfiguratorPreview
        productImage="💻"
        productName="ProBook Ultra 16\""
        configSummary={configSummary}
        currentColor={config.color}
      />

      <ConfiguratorOptions
        config={config}
        onChange={setConfig}
      />
    </div>
  );
}

PC02: Configurator Preview

3D product preview card with real-time visual updates as configuration changes

Step 1: Component Overview

What is PC02?

PC02 (Configurator Preview) is a visual preview card that displays the configured product in real-time. It updates dynamically as users make selections (color, specs, accessories), providing instant visual feedback before purchase. This component dramatically reduces purchase hesitation by letting users "see" their custom product.

Conversion Impact: Apple's configurators with live previews increase conversion by 15-25% compared to static product images. Users need to visualize their customization choices.

Step 2: When to Use

Use PC02 when:

  • Product has visible customization options (color, size, components)
  • High-value products (>€500) where visualization reduces uncertainty
  • Products with 3+ customization steps (justify the preview space)
  • Sticky positioning on desktop (preview stays visible while scrolling options)
  • Desktop-first experience (collapse/hide on mobile)

Don't use when:

  • Product appearance doesn't change based on options (specs-only config)
  • Low-value products (<€100) — overkill for simple items
  • Mobile-first experience (takes too much screen space)

Step 5: Layout Rules

Positioning & Behavior

  • Desktop: Sticky positioning (top: 160px) — stays visible while scrolling options
  • Mobile: Static positioning at top, collapses to 240px height
  • Layout integration: Left column in 2-column grid (with PC07: ConfiguratorLayout)
  • Margin bottom: 20px gap before next element

Step 6: Accessibility

WCAG 2.1 AA Compliance

  • role="img" on preview visual with aria-label="Product preview: {productName} in {colorName}"
  • Rotate/zoom buttons have title tooltips + aria-label
  • Color indicator includes text (not just visual dot) for screen readers
  • Focus ring visible on control buttons (teal glow)
Step 7: Implementation Notes

React Component Example 

import { RotateCcw, Maximize2 } from 'lucide-react'

export function ConfiguratorPreview({
  productName,
  productImage,
  configSummary,
  colorName,
  colorHex,
  onRotate,
  onZoom,
  showControls = true,
  isSticky = true,
  stickyTop = 160,
}: ConfiguratorPreviewProps) {
  return (
    <div
      className="preview-card"
      style={{
        position: isSticky ? 'sticky' : 'static',
        top: isSticky ? `${stickyTop}px` : undefined,
      }}
    >
      <div
        className="preview-visual"
        role="img"
        aria-label={`Product preview: ${productName} in ${colorName}`}
      >
        {/* Color indicator */}
        <div className="preview-color-indicator">
          <div
            className="pci-dot"
            style={{ background: colorHex }}
          />
          {colorName}
        </div>

        {/* Product visual */}
        <div className="preview-product">
          {productImage}
        </div>

        {/* Controls */}
        {showControls && (
          <div className="preview-rotate">
            <button
              className="pr-btn"
              onClick={onRotate}
              title="Roteren"
              aria-label="360° roteren"
            >
              <RotateCcw size={16} />
            </button>
            <button
              className="pr-btn"
              onClick={onZoom}
              title="Vergroten"
              aria-label="Fullscreen weergave"
            >
              <Maximize2 size={16} />
            </button>
          </div>
        )}
      </div>

      {/* Preview bar */}
      <div className="preview-bar">
        <div>
          <div className="pvb-name">{productName}</div>
          <div className="pvb-config">{configSummary}</div>
        </div>
      </div>
    </div>
  )
}
Step 8: Variations & Related

Implementation Approaches

  • Simple: Emoji or icon (fastest, works for all products)
  • Images: Pre-rendered product images per color/variant
  • 3D (Three.js): Real 3D model with rotation/zoom (high-end products)
  • Canvas: 2D canvas rendering for dynamic customization layers

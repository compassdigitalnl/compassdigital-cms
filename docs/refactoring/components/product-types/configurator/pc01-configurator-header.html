STEP 1: Component Overview

Purpose

ConfiguratorHeader provides a sticky navigation header for product configuration flows, allowing users to track their progress, save their configuration, and navigate back to the product catalog while building custom products.

Key Features

  • ✅ Sticky positioning: Stays visible during scroll (z-index: 200)
  • ✅ Brand logo: Left-aligned, links back to homepage
  • ✅ Product title: Centered, shows current product being configured
  • ✅ Back navigation: Return to product listing or close configurator
  • ✅ Save status indicator: Shows "Saved", "Saving...", or "Auto-save enabled"
  • ✅ Action buttons: Share, favorite, PDF export
  • ✅ Mobile responsive: Collapses to essential elements only
  • ✅ Loading states: Spinning icon for save operations

Visual Design

  • Container: White background, 64px height, 1px grey bottom border
  • Shadow: Subtle drop shadow for depth (--shadow-sm)
  • Logo: 20px font, 800 weight, teal accent color
  • Product title: 14px, 700 weight, center-aligned, icon support
  • Back link: Grey text, teal on hover, left arrow icon
  • Action buttons: 36×36px squares, grey border, teal on hover
  • Save status: Pill shape (6px radius), grey-light bg, green when saved

Use Cases

  1. Custom laptops/PCs: Multi-step hardware configuration
  2. Medical equipment: Configure diagnostic kits or surgical tools
  3. Custom sneakers: Choose colors, materials, personalization
  4. Furniture: Customize dimensions, fabrics, finishes
  5. Gift boxes: Build personalized product bundles
  6. B2B bulk orders: Configure multiple SKU variations at once

STEP 2: When to Use This Component

✅ Use ConfiguratorHeader when:

  • Product has 3+ customization options (multi-step flow needed)
  • Configuration process takes multiple screens/pages
  • Users need persistent navigation while configuring
  • Save/share functionality is important (complex configurations)
  • Product value is high (users want to save progress)
  • B2B use case where configurations are shared internally
  • Configurator is full-screen (not embedded in product page)

❌ Don't use ConfiguratorHeader when:

  • Product has <3 options (simple variant selector is enough)
  • Configuration is single-screen (no need for sticky header)
  • Configurator is embedded modal (use modal header instead)
  • Product is low-value (users won't save configurations)

STEP 4: Props & Configuration

TypeScript Interface

interface ConfiguratorHeaderProps {
  // Logo configuration
  logo: {
    text: string;          // e.g., "techshop"
    accent?: string;       // e.g., "shop" (will be styled with teal)
    icon?: string;         // Lucide icon name
    href: string;          // Link URL
  };

  // Product info
  productTitle: string;    // e.g., "ProBook Ultra 16\""
  productIcon?: string;    // Emoji or Lucide icon

  // Back navigation
  backLink: {
    text: string;          // e.g., "Terug naar laptops"
    href: string;
  };

  // Save status
  saveStatus?: {
    state: 'saved' | 'saving' | 'error';
    text?: string;         // Custom text (defaults based on state)
  };

  // Action buttons
  actions?: Array<{
    icon: string;          // Lucide icon name
    label: string;         // For aria-label
    onClick: () => void;
  }>;

  // Sticky behavior
  sticky?: boolean;        // Default: true

  // Mobile behavior
  hideOnMobile?: {
    logo: boolean;         // Hide logo text
    backLinkText: boolean; // Hide "Terug naar" text
    saveStatus: boolean;   // Hide save status
  };
}

const defaultProps: Partial<ConfiguratorHeaderProps> = {
  sticky: true,
  hideOnMobile: {
    logo: true,
    backLinkText: true,
    saveStatus: true
  }
};

Layout Grid

The header uses flexbox with justify-content: space-between for left-center-right alignment.

  • Left: Logo (flex-start, natural width)
  • Center: Product title (absolute positioning at 50%, transform: translateX(-50%))
  • Right: Actions (flex-end, natural width)

Responsive Behavior

Breakpoint Changes
> 768px (Desktop) • Height: 64px
• Logo: icon + full text
• Product title: centered (absolute positioning)
• Back link: full text ("Terug naar...")
• Save status: visible

< 768px (Mobile) • Height: 56px

• Back link: icon only (text hidden)

STEP 7: Implementation Notes

Save Status State Management

// Auto-save with debounce
import { useState, useEffect } from 'react';
import { debounce } from 'lodash';

const [saveStatus, setSaveStatus] = useState<'saved' | 'saving' | 'error'>('saved');
const [configuration, setConfiguration] = useState({});

// Debounced save function (1 second delay)
const saveConfiguration = debounce(async (config) => {
  setSaveStatus('saving');

  try {
    await fetch('/api/save-configuration', {
      method: 'POST',
      body: JSON.stringify(config)
    });
    setSaveStatus('saved');
  } catch (error) {
    setSaveStatus('error');
  }
}, 1000);

// Auto-save on configuration changes
useEffect(() => {
  saveConfiguration(configuration);
}, [configuration]);

📋 PC03: ConfiguratorOptionSelector

Component Type: Product Type Component — Configurator Option Selection
Category: Form Input / Selection Controls
Complexity: Medium

💡 Purpose: Provides 3 visual selection patterns for product configuration: Color Swatches (visual), Option Cards (radio with details), and Toggle Options (checkbox for extras). Enables users to customize products by selecting from available options.

STEP 1: Component Overview

What is ConfiguratorOptionSelector?

ConfiguratorOptionSelector is a family of 3 related selection components designed for product configurators. Each variant is optimized for different types of options: visual choices (colors/materials), detailed specifications (processor/RAM/storage), and optional extras (accessories/services).

The 3 Variants

1. Color Swatches (Visual Selection)

  • Use case: Colors, materials, finishes, patterns
  • Visual: 48×48px colored squares with checkmark overlay
  • Selection: Single selection (radio behavior)
  • Layout: Flex-wrap horizontal grid

2. Option Cards (Radio with Details)

  • Use case: Processor, RAM, storage, size variants, subscription tiers
  • Visual: Full-width cards with radio button + info section + price
  • Selection: Single selection (radio behavior)
  • Layout: Vertical stack
  • Features: Title, description, spec tags, badges, price deltas

3. Toggle Options (Checkbox for Extras)

  • Use case: Accessories, services, warranties, add-ons
  • Visual: 2-column grid cards with checkbox + name/desc + price
  • Selection: Multiple selection (checkbox behavior)
  • Layout: 2-column grid (1-col on mobile)

Key Features

  • Visual feedback: Teal border + light teal background on selected state
  • Hover effects: Teal border on hover, scale animation on color swatches
  • Badge support: "Populairst", "Aanbevolen", "Pro" labels
  • Price display: Base price, included ("Incl."), or delta ("+€200")
  • Spec tags: Icon + text tags for technical specs (CPU cores, GPU, etc.)
  • Accessibility: Keyboard navigation, ARIA attributes, focus management
  • Responsive design: Mobile-optimized layouts

STEP 2: When to Use This Component

Use Color Swatches When:

  • Option is primarily VISUAL (color, material, texture, pattern)
  • You have 3-12 options to display
  • Users benefit from seeing the actual color/material
  • Labels are simple (1-2 words: "Space Grey", "Zilver")

Use Option Cards When:

  • Option has COMPLEX DETAILS (specs, features, descriptions)
  • You need to show pricing differences clearly
  • Options have technical specifications (CPU cores, RAM size, storage)
  • You want to guide users with badges ("Populairst", "Aanbevolen")
  • You have 2-6 mutually exclusive options

Use Toggle Options When:

  • Options are EXTRAS/ADD-ONS (not core configuration)
  • Users can select MULTIPLE items
  • Options are independent (selecting one doesn't affect others)
  • You have 2-8 optional extras
  • Price is the main decision factor (warranties, accessories)

When NOT to Use

  • Simple text options: Use native <select> dropdown instead
  • 20+ options: Use searchable dropdown or filterable list
  • Multi-step dependencies: Use ConfiguratorStepNav to break into separate steps
  • Quantity selection: Use QuantityStepper (C23) instead

Component Pairing

ConfiguratorOptionSelector works best when combined with:

  • PC02: ConfiguratorStepNav — Shows which step user is on
  • PC04: ConfiguratorPreview — Live preview updates as user selects options
  • PC05: ConfiguratorPriceBreakdown — Running total updates with each selection
  • PC06: ConfiguratorValidation — Shows errors if required options not selected

React/Next.js Component (Color Swatches)

// components/ColorSwatches.tsx
import React from 'react';
import { Palette } from 'lucide-react';

interface ColorOption {
  id: string;
  label: string;
  colorValue: string;
}

interface ColorSwatchesProps {
  options: ColorOption[];
  selectedId: string;
  onChange: (optionId: string) => void;
  title?: string;
  description?: string;
}

export function ColorSwatches({
  options,
  selectedId,
  onChange,
  title = 'Kleur',
  description,
}: ColorSwatchesProps) {

  const handleKeyDown = (e: React.KeyboardEvent, index: number) => {
    let newIndex = index;

    if (e.key === 'ArrowLeft' && index > 0) {
      newIndex = index - 1;
    } else if (e.key === 'ArrowRight' && index < options.length - 1) {
      newIndex = index + 1;
    }

    if (newIndex !== index) {
      const nextButton = document.querySelectorAll('.color-opt')[newIndex] as HTMLButtonElement;
      nextButton?.focus();
      e.preventDefault();
    }
  };

  return (
    <div className="cfg-section" role="radiogroup" aria-labelledby="color-label">
      <div id="color-label" className="cfg-title">
        <Palette size={18} /> {title}
      </div>
      {description && <div className="cfg-desc">{description}</div>}

      <div className="color-grid">
        {options.map((option, index) => (
          <button
            key={option.id}
            className={`color-opt ${selectedId === option.id ? 'selected' : ''}`}
            role="radio"
            aria-checked={selectedId === option.id}
            aria-label={option.label}
            tabIndex={selectedId === option.id ? 0 : -1}
            onClick={() => onChange(option.id)}
            onKeyDown={(e) => handleKeyDown(e, index)}
          >
            <div
              className="co-swatch"
              style={{ background: option.colorValue }}
              aria-hidden="true"
            />
            <div className="co-label">{option.label}</div>
          </button>
        ))}
      </div>
    </div>
  );
}

React Component (Option Cards)

// components/OptionCards.tsx
import React from 'react';
import * as Icons from 'lucide-react';

interface OptionCardOption {
  id: string;
  title: string;
  description: string;
  specs?: Array<{ icon: string; label: string }>;
  badge?: { label: string; color: string };
  price: {
    amount: string;
    delta?: string;
    deltaType?: 'up' | 'neutral';
  };
}

interface OptionCardsProps {
  options: OptionCardOption[];
  selectedId: string;
  onChange: (optionId: string) => void;
  title: string;
  icon: string;
  description?: string;
}

export function OptionCards({
  options,
  selectedId,
  onChange,
  title,
  icon,
  description,
}: OptionCardsProps) {
  const IconComponent = Icons[icon as keyof typeof Icons] as React.ElementType;

  return (
    <div className="cfg-section" role="radiogroup" aria-labelledby={`${icon}-label`}>
      <div id={`${icon}-label`} className="cfg-title">
        <IconComponent size={18} /> {title}
      </div>
      {description && <div className="cfg-desc">{description}</div>}

      <div className="opt-grid">
        {options.map((option) => (
          <button
            key={option.id}
            className={`opt-card ${selectedId === option.id ? 'selected' : ''}`}
            role="radio"
            aria-checked={selectedId === option.id}
            onClick={() => onChange(option.id)}
          >
            <div className="opt-radio" aria-hidden="true" />

            <div className="opt-info">
              <h4>
                {option.title}
                {option.badge && (
                  <span
                    className="opt-badge"
                    style={{ background: option.badge.color }}
                  >
                    {option.badge.label}
                  </span>
                )}
              </h4>
              <div className="opt-desc">{option.description}</div>

              {option.specs && option.specs.length > 0 && (
                <div className="opt-spec">
                  {option.specs.map((spec, idx) => {
                    const SpecIcon = Icons[spec.icon as keyof typeof Icons] as React.ElementType;
                    return (
                      <span key={idx} className="os-tag">
                        <SpecIcon size={10} /> {spec.label}
                      </span>
                    );
                  })}
                </div>
              )}
            </div>

            <div className="opt-price">
              <div className="op-amount">{option.price.amount}</div>
              {option.price.delta && (
                <div className={`op-delta ${option.price.deltaType || ''}`}>
                  {option.price.delta}
                </div>
              )}
            </div>
          </button>
        ))}
      </div>
    </div>
  );
}

Common Pitfalls

  • ❌ Mixing selection types: Don't use radio behavior for multiple selection or checkbox for single selection
  • ❌ Too many color swatches: 12+ colors becomes overwhelming — consider grouping or dropdown
  • ❌ Missing price context: Always show "Incl." or "Standaard" for base option, not just blank
  • ❌ Unclear badges: Use "Populairst" or "Aanbevolen", not vague terms like "Best"
  • ❌ Inconsistent pricing format: Stick to "€1.599" (with dot, not comma) for consistency
  • ❌ Missing descriptions: Option cards without descriptions are harder to understand

💡 Best Practice: For color swatches, always provide text labels below — don't rely solely on color for accessibility (color-blind users need text confirmation).

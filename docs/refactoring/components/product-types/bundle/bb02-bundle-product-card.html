BB02: Bundle Product Card

Selectable product cards for custom bundle builder with visual selection indicators

STEP 1: Component Overview

Purpose: Allow users to select individual products to build a custom bundle. Each card shows product details, pricing, ratings, and a visual checkmark when selected.

Key Features

  • Toggle Selection: Click to select/deselect product for bundle
  • Visual Feedback: Teal border + light teal background + checkmark when selected
  • Product Image: Emoji or image placeholder with gradient background
  • Product Info: Name, description, price (with optional old price), star rating
  • Category Badge: Optional category label (e.g., "Design", "Data")
  • Hover States: Border color change, subtle elevation
  • Disabled State: Greyscale when product unavailable or max bundle limit reached

Usage Context

Use in bundle builder interfaces where users can select multiple products to create a custom package. Works alongside BundleItemRow (BB03) to show selected items and BundleTotalCalculator (BB05) to display pricing.

Visual Description

Vertical card with product image at top, followed by name, description, and bottom row with price (left) and star rating (right). When selected, shows teal border, light teal background, and white checkmark in teal circle (top-right corner).

STEP 2: When to Use

✅ Primary Use Cases

  1. Bundle Builder Pages: Allow users to select products across categories (Design, Data, Assets, etc.)
  2. Custom Package Creation: Let users build their own product sets (e.g., software bundles, course packages)
  3. Multi-Select Product Grids: Any interface where users choose multiple products simultaneously
  4. Comparison Shopping: Select products to compare features/pricing (with max selection limit)
  5. Wishlist Building: Multi-select to add products to wishlist

❌ When NOT to Use

  • Single Product Selection: Use radio buttons or standard product cards (EC01) instead
  • Pre-Made Bundles: Use BundleOverviewCard (BB01) for fixed bundle packages
  • Cart Items: Use CartLineItem (EC06) for shopping cart
  • Product Variants: Use VariantColorSwatches (VP01) or VariantSizeSelector (VP02) for single product variations
💡 Pro Tip: Limit bundle selection to 3-5 products to avoid overwhelming users. Show a counter ("3/5 selected") and disable remaining cards when limit is reached.

STEP 7: Implementation Notes

React/Next.js Component Example

import React from 'react';

interface BundleProductCardProps {
  id: string;
  name: string;
  description: string;
  image: string;
  price: number;
  originalPrice?: number;
  rating?: number;
  reviewCount?: number;
  category?: string;
  showCategory?: boolean;
  isSelected: boolean;
  isDisabled?: boolean;
  onToggle: (id: string) => void;
  className?: string;
}

export const BundleProductCard: React.FC<BundleProductCardProps> = ({
  id,
  name,
  description,
  image,
  price,
  originalPrice,
  rating,
  reviewCount,
  category,
  showCategory = false,
  isSelected,
  isDisabled = false,
  onToggle,
  className = ''
}) => {
  const handleClick = () => {
    if (!isDisabled) {
      onToggle(id);
    }
  };

  const handleKeyDown = (e: React.KeyboardEvent) => {
    if ((e.key === 'Enter' || e.key === ' ') && !isDisabled) {
      e.preventDefault();
      onToggle(id);
    }
  };

  const renderStars = (rating: number) => {
    return '★'.repeat(Math.floor(rating)) + '☆'.repeat(5 - Math.floor(rating));
  };

  return (
    <div
      className={`bundle-product-card ${isSelected ? 'selected' : ''} ${isDisabled ? 'disabled' : ''} ${className}`}
      role="checkbox"
      aria-checked={isSelected}
      aria-label={`Select ${name}${isSelected ? ', currently selected' : ''}`}
      tabIndex={isDisabled ? -1 : 0}
      onClick={handleClick}
      onKeyDown={handleKeyDown}
    >
      {showCategory && category && (
        <div className="bpc-category">{category}</div>
      )}

      <div className="bpc-img" aria-hidden="true">
        {image}
      </div>

      <h3 className="bpc-name">{name}</h3>
      <p className="bpc-desc">{description}</p>

      <div className="bpc-bottom">
        <div className="bpc-price">
          €{price}
          {originalPrice && (
            <span className="old"> €{originalPrice}</span>
          )}
        </div>

        {rating && (
          <div
            className="bpc-rating"
            aria-label={`${rating} out of 5 stars${reviewCount ? `, ${reviewCount} reviews` : ''}`}
          >
            {renderStars(rating)}
            {reviewCount && <span> ({reviewCount})</span>}
          </div>
        )}
      </div>
    </div>
  );
};

State Management

// Parent component: Track selected products
const [selectedProducts, setSelectedProducts] = useState<string[]>([]);
const MAX_BUNDLE_SIZE = 5;

const handleToggle = (productId: string) => {
  setSelectedProducts(prev => {
    if (prev.includes(productId)) {
      // Deselect
      return prev.filter(id => id !== productId);
    } else {
      // Select (if under limit)
      if (prev.length < MAX_BUNDLE_SIZE) {
        return [...prev, productId];
      }
      return prev; // Don't add if at limit
    }
  });
};

Performance Optimization

  • Memoization: Memoize card component to prevent re-renders when siblings change
  • Virtual Scrolling: Use react-window for 50+ products
  • Image Loading: Lazy load product images with IntersectionObserver

Edge Cases

  • Max Bundle Limit: Disable unselected cards when limit reached
  • Out of Stock: Show disabled state + "Tijdelijk uitverkocht" overlay
  • Long Product Names: Truncate at 2 lines with ellipsis
  • Missing Rating: Don't show rating section if undefined

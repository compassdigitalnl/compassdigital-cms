DD01: DeliveryHero

Success hero component displayed after purchase of digital products (software, SaaS access, courses, templates). Confirms delivery, provides download links or access credentials, and sets expectations for next steps.

STEP 1: Component Overview

Purpose

DeliveryHero is the first thing users see after completing a digital product purchase. It celebrates the successful transaction, provides immediate access to the product, and reduces anxiety by clearly communicating what happens next. This component is critical for conversion completion and customer satisfaction.

Key Features

  • Success Celebration: Animated checkmark icon with pulsing green glow
  • Product Summary: Name, version, license type, order number
  • Primary CTA: Download button or login link (context-dependent)
  • Secondary Info: Email confirmation, support period, update policy
  • Expiry Notice: Download link validity period (if applicable)
  • Alternative Access: Link to account downloads page
  • 4 Use Cases: Download, SaaS Access, Course Access, Compact

Business Impact

  • Reduces Support Tickets: Clear instructions reduce "How do I access?" inquiries by 40-50%
  • Increases Satisfaction: Immediate access creates positive first impression
  • Prevents Churn: Clear next steps prevent abandoned downloads/logins
  • Upsell Opportunity: Can highlight additional features or upgrade paths

STEP 2: When to Use This Component

✅ Perfect For

  • Downloadable Products: Software, plugins, templates, design files
  • SaaS Account Creation: After signup for web-based platforms
  • Online Courses: After enrollment, providing access to learning platform
  • License Keys: After purchase of software licenses
  • Digital Subscriptions: After subscribing to recurring digital services

❌ Not Needed For

  • Physical Products: Use standard order confirmation (OC01: OrderSuccessHero)
  • In-Person Services: Use appointment confirmation instead
  • Free Downloads: Simpler thank-you page sufficient

  };

    text: string;                    // Plain text or HTML
    links?: Array<{ text: string; href: string; }>;
  };
}

React/Next.js Component Example

'use client'

import { Check, MailCheck, GraduationCap, Download, LogIn } from 'lucide-react'

const iconMap = {
  check: Check,
  'mail-check': MailCheck,
  'graduation-cap': GraduationCap,
  download: Download,
  'log-in': LogIn,
}

export function DeliveryHero({
  variant = 'download',
  badge,
  title,
  subtitle,
  product,
  primaryAction,
  secondaryAction,
  infoItems,
  expiry,
  footer
}: DeliveryHeroProps) {
  const IconComponent = iconMap[variant === 'email-sent' ? 'mail-check' :
                                variant === 'course' ? 'graduation-cap' : 'check']

  return (
    <div className={`delivery-hero ${variant}`}>
      <div className="dh-icon">
        <IconComponent />
      </div>

      {badge && <div className="dh-badge">{badge}</div>}

      <h1 className="dh-title">{title}</h1>
      <p className="dh-subtitle">{subtitle}</p>

      <div className="dh-product">
        <div className="dh-product-icon">
          {/* Product icon */}
        </div>
        <div className="dh-product-info">
          <div className="dh-product-name">{product.name}</div>
          <div className="dh-product-meta">{product.meta}</div>
          {product.license && (
            <div className="dh-product-license">
              {/* License info */}
            </div>
          )}
        </div>
      </div>

      <div className="dh-actions">
        <a href={primaryAction.href} className="dh-btn dh-btn-primary">
          {/* Primary button */}
        </a>
        {secondaryAction && (
          <a href={secondaryAction.href} className="dh-btn dh-btn-secondary">
            {/* Secondary button */}
          </a>
        )}
      </div>

      {infoItems && (
        <div className="dh-info-grid">
          {infoItems.map((item, i) => (
            <div key={i} className="dh-info-item">
              {/* Info item */}
            </div>
          ))}
        </div>
      )}

      {expiry && (
        <div className="dh-expiry">
          Download link geldig tot <strong>{expiry.date}</strong>
        </div>
      )}

      {footer && <div className="dh-footer">{footer.text}</div>}
    </div>
  )
}

  </nav>

ARIA Attributes

  • Section Role: role="status" announces success to screen readers
  • Live Region: aria-live="polite" for dynamic content updates
  • Icon Hidden: aria-hidden="true" (decorative, meaning conveyed by text)
  • Actions Nav: aria-label="Download acties" for button group

Keyboard Navigation

  • Tab Order: Primary button → Secondary button → Info links → Footer links
  • Focus Indicators: 3px teal glow ring on all interactive elements
  • Skip Links: Consider "Skip to download" link for power users

Screen Reader Support

  • Success Announcement: Use role="status" to announce hero on page load
  • Button Labels: "Download nu" is clear (avoid icon-only buttons)
  • Expiry Date: Use <time datetime="2026-02-28"> for dates
  • Product Info: Use semantic headings (H1 for page title, H2 for product name)

STEP 7: Implementation Notes

Dynamic Content Generation

// Server-side example (Next.js API route)
export async function POST(request: Request) {
  const { orderId } = await request.json()

  // Fetch order details from database
  const order = await db.order.findUnique({ where: { id: orderId } })
  const product = await db.product.findUnique({ where: { id: order.productId } })

  // Generate download link (signed URL, expires in 3 days)
  const downloadUrl = await generateSignedUrl({
    productId: product.id,
    userId: order.userId,
    expiresIn: 3 * 24 * 60 * 60  // 3 days
  })

  // Send confirmation email
  await sendEmail({
    to: order.email,
    template: 'digital-delivery',
    data: {
      productName: product.name,
      downloadUrl,
      licenseKey: order.licenseKey,
      expiryDate: new Date(Date.now() + 3 * 24 * 60 * 60 * 1000)
    }
  })

  return { success: true, downloadUrl }
}

Download Link Security

  • Signed URLs: Use expiring, signed download URLs (not direct file links)
  • Rate Limiting: Prevent abuse with download limits (e.g., 5 downloads per purchase)
  • IP Tracking: Optionally track download IPs for fraud detection
  • Account Requirement: Require login for repeat downloads (after initial 3-day window)

Customization Options

  • Custom Success Icon: Replace checkmark with product-specific icon (e.g., rocket for launch, key for licenses)
  • Video Tutorial: Embed onboarding video below hero ("How to get started")
  • Referral Incentive: Add "Share with friends, get 20% off" section below
  • Upsell Banner: "Upgrade to Enterprise for team features" (subtle, non-intrusive)
  • Social Proof: "Join 12,000+ customers using this product"

Performance Considerations

  • Page Load: This is often the first page after payment — optimize for speed (<1s LCP)
  • Download Button: Pre-generate signed URLs during checkout (don't make users wait)
  • Email Sending: Queue emails asynchronously (don't block page render)
  • Animation: Use CSS animations (not JS) for success pulse

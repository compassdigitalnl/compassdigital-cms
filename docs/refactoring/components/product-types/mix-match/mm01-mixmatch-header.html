📦 MM01: MixMatchHeader — Component Documentation

STEP 1: Component Overview

Component: MixMatchHeader

Category: Product Types → Mix & Match

Purpose: Hero banner for Mix & Match product pages that introduces the customization concept, displays key product information, and motivates users with visual stats.

Key Features:

  • Dark gradient background (navy → navy-light) with decorative teal radial glow
  • Badge pill with icon indicating "Mix & Match" product type
  • Large title with highlighted product name (teal accent)
  • Descriptive text explaining the selection process
  • Three stat cards displaying: item count to select, fixed box price, and total available products
  • Fully responsive: hides stats on mobile, stacks layout vertically

Visual Description:

The header creates an immersive introduction with a dark navy gradient background. A subtle teal glow in the top-right adds visual interest. Content is split: left side shows badge + title + description, right side displays three centered stat cards with large teal numbers separated by subtle dividers. The entire section is contained within the standard 1240px container.

STEP 2: When to Use

Use this component when:

  • Build-your-own product pages — Any product where users select X items from Y available options (e.g., "Pick 6 lunch items", "Create your gift box")
  • Fixed-price bundles — Products with a flat price regardless of specific selections made
  • Mix & Match campaigns — Promotional pages encouraging product variety selection
  • Subscription box builders — Monthly boxes where users customize their selection
  • Sample packs — "Try before you buy" packs with multiple product choices

Do NOT use when:

  • Product has variant options (size, color) — use VariableProduct instead
  • Bundle items are pre-selected without customization — use BundleBuilder instead
  • Price changes based on specific selections — this header shows fixed pricing
  • Simple product page without customization options

STEP 7: Implementation Notes

React/Next.js Component Example:

import React from 'react';
import { Package } from 'lucide-react';

interface MixMatchHeaderProps {
  badgeText?: string;
  badgeIcon?: React.ReactNode;
  title: string;
  highlightedText?: string;
  description: string;
  stats: Array<{ value: string | number; label: string }>;
  showStats?: boolean;
  className?: string;
}

export const MixMatchHeader: React.FC<MixMatchHeaderProps> = ({
  badgeText = 'Mix & Match',
  badgeIcon = <Package size={12} />,
  title,
  highlightedText,
  description,
  stats = [],
  showStats = true,
  className = '',
}) => {
  // Parse title to wrap highlighted text in <em>
  const parseTitle = (titleText: string, highlight?: string) => {
    if (!highlight) return titleText;
    const parts = titleText.split(highlight);
    return (
      <>
        {parts[0]}
        <em>{highlight}</em>
        {parts[1]}
      </>
    );
  };

  return (
    <header className={`mm-header ${className}`} role="banner">
      <div className="mmh-container">
        <div className="mmh-inner">
          <div className="mmh-left">
            <span className="mmh-badge" aria-label={`Product type: ${badgeText}`}>
              <span aria-hidden="true">{badgeIcon}</span>
              {badgeText}
            </span>
            <h1 className="mmh-title">
              {parseTitle(title, highlightedText)}
            </h1>
            <p className="mmh-desc">{description}</p>
          </div>
          {showStats && stats.length > 0 && (
            <div className="mmh-right" aria-label="Product statistics">
              {stats.map((stat, idx) => (
                <div
                  key={idx}
                  className="mmh-stat"
                  aria-label={`${stat.value} ${stat.label}`}
                >
                  <div className="mmhs-num">{stat.value}</div>
                  <div className="mmhs-label" aria-hidden="true">
                    {stat.label}
                  </div>
                </div>
              ))}
            </div>
          )}
        </div>
      </div>
    </header>
  );
};

Performance Optimization:

  • CSS containment: Add contain: layout style paint; to .mm-header to isolate rendering
  • Lazy glow: Decorative glow can be disabled on low-end devices with prefers-reduced-motion
  • Font loading: Ensure Plus Jakarta Sans 800 weight is preloaded for title
  • Image optimization: If badge uses custom icon, use SVG or optimized WebP

Edge Cases:

  • Long title: Titles over 60 characters should wrap (no truncation), tested up to 100 characters
  • No stats: If showStats={false}, left content centers horizontally on desktop
  • Many stats: Max 4 stats recommended (more causes overflow on tablets)
  • RTL languages: Entire layout mirrors for Arabic/Hebrew (glow moves to top-left)
  • Dark mode: Not applicable (header is inherently dark, but ensure glow visibility if page bg changes)

Browser Compatibility:

  • Modern browsers: Full support (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
  • Gradient fallback: Solid navy background for IE11 (if still supported)
  • Radial gradient: Supported in all modern browsers, degrades gracefully to solid background
  • Flexbox: Use autoprefixer for older browsers (-webkit-flex)

📦 MM03: MixMatchProductGrid — Component Documentation

STEP 1: Component Overview

Component: MixMatchProductGrid

Category: Product Types → Mix & Match

Purpose: Responsive grid container that displays selectable products for Mix & Match boxes. Provides consistent layout and spacing for product cards (MM04) with automatic responsive behavior.

Key Features:

  • 3-column grid on desktop (>900px), 2-column on tablet (640-900px), 1-column on mobile (<640px)
  • 10px gap between items (consistent spacing)
  • Empty state support with icon + message
  • Auto-fill behavior for even card heights
  • Fully responsive without JavaScript

Visual Description:

A simple CSS Grid layout that adapts to screen size. Products are displayed in equal-width columns with consistent 10px gaps. When no products are available, a centered empty state message appears spanning the full grid width.

STEP 2: When to Use

Use this component when:

  • Mix & Match product pages — Primary use case for displaying selectable products
  • Product selection grids — Any scenario where users choose multiple products from a set
  • Category-filtered views — When products are filtered by category (works with MM06)
  • Search results — When displaying Mix & Match products in search

Do NOT use when:

  • Displaying shop archive products — use EC01 ProductCard grid instead
  • Single product display — no grid needed
  • Horizontal scrolling carousel — use different layout pattern

STEP 3: Design Tokens

Layout:

  • Display: grid
  • Grid columns (desktop): repeat(3, 1fr)
  • Grid columns (tablet): repeat(2, 1fr) — @media (max-width: 900px)
  • Grid columns (mobile): 1fr — @media (max-width: 640px)
  • Gap: 10px (all breakpoints)

Empty State:

  • Icon size: 48px
  • Icon opacity: 0.5
  • Text size: 14px, 600 weight
  • Text color: #94A3B8 (grey-mid)
  • Padding: 64px 24px
  • Grid span: 1 / -1 (full width)

STEP 4: Props & Configuration

TypeScript Interface:

interface MixMatchProductGridProps {
  // Content
  products: Product[];

  // Layout
  columns?: {
    desktop: number;  // Default: 3
    tablet: number;   // Default: 2
    mobile: number;   // Default: 1
  };
  gap?: number;       // Default: 10 (px)

  // Empty state
  emptyIcon?: string;
  emptyText?: string;
  showEmptyState?: boolean; // Default: true

  // Styling
  className?: string;

  // Children (product cards)
  children?: React.ReactNode;
}

React/Next.js Component Example:

import React from 'react';
import { MixMatchProductCard } from './MixMatchProductCard';

interface MixMatchProductGridProps {
  products: Product[];
  selectedIds: string[];
  onToggle: (product: Product) => void;
  maxItems: number;
  currentCount: number;
  className?: string;
}

export const MixMatchProductGrid: React.FC<MixMatchProductGridProps> = ({
  products,
  selectedIds,
  onToggle,
  maxItems,
  currentCount,
  className = '',
}) => {
  if (products.length === 0) {
    return (
      <div className={`products-grid ${className}`}>
        <div className="products-grid-empty">
          <div className="products-grid-empty-icon">📦</div>
          <div className="products-grid-empty-text">
            Geen producten beschikbaar
          </div>
        </div>
      </div>
    );
  }

  return (
    <div className={`products-grid ${className}`}>
      {products.map((product) => {
        const isSelected = selectedIds.includes(product.id);
        const isDisabled = !isSelected && currentCount >= maxItems;

        return (
          <MixMatchProductCard
            key={product.id}
            product={product}
            isSelected={isSelected}
            isDisabled={isDisabled}
            onToggle={() => onToggle(product)}
          />
        );
      })}
    </div>
  );
};

STEP 6: Accessibility

Semantic HTML: Use <div class="products-grid" role="list"> with product cards as role="listitem".

ARIA: Grid container has role="list", each child card has role="listitem" for screen reader navigation.

STEP 7: Implementation Notes

Performance: Use grid-auto-rows: 1fr; for equal height cards. CSS Grid is performant — no JavaScript needed for layout.

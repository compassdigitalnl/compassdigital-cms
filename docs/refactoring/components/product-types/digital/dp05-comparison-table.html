📖 DP05: ComparisonTable — Complete Documentation

STEP 1: Component Overview

Component Name: DP05: ComparisonTable

Category: Digital Product Components (Product Types)

Usage: Side-by-side comparison of pricing tiers/license types with detailed feature breakdown

Description:

ComparisonTable displays a comprehensive feature comparison across multiple pricing tiers in a tabular format. Used on pricing pages and product pages to help users choose the right plan by showing exactly what's included in each tier.

Key Features:

  • Multi-column comparison (typically 3-4 tiers)
  • Tier header cards (name, price, description, badge)
  • Highlighted "Popular" or "Recommended" column (teal background)
  • Categorized feature rows (Core, AI, Collaboration, Support)
  • Feature availability indicators (checkmarks, crosses, text values)
  • Sticky column headers on scroll
  • Sticky first column (feature names) on horizontal scroll
  • CTA buttons in footer row
  • Responsive horizontal scroll on mobile

STEP 2: When to Use This Component

✅ Use ComparisonTable when:

  • Comparing 3-4 pricing tiers with 10+ features
  • Users need to see exact differences between plans
  • Feature availability varies significantly across tiers
  • Showing quantitative differences (storage, users, projects)
  • Highlighting "most popular" or "recommended" tier
  • Providing detailed feature transparency for informed decisions

❌ Don't use when:

  • Only 1-2 pricing options → use DP02 (LicenseSelector) instead
  • Features are simple/few (≤5) → use DP04 (FeatureGrid) or DP02 (LicenseSelector) with feature lists
  • Comparing non-pricing items → use generic comparison table
  • Mobile-first audience (tables are hard on small screens) → consider alternative layouts

💡 Common Use Cases:

  • SaaS pricing page: Free vs Starter vs Pro vs Enterprise comparison
  • Software product page: Personal vs Professional vs Team license comparison
  • Subscription plans: Monthly vs Annual vs Lifetime feature differences
  • API tiers: Rate limits, endpoints, support levels across tiers
  • Hosting plans: Storage, bandwidth, features comparison

STEP 4: Props & Configuration

TypeScript Interface:

interface ComparisonTableProps {
  header?: {
    overline?: string;
    title: string;
    description?: string;
  };
  tiers: PricingTier[];
  features: FeatureCategory[];
  highlightedTier?: string;  // Tier ID to highlight
}

interface PricingTier {
  id: string;
  name: string;
  price: string | number;
  period?: string;  // e.g., "eenmalig", "per maand"
  description: string;
  badge?: string;  // e.g., "Populairst"
  cta: {
    text: string;
    variant: 'primary' | 'outline';
    onClick: () => void;
  };
}

interface FeatureCategory {
  name: string;  // Category label (e.g., "Core Features")
  features: Feature[];
}

interface Feature {
  name: string;  // Feature name (e.g., "Aantal gebruikers")
  values: {
    [tierId: string]: boolean | string | number;
    // boolean → checkmark/cross
    // string/number → display as text
  };
}

STEP 5: Layout Rules

Table Structure:

  • Column 1: Feature names (35% width, sticky left on mobile scroll)
  • Columns 2-4: Tier values (equal width, auto-distribute remaining space)
  • Header row: Sticky top on vertical scroll
  • Category rows: Full-width colspan, grey background
  • Footer row: CTA buttons, sticky bottom optional

Responsive Behavior:

  • Desktop (≥901px): Full table visible, no scroll
  • Tablet/Mobile (≤900px): Horizontal scroll enabled:
    • Table min-width: 700px
    • First column sticky left (feature names always visible)
    • Header sticky top
    • Shadow on sticky column edge

Highlight Column:

  • Header cell: Teal gradient background, white text
  • Body cells: Light teal background (teal-glow)
  • Visual prominence guides user to recommended tier

    <thead role="rowgroup">
      <tr role="row">
        <th scope="col"></th>
        <th scope="col">Personal</th>
        <th scope="col" class="highlight">Professional</th>
        <th scope="col">Team</th>
      </tr>
    </thead>
    <tbody role="rowgroup">
      <tr role="row">
        <th scope="row">Aantal gebruikers</th>
        <td>1</td>
        <td class="highlight">1</td>
        <td>Tot 10</td>
      </tr>
    </tbody>
  </table>
</section>

ARIA Attributes:

  • <th scope="col"> for column headers (tier names)
  • <th scope="row"> for row headers (feature names)
  • Checkmark/cross icons: aria-label="Inbegrepen" or aria-label="Niet inbegrepen"
  • Screen readers announce: "Aantal gebruikers, Personal: 1, Professional: 1, Team: Tot 10"

Keyboard Navigation:

  • Table is keyboard-navigable (Tab through CTA buttons)
  • Scrollable area accessible via keyboard (Arrow keys or Tab + Enter)
  • CTA buttons focusable with visible focus ring

Color Contrast:

  • Navy text on white: 13.4:1 (AAA)
  • White text on teal highlight: 4.7:1 (AA)
  • Grey-mid on white: 4.8:1 (AA)
  • Checkmark green on white: sufficient for icons (non-text)

STEP 7: Implementation Notes

Dynamic Table Rendering (React):

function ComparisonTable({ tiers, features, highlightedTier }: Props) {
  return (
    <table className="comparison-table">
      <thead>
        <tr>
          <th></th>
          {tiers.map(tier => (
            <th
              key={tier.id}
              className={tier.id === highlightedTier ? 'highlight' : ''}
            >
              <TierCard tier={tier} />
            </th>
          ))}
        </tr>
      </thead>
      <tbody>
        {features.map(category => (
          <>
            <tr className="ct-category-row">
              <td colSpan={tiers.length + 1}>{category.name}</td>
            </tr>
            {category.features.map(feature => (
              <tr key={feature.name}>
                <td>{feature.name}</td>
                {tiers.map(tier => (
                  <td
                    key={tier.id}
                    className={tier.id === highlightedTier ? 'highlight' : ''}
                  >
                    <FeatureValue value={feature.values[tier.id]} />
                  </td>
                ))}
              </tr>
            ))}
          </>
        ))}
      </tbody>
      <tfoot>
        <tr>
          <td></td>
          {tiers.map(tier => (
            <td key={tier.id}>
              <button
                className={`ct-btn-${tier.cta.variant}`}
                onClick={tier.cta.onClick}
              >
                {tier.cta.text}
              </button>
            </td>
          ))}
        </tr>
      </tfoot>
    </table>
  );
}

Feature Value Component:

function FeatureValue({ value }: { value: boolean | string | number }) {
  if (typeof value === 'boolean') {
    return value ? (
      <div className="ct-check" aria-label="Inbegrepen">
        <Check size={16} />
      </div>
    ) : (
      <div className="ct-cross" aria-label="Niet inbegrepen">
        <X size={16} />
      </div>
    );
  }

  return <span className="ct-value">{value}</span>;
}

STEP 8: Variations & Related Components

Related Components:

Component When to Use Instead
DP02: LicenseSelector Simple tier selection without detailed feature comparison (2-4 tiers, ≤5 key features)
DP04: FeatureGrid Showcase features without pricing comparison (educational, not decisional)
Pricing (CMS Block) Pricing cards without detailed feature breakdown (high-level comparison)

Combination Patterns:

  • DP02 + DP05: License selector (quick choice) → Comparison table link ("See full comparison")
  • DP05 + FAQ: Comparison table → FAQ addressing common tier questions
  • Hero + DP05: Pricing hero intro → Full comparison below
  • DP05 + Testimonials: Comparison table → Social proof per tier

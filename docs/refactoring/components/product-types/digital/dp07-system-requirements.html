DP07: SystemRequirements

Display technical requirements and specifications for digital products (software, SaaS platforms, templates, courses). This component helps users verify compatibility before purchase or download.

STEP 1: Component Overview

Purpose

SystemRequirements provides a clear, scannable overview of technical specifications needed to run digital products. It reduces support requests by helping users verify compatibility upfront, and builds trust by being transparent about product requirements.

Key Features

  • 4 Layout Variants: Grid (default), Table, Min/Recommended Comparison, Compact
  • Category-based Organization: OS, Browser, Hardware, Software dependencies
  • Minimum vs. Recommended Specs: Two-tier system for clarity
  • Icon Support: Visual category identifiers (Lucide icons)
  • Responsive Design: Optimized for mobile with horizontal scroll tables
  • Info Banners: Additional context and warnings
  • Copy-to-Clipboard: Easy sharing of technical specs (optional)

Business Impact

  • Reduces Support Tickets: 30-40% fewer "Does this work on my system?" inquiries
  • Increases Conversion: Users confident their system is compatible are more likely to purchase
  • Builds Trust: Transparency about requirements shows professionalism
  • SEO Value: Structured data markup improves search visibility for technical queries

STEP 2: When to Use This Component

✅ Perfect For

  • Software Products: Desktop apps, plugins, extensions
  • SaaS Platforms: Web applications with browser/hardware requirements
  • Digital Templates: Design files requiring specific software versions (e.g., "Requires Figma 99+")
  • Online Courses: Courses requiring software installations
  • Developer Tools: SDKs, frameworks, libraries with dependency requirements
  • Game Downloads: PC/Mac games with hardware specs

❌ Not Needed For

  • Physical Products: Use product specifications table instead
  • Simple Digital Downloads: PDFs, images that work on any device
  • Mobile Apps: Use app store metadata instead (handled by platform)

STEP 4: Props & Configuration

TypeScript Interface

interface SystemRequirementsProps {
  header?: {
    overline?: string;           // e.g., "Technische specificaties"
    title: string;                // e.g., "Systeemvereisten"
    description?: string;         // Optional helper text
    align?: 'left' | 'center';    // Default: 'left'
  };
  categories: RequirementCategory[];
  variant?: 'grid' | 'table' | 'comparison' | 'compact';
  infoBanner?: {
    text: string;
    icon?: string;               // Lucide icon name, default: 'info'
  };
  enableCopy?: boolean;          // Show copy buttons for specs
}

interface RequirementCategory {
  name: string;                  // e.g., "Besturingssysteem"
  icon: string;                  // Lucide icon name
  specs: RequirementSpec[];
}

interface RequirementSpec {
  label: string;                 // e.g., "Windows", "Processor"
  value: string;                 // e.g., "Windows 10 (64-bit) of nieuwer"
  recommended?: string;          // For table/comparison variants
  mono?: boolean;                // Use monospace font (for version numbers)
}

React/Next.js Component Example

'use client'

import { Monitor, Globe, Cpu, Package, Info } from 'lucide-react'

const iconMap = {
  monitor: Monitor,
  globe: Globe,
  cpu: Cpu,
  package: Package,
}

export function SystemRequirements({
  header,
  categories,
  variant = 'grid',
  infoBanner
}: SystemRequirementsProps) {
  return (
    <div className={`system-requirements ${variant}`}>
      {header && (
        <div className={`sr-header ${header.align || 'left'}`}>
          {header.overline && (
            <div className="sr-overline">{header.overline}</div>
          )}
          <div className="sr-title">{header.title}</div>
          {header.description && (
            <div className="sr-description">{header.description}</div>
          )}
        </div>
      )}

      {variant === 'grid' && (
        <div className="sr-grid">
          {categories.map((cat, i) => {
            const Icon = iconMap[cat.icon]
            return (
              <div key={i} className="sr-category">
                <div className="sr-category-header">
                  <div className="sr-category-icon">
                    <Icon size={20} />
                  </div>
                  <div className="sr-category-name">{cat.name}</div>
                </div>
                <div className="sr-specs">
                  {cat.specs.map((spec, j) => (
                    <div key={j} className="sr-spec">
                      <div className="sr-spec-label">{spec.label}</div>
                      <div className={`sr-spec-value ${spec.mono ? 'mono' : ''}`}>
                        {spec.value}
                      </div>
                    </div>
                  ))}
                </div>
              </div>
            )
          })}
        </div>
      )}

      {infoBanner && (
        <div className="sr-info-banner">
          <Info size={20} />
          <div className="sr-info-text">{infoBanner.text}</div>
        </div>
      )}
    </div>
  )
}

STEP 5: Layout Rules

Container

  • Background: White card with 1px grey border, 12px radius
  • Padding: 32px (default), 24px (compact), 16px (mobile)
  • Max-Width: No max-width (inherits parent container)

Grid Variant Layout

  • Grid: grid-template-columns: repeat(2, 1fr)
  • Gap: 24px between categories (16px on mobile)
  • Responsive: 2-col → 1-col at 900px breakpoint
  • Category Cards: Grey-light bg, 1px grey border, hover: teal border + shadow

Table Variant Layout

  • Table Structure: 3 columns (Component, Minimum, Recommended)
  • Sticky Header: Table head stays visible on vertical scroll
  • Category Rows: Full-width cells with teal bg, 2px bottom border
  • Zebra Striping: Alternate row backgrounds on hover (grey-light)
  • Horizontal Scroll: On mobile, table scrolls horizontally with min-width

Comparison Variant Layout

  • Grid: grid-template-columns: 1fr 1fr
  • Gap: 24px (16px mobile)
  • Recommended Column: Teal glow bg, teal border, "Best" badge
  • Responsive: 2-col → 1-col (stacked) at 900px

Spacing Hierarchy

  • Header → Content: 32px gap
  • Category Icon → Name: 12px gap
  • Category Header → Specs: 16px gap
  • Spec Label → Value: 4px gap
  • Spec Items: 12px gap (8px compact)

Icon Boxes

  • Size: 40×40px (default), 32×32px (compact)
  • Background: Teal glow (rgba(0,137,123,0.12))
  • Icon Color: Teal (#00897B)
  • Border Radius: 8px (--radius-sm)

<section className="system-requirements" aria-labelledby="sr-title">
  <header className="sr-header">

        </div>
      </dl>
    </article>
  </div>
</section>

ARIA Attributes

  • Section Label: aria-labelledby="sr-title" on container
  • Category List: role="list" on grid/table for screen readers
  • Category Items: role="listitem" for each category card
  • Table Semantics: Use native <table>, <thead>, <tbody> (no ARIA needed)
  • Info Banner: role="status" or role="alert" for important warnings

Keyboard Navigation

  • Focusable Elements: Copy buttons (if enabled)
  • Tab Order: Header → Categories (top-to-bottom, left-to-right) → Copy buttons → Info banner links
  • Focus Indicators: 3px teal glow ring on interactive elements

Screen Reader Support

  • Definition Lists: Use <dl>, <dt>, <dd> for spec label/value pairs
  • Icon Labels: Add aria-label to category icons (e.g., aria-label="Besturingssysteem")
  • Monospace Values: No special treatment (numbers/versions read correctly)
  • Table Headers: Use <th scope="col"> for column headers

  • Touch Targets: Category cards min 44×44px on mobile (entire card is tappable if interactive)
  • Font Scaling: Use relative units (rem) for accessibility zoom support
  • Horizontal Scroll: Table variant scrolls left/right on small screens (indicated by scroll shadows)

STEP 7: Implementation Notes

Content Strategy

  • Be Specific: "Windows 10 (64-bit)" > "Windows 10"
  • Version Numbers: Use monospace font for clarity (v16.0.0)
  • Avoid Jargon: "8 GB RAM" > "8192 MB of system memory"
  • Positive Language: "Recommended" > "You should really use this"
  • Future-Proof: "macOS 11.0 or newer" (not "macOS 11.0, 12.0, 13.0")

Copy-to-Clipboard Feature

function copySpec(text: string, button: HTMLButtonElement) {
  navigator.clipboard.writeText(text).then(() => {
    // Show success feedback
    button.classList.add('copied')
    button.querySelector('i')?.setAttribute('data-lucide', 'check')
    lucide.createIcons()

    // Reset after 2 seconds
    setTimeout(() => {
      button.classList.remove('copied')
      button.querySelector('i')?.setAttribute('data-lucide', 'copy')
      lucide.createIcons()
    }, 2000)
  })
}

Customization Options

  • Custom Icons: Replace default icons with product-specific ones (e.g., Docker logo for containerized apps)
  • Color Themes: Override --teal with vertical-specific accent color
  • Additional Columns: Add "Enterprise" tier to table variant (3 → 4 columns)
  • Collapsible Categories: For long spec lists, add expand/collapse per category
  • Compatibility Checker: Add interactive form to auto-detect user's system and highlight compatible specs

Performance Considerations

  • Icon Loading: Use Lucide React (tree-shakeable) instead of CDN for production
  • Table Rendering: For 20+ rows, consider virtualization or pagination
  • Mobile Tables: Use CSS overflow-x: auto with scroll shadows for better UX

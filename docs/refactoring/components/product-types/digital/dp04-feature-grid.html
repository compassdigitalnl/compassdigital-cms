📖 DP04: FeatureGrid — Complete Documentation

STEP 1: Component Overview

Component Name: DP04: FeatureGrid

Category: Digital Product Components (Product Types)

Usage: Display key product features in a scannable grid layout with icons and descriptions

Description:

FeatureGrid showcases digital product features in a structured grid with icons, titles, and descriptions. Used on product pages, landing pages, and feature comparison sections to highlight capabilities and benefits.

Key Features:

  • Flexible grid layouts (2/3/4 columns)
  • Icon box with color variants (teal, green, blue, purple, amber)
  • Hover effects: icon scale + color fill
  • Optional section header (overline + title + description)
  • 3 layout variants: Default (vertical icons), Card (bordered boxes), Horizontal (icon + text inline)
  • Compact variant for dense feature lists
  • Fully responsive (auto-stacks on mobile)

STEP 2: When to Use This Component

✅ Use FeatureGrid when:

  • Showcasing 4-12 product features or benefits
  • Highlighting SaaS capabilities (integrations, security, collaboration)
  • Displaying software features (AI tools, export options, analytics)
  • Presenting template/theme features (responsive, customizable, performant)
  • Creating "Why choose us?" or "What's included?" sections
  • Comparing feature sets across pricing tiers (with checkmarks)

❌ Don't use when:

  • Only 1-3 features → use Features CMS block instead (larger cards)
  • 20+ features → too many, group into categories or use tabs
  • Feature comparison table → use DP05 (ComparisonTable) instead
  • Step-by-step process → use numbered steps or timeline

💡 Common Use Cases:

  • SaaS landing page: Core features (AI, collaboration, security, analytics)
  • Software product page: Key capabilities (import/export, plugins, customization)
  • Template marketplace: Template features (responsive, SEO-ready, 1-click install)
  • App store listing: App features (offline mode, notifications, sync)
  • Pricing page: "All plans include..." feature list

STEP 3: Design Tokens

Colors:

  • --teal (#00897B) — Default icon background + text (hover: fill)
  • --teal-glow (rgba(0,137,123,0.12)) — Icon box background
  • --green (#00C853) — Green icon variant
  • --blue (#2196F3) — Blue icon variant
  • --purple (#7C3AED) — Purple icon variant
  • --amber (#F59E0B) — Amber icon variant
  • --navy (#0A1628) — Title text
  • --grey-dark (#64748B) — Description text
  • --grey (#E8ECF1) — Card border

Typography:

  • Section overline: 11px, 700 weight, uppercase, teal
  • Section title: 28px, 800 weight, navy
  • Section description: 15px, grey-dark
  • Feature title: 16px, 700 weight, navy
  • Feature description: 13px, grey-dark
  • Compact title: 14px, 700 weight
  • Compact description: 12px

Spacing:

  • Grid gap: 24px (default) / 16px (compact)
  • Icon box: 48px × 48px (default) / 40px × 40px (compact/horizontal)
  • Icon size: 24px (default) / 20px (compact/horizontal)
  • Item gap: 12px (default) / 8px (compact)
  • Card padding: 24px (when card variant)

STEP 4: Props & Configuration

TypeScript Interface:

interface FeatureGridProps {
  header?: {
    overline?: string;           // Optional eyebrow text
    title: string;               // Section title
    description?: string;        // Section description
    align?: 'center' | 'left';   // Text alignment (default: center)
  };
  features: Feature[];           // Array of features
  columns?: 2 | 3 | 4;          // Grid columns (default: 2)
  variant?: 'default' | 'card' | 'horizontal';  // Layout variant
  compact?: boolean;             // Use compact spacing/sizing
}

interface Feature {
  icon: string;                  // Lucide icon name
  iconColor?: 'teal' | 'green' | 'blue' | 'purple' | 'amber';  // Icon theme
  title: string;                 // Feature title
  description: string;           // Feature description
}

STEP 5: Layout Rules

Grid Layouts:

  • 2-column: Best for detailed features (4-6 items)
  • 3-column: Balanced layout (6-9 items)
  • 4-column: Compact grid for many features (8-12 items)

Responsive Behavior:

  • Desktop (≥901px): User-specified columns (2/3/4)
  • Tablet (641-900px): 4-col → 2-col, 3-col → 2-col, 2-col unchanged
  • Mobile (≤640px): All variants → 1 column

Variant Differences:

Variant Layout Best For
Default Vertical flex (icon → title → desc) Scannable feature lists, landing pages
Card Same as default + border + padding Visual separation, distinct features
Horizontal Horizontal flex (icon ↔ content) Dense layouts, checklist-style features
Compact Smaller icons/text, tighter gaps Long feature lists, secondary features

STEP 6: Accessibility

Semantic HTML:

<section class="feature-grid-section" aria-labelledby="features-title">
  <header class="fg-header">
    <div class="fg-overline">Krachtige Features</div>
    <h2 id="features-title" class="fg-title">
      Alles wat je nodig hebt om te groeien
    </h2>
    <p class="fg-desc">...</p>
  </header>

  <div class="feature-grid cols-2">
    <article class="feature-item">
      <div class="fi-icon" aria-hidden="true">
        <i data-lucide="cpu"></i>
      </div>
      <div class="fi-content">
        <h3 class="fi-title">AI Content Generation</h3>
        <p class="fi-desc">...</p>
      </div>
    </article>
  </div>
</section>

ARIA Notes:

  • Icons marked aria-hidden="true" (decorative only)
  • Section labeled with aria-labelledby pointing to title
  • Each feature wrapped in <article> for semantic grouping
  • Feature titles use <h3> for proper heading hierarchy

Color Contrast:

  • Title (navy) on white: 13.4:1 (AAA)
  • Description (grey-dark) on white: 6.5:1 (AAA)
  • Icon colors on light backgrounds: minimum 3:1 (non-text)
  • Hover states maintain contrast requirements

STEP 7: Implementation Notes

Dynamic Icon Rendering (React):

import * as LucideIcons from 'lucide-react';

function FeatureIcon({ icon, color }: { icon: string; color: string }) {
  const IconComponent = LucideIcons[icon as keyof typeof LucideIcons];

  if (!IconComponent) {
    console.warn(`Icon "${icon}" not found`);
    return null;
  }

  return (
    <div className={`fi-icon ${color}`}>
      <IconComponent size={24} />
    </div>
  );
}

STEP 8: Variations & Related Components

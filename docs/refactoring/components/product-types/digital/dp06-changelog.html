📖 DP06: Changelog — Complete Documentation

STEP 1: Component Overview

Component Name: DP06: Changelog

Category: Digital Product Components (Product Types)

Usage: Display version history, release notes, and updates for digital products

Description:

Changelog displays a chronological timeline of product updates with version numbers, dates, and categorized changes (New features, Improvements, Bug fixes, Security patches). Used on product pages, documentation sites, and release notes pages to keep users informed about product evolution.

Key Features:

  • Chronological timeline (newest first)
  • Version numbers with semantic versioning (v4.2.0)
  • Release dates in readable format
  • Version badges (New, Major, Security, Beta)
  • Categorized changes (New, Improved, Fixed, Security)
  • Optional version summary (one-sentence description)
  • Collapsible versions (hide/show details)
  • Color-coded change types (purple, teal, green, coral)
  • Compact variant for dense listings

STEP 2: When to Use This Component

✅ Use Changelog when:

  • Showing version history for software, apps, or SaaS products
  • Documenting updates for templates, themes, or plugins
  • Providing transparency about product improvements
  • Building trust by showing active development
  • Highlighting new features to drive upgrades
  • Communicating security patches to users

❌ Don't use when:

  • Only 1-2 versions exist → just show release notes inline
  • Internal version tracking → use dev tools (Git, JIRA)
  • Non-software products (physical goods)

💡 Common Use Cases:

  • Product page tab: "What's New" or "Changelog" tab on product pages
  • Documentation site: Dedicated changelog page (/changelog)
  • Dashboard notification: "See what's new in v4.2.0" modal after login
  • Email campaigns: Highlight major updates to users
  • GitHub releases: Mirrored changelog for open-source projects

Spacing:

  • Version gap: 32px between versions

  • Change item gap: 6px between items

Icons (Lucide):

STEP 4: Props & Configuration

TypeScript Interface:

interface ChangelogProps {
  header?: {
    overline?: string;
    title: string;
    description?: string;
    align?: 'center' | 'left';
  };
  versions: Version[];
  variant?: 'default' | 'collapsible' | 'compact';
  defaultExpanded?: boolean;  // For collapsible variant
}

interface Version {
  number: string;              // e.g., "v4.2.0"
  date: string;                // e.g., "25 Februari 2026"
  badges?: VersionBadge[];
  summary?: string;            // One-sentence description
  changes: ChangeGroup[];
}

interface VersionBadge {
  text: string;                // e.g., "Nieuwe features"
  variant: 'new' | 'major' | 'security' | 'beta';
}

interface ChangeGroup {
  type: 'new' | 'improved' | 'fixed' | 'security';
  items: string[];             // List of change descriptions
}

STEP 5: Layout Rules

Version Entry Structure:

  • Header: Version number + date + badges (flex row, wrap)
  • Summary: Optional one-sentence description
  • Changes: Grouped by type (New, Improved, Fixed, Security)
  • Change items: Bulleted list with colored labels

Variant Differences:

Variant Behavior Best For
Default All versions expanded, full details visible Dedicated changelog page, few versions (≤10)
Collapsible Click header to expand/collapse, bordered cards Many versions (10+), product page tab
Compact Smaller text/spacing, condensed layout Sidebar widget, modal, limited space

Responsive Behavior:

  • Desktop (≥641px): Version header horizontal (number, date, badges inline)
  • Mobile (≤640px): Version header stacks vertically, toggle icon absolute positioned

ARIA Attributes:

  • Dates: <time datetime="2026-02-25"> for machine-readable dates
  • Collapsible: aria-expanded="true/false" on header
  • Toggle button: role="button", aria-label="Toggle version details"

Keyboard Navigation:

  • Tab: Focus collapsible headers
  • Enter/Space: Expand/collapse version
  • All interactive elements keyboard-accessible

Color Contrast:

  • Version number (navy) on white: 13.4:1 (AAA)
  • Change text (grey-dark) on white: 6.5:1 (AAA)
  • Badge text on colored backgrounds: minimum 4.5:1

STEP 7: Implementation Notes

Version Collapse/Expand (React):

const [expandedVersions, setExpandedVersions] = useState<Set<string>>(
  new Set(defaultExpanded ? versions.map(v => v.number) : [])
);

function toggleVersion(versionNumber: string) {
  setExpandedVersions(prev => {
    const next = new Set(prev);
    if (next.has(versionNumber)) {
      next.delete(versionNumber);
    } else {
      next.add(versionNumber);
    }
    return next;
  });
}

Date Formatting:

function formatDate(date: Date, locale: string = 'nl-NL'): string {
  return new Intl.DateTimeFormat(locale, {
    day: 'numeric',
    month: 'long',
    year: 'numeric',
  }).format(date);
}

// Usage
const formattedDate = formatDate(new Date('2026-02-25')); // "25 februari 2026"

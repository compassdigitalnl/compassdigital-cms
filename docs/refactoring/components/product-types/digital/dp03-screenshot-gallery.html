📖 DP03: ScreenshotGallery — Complete Documentation

STEP 1: Component Overview

Component Name: DP03: ScreenshotGallery

Category: Digital Product Components (Product Types)

Usage: Display product screenshots in a grid with lightbox viewer for digital products (software, apps, SaaS, templates)

Description:

ScreenshotGallery displays product screenshots in a responsive grid layout with click-to-zoom lightbox functionality. Used on digital product pages to showcase UI/UX, features, and workflows through visual examples.

Key Features:

  • Responsive grid layout (2/3/4 columns, user-selectable)
  • 16:10 aspect ratio thumbnails (optimized for screen captures)
  • Hover effects: border highlight, shadow elevation, caption reveal, zoom icon
  • Optional feature badges (e.g., "Nieuwe UI", "AI Tools")
  • Full-screen lightbox modal with keyboard navigation
  • Prev/Next navigation arrows
  • Thumbnail strip in lightbox for quick jumping
  • Caption display (title + counter "X van Y")
  • ESC to close, arrow keys to navigate
  • Click outside lightbox to close (backdrop click)

STEP 2: When to Use This Component

✅ Use ScreenshotGallery when:

  • Displaying software UI screenshots (dashboards, panels, features)
  • Showcasing SaaS app interfaces across different screens
  • Presenting template designs (WordPress themes, Figma templates, etc.)
  • Demonstrating mobile app screens or responsive designs
  • Showing before/after comparisons of digital products
  • Illustrating workflow steps or feature tutorials
  • You have 4+ screenshots to display (minimum recommended)

❌ Don't use when:

  • Only 1-2 images → use standard product image gallery (PD01) instead
  • Displaying physical product photos → use ProductGallery (PD01) with 1:1 aspect ratio
  • Showing video content → use ProductVideoPlayer (PD08) instead
  • Large image sets (20+) → consider pagination or filtering

💡 Common Use Cases:

  • Software product page: Dashboard, settings, reports, mobile app views
  • SaaS landing page: Feature highlights, user workflows, integrations
  • Template marketplace: Homepage, inner pages, components, responsive views
  • App store listing: 5-8 app screenshots showing key features
  • Plugin/extension: Admin panels, frontend output, configuration screens

STEP 4: Props & Configuration

TypeScript Interface:

interface ScreenshotGalleryProps {
  title?: string;                   // Section title (e.g., "Product Screenshots")
  screenshots: Screenshot[];        // Array of screenshot items
  defaultColumns?: 2 | 3 | 4;      // Initial grid columns (default: 3)
  showViewToggle?: boolean;         // Show column toggle buttons (default: true)
  aspectRatio?: '16:10' | '4:3' | '9:16';  // Thumbnail aspect ratio
}

interface Screenshot {
  id: string;                       // Unique identifier
  src: string;                      // Image URL (high-res for lightbox)
  thumbnail?: string;               // Optional thumbnail URL (optimized)
  alt: string;                      // Alt text for accessibility
  caption?: string;                 // Image caption/title
  badge?: {
    text: string;                   // Badge text (e.g., "Nieuwe UI")
    color?: string;                 // Custom badge color (default: teal)
  };
}

STEP 5: Layout Rules

Grid Layout:

  • 2-column: grid-template-columns: repeat(2, 1fr)
  • 3-column: grid-template-columns: repeat(3, 1fr) (default)
  • 4-column: grid-template-columns: repeat(4, 1fr)
  • Gap: 16px between items

Responsive Breakpoints:

  • Desktop (≥901px): User-selected columns (2/3/4)
  • Tablet (641-900px): 4-col → 3-col, else unchanged
  • Mobile (≤640px): Force 2 columns for all variants

Lightbox Layout:

  • Backdrop: Full viewport, fixed positioning, z-index 1000
  • Content: Centered flex column, max-width 1200px
  • Image: max-height calc(90vh - 120px) — leaves space for caption + thumbnails
  • Navigation: Absolute left/right at 50% vertical, 24px from edges
  • Close button: Absolute top-right, 24px from edges
  • Thumbnails: Horizontal scroll strip, 80×50px each, 8px gap

Item Hover States:

  • Border: grey → teal
  • Shadow: none → shadow-md
  • Transform: translateY(0) → translateY(-2px)
  • Overlay: opacity 0 → 1 (gradient caption)
  • Zoom icon: opacity 0 → 1, scale 1.05

Keyboard Navigation:

  • ESC: Close lightbox
  • Arrow Left: Previous image
  • Arrow Right: Next image
  • Home: Jump to first image
  • End: Jump to last image
  • Tab: Focus through thumbnails, close button, nav buttons

Focus Management:

  • When lightbox opens: trap focus inside modal
  • When lightbox closes: return focus to clicked thumbnail
  • Focus indicator visible on all interactive elements

Screen Reader Support:

  • Lightbox announced as "Dialog" when opened
  • Image changes announced: "Afbeelding 2 van 8: Analytics Dashboard"
  • Navigation buttons have descriptive labels
  • Close button labeled "Sluit lightbox"

STEP 7: Implementation Notes

Lightbox State Management (React):

const [lightboxOpen, setLightboxOpen] = useState(false);
const [currentIndex, setCurrentIndex] = useState(0);

function openLightbox(index: number) {
  setCurrentIndex(index);
  setLightboxOpen(true);
  document.body.style.overflow = 'hidden';  // Prevent scroll
}

function closeLightbox() {
  setLightboxOpen(false);
  document.body.style.overflow = '';
}

function navigateLightbox(direction: number) {
  const newIndex = currentIndex + direction;
  if (newIndex >= 0 && newIndex < screenshots.length) {
    setCurrentIndex(newIndex);
  }
}

Keyboard Event Handling:

useEffect(() => {
  if (!lightboxOpen) return;

  function handleKeyDown(e: KeyboardEvent) {
    switch (e.key) {
      case 'Escape':
        closeLightbox();
        break;
      case 'ArrowLeft':
        navigateLightbox(-1);
        break;
      case 'ArrowRight':
        navigateLightbox(1);
        break;
      case 'Home':
        setCurrentIndex(0);
        break;
      case 'End':
        setCurrentIndex(screenshots.length - 1);
        break;
    }
  }

  window.addEventListener('keydown', handleKeyDown);
  return () => window.removeEventListener('keydown', handleKeyDown);
}, [lightboxOpen, currentIndex]);

STEP 8: Variations & Related Components

Aspect Ratio Variants:

  • 16:10 (Default): Desktop screenshots, widescreen apps
  • 4:3: Tablet/iPad screenshots, traditional software
  • 9:16: Mobile phone screenshots (portrait), app stores
  • 1:1: Instagram-style square crops, social media tools

Grid Variants:

  • 2-column: Large screenshots, detailed views (800px+ per image)
  • 3-column: Default, balanced layout (600px per image)
  • 4-column: Compact grid, more screenshots visible (400px per image)
  • Masonry: Alternative layout with varying heights (Pinterest-style)

Related Components:

Component When to Use Instead
PD01: ProductGallery Main product photos (1:1 ratio), physical products, 360° views
PD08: ProductVideoPlayer Show video demos, tutorials, walkthrough (not static screenshots)
ImageGallery (CMS Block) Content images on blog posts, case studies (not product-specific)
PortfolioGrid Design portfolios, creative work samples (different layout pattern)

Combination Patterns:

  • DP01 (Hero) + DP03 (Gallery): Hero mockup above, full screenshot gallery below
  • DP03 + DP04 (FeatureGrid): Screenshots showing features, followed by feature descriptions
  • DP03 + PD03 (ProductTabs): Screenshots in a dedicated "Screenshots" tab
  • DP03 + DP08 (VideoPlayer): Video demo followed by static screenshot gallery

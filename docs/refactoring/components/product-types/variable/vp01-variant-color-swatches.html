📋 STEP 1: Component Overview

Component Name: VariantColorSwatches (VP01)

Category: Product Types → Variable Product

Purpose: Visual color selector for product variants, allowing customers to choose from multiple color options with immediate visual feedback.

Key Features

  • ✅ Visual Color Swatches — 44×44px circular color chips with exact product colors
  • ✅ Selected State — Navy border + teal glow ring + white checkmark icon
  • ✅ Hover Tooltips — Color name appears below swatch on hover (with price suffix if applicable)
  • ✅ Gradient Support — Supports solid colors AND CSS gradients for premium finishes
  • ✅ Disabled State — Out-of-stock colors shown with reduced opacity + diagonal line
  • ✅ Selection Indicator — Selected color name displayed in teal badge at top right
  • ✅ Price Variants — Optional "+€X" suffix for premium colors (shown in tooltip)
  • ✅ Responsive — Flex-wrap layout adapts to container width

Visual Description

The component consists of a header row with label (palette icon + "Kleur" text) and meta information (type badge + selected color name), followed by a horizontal row of color swatches. Each swatch is a 44×44px rounded square with the exact product color. On hover, the swatch scales up (1.08×) and shows a tooltip with the color name. When selected, the swatch gets a 2.5px navy border and a 3px teal glow shadow, with a white circular checkmark icon appearing in the center.

Usage Context

Use this component on variable product pages where color is a key differentiator (clothing, shoes, accessories, tech products). Ideal for 5-12 color options. For more than 15 colors, consider using a dropdown selector (VP03) instead.

🎯 STEP 2: When to Use

Primary Use Cases

  1. Apparel & Footwear — T-shirts, sneakers, jackets with multiple colorways
  2. Accessories — Bags, wallets, phone cases with color options
  3. Tech Products — Headphones, laptops, phone covers in different finishes
  4. Home Goods — Cushions, curtains, tableware with color variants
  5. Custom Products — Made-to-order items where color is configurable

When NOT to Use

  • ❌ More than 15 colors — Use dropdown selector (VP03) for better UX
  • ❌ Subtle color differences — If colors are too similar to distinguish visually, use text labels instead
  • ❌ Complex patterns/prints — Use image-based selector (VP04) for patterns
  • ❌ Non-visual variants — For variants like "material" or "scent", use dropdown or radio buttons
  • ❌ Mobile-only products — 44px swatches may be too small; consider larger touch targets

Guidelines

  • Always display selected color name in the header (helps screen readers and confirms selection)
  • Use exact brand colors (not approximations) to match physical products
  • For premium finishes (metallic, chrome, gradient), use CSS gradients to indicate special treatment
  • Include price suffix (+€X) in tooltip if color affects price
  • Disable out-of-stock colors but keep them visible (don't hide) to show full range
  • Order swatches logically: neutrals first, then colors by rainbow spectrum, premiums last

TypeScript Interface

interface ColorSwatch {
  id: string
  name: string              // Display name (e.g., "Midnight Black")
  value: string             // CSS color value (hex, rgb, gradient)
  price?: number            // Additional price (e.g., 10 for "+€10")
  available: boolean        // Stock availability
  image?: string            // Optional: change main product image on select
  sku?: string              // Variant SKU for inventory tracking
}

interface VariantColorSwatchesProps {
  label?: string                    // Default: "Kleur"
  icon?: string                     // Lucide icon name, default: "palette"
  swatches: ColorSwatch[]           // Array of color options
  selected: string                  // ID of selected swatch
  onSelect: (swatch: ColorSwatch) => void  // Selection handler
  showTypeBadge?: boolean           // Show "Swatches" badge, default: true
  compact?: boolean                 // Smaller 36×36px swatches, default: false
  required?: boolean                // Show required indicator (*), default: false
  disabled?: boolean                // Disable all swatches, default: false
}

Validation Rules

  • Minimum 2 swatches required (no single-color products)
  • Maximum 20 swatches recommended (UX degradation beyond that)
  • At least one swatch must be available: true
  • Color values must be valid CSS (hex, rgb, rgba, hsl, or gradient)
  • Swatch names must be unique within a product

Keyboard Navigation

Key Action
Tab Focus enters/exits the swatch group (only selected swatch is tabbable)
Arrow Right/Down Move to next swatch (skip disabled), wrap to first
Arrow Left/Up Move to previous swatch (skip disabled), wrap to last
Space / Enter Select focused swatch
Home Jump to first available swatch
End Jump to last available swatch

Focus Management

  • Focus ring: box-shadow: 0 0 0 3px var(--teal-glow) — same as selected state for consistency
  • Roving tabindex: Only the selected (or first available) swatch has tabindex="0", others have tabindex="-1"
  • Focus visible: Use :focus-visible to show focus ring only for keyboard navigation (not mouse clicks)

Screen Reader Announcements

  • On focus: "Midnight Black, radio button, checked, 1 of 8"
  • On select: "Midnight Black selected" (via aria-live="polite" on selected badge)
  • Disabled: "Arctic White, radio button, disabled, unavailable"
  • Price suffix: "Silver Chrome, plus ten euros, radio button, 8 of 8"

Color Contrast

  • Label text (14px): Navy (#0A1628) on white — 12.5:1 contrast (WCAG AAA ✓)
  • Selected badge (12px): Teal text (#00897B) on teal-glow bg — needs dark text; use navy instead for small text
  • Tooltip (10px): Navy on white — 12.5:1 (WCAG AAA ✓)
  • Checkmark: Navy icon on white circle — high contrast regardless of swatch color

Touch Targets (Mobile)

  • Minimum touch target: 44×44px (default swatch size) — meets WCAG 2.5.5 (AAA) ✓
  • Spacing between targets: 6px gap — sufficient for fat-finger avoidance
  • Compact mode (36×36px): Falls short of AAA (44px minimum), use only when necessary

      case 'ArrowUp':
        e.preventDefault()

        return

  }

State Management

  • Parent component manages selected color ID (controlled component pattern)
  • onSelect callback receives full swatch object (not just ID) for price/image updates
  • Roving tabindex managed via React state (focusedIndex)
  • Image switching handled by parent: when onSelect fires, update main product image to swatch.image

API Integration (Payload CMS)

Performance Optimization

  • Lazy load variant images: Use Next.js <Image priority={false} /> for non-default colors
  • Memoize swatch rendering: Wrap swatch buttons in React.memo to prevent re-renders
  • Debounce image switching: If user rapidly clicks swatches, debounce main image update (150ms)
  • Preload next color: On hover, preload that color's image for instant switching

Edge Cases & Error Handling

  1. All colors out of stock: Show "Tijdelijk niet beschikbaar" message, disable add-to-cart
  2. Selected color becomes unavailable: Auto-select first available color, show notification
  3. Invalid CSS color: Fallback to grey swatch with warning icon
  4. Missing color name: Use "Kleur {index}" as fallback
  5. Duplicate color names: Append SKU to disambiguate (e.g., "Black (SKU-001)", "Black (SKU-002)")

Browser Compatibility

  • CSS Gradients: Supported in all modern browsers (IE11 needs -ms- prefix)
  • Flexbox: Full support (IE11+)
  • :focus-visible: Polyfill required for Safari <15.4
  • CSS Custom Properties: IE11 needs PostCSS fallback

🔀 STEP 8: Variations & Related Components

Component Variants

1. Compact Mode

Use case: Mobile layouts, sidebar filters, or products with 15+ colors

  • Swatch size: 36×36px (instead of 44×44px)
  • Checkmark circle: 16×16px
  • Tooltip font-size: 9px
  • Gap: 4px (instead of 6px)

2. No Header Mode

Use case: Minimalist product cards, quick-add modals

  • Hide label and badges entirely
  • Swatches only, with tooltips for identification
  • Useful when color is the ONLY variant (no size/material to distinguish)

3. Swatch + Text Mode

Use case: Accessibility enhancement, complex color names

  • Show color name text NEXT to each swatch (not just in tooltip)
  • Layout: [● Midnight Black] [● Arctic White] ...
  • Better for users who can't distinguish colors visually

4. Grid Layout (3-4 columns)

Use case: Many colors (15+), better organization

  • Use CSS Grid instead of Flexbox: grid-template-columns: repeat(auto-fill, 44px)
  • Align swatches in neat columns instead of flowing wrap

Migration Notes

If replacing an old dropdown-based color selector:

  1. Map dropdown options to swatches array (value = color code, label = name)
  2. Replace <select> with <VariantColorSwatches />
  3. Update onSelect handler to accept swatch object (not just string value)
  4. Add color values to database (if not already stored) — use Figma/brand guidelines for exact codes
  5. Test gradient colors (old system may not have supported them)
  6. Update analytics events: change from "dropdown_select" to "color_swatch_select"

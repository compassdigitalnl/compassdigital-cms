STEP 1: Component Overview
VariantGridContainer is a pure CSS layout pattern that creates a responsive 6-column grid for displaying multiple product variants (sizes, colors, materials) simultaneously. This container enables customers to view and select multiple variants at once in a compact, scannable grid format.
Key Features:
  • 6-column desktop grid — Fixed 6 columns at 1024px+ for consistent visual rhythm
  • Auto-fill mobile grid — Responsive minmax(120px, 1fr) for flexible column count
  • Minimal gap spacing — 10px gap (8px on mobile) for compact, scannable layout
  • Zero JavaScript — Pure CSS Grid, no JS required for layout
  • Child-agnostic — Works with any variant card component (VP08, custom cards)
STEP 2: When to Use
Use VariantGridContainer when:
  • Displaying 6+ product variants (sizes, colors, materials) for multi-select ordering
  • Customers need to bulk order multiple SKUs (e.g., "6 XS + 12 M + 18 L")
  • You want a compact, scannable grid view (vs. list view for detailed info)
  • Product variants have minimal info (size/color name, price, stock status)
  • B2B scenarios where customers order full size runs
Do NOT use when:
  • You have <5 variants — use single-row selection (VP01-VP05) instead
  • Variants require detailed descriptions — use list view (VP13) instead
  • You need single variant selection — use radio buttons (VP02) or swatches (VP01)
  • Mobile experience is critical and screen space is limited — consider list view first
STEP 3: Design Tokens
Layout Specifications:
Property Desktop (≥1024px) Tablet (768-1023px) Mobile (<768px)
Grid Columns 6 fixed (repeat(6, 1fr)) auto-fill minmax(120px, 1fr) auto-fill minmax(100px, 1fr)
Gap 10px 10px 8px
Min Card Width ~160px (1fr of 6) 120px 100px
Spacing Grid (4px base):
  • Gap (desktop/tablet): 10px — slightly off-grid for tighter packing
  • Gap (mobile): 8px — standard 4px grid increment
Why 10px gap?
The 10px gap is a deliberate design decision for compact multi-variant grids. It's tight enough to show 6 columns on standard desktops (1200px container) while maintaining visual breathing room. On mobile, we reduce to 8px (4px grid) for optimal space usage.
STEP 4: Props & Configuration
CSS Class: .variant-grid-compact
HTML Usage:
<!-- Wrap any variant cards in this container --> <div class="variant-grid-compact"> <!-- Child cards auto-arrange in 6-column grid --> <div class="variant-card-compact">...</div> <div class="variant-card-compact">...</div> <div class="variant-card-compact">...</div> <!-- ... more cards ... --> </div>
React/Next.js Component:
interface VariantGridContainerProps { children: React.ReactNode className?: string } export function VariantGridContainer({ children, className = '' }: VariantGridContainerProps) { return ( <div className={`variant-grid-compact ${className}`}> {children} </div> ) }
Tailwind CSS Alternative:
<!-- If you prefer Tailwind over custom CSS --> <div class="grid grid-cols-[repeat(auto-fill,minmax(120px,1fr))] lg:grid-cols-6 gap-2.5 lg:gap-[10px]"> <!-- Variant cards --> </div>
STEP 5: Layout Rules
Component Anatomy:
  • Container: CSS Grid with auto-fill/fixed columns
  • Responsive behavior:
    • Desktop (≥1024px): Always 6 columns (repeat(6, 1fr))
    • Tablet (768-1023px): Auto-fill, min 120px per card
    • Mobile (<768px): Auto-fill, min 100px per card
    • Small mobile (<480px): Fixed 2 columns
  • Gap spacing: 10px (desktop/tablet), 8px (mobile)
Grid Behavior Examples:
Container Width Columns Card Width Total Cards Visible
1200px (desktop) 6 fixed ~190px 6 per row
900px (tablet) auto-fill ~140px 5-6 per row
640px (mobile) auto-fill ~120px 4-5 per row
<480px (small) 2 fixed ~160px 2 per row
Why 6 columns on desktop?
6 columns is optimal for apparel/fashion products with standard size runs (XS, S, M, L, XL, XXL). It also works well for 12-variant grids (2 rows), 18-variant grids (3 rows), etc. The fixed 6-column layout prevents awkward 7-column or 5-column breakpoints.
STEP 6: Accessibility
Container-Level Accessibility:
  • Semantic HTML: Use <div role="group"> or <fieldset> for multi-select grids
  • ARIA Label: Add aria-label="Product variant selection" to container
  • Focus order: Grid maintains natural left-to-right, top-to-bottom tab order
  • Screen reader announcements: "Grid with X variants, select one or more"
Example with ARIA:
<div class="variant-grid-compact" role="group" aria-label="Product variant selection - choose one or more sizes" > <!-- Variant cards with proper ARIA --> </div>
Child Component Requirements:
  • Each card must be keyboard-focusable (tabindex="0")
  • Each card must have role="checkbox" or role="radio"
  • Each card must announce selection state (aria-checked="true/false")
STEP 7: Implementation Notes
Complete CSS Pattern:
.variant-grid-compact { display: grid; grid-template-columns: repeat(auto-fill, minmax(120px, 1fr)); gap: 10px; } /* Desktop: always 6 columns */ @media (min-width: 1024px) { .variant-grid-compact { grid-template-columns: repeat(6, 1fr); } } /* Tablet: tighter min-width */ @media (max-width: 768px) { .variant-grid-compact { grid-template-columns: repeat(auto-fill, minmax(100px, 1fr)); gap: 8px; } } /* Small mobile: force 2 columns */ @media (max-width: 480px) { .variant-grid-compact { grid-template-columns: repeat(2, 1fr); gap: 8px; } }
Performance Tip
CSS Grid is highly performant and GPU-accelerated. No JavaScript is needed for layout calculations. The browser handles all responsive behavior natively. For 100+ variant cards, consider virtualization with libraries like react-window or react-virtuoso.
Integration with VP08 (VariantCardCompact):
<!-- Perfect pairing: VP12 container + VP08 cards --> <div class="variant-grid-compact"> {variants.map(variant => ( <VariantCardCompact key={variant.id} variant={variant} onSelect={handleSelect} /> ))} </div>
Browser Support:
  • CSS Grid: Supported in all modern browsers (Chrome 57+, Firefox 52+, Safari 10.1+, Edge 16+)
  • auto-fill: Full support in all grid-compatible browsers
  • minmax(): Full support, no fallback needed

STEP 1: Component Overview
VariantListContainer is a pure CSS layout pattern that creates a vertical list layout for displaying product variants in detailed rows. This container enables customers to view comprehensive variant information (SKU, stock, price, quantity) in a scannable list format optimized for data-heavy displays.
Key Features:
  • Flexbox column layout — Vertical stacking with consistent gap spacing
  • 8px gap spacing — Tight, scannable rows without visual clutter
  • Full-width rows — Maximizes horizontal space for detailed info columns
  • Zero JavaScript — Pure CSS Flexbox, no JS required for layout
  • Child-agnostic — Works with any variant row component (VP09, custom rows)
STEP 2: When to Use
Use VariantListContainer when:
  • Displaying detailed variant info (SKU, full names, stock counts, notes)
  • Customers need to compare variant attributes side-by-side in table format
  • You have 10+ variants that benefit from vertical scrolling (vs. grid wrapping)
  • B2B scenarios where SKUs and stock levels are critical decision factors
  • Mobile users need a simplified single-column view (list is more mobile-friendly than grid)
Do NOT use when:
  • You have <6 variants — grid view (VP12) is more visual and scannable
  • Variants have minimal info (just size/color + price) — use grid instead
  • You want a compact, space-efficient layout — grid packs better on desktop
  • Visual product images are the primary decision factor — grid shows more images at once
STEP 3: Design Tokens
Layout Specifications:
Property Value Notes
Display flex Flexbox column layout
Flex Direction column Vertical stacking
Gap 8px 4px grid spacing
Width 100% Full container width
Max Width (optional) 900px For readability on large screens
Spacing Grid (4px base):
  • Gap: 8px — standard 4px grid increment for clean vertical rhythm
  • Consistent with grid view: VP12 uses 8px gap on mobile, ensuring seamless view switching
Why 8px gap?
The 8px gap provides optimal density for list views. It's tight enough to show 8-10 rows above the fold (on 1080p screens) while maintaining visual separation. Consistent with the 4px grid system used throughout the design system.
STEP 4: Props & Configuration
CSS Class: .variant-list-compact
HTML Usage:
<!-- Wrap any variant rows in this container --> <div class="variant-list-compact"> <!-- Child rows auto-stack vertically --> <div class="variant-row-compact">...</div> <div class="variant-row-compact">...</div> <div class="variant-row-compact">...</div> <!-- ... more rows ... --> </div>
React/Next.js Component:
interface VariantListContainerProps { children: React.ReactNode className?: string constrained?: boolean // Adds max-width: 900px } export function VariantListContainer({ children, className = '', constrained = false }: VariantListContainerProps) { return ( <div className={`variant-list-compact ${constrained ? 'constrained' : ''} ${className}`}> {children} </div> ) }
Tailwind CSS Alternative:
<!-- If you prefer Tailwind over custom CSS --> <div class="flex flex-col gap-2 max-w-[900px]"> <!-- Variant rows --> </div>
STEP 5: Layout Rules
Component Anatomy:
  • Container: CSS Flexbox with column direction
  • Responsive behavior: Always single-column (inherently responsive)
  • Gap spacing: 8px between all rows
  • Optional constraint: max-width: 900px for readability on large screens
Row Anatomy (VP09 VariantRowCompact):
Column Width Content
1. Checkbox 24px Selection checkbox
2. Thumbnail 80px Product image (60×60px)
3. Info 1fr (flexible) Name + SKU + attributes
4. Stock auto Stock indicator badge
5. Price auto (~80px) Price per unit
6. Quantity 110px Qty stepper (−/input/+)
Mobile Responsiveness
The list container itself is always single-column, but child rows (VP09) collapse their internal grid on mobile: checkbox + info + qty stepper (3-column), with thumbnail/stock/price repositioned or hidden. See VP09 documentation for row-level responsive behavior.
STEP 6: Accessibility
Container-Level Accessibility:
  • Semantic HTML: Use <div role="list"> for list semantics
  • ARIA Label: Add aria-label="Product variant list" to container
  • Focus order: Top-to-bottom natural tab order through rows
  • Screen reader announcements: "List with X variants, use arrow keys to navigate"
Example with ARIA:
<div class="variant-list-compact" role="list" aria-label="Product variant list - select one or more variants" > <!-- Each row has role="listitem" --> <div class="variant-row-compact" role="listitem">...</div> </div>
Child Component Requirements:
  • Each row must have role="listitem" for screen reader context
  • Each row's interactive elements (checkbox, qty buttons) must be keyboard-focusable
  • Each row must announce variant name + price when focused
STEP 7: Implementation Notes
Complete CSS Pattern:
.variant-list-compact { display: flex; flex-direction: column; gap: 8px; } /* Optional: constrain width for readability */ .variant-list-compact.constrained { max-width: 900px; margin: 0 auto; /* Center on large screens */ }
Performance Tip
CSS Flexbox is highly performant. No JavaScript is needed for layout. For 100+ variant rows, consider virtualization with react-window or react-virtuoso to render only visible rows. Virtual scrolling can reduce initial render time by 80-90%.
Integration with VP09 (VariantRowCompact):
<!-- Perfect pairing: VP13 container + VP09 rows --> <div class="variant-list-compact"> {variants.map(variant => ( <VariantRowCompact key={variant.id} variant={variant} onSelect={handleSelect} /> ))} </div>
Virtualization Example (react-window):
import { FixedSizeList } from 'react-window' <FixedSizeList height={600} itemCount={variants.length} itemSize={72} // Row height (60px + 8px gap + 4px padding) width="100%" className="variant-list-compact" > {({ index, style }) => ( <div style={style}> <VariantRowCompact variant={variants[index]} /> </div> )} </FixedSizeList>
Browser Support:
  • CSS Flexbox: Universal support (IE10+, all modern browsers)
  • flex-direction: column: Full support in all flex-compatible browsers
  • gap property: Supported in Chrome 84+, Firefox 63+, Safari 14.1+, Edge 84+
STEP 8: Variations & Related Components
Layout Variants:
  • VP13: VariantListContainer — This component (list view)
  • VP12: VariantGridContainer — Grid view alternative (6-column responsive)
Child Components (use inside list):
  • VP09: VariantRowCompact — Horizontal row with 6-column grid (checkbox, thumb, info, stock, price, qty)
  • Custom rows — Any row component with consistent height
Related Components:
  • VP11: VariantToolbar — Grid/list toggle + bulk actions (used above VP12/VP13)
  • VP10: VariantSelectionSidebar — Sticky summary of selected variants + totals
  • VP08: VariantCardCompact — Card component for grid view (use with VP12)
View Toggle Pattern:
const [viewMode, setViewMode] = useState('grid') return ( <> <VariantToolbar viewMode={viewMode} onViewChange={setViewMode} /> {viewMode === 'grid' ? ( <VariantGridContainer> {variants.map(v => <VariantCardCompact variant={v} />)} </VariantGridContainer> ) : ( <VariantListContainer> {variants.map(v => <VariantRowCompact variant={v} />)} </VariantListContainer> )} </> )

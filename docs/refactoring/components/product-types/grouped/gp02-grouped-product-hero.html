GP02: Grouped Product Hero

Hero section for grouped product pages with badge, product title, description, and key statistics (item count + savings). Features dark gradient background with decorative teal glow.

1 Component Overview

Purpose

The Grouped Product Hero introduces grouped/bundled product pages with a prominent dark hero section. It communicates the value proposition (buy individually OR get complete set discount) and displays key metrics (item count, total savings).

Key Features

  • Product Type Badge: "Grouped Product" pill with package icon (teal styling)
  • Large Product Title: 36px serif font (DM Serif Display) with white color
  • Description Text: Max-width 600px, semi-transparent white (70% opacity)
  • Statistics Grid: 2-column flex layout showing item count + savings amount
  • Dark Gradient Background: Navy → navy-light gradient (135deg)
  • Decorative Glow: Teal radial gradient (pseudo-element, top-right)
  • Responsive Behavior: Stats stack vertically on mobile (<768px)

2 When to Use

✅ Use this component when:

  • Product page displays a grouped product (multiple related items sold individually OR as a set)
  • You want to highlight bundle savings (e.g., "Save €45 when buying complete set")
  • You need to communicate flexibility ("Buy all OR pick individual items")
  • Product has multiple components (3+ items) that can be purchased separately

❌ Don't use when:

  • Product is a fixed bundle (cannot buy individual items) → Use Bundle Hero instead
  • Product has variants (sizes/colors) → Use Variable Product Hero (PD02)
  • It's a regular simple product → Use standard product meta (PD02)
  • Page is not a product page (e.g., category page) → Use CategoryHero (EC02)

6 Accessibility

♿ Accessibility Checklist

  • Semantic HTML: Use <h1> for title (main heading of page)
  • Color Contrast: White text on dark gradient = 15:1 ratio (AAA compliant)
  • Icon Accessibility: Badge icon is decorative, no alt text needed
  • Reading Order: Badge → Title → Description → Stats (logical flow)
  • Focus Management: No interactive elements (pure presentational)
  • Screen Reader: Stats announced as "6 Items in set" and "€45 Besparing bij complete set"

ARIA Attributes

None required. This is a purely presentational component with semantic HTML. The badge is decorative and does not convey essential information beyond the visible text.

7 Implementation Notes

Payload CMS Integration

Payload Collection Schema
// collections/GroupedProducts.ts
{
  slug: 'grouped-products',
  fields: [
    {
      name: 'title',
      type: 'text',
      required: true,
      label: 'Product Set Name'
    },
    {
      name: 'description',
      type: 'textarea',
      required: true,
      label: 'Hero Description',
      maxLength: 200
    },
    {
      name: 'items',
      type: 'relationship',
      relationTo: 'products',
      hasMany: true,
      label: 'Included Products'
    },
    {
      name: 'setDiscount',
      type: 'number',
      required: true,
      label: 'Set Discount (%)',
      min: 0,
      max: 50
    }
  ],
  hooks: {
    beforeChange: [
      ({ data }) => {
        // Auto-calculate savings
        const subtotal = data.items.reduce((sum, item) => sum + item.price, 0)
        data.savings = subtotal * (data.setDiscount / 100)
        data.itemCount = data.items.length
        return data
      }
    ]
  }
}

Dynamic Savings Calculation

Common Pitfalls

  • Overflow on mobile: Ensure overflow: hidden on .grouped-hero to clip decorative glow
  • Description too long: Limit description to 1-2 sentences (max 200 chars) for visual balance
  • Incorrect savings: Always show TOTAL savings (not percentage) for clarity
  • Missing currency symbol: Always prefix savings with € (or locale currency)
  • Hardcoded label text: Make "Items in set" and "Besparing" translatable (i18n)

Performance Considerations

  • Decorative glow: Use CSS pseudo-element (not extra DOM node) for performance
  • Font loading: Preload DM Serif Display for hero title (critical text)
  • Background gradient: Use CSS gradient (not image) for better performance
  • Item count calculation: Pre-calculate in CMS hook (not on every render)

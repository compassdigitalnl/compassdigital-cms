📦 STEP 1: Component Overview

Component: SP02: SubscriptionFilterToggle

Purpose: Binary toggle switch for filtering between "Persoonlijk" and "Cadeau" subscription types. Alternative to pill-style tabs when you have exactly 2 mutually exclusive options.

Key Features:

  • iOS-style toggle switch (56×28px) with animated sliding dot
  • Left/right labels with icons (Persoonlijk ↔ Cadeau)
  • Active label: navy text (bold)
  • Inactive label: grey text
  • Switch background: grey (Persoonlijk) → teal (Cadeau)
  • Smooth transition animation (0.2s cubic-bezier)
  • Clickable labels for easier interaction
  • Centered layout in white card container

Visual Description: Centered toggle switch with icon labels on both sides. Grey switch indicates "Persoonlijk", teal switch indicates "Cadeau". White dot slides left/right on toggle.

📋 STEP 2: When to Use

Primary Use Cases:

  • Binary Subscription Types: When filtering between exactly 2 options (Personal vs Gift)
  • Clear Opposite States: When options are mutually exclusive and conceptually opposite
  • Cleaner UI Alternative: When pill tabs feel too busy for just 2 options
  • Mobile-Friendly: Toggle switches work better on touch devices than small tabs

When NOT to Use:

  • ❌ For 3+ options (use SP01 pill tabs or SP03 sidebar)
  • ❌ For multi-select filtering (use checkboxes in sidebar)
  • ❌ When you need product counts per category (use SP01 with count badges)
  • ❌ For non-boolean states (use radio buttons or dropdown)

ARIA Attributes:

  • role="switch" on toggle element
  • aria-checked="true/false" reflects current state
  • aria-labelledby links to both label IDs
  • aria-hidden="true" on decorative icons
  • tabindex="0" makes switch keyboard focusable

Keyboard Navigation:

  • Tab - Focus on switch
  • Space/Enter - Toggle switch
  • ← - Switch to left (Persoonlijk)
  • → - Switch to right (Cadeau)

  • Focus remains on switch after activation

💻 STEP 7: Implementation Notes

React/Next.js Component:

'use client';

import { useState } from 'react';
import { User, Gift } from 'lucide-react';

export const SubscriptionFilterToggle = ({
  leftLabel = 'Persoonlijk',
  rightLabel = 'Cadeau',
  value,
  onChange
}) => {
  const isRight = value === 'right';

  const toggle = () => {
    onChange(isRight ? 'left' : 'right');
  };

  const handleKeydown = (e) => {
    if (e.key === ' ' || e.key === 'Enter') {
      e.preventDefault();
      toggle();
    } else if (e.key === 'ArrowLeft') {
      onChange('left');
    } else if (e.key === 'ArrowRight') {
      onChange('right');
    }
  };

  return (
    <div className="toggle-filter">
      <span
        className={`toggle-label ${!isRight ? 'active' : ''}`}
        onClick={toggle}
      >
        <User size={18} aria-hidden="true" />
        {leftLabel}
      </span>

      <div
        className={`toggle-switch ${isRight ? 'cadeau' : ''}`}
        role="switch"
        aria-checked={isRight}
        tabIndex={0}
        onClick={toggle}
        onKeyDown={handleKeydown}
      >
        <div className="toggle-dot" />
      </div>

      <span
        className={`toggle-label ${isRight ? 'active' : ''}`}
        onClick={toggle}
      >
        <Gift size={18} aria-hidden="true" />
        {rightLabel}
      </span>
    </div>
  );
};

Usage Example:

const [subscriptionType, setSubscriptionType] = useState('left');

<SubscriptionFilterToggle
  value={subscriptionType}
  onChange={setSubscriptionType}
/>

State Management:

  • Store toggle state in component state or URL query params
  • Filter product list based on toggle value
  • Update URL with ?type=gift for shareable links
  • Persist user preference in LocalStorage (optional)

Performance Optimization:

  • Use CSS transitions (hardware accelerated) instead of JS animations
  • Memoize callback functions with useCallback
  • Debounce product filtering if fetching from API (100-200ms)

Edge Cases:

  • No products for selected type: Show empty state message
  • Loading state: Disable toggle during product fetch
  • Default selection: Always default to "Persoonlijk" (left)
  • URL sync: Read initial state from URL query param

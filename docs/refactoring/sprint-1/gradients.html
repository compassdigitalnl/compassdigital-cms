Design Token
Waarom gradients?
Gradients voegen depth en visual interest toe aan buttons, hero sections en backgrounds. Ze zijn essentieel voor multi-tenant branding waarbij elke vertical zijn eigen gradient identiteit krijgt.
Primary Gradient
Primary Gradient
linear-gradient(135deg, #00897B 0%, #26A69A 100%)

Gebruik: Primary buttons, CTAs, hover states, interactive elementen

Multi-tenant: Beauty: pink gradient • Horeca: gold gradient • Medisch: teal gradient (default)

Secondary Gradient
Secondary Gradient
linear-gradient(135deg, #0A1628 0%, #1B2B45 100%)

Gebruik: Dark sections, footers, navigation backgrounds, contrast areas

Multi-tenant: Consistent across verticals (dark foundation)

Hero Gradient
Hero Overlay (Subtle)
linear-gradient(135deg, rgba(0,137,123,0.08) 0%, rgba(38,166,154,0.12) 100%)

Gebruik: Hero section overlays, subtle backgrounds, accent areas

Multi-tenant: Matches primary color with low opacity (8-12%)

Accent Gradient (Optional)
Accent Gradient
linear-gradient(135deg, #7C3AED 0%, #A855F7 100%)

Gebruik: Special promotions, badges, accent elements, "new" tags

Multi-tenant: Optional - can be customized per vertical for special campaigns

Step 3: CSS Variable Generation

Generate CSS custom properties in app/layout.tsx

// app/layout.tsx import { getPayloadHMR } from '@payloadcms/next/utilities' import configPromise from '@payload-config' import type { Theme } from '@/types/theme' export default async function RootLayout({ children }) { const payload = await getPayloadHMR({ config: configPromise }) // Fetch theme global const theme = await payload.findGlobal({ slug: 'theme', }) as Theme // Generate gradient CSS variables const gradients = theme.gradients || {} const gradientVars = ` --gradient-primary: ${gradients.primaryGradient || 'linear-gradient(135deg, #00897B 0%, #26A69A 100%)'}; --gradient-secondary: ${gradients.secondaryGradient || 'linear-gradient(135deg, #0A1628 0%, #1B2B45 100%)'}; --gradient-hero: ${gradients.heroGradient || 'linear-gradient(135deg, rgba(0,137,123,0.08) 0%, rgba(38,166,154,0.12) 100%)'}; --gradient-accent: ${gradients.accentGradient || 'linear-gradient(135deg, #7C3AED 0%, #A855F7 100%)'}; ` return ( <html> <head> <style dangerouslySetInnerHTML={{ __html: `:root { ${gradientVars} }` }} /> </head> <body>{children}</body> </html> ) }

Multi-Tenant Vertical-Specific Gradients
Multi-tenant Power!
Each vertical can override gradients via the Themes collection. Beauty gets pink gradients, Horeca gets gold, Medisch keeps teal. All automatic via data-theme attribute!

Best Practices Guidelines & Tips

✅ DO

  • Use 135deg for consistent diagonal direction
  • Keep gradients subtle (max 2 colors, close hues)
  • Test gradients on white AND dark backgrounds
  • Ensure text contrast meets WCAG AA standards (4.5:1)
  • Use rgba() for hero gradients (8-12% opacity)
  • Match hero gradient to primary color with low opacity

❌ DON'T

  • Use more than 2 colors in a single gradient (rainbow effect)
  • Mix warm and cool tones (e.g., blue + orange)
  • Use gradients on body text (readability issues)
  • Forget fallback colors for browsers without gradient support
  • Use high opacity hero gradients (>20%) - too overpowering

💡 Tips

  • Consistency: All gradients should flow 135deg (top-left to bottom-right)
  • Performance: Gradients are CSS-only, no JavaScript needed
  • Accessibility: Ensure 4.5:1 contrast for white text on gradient buttons
  • Testing: Preview all verticals before deployment
  • Fallback: Always provide solid color fallback: background: #00897B; background: var(--gradient-primary);

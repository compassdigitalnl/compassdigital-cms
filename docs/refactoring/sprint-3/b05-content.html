5 Testing Checklist
  • Lexical Features: Test all editor features: headings (H2-H4), bold, italic, underline, strikethrough. Verify formatting renders correctly on frontend.
  • Lists: Create ordered (numbered) and unordered (bulleted) lists. Test nested lists (2-3 levels deep). Verify list styling matches design system.
  • Links: Add internal links (/about), external links (https://...), and hash links (#section). Test link color (teal), underline, hover states.
  • Code Blocks: Test inline code (`code`) and code blocks with syntax highlighting. Verify monospace font (JetBrains Mono), background color, border radius.
  • Width Variants: Test narrow (640px), wide (900px), full (100%). Verify responsive behavior on mobile. Test long words/URLs don't overflow.
  • Typography Spacing: Verify vertical rhythm: paragraphs, headings, lists have correct spacing. Test long content (2000+ words) for readability.
  • Empty State: Test block with minimal content (1 paragraph). Test with very long content. Verify no layout breaks or performance issues.
6 Database Migration
SQL migrations/YYYYMMDDHHMMSS_add_content_block.sql 
-- Add Content block type
ALTER TYPE block_type ADD VALUE IF NOT EXISTS 'content';

-- Create content_blocks table
CREATE TABLE IF NOT EXISTS content_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  block_id UUID NOT NULL REFERENCES blocks(id) ON DELETE CASCADE,

  content JSONB NOT NULL, -- Lexical rich text JSON
  max_width VARCHAR(20) NOT NULL DEFAULT 'narrow'
    CHECK (max_width IN ('narrow', 'wide', 'full')),

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create indexes
CREATE INDEX idx_content_blocks_block_id ON content_blocks(block_id);
CREATE INDEX idx_content_blocks_max_width ON content_blocks(max_width);

-- Full-text search index on content (for search functionality)
-- Extracts plain text from Lexical JSON for search
CREATE INDEX idx_content_blocks_content_search ON content_blocks
  USING gin(to_tsvector('english', content::text));

-- Trigger for updated_at
CREATE OR REPLACE FUNCTION update_content_blocks_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_content_blocks_updated_at
  BEFORE UPDATE ON content_blocks
  FOR EACH ROW EXECUTE FUNCTION update_content_blocks_updated_at();

-- Comments
COMMENT ON TABLE content_blocks IS 'Rich text content blocks with Lexical editor';
COMMENT ON COLUMN content_blocks.content IS 'Lexical editor JSON with full rich text support (headings, lists, links, code, etc.)';
COMMENT ON COLUMN content_blocks.max_width IS 'Content width: narrow (640px, optimal reading), wide (900px), full (100%)';

-- Rollback script

7 Troubleshooting Guide
Rich text renders as [object Object]
Cause: Lexical JSON not serialized to React components.
Fix: Import serializeLexical from @/utilities/serializeLexical. Call serializeLexical({ nodes: content }). Verify content is not null/undefined. Check Lexical JSON structure.
Typography styles not applying
Cause: Tailwind Typography plugin not installed or configured.
Fix: Run npm install @tailwindcss/typography. Add to plugins in tailwind.config.js. Verify prose class is applied. Test by inspecting computed styles in DevTools.
Headings too large/small
Cause: Typography plugin default sizes or missing custom typography config.
Fix: Customize typography theme in tailwind.config.js. Set H2 (24px), H3 (20px), H4 (16px). Test with prose-sm or prose-lg variants. Verify font-serif applies to headings.
Links not teal color
Cause: Typography prose styles override link color.
Fix: Add prose-navy custom theme with --tw-prose-links: #00897B. Apply prose-navy class alongside prose. Test hover state (darker teal). Verify underline appears.
Code blocks have no syntax highlighting
Cause: Lexical CodeBlockFeature doesn't include syntax highlighting by default.
Fix: Install Prism.js or highlight.js. Add custom code block serializer in serializeLexical utility. Apply syntax theme CSS. Test with multiple languages (JS, Python, SQL).
Content width not responsive
Cause: Missing mx-auto or px-6 classes, or max-width not applying.
Fix: Verify parent div has mx-auto px-6. Check max-w-2xl/max-w-4xl classes apply. Test at different breakpoints. Ensure no parent overflow:hidden cutting off content.
Long words/URLs overflow container
Cause: No word-break or overflow-wrap CSS.
Fix: Add break-words class to content wrapper. Use overflow-wrap: break-word in custom CSS. Test with very long URLs (100+ chars). Consider using word-break: break-word for aggressive breaking.
Blockquotes not styled
Cause: Typography plugin blockquote styles not applying or custom styles needed.
Fix: Verify BlockquoteFeature enabled in Lexical config. Add custom blockquote styles: border-left (teal), padding-left, italic text. Test quote marks display. Check grey color for quote text.

Usage Guidelines
  • Optimal Reading Width: Use "narrow" (640px) for blog posts, articles, documentation. This provides 45-75 characters per line - optimal for reading comprehension.
  • Wide Width: Use "wide" (900px) for content with images, tables, or code examples that need more horizontal space.
  • Full Width: Use "full" sparingly for special layouts, landing pages, or when content should span entire container.
  • Headings: H2 for main sections, H3 for subsections, H4 for minor points. Keep heading hierarchy logical (don't skip levels).
  • Lists: Use unordered lists for items without sequence. Use ordered lists for steps, rankings, or sequential items. Keep list items concise (1-2 lines).
  • Code Blocks: Use inline code for short snippets (variable names, commands). Use code blocks for multi-line code examples. Specify language for syntax highlighting.
  • Links: Link color is teal (#00897B) with underline. Use descriptive link text (not "click here"). External links can optionally show icon.

5 Testing Checklist

Comprehensive testing checklist covering all variants, icon loading, and responsive behavior.

  • Grid Layouts: Test all 3 variants (grid-3, grid-4, list) at desktop/tablet/mobile breakpoints. Verify grid-3 shows 3→2→1 columns, grid-4 shows 4→2→1, list stays single column.
  • Icon Loading: Test with valid Lucide icon names (zap, palette, shield-check). Verify icons load dynamically from lucide-react package. Check fallback "?" displays for invalid icon names.
  • Icon Styles: Verify all 3 icon styles render correctly: glow (teal bg with 12% opacity), solid (teal bg + white icon), outlined (2px teal border + teal icon).
  • Content Alignment: Test center vs left alignment for both grid and list variants. Confirm icons align correctly (centered in grid, left in list). Check text alignment matches setting.
  • Feature Count: Test with minimum 2 features and maximum 12 features. Verify grid wraps correctly with odd numbers (e.g., 5 features in grid-3 layout). Check spacing consistency.
  • Responsive Behavior: Confirm mobile (< 640px) always shows 1 column. Tablet (640-1024px) shows 2 columns. Desktop (>= 1024px) respects variant setting (3 or 4 columns).
  • Typography & Spacing: Verify title uses DM Serif Display at 24-28px. Feature titles use Plus Jakarta Sans Bold 16px. Descriptions use 13px with grey-dark color. Check consistent 24px gaps between cards.
6 Database Migration

SQL migration to add the Features block fields to the blocks table and create necessary indexes.

SQL migrations/YYYYMMDDHHMMSS_add_features_block.sql 
-- Add Features block type to blocks table

ALTER TYPE block_type ADD VALUE IF NOT EXISTS 'features';

-- Create features_blocks table
CREATE TABLE IF NOT EXISTS features_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  block_id UUID NOT NULL REFERENCES blocks(id) ON DELETE CASCADE,

  -- Content fields
  title TEXT,
  description TEXT,

  -- Layout fields
  variant VARCHAR(20) NOT NULL DEFAULT 'grid-3'
    CHECK (variant IN ('grid-3', 'grid-4', 'list')),
  icon_style VARCHAR(20) NOT NULL DEFAULT 'glow'
    CHECK (icon_style IN ('glow', 'solid', 'outlined')),
  alignment VARCHAR(10) NOT NULL DEFAULT 'center'
    CHECK (alignment IN ('center', 'left')),

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT fk_block FOREIGN KEY (block_id) REFERENCES blocks(id) ON DELETE CASCADE
);

-- Create junction table for features array
CREATE TABLE IF NOT EXISTS features_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  features_block_id UUID NOT NULL REFERENCES features_blocks(id) ON DELETE CASCADE,
  icon TEXT NOT NULL, -- Lucide icon name
  title TEXT NOT NULL,
  description TEXT,
  order_index INTEGER NOT NULL DEFAULT 0,

  created_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT unique_feature_order UNIQUE (features_block_id, order_index),
  CONSTRAINT min_2_features CHECK (
    (SELECT COUNT(*) FROM features_items WHERE features_block_id = features_items.features_block_id) >= 2
  ),
  CONSTRAINT max_12_features CHECK (
    (SELECT COUNT(*) FROM features_items WHERE features_block_id = features_items.features_block_id) <= 12
  )
);

-- Create indexes for performance
CREATE INDEX idx_features_blocks_block_id
  ON features_blocks(block_id);

CREATE INDEX idx_features_blocks_variant
  ON features_blocks(variant);

CREATE INDEX idx_features_items_block_id
  ON features_items(features_block_id);

CREATE INDEX idx_features_items_order
  ON features_items(features_block_id, order_index);

-- Add trigger for updated_at timestamp
CREATE OR REPLACE FUNCTION update_features_blocks_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_features_blocks_updated_at
  BEFORE UPDATE ON features_blocks
  FOR EACH ROW
  EXECUTE FUNCTION update_features_blocks_updated_at();

7 Troubleshooting Guide
Icons not loading or showing "?"
Cause: Invalid Lucide icon name, incorrect PascalCase conversion, or lucide-react not installed.
Fix: Verify icon names at lucide.dev. Ensure lucide-react package is installed: npm install lucide-react. Check PascalCase conversion logic handles hyphens correctly (shield-check → ShieldCheck). Add console.log to debug IconComponent lookup.
Grid not responsive on mobile
Cause: Tailwind breakpoint classes not applying or CSS conflicts.
Fix: Verify Tailwind config includes md: and lg: breakpoints. Check grid-cols-1 md:grid-cols-2 lg:grid-cols-3 classes are correct. Test at exact breakpoints (640px, 1024px). Clear Tailwind cache and rebuild.
Icon background glow not showing teal color
Cause: Tailwind not recognizing rgba(0,137,123,0.12) or custom class purged.
Fix: Use Tailwind arbitrary values: bg-[rgba(0,137,123,0.12)]. Alternatively, add to tailwind.config.js theme.extend.colors: 'teal-glow': 'rgba(0, 137, 123, 0.12)'. Whitelist class in safelist if using JIT mode.
List layout icons not aligning left
Cause: Flex shrink causing icon size changes or alignment classes missing.
Fix: Add flex-shrink-0 to icon container div. Verify parent has flex gap-4 items-start classes. Check icon width/height are fixed (w-14 h-14). Remove any margin-auto classes that override alignment.
Feature descriptions too long breaking layout
Cause: No max length validation or line clamp on descriptions.
Fix: Add validation in Payload config: validate: (val) => val.length <= 200. Apply CSS line clamp: className="line-clamp-3" or max-h-[4.5rem] overflow-hidden. Add character counter in admin panel for UX.
Uneven card heights in grid layout
Cause: Variable description lengths creating inconsistent heights.
Fix: Add min-h-[200px] to feature-card div for consistent baseline. Alternatively, use CSS Grid with grid-auto-rows: 1fr. Or add line-clamp-3 to descriptions to equalize content height.
Icons pixelated or blurry at high DPI
Cause: SVG icons not scaling correctly or size mismatch.
Fix: Ensure IconComponent receives className with w-7 h-7 (28px). Verify SVG viewBox is correct. Use strokeWidth={2} prop for consistent line weights. Test on Retina/HiDPI displays. Avoid rasterized icon formats (PNG/JPG).
Section title not using DM Serif Display font
Cause: font-serif class not applying or font not loaded.
Fix: Verify Google Fonts import includes DM Serif Display in layout/head. Check tailwind.config.js has fontFamily.serif: ['DM Serif Display', 'serif']. Apply font-serif class to h2 element. Inspect computed styles in DevTools.

Usage Guidelines
  • Title: DM Serif Display, 24-28px, centered or left-aligned
  • Icons: Lucide icons only, 28px size, 3 styles (glow/solid/outlined)
  • Features: Min 2, max 12 features per block, keep descriptions under 200 chars
  • Variants: Grid-3 (3 cols), Grid-4 (4 cols), List (horizontal icon + text)
  • Grid: Desktop: 3/4 cols, Tablet (640px): 2 cols, Mobile: 1 col
  • Spacing: 24px gap between cards, 32px bottom margin for section
  • Alignment: Center (icons+text centered) or Left (icons+text left-aligned)

5 Testing Checklist
  • Accordion Open/Close: Click question to open answer. Click again to close. Verify smooth transition. Test clicking different questions (previous should close).
  • Icon Rotation: Plus icon should rotate 45° to form X when open. Verify icon background changes from grey-light to teal. Test transition smoothness (0.2s).
  • Keyboard Navigation: Tab through FAQ items. Press Enter/Space to toggle open. Verify focus state (teal outline ring). Test with screen reader.
  • Rich Text Rendering: Add answer with bold, italic, lists, links. Verify all Lexical features render correctly. Test code blocks and images if enabled.
  • Two-Column Layout: Test two-column variant splits FAQs evenly. Verify columns stack vertically on mobile (< 768px). Test odd number of FAQs (5, 7, 9).
  • Multiple Open Items: Current implementation allows only one open at a time. Test that opening new item closes previous. Modify if multiple-open is desired.
  • Empty States: Test with no title/description. Test with 1 FAQ, 2 FAQs, 10 FAQs. Verify layout doesn't break with very long questions or answers.

-- Add FAQ block type
ALTER TYPE block_type ADD VALUE IF NOT EXISTS 'faq';

-- Create faq_blocks table
CREATE TABLE IF NOT EXISTS faq_blocks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  block_id UUID NOT NULL REFERENCES blocks(id) ON DELETE CASCADE,

  title TEXT,
  description TEXT,
  variant VARCHAR(20) NOT NULL DEFAULT 'single-column'
    CHECK (variant IN ('single-column', 'two-column')),

  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create junction table for FAQ items
CREATE TABLE IF NOT EXISTS faq_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  faq_block_id UUID NOT NULL REFERENCES faq_blocks(id) ON DELETE CASCADE,
  question TEXT NOT NULL,
  answer JSONB NOT NULL, -- Lexical rich text JSON
  order_index INTEGER NOT NULL DEFAULT 0,

  created_at TIMESTAMPTZ DEFAULT NOW(),

  CONSTRAINT unique_faq_order UNIQUE (faq_block_id, order_index)
);

-- Create indexes
CREATE INDEX idx_faq_blocks_block_id ON faq_blocks(block_id);
CREATE INDEX idx_faq_blocks_variant ON faq_blocks(variant);
CREATE INDEX idx_faq_items_faq_block_id ON faq_items(faq_block_id);
CREATE INDEX idx_faq_items_order ON faq_items(faq_block_id, order_index);

-- Full-text search index on questions (for search functionality)
CREATE INDEX idx_faq_items_question_search ON faq_items
  USING gin(to_tsvector('english', question));

-- Trigger for updated_at
CREATE OR REPLACE FUNCTION update_faq_blocks_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_faq_blocks_updated_at
  BEFORE UPDATE ON faq_blocks
  FOR EACH ROW EXECUTE FUNCTION update_faq_blocks_updated_at();

7 Troubleshooting Guide
Accordion doesn't open/close on click
Cause: Client component not marked with 'use client' or state not updating.
Fix: Verify FAQAccordion.tsx has 'use client' directive at top. Check useState is imported from 'react'. Confirm toggleItem function is called correctly. Test in browser console: check React DevTools for state updates.
Icon not rotating to X when open
Cause: Missing rotate-45 class or CSS transform not applying.
Fix: Verify isOpen condition: isOpen ? 'rotate-45' : ''. Check Tailwind includes rotate-45 utility. Test transition-all class is present. Use Chrome DevTools to inspect computed transform value.
Rich text answer renders as [object Object]
Cause: Lexical JSON not serialized to React components.
Fix: Import serializeLexical from @/utilities/serializeLexical. Call serializeLexical({ nodes: faq.answer }). Verify answer field is JSONB in database. Check Lexical JSON structure is valid.
Multiple items can be open at once
Cause: This is by design if using array state instead of single index.
Fix: Current implementation (openIndex: number | null) allows only one open. To allow multiple: change state to openIndexes: number[] and toggle by adding/removing from array. Update isOpen check to openIndexes.includes(globalIndex).
Two-column layout not splitting evenly
Cause: Math.ceil calculation incorrect or columns not distributing properly.
Fix: Verify itemsPerColumn = Math.ceil(faqs.length / columns). Test with odd numbers (5 FAQs = 3 in col 1, 2 in col 2). Check slice logic: faqs.slice(i * itemsPerColumn, (i + 1) * itemsPerColumn). Test md:grid-cols-2 applies at 768px breakpoint.
Clicking one FAQ opens wrong item in two-column
Cause: globalIndex calculation incorrect.
Fix: Verify globalIndex = colIndex * itemsPerColumn + itemIndex. Test by console.log(globalIndex) on click. Ensure key prop uses globalIndex, not itemIndex. Check openIndex comparison matches globalIndex.
Answer content jumps/flickers when opening
Cause: Missing transition or height animation not smooth.
Fix: Add max-height animation: use max-h-0 when closed, max-h-screen when open. Include transition-all duration-200 classes. Consider using react-spring or framer-motion for smoother accordion animation.
Focus state not visible on keyboard navigation
Cause: Browser default focus outline removed or not styled.
Fix: Add focus:ring-2 focus:ring-teal focus:ring-offset-2 to button. Test with Tab key navigation. Ensure focus-visible pseudo-class works. Never use outline-none without replacement focus state.

Usage Guidelines
  • Questions: Keep questions concise (under 100 characters). Use natural language as users would ask them. Start with question words (Hoe, Wat, Waarom, etc.).
  • Answers: Full Lexical rich text support. Use bold for emphasis, lists for steps, links for references. Keep answers under 200 words for scannability.
  • Icon Behavior: Plus icon rotates 45° to form X when open. Background changes from grey to teal. Provides clear visual feedback of state.
  • Layout Variants: Single-column (default, best for 5-10 FAQs, max 720px), Two-column (side-by-side on desktop, stacks on mobile, best for 6+ FAQs).
  • Interaction: Current implementation allows only one FAQ open at a time (accordion pattern). Opening new item closes previous. This is standard UX practice.
  • SEO Benefits: FAQ blocks automatically generate FAQPage schema.org markup for rich snippets in Google. Keep questions natural and answers complete.
  • Accessibility: Fully keyboard accessible (Tab, Enter/Space). Screen reader friendly with proper ARIA labels. Focus states clearly visible.
